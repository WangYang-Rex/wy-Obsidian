# OpenClaw 产品调研报告

## 概述

[[OpenClaw]] 是一款**开源的本地优先 AI 代理与自动化平台**，于 2026 年初在 GitHub 迅速走红。它的核心理念是：**"AI 不应只是回答问题，而应主动帮你完成任务"**。

与传统云端 AI 助手（如 ChatGPT、Claude.ai）不同，OpenClaw 运行在用户自己的设备上（笔记本、私有服务器、家庭实验室），通过本地网关处理所有请求，数据永不上传第三方服务器，真正实现**隐私优先、自主可控**。

---

## 核心特性

### 1. 本地优先架构

[[本地优先]]（Local-First）是 OpenClaw 的核心设计理念：

- **数据主权**：所有对话历史、个人偏好、文件内容都存储在用户本地
- **隐私保护**：数据不经过第三方服务器，仅在与 LLM API 交互时发送必要内容
- **离线可用**：核心功能可在无网络环境运行（配合本地模型）

### 2. 多渠道通信集成

OpenClaw 支持通过多种消息渠道与 AI 交互：

| 渠道 | 状态 | 用途 |
|------|------|------|
| [[Telegram]] | ✅ 稳定 | 国际即时通讯 |
| [[WhatsApp]] | ✅ 稳定 | 国际即时通讯 |
| [[Discord]] | ✅ 稳定 | 社区/游戏聊天 |
| [[飞书]] | ✅ 官方插件 | 企业协作 |
| [[钉钉]] | ✅ 支持 | 企业协作 |
| [[Signal]] | ✅ 稳定 | 加密通讯 |
| [[iMessage]] | ✅ 支持 | 苹果生态 |
| [[Slack]] | ✅ 稳定 | 团队协作 |
| Web Chat | ✅ 内置 | 网页界面 |

### 3. 多模型支持

OpenClaw 支持接入多种 [[大语言模型]]（LLM）：

- **OpenAI**：GPT-4o、GPT-4-turbo、GPT-3.5
- **Anthropic**：Claude 3.5 Sonnet、Claude 3 Opus
- **阿里云百炼**：通义千问系列
- **本地模型**：Ollama、vLLM、SGLang
- **其他**：Google Gemini、DeepSeek、智谱 GLM

### 4. [[MCP]] 协议支持

OpenClaw 原生支持 [[Model Context Protocol]]（MCP），这是 Anthropic 推出的标准化工具调用协议：

- 通过 MCP 服务器扩展 AI 能力
- 支持 _stdio_ 和 _Streamable-HTTP_ 两种传输模式
- 内置 `mcporter` CLI 工具管理 MCP 服务器

### 5. [[Agent Skills]] 系统

OpenClaw 采用 [[AgentSkills]] 架构来扩展 AI 能力：

```
~/.openclaw/workspace/skills/
├── skill-name/
│   ├── SKILL.md      # 技能定义文件
│   ├── scripts/      # 可执行脚本
│   └── references/   # 参考文档
```

- **SKILL.md**：定义技能触发条件和使用方式
- **按需加载**：仅在相关任务时加载技能
- **社区共享**：通过 [[ClawHub]] 分享和安装技能

### 6. 持久记忆系统

[[持久记忆]]（Persistent Memory）让 AI 助手具备长期记忆能力：

- **MEMORY.md**：长期记忆存储
- **memory/*.md**：每日日志记录
- **向量检索**：基于语义搜索的记忆召回
- **隐私隔离**：不同会话的记忆可隔离

## 部署方式

### 1. 本地部署（推荐）

```bash
# macOS/Linux
npm install -g openclaw
openclaw gateway start

# 或使用 Docker
docker run -d openclaw/gateway
```

### 2. 云服务器部署

支持阿里云、腾讯云、AWS 等主流云平台：

```bash
# 一键安装脚本
curl -fsSL https://get.openclaw.ai | bash
```

### 3. 阿里云镜像市场

阿里云轻量应用服务器提供 OpenClaw 应用镜像，可一键部署。

## 总结

### 优势

1. **隐私优先**：本地部署，数据完全自主
2. **高度可定制**：开源架构，可任意扩展
3. **多渠道覆盖**：支持 10+ 主流通讯平台
4. **主动执行**：不只是对话，能真正完成任务
5. **社区活跃**：快速迭代，生态丰富

### 挑战

1. **部署门槛**：需要一定的技术能力
2. **模型成本**：需自备 LLM API Key
3. **稳定性**：依赖第三方 API，可能有波动

### 适用人群

- ✅ 注重隐私的个人用户
- ✅ 有技术背景的开发者
- ✅ 需要定制化 AI 的企业
- ❌ 追求开箱即用的普通用户

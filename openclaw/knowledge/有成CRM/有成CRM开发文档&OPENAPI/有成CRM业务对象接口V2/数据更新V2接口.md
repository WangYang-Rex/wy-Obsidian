# 数据更新V2接口

> **原文**：[数据更新V2接口 - 有成CRM官网帮助中心](https://www.ycbg.com/help-center.html?tab=1&id=641)  
> **导出时间**：2026-06-23

接口说明：更新CRM业务对象数据

**请求说明**

请求方式：post

请求参数数据类型：application/json

请求路径：/oapi/corp/v2/data/update.json

**请求参数说明：**

| 参数 | 参数类型 | 参数必填 | 说明 |
|----|----|----|----|
| corpId | String | 是 | 企业ID |
| corpAccessToken | String | 是 | 访问企业数据合法性票据 |
| deployId | String | 是 | 业务对象deployId |
| userId | String | 是 | 用户ID，执行更新操作的用户 |
| dataId | Long | 是 | 数据唯一ID，要更新的数据主键 |
| templateId | String | 否 | 业务对象模板ID |
| incrUpd | Boolean | 否 | 是否增量更新，默认false。true时仅更新传入的字段，不覆盖未传入的字段 |
| needSubmit | Boolean | 否 | 是否提交审批，默认false |
| data | Map\<String,Object\> | 是 | 主表单数据集合，key为字段名，value为字段值 |
| relationDatas | Map\<String,List\<Map\>\> | 否 | 子表单数据集合，key为子表单deployId，value为数据列表 |
| contractId | Long | 否 | 关联合同订单ID |
| saleOrderId | Long | 否 | 关联合同订单ID |
| quotationId | Long | 否 | 关联报价单ID |

**增量更新逻辑说明：**

当 incrUpd=true 时，系统将按以下规则进行增量更新：

**1. 主表单数据增量更新规则：**

| 场景             | 处理逻辑                                       |
|------------------|------------------------------------------------|
| 传入字段有值     | 仅更新传入的字段，其他未传入字段保持原有值不变 |
| 传入字段值为null | 将该字段清空（设置为null或默认值）             |
| 不传入某字段     | 该字段保持原有值不变（不做任何处理）           |

**2. 子表单数据增量更新规则：**

| 子表单数据行id值       | 处理逻辑                                 |
|------------------------|------------------------------------------|
| id为null或空           | 视为新增操作，新增一条子表单数据         |
| id有值（如id=1）       | 视为更新操作，更新该id对应的子表单数据行 |
| 原有数据行不传入       | 视为删除操作，删除该id对应的子表单数据行 |
| 子表单列表为空数组\[\] | 删除该子表单的所有数据行                 |
| 不传relationDatas参数  | 保持所有子表单数据不变（不做任何处理）   |

**响应结果说明：**

| 参数    | 参数类型 | 参数必填 | 说明         |
|---------|----------|----------|--------------|
| success | Boolean  | 是       | 请求成功状态 |
| data    | Object   | 是       | 更新结果对象 |
| code    | String   | 是       | 返回码       |
| msg     | String   | 是       | 返回码描述   |

**响应结果结构示例：**

a.正常结果

```json
{
    "success": true,
    "code": "0",
    "msg": "success",
    "data": {}
}
```

b.错误结果

```json
{
    "success": false,
    "code": "12002",
    "msg": "corpAccessToken不存在或已过期"
}
```

**注意事项：**

1\. incrUpd=false（默认）时，会使用编辑表单字段列表进行全量更新，未传入的编辑表单字段会被清空或设为默认值。

2\. incrUpd=true时，仅更新传入的字段，未传入的字段保持原有值不变；传入null值的字段会被清空。

3\. incrUpd=true时，子表单数据支持增量操作：id为空视为新增、id有值视为更新、不传入的数据行视为删除。

4\. incrUpd=true时，若不传relationDatas参数，则所有子表单数据保持不变。

5\. incrUpd=false时，子表单数据会整体替换（非增量），传入的数据完全覆盖原有数据。

6\. 该接口有请求频率限制，每分钟最多60次。

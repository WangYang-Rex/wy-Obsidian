# 数据新增V2接口

> **原文**：[数据新增V2接口 - 有成CRM官网帮助中心](https://www.ycbg.com/help-center.html?tab=1&id=641)  
> **导出时间**：2026-06-23

接口说明：新增CRM业务对象数据

**请求说明**

请求方式：post

请求参数数据类型：application/json

请求路径：/oapi/corp/v2/data/insert.json

**请求参数说明：**

| 参数 | 参数类型 | 参数必填 | 说明 |
|----|----|----|----|
| corpId | String | 是 | 企业ID |
| corpAccessToken | String | 是 | 访问企业数据合法性票据 |
| deployId | String | 是 | 业务对象deployId |
| userId | String | 是 | 用户ID，新增数据的创建人 |
| templateId | String | 否 | 业务对象模板ID |
| needSubmit | Boolean | 否 | 是否提交审批，默认false |
| data | Map\<String,Object\> | 是 | 主表单数据集合，key为字段名，value为字段值 |
| relationDatas | Map\<String,List\<Map\>\> | 否 | 子表单数据集合，key为子表单deployId，value为数据列表 |
| contractId | Long | 否 | 关联合同订单ID（仅订单相关对象使用） |
| saleOrderId | Long | 否 | 关联合同订单ID（仅订单相关对象使用） |
| quotationId | Long | 否 | 关联报价单ID（仅报价单相关对象使用） |

**响应结果说明：**

| 参数    | 参数类型 | 参数必填 | 说明             |
|---------|----------|----------|------------------|
| success | Boolean  | 是       | 请求成功状态     |
| data    | Object   | 是       | 新增结果对象     |
| data.id | Long     | 是       | 新增数据的主键ID |
| code    | String   | 是       | 返回码           |
| msg     | String   | 是       | 返回码描述       |

**响应结果结构示例：**

a.正常结果

```json
{
    "success": true,
    "code": "0",
    "msg": "success",
    "data": {
        "id": 1155
    }
}
```

b.错误结果

```json
{
    "success": false,
    "code": "12002",
    "msg": "corpAccessToken不存在或已过期"
}
```

**注意事项：**

1\. data中的字段名需使用字段的英文名称（fieldName），对于选项类字段建议使用value值（如custom_level_value）。

2\. relationDatas中的key为子表单的deployId，value为子表单数据列表。

3\. 部分业务对象有必填字段校验，请确保传入必要的字段值。

4\. 该接口有请求频率限制，每分钟最多60次。

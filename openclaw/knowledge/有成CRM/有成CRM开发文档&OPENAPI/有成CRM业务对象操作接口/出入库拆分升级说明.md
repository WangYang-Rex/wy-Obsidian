# 出入库拆分升级说明

> **原文**：[出入库拆分升级说明 - 有成CRM官网帮助中心](https://www.ycbg.com/help-center.html?tab=1&id=641)  
> **导出时间**：2026-06-23

**1、背景**

之前系统只有入库单、入库单产品明细、出库单、出库单产品明细四个业务对象，其中每个业务对象根据其字段属性又分为不同的业务单据，随着业务日渐复杂，需要将这不同的业务单据拆分成独立的业务对象，那么对应开放接口的入参和出参就会有调整。

**2、涉及到的接口**

2.1、开放业务对象数据列表接口：/oapi/corp/v1/data/list.json

2.2、开放业务对象数据详情接口：/oapi/corp/v1/data/detail.json

2.3、开放业务对象字段列表接口：/oapi/corp/v1/object/field/list.json

**3、调整**

3.1、之前请求某个业务单据的数据，需要 deployId + stockType 确定，现在只需要 deployId 便可以确定。

举个列子，采购入库单之前的入参：

```json
{
"deployId":"stock_in",
"conditionConfig": [
{
"conditions": [
{
"comparison": "IN_ARRAY",
"deployId": "stock_in",
"fieldName": "stock_in_type",
"fieldTitle": "入库类型",
"fieldValue": "PURCHASE_ORDER",
"domType": "SELECT"
}
]
}
]
}
```

现在的入参：

```json
{
"deployId": "stock_in_po"
}
```

3.2、拆分对象的前后参数对比

|  | 之前的入参 | 现在的入参 |

| 其他入库单产品明细 | deployId = stock_in_product_particular stockInType = OTHER_IN | deployId = stock_in_other_dtl |

# 采购单明细中关联申请明细优化

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2023-11-21 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

采购单明细中，要配置数据联动，带入采购申请明细中字段，目前不支持，原因是：

采购单明细中，源单明细行编码目前只关联了合同订单产品的‘明细行编号’

（原先考虑采购申请产品不能重复，回刷用产品明细匹配即可，故不用存储明细行编号）

## 2、需求详情

| **需求** | **描述** |
|----------|----------|
| 采购需求池 | 字段‘明细行编码’字段取值优化：针对来源于采购申请明细的数据，取该对象的\{明细编号\}备注：历史数据在上线时需要按照上述逻辑刷上 |
| 采购单 | 一、采购单产品\+ 源单类型：新增字段，单选（枚举‘合同订单产品’‘采购申请产品’），非必填，可隐藏，名称不可更改备注：字段属性配置不在页面上展示\+ 源单明细行编码：改为指定关联对象，关联的对象为‘合同订单产品’‘采购申请产品’备注：历史数据处理- 源单类型\* \{来源类型\}=‘合同订单’明细行，刷‘合同订单产品’\* \{来源类型\}=‘采购申请’明细行，刷‘采购申请产品’- 源单明细行编码\* 针对\{来源类型\}=‘采购申请’的明细行，结合对应的\{需求来源单据\}\{产品明细\}匹配查找，刷\{源单明细行编码\}值二、回刷采购申请、采购需求池的采购数量、状态逻辑优化之前：根据\{来源类型\}\{需求来源单据\}\{产品明细\}匹配回刷优化：根据\{源单明细行编码\}\{源单类型\}匹配回刷 |

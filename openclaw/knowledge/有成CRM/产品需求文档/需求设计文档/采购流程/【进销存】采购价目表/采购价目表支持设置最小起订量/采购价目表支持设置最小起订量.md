# 采购价目表支持设置最小起订量

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 20220525 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

1、针对固定供应商，存在合同中约定，某些产品的最小起订量，低于该量则不允许下单的情况；

2、价目表支持录入税后价，反算出税前价；

## 2、业务流程/架构

无。

## 3、功能清单

| **模块** | **二级模块** | **功能点** |
|----------|----------------|-------------|
| 采购流程 | 采购价目表 | 1、支持设置最小起订量2、支持录入税后价，反算出税前价 |
|  | 采购单 | 采购数量低于最小起订量，则不允许下单 |

## 4、需求详情

### 4.1、需求说明（pc端）

#### 4.1.1 采购价目表

字段需求

| **字段名称** | **变更类型** | **字段属性** | **字段类型** | **是否隐藏** | **是否必填** | **字段名称是否可改** | **用户是否可设置默认值** | **新建编辑页是否可见** | **备注** |
|----------------|----------------|----------------|----------------|----------------|----------------|----------------------------|----------------------------------|-------------------------------|----------|
| 含税单价 |  |  |  |  |  |  |  |  | 用户可编辑该字段值，编辑后反算采购价格采购价格=含税单价/（1\+专项税率) |
| 是否最小起订量 | 新增字段 | 系统字段 | 单选 | 不可隐藏 | 必填（不可更改） | 否 | 1、默认为‘否’2、不支持编辑默认值；3、支持选择默认值； | 是 | 1、枚举值：是/否2、功能上线时，历史单据该字段统一刷成‘否’ |

其他需求：

| **需求** | **描述** |
|----------|----------|
| 新建编辑页—增加字段‘是否最小起订量’说明 | 说明内容如下：为‘是’，则表明该行‘采购量从’的数量，为最小起订量，下达采购单时若低于该数量则会进行提示或者无法提交； |
| 新建/编辑页：保存/提交单据增加校验 | 在已有的校验基础上，增加如下校验：本单据中，若存在同一产品明细有\>1行\{是否最小起订量\}=‘是’，则toast提示‘\{产品名称\}（\{系统编码\}）不能重复设置最小起订量’ |

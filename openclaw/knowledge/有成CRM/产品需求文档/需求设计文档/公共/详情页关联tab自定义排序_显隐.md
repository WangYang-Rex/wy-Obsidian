# 详情页关联tab自定义排序/显隐

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2023-06-13 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

不同用户对详情页关联tab显示及排序诉求不同，目前固定展示逻辑灵活性过低，需支持用户自定义，从而增强系统的业务自定义能力

## 2、需求详情



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4jKqm0bvw3P4Nnw1/img/ad718c21-2f89-46a6-b359-8f75fab55e1e.png?Expires=1774937649&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=V4bFEt%2BzViQGhDdune4C2kbSXIA%3D "")

| **需求** | **说明** |
|----------|----------|
| 增加tab页自定义设置 | 一、支持设置的对象范围：所有对象二、详情菜单设置页\+ 说明文案打开默认页：即打开数据详情，默认展示的页面；当默认页因功能被关闭或者其他原因被隐藏时，默认打开‘详情’生效范围：该设置对本人\+本对象生效，即不影响他人的详情菜单，且所有模板均生效。\+ tab项：该对象所有关联tab（要结合企业版本\+应用开关（关闭的不显示）），无须判断其他）- ~~子表单：展示~~~~‘\{子表单字段名称-不带模板\}（\{子表单字段名称-当前模板\}）’~（若当前模板中设置隐藏了本子表单，则只展示不带模板的字段名称），举例该子表单在模板1中名称为‘合同产品’，模板2中隐藏了该子表单，则：~\* ~~在模板1所属数据详情中设置时展示：合同订单产品（合同产品）~~\* ~~在模板1所属数据详情中设置时展示：合同订单产品~~- 关联对象：考虑到重复关联同一对象的情况，例如跟进记录中，增加了n个\{关联对象\}='客户'字段，则在客户下，会有n个跟进记录tab，会导致无法预知排序结果的情况，故展示为‘\{对象类型名称\}（\{该对象在下游对象中列表字段名\}）’，举例跟进记录中增加了2个关联客户字段，不带模板的字段名分别为‘客户1’‘客户2’，则展示的两个tab名分别为\* 跟进记录（客户1）\* 跟进记录（客户2）\+ 支持操作- 调整顺序- tab显隐\* 详情tab不支持隐藏（【显隐设置】按钮隐藏）\* 默认打开页不支持隐藏（【显隐设置】按钮隐藏）\* 历史tab、新增tab均默认‘显示’，排序按照现有逻辑）- 设置打开默认页（打开详情默认定位到的tab页面）\* 已设置的默认打开页tab，【设为默认】按钮隐藏\* 所有对象，所有用户，默认打开页均为‘详情’页\+ 处理逻辑- 须记住该企业\+该用户\+该对象的上述设置（设置是针对对象，所有模板均生效）- tab 最终显隐：取用户、该单据最新可见范围内所有关联tab（结合菜单权限、单据审批状态、是否共享、是否开启审批、单据模板） 和 设置页中显示的tab项的交集- 设置的默认打开页，经判断，该用户最终不可见，则默认打开‘详情页’- 版本开通/菜单开启：新增加的模块，默认排序在最后- 移动端：有增加‘协作人’tab，此tab在pc端没有，故在设置列表中不展示，在移动端展示的逻辑为 显隐及整体排序同‘详情’（固定排在‘详情’tab 后） |

## 1、需求背景

录入自定义对象\+自定义子表单数据后，支持打印。

## 2、业务流程/架构

说明：业务中的关联对象，为该对象的上游对象，如合同订单中有关联对象‘客户’，那么合同订单为客户的下游对象。

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 业务模板 | 针对业务对象、业务模板操作可触发打印设置变更 |  |
| 菜单设置 | 调整菜单顺序，打印设置同步调整 |  |
| 打印模版 | 支持设置自定义对象\+自定义子表单 |  |
| 详情&详情页 | 支持单个打印、批量打印 |  |
| 角色设置 | 支持设置打印权限 |  |

## 4、需求详情

### 4.1 设置-业务模板

| **模块** | **说明** |
|----------|----------|
| 新建/编辑/删除对象 | 一、新建对象结果：接入打印设置（无系统预设打印模板）二、编辑对象结果：对应业务对象名称变更（包含模板归属业务对象名称、模板设置中对应对象名称）三、删除对象结果：1）在打印设置中删除；2）删除该对象下所有设置好的打印模板； |
| 业务模板 | 一、编辑业务模板1\.  若有新增\{类型\}=‘关联对象’的字段（含在子对象中新增）结果：打印模板中，本对象及子表单对象中增加关联对象信息（如其他入库单中增加了关联对象‘客户’，则在打印模板中，可以插入客户关联的客户信息）   2\. 若有新增\{类型\}=‘子表单’结果：打印模板中，增加子表单对象；二、删除\{类型\}=‘关联对象’的字段（含在子对象中删除）结果：打印模板中，本对象及子表单对象中清除关联对象信息（已设置好的打印模板，也要将关联对象的字段删除）三、删除\{类型\}=‘子表单’的字段结果：打印模板中，删除子表单对象信息（已设置好的打印模板，也要将子表单相关的动态表单删除）四、编辑子表单的名称结果：对应业务对象名称变更（模板设置中对应对象名称） |

### 4.3设置-打印模版

| **模块** | **说明** |

| 打印设置列表 | 一、功能上线时，所有业务对象均接入打印，打印设置列表取的是主业务对象，新接   入打印设置的模块无默认打印模版   备注：1\. 以下模块，不是业务对象，它们有字段配置，但没有按业务对象去实现，正常是有DB数据，可能没有solr/es数据，也没有pg数据1. 期初应收款 sale\_return\_order\_product2. 应收款明细 sale\_order\_opening\_payment3. 期初应付款 purchase\_opening\_payment4. 应付款明细 purchase\_due\_payme5. 实际付款明细 purchase\_real\_payment\_detail （还未对实际付款铺开，后续可以作为实际付款的子表单，但是本期就要支持打印--目前线上就可以）6. 实际回款明细 real\_payment\_detail （还未对实际回款铺开，后续可以作为实际回款的子表单，但是本期就要支持打印--目前线上就可以）7. 日报/周报/月报 work\_daily/work\_weekly/work\_monthly8. 报销/报销费用 reimburse/reimburse\_cost9. 库存查询 inventory10. 期初库存 inventory\_opening11. 期初存货 inventory\_account12. 批次保质期查询 inventory\_expiration13. 库存预警设置 inventory\_warning14. 库存预警记录 inventory\_warning\_record15. 出入库记录 stock\_record2\. 产品-产品明细：产品明细是产品的子业务对象，但是在打印设置中，产品不需要支持打印，产品明细要作为主对象支持打印（按照目前线上的逻辑） |
| 新建/编辑打印模版（针对所有对象） | 展示顺序：全局变量----\>本对象----\>子表单----\>下游对象-----\>审批流程一、支持插入全局变量二、支持插入本对象、本对象关联对象字段（标题展示关联对象在主对象中字段名）1\. 当关联对象为单选时：本对象-关联多个相同关联对象的情况（如合同订单中有多个关联客户字段：客户1，客户2），此时在打印中标题应为‘关联对象在主对象中字段名’，如下示意图2\. 当关联对象为多选时，对应关联对象只能插入动态表格中，另同样要考虑本对象-关联多个相同关联对象的情况1. 标题应为‘关联对象（主对象.关联对象在主对象中字段名）’举例合同订单关联客户为多选，且若支持多次关联客户，那么在合同订单打印模板设置时，可以插入动态表格：客户.（合同订单.客户1）客户.（合同订单.客户2）...客户.（合同订单.客户N）b.目前系统中存在多选的对象包括1. 销售机会：关联联系人（客户）2. 销售出库单：关联合同订单3. 开票申请：关联合同订单、计划回款、实际回款4. 采购发票：关联采购单、计划付款、实际付款三、支持插入本对象下游对象字段（查找主对象中，关联的对象包含本对象的）~~1、当下游对象的关联对象支持多选时，不支持在打印模板中插入该关联对象字段~~2、存在多个关联对象关联同一对象的情况（如合同订单中有多个关联客户字段：客户1，客户2），此时在打印中标题应为‘下游对象.主对象在下游对象中字段名’，如下示意图四、支持插入子表单、子表单关联对象字段备注：当子表单关联对象支持多选时，不支持在打印模板中插入该关联对象字段五、支持插入审批流程 |

## 1、需求背景

很多客户都提出需要二次编辑关联对象，如开票申请支持补录实际回款，联系人支持变更关联客户等，本期做对单据编辑时关联字段限制统一优化处理。

## 2、业务流程/架构

## 3、需求详情

### 4.1、需求说明1

针对关联对象字段，是否支持补全、是否支持变更、是否支持清空，实现属性可配置化（不开放用户）

1、支持为空时补全：关联对象字段原值为空，支持编辑填写

2、支持变更：关联对象字段原值不为空，支持拜编辑修改

3、支持清空：关联对象字段原值不为空，支持情况值

| **对象-字段** | **支持为空时补全** | **支持变更** | **支持清空** |
|-----------------|--------------------------------------------------------------|-----------------------------------------------------|----------------|
| 联系人-客户 | 是 | 是 | 是 |
| 联系人-项目 | 是 | 是 | 是 |
| 销售机会-客户 | 是 | 是 | 是 |
| 开票申请-实际回款 | 是 | 否 | 否 |
| 合同订单-客户 | 是 | 否 | 否 |
| 协议-客户 | 是 | 否 | 否 |
| 项目-客户 | 是 | 是 | 是 |
| 开票申请-关联回款类型 | 是 | 否 | 否 |
| 开票申请-关联回款 | 是 | 否 | 否 |
| 其他系统关联对象字段 | 是 | 否 | 否 |
| 自定义关联对象 | 是 | 是 | 是 |

# 字段扫码录入

| **修订时间** | **版本** | **修订说明** |

| 2022-05-17 | v1.0 | 新建 |
|  |  |  |

## 1、需求背景

1、一些售卖设备的企业，在录入合同订单等单据时候，往往需要同步录入产品的设备号。如一家卖电瓶车电池的企业，录入订单时，需要扫描电池的条形码，录入电池的设备号。

2、在维护产品明细信息的时候，需要扫码录入产品的条形码（PC端可以直接连接扫码枪录入，无需开发；MB端等后续开放产品明细新增、编辑入口后再实现此场景，本期不考虑）

## 2、业务流程/架构

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 业务模板 | 支持设置字段是否支持扫码填写 |  |
| 移动端新建、编辑页面 | 移动端新建、编辑页面，对应字段支持扫码录入 |  |

## 4、需求详情

### 4.1、业务模板设置
1. 字段类型为：单行文本、多行文本。字段设置处新增“录入设置”：手动录入、扫码录入、手动录入\+扫码录入
2. 线索-姓名、客户-客户名称字段不支持设置扫码录入
3. 默认为“手动录入”
4. 历史企业也统一刷为“手动录入”

### 4.2、移动端新建、编辑页面。根据业务模板的设置不同，展示不同的样式
5. 手动录入：和现有一致
6. 扫码录入：（1）输入框后面，展示“扫码”入口，点击后打开摄像头扫码。扫码识别后返回并输入字段值。如下图（2）此时不支持手动录入，点击输入框，或者点击“扫码”后都跳转到扫码页面

（2）同一个字段，连续多次发起扫码。每次扫码之后，覆盖原来的字段值。如下图

![Picture 3](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45Kb8KooyqDQ/img/b5f1d2a8-af35-4c86-befb-bad8024bc66e.png?Expires=1774937933&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=JV3rHMab3K9PilaWY4TpaRepp4k%3D "")
7. 手动录入\+扫码录入：（1）此时支持手动录入、扫码录入。点击输入框，则手动录入；点击扫码，则进入扫码录入

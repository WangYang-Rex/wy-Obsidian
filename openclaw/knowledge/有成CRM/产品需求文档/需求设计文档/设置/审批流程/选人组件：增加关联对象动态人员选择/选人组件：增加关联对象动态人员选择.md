# 选人组件：增加关联对象动态人员选择

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|

|  |  |  |  |

## 1、需求背景

历史选人组件需求：[<u>https://tb.raycloud.com/task/6107599974c0072a0c926ea5</u>](https://tb.raycloud.com/task/6107599974c0072a0c926ea5)

场景：
1. 审批扩展：例如可以设置项目-人员映射关系，并在审批流程条件分支里面使用，如合同订单，当合同所属项目1负责人为a，那么应该给a审批；
2. 消息：任务结束后，需要给主流程的相关人员发送消息；

## 2、需求详情

| **需求** | **描述** |
|----------|----------|
| 选人组件扩展 | 一、动态人员优化1\. 原动态人员：根目录变更为 动态人员-\{本对象名称\}- 阶段推进器：展示的是 该阶段推进器 流程发起对象名；- 审批/自定义消息（还未上线）：展示的是审批/消息对应业务对象名；2\. 新增本单据关联对象动态人员1. 关联对象：取本业务对象中，所有 \{字段类型\}=‘关联对象’ （支持多选关联对象、指定关联对象、相同对象），字段名取字段列表中\{字段名称\}1. 针对指定关联对象，展示\{字段名称\}（\{指定关联对象名\}），举例在开票申请中，关联回款是指定对象，指定了计划回款、实际回款，那么就展示：关联回款（计划回款）关联回款（实际回款）2. 针对关联多选，须取查找到的多个关联对象中人员信息，举例：销售出库单审批流程节点，设置审批人为关联合同的负责人，当关联合同为多选时，负责人分别为员工A/员工B,那么此时该节点的审批人取员工A和员工B2. 人员：取关联对象中，关联选择人员组件字段；三、选中后展示优化\+ 展示为 ‘\{关联对象\}  \{人员\}  \{层级\}’，历史已保存配置信息按新的样式展示四、关联对象被删除：参考历史处理方式五、处理进行以下处理时，增加关联查找本对象关联对象的人员及上级：\+ 审批取审批及抄送人\+ 消息取消息接收人/无须提醒人\+ 任务生成取负责人/协作人\+ 工单模板：工单流程节点选择受理人 |

# 数据共享支持数据范围设置

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2022-11-29 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

'员工A' 在两个子公司任职，直属上级分别是'领导A'、'领导B'，针对员工在系统中建的不同的数据，需要区分是归属于A/B哪家公司的，且上级只能看到该员工为各自公司建的数据。

## 2、需求详情

### 2.1、需求说明

| **需求** | **描述** |
|----------|----------|
| 数据共享优化 | 一、增加数据范围设置：同大列表的过滤筛选（不区分用户，可选字段包含该对象下所有字段---去除附件、图片类型字段）备注：1、中文案：共享对象为空或者选择了多个，不支持设置数据范围。2、共享对象为空/多个、对象变更：数据范围设置清空，且不允许用户编辑（禁用效果）3、过滤条件：最多可添加6个条件（不区分or and，真实最多可添加15个），在添加第7个条件时（点击“\+” 添加单个筛选条件 or '添加筛选器'），toast提示‘最多只能添加6个条件’二、以下对象接入数据共享采购需求池、应付款明细、计划付款、实际付款、计划回款、实际回款、应收款明细、开票申请、采购发票、期初应付款、期初应收款三、列表优化1、增加展示数据范围设置：只展示是全部/部分（不用展示过滤条件设置）2、字段顺序：数据来源、共享对象、共享范围、共享权限、数据范围、状态 |
| 采购需求池增加数据权限 | 系统默认场景：1、已有-全部2、我负责的3、我协作的4、本部门的5、本部门及下属部门的6、已有-待采购：权限范围下\{采购状态\}不为‘全部采购’的明细行7、已有-已采购：权限范围下\{采购状态\}为‘全部采购’的明细行8、已有-合同订单来源：权限范围下\{来源类型\}=‘合同订单’的明细行9、已有-采购申请来源：权限范围下\{来源类型\}=‘采购申请’的明细行10、共享给我的 |
| 以下对象增加核算维度字段（包括新开通的企业） | 一、字段需求字段属性：系统字段字段名称：核算维度（可更改）是否必填/隐藏：非必填（可更改）、隐藏（可更改）字段类型：关联对象（核算维度）新建编辑页可见：是数据范围：无默认值：自定义二、需要加上该字段的模块：线索、客户、跟进记录、拜访签到、~~bom单、供应商、联系人（供应商）、报销（暂不做）~~ |
| 版本改动 | 单独进销存版本：包含数据共享模块备注：历史进销存独立版客户，刷上数据共享模块 |

## 1、需求背景

客户陕西骏景，工程类项目的回款需要走内部审批流程

## 2、业务流程/架构

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 审批流 | 计划回款支持审批流 |  |
|  |  |  |

## 4、需求详情

### 4.1、审批流设置
1. 审批流程设置-对象列表，增加【计划回款】，排序同菜单排序
2. 审批摘要字段默认值：客户名称、期数、计划回款金额


![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/Pd6l2Z7Qg2a6yl7M/img/0f873897-aaff-41f0-96fe-77c10e162e90.png?Expires=1774165263&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=B38yUU5qSpyTclSmGs83EFAftVc%3D "")
3. 审批流默认关闭
4. 审批流通用设置功能全部接入支持（包含新建触发审批、编辑触发审批等）

### 4.2、计划回款业务模板

1、增加系统字段【审批状态】，字段位置在「负责人」前面

| **是否隐藏** | **属性** | **类型** | **新建/编辑是否隐藏** |
|----------------|----------|----------|-----------------------------|
| 否 | 其他字段 | 单选 | 是 |

2、历史数据
- 新老用户都增加，历史的单据的审批状态统一订正为「已通过」

### 4.3、审批列表及详情


![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/Pd6l2Z7Qg2a6yl7M/img/3dc6f345-8d0f-4448-9e00-14956e600c00.png?Expires=1774165263&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=hMD%2BXIxvy4i%2BLkIsMGVt1Jr4q5o%3D "")

| **模块** | **说明** |
|----------|----------|
| 审批列表 | 1\. 审批类型筛选项，同步增加【计划回款】，位置与设置页面一致2\. 列表字段默认增加【审批状态】 |
| 审批单 | 1\. 标题字段：取计划回款的「合同订单编号」2\. 审批详情关联列表：\+ 详情：计划回款单详情\+ 审批记录\+ 审批详情 |

### 4.3、关联业务逻辑说明

| **模块** | **说明** |
|----------|----------|
| 计划回款详情 | 审批中状态，隐藏【编辑】【删除】，只展示【审批详情】入口 |
| 计划回款编辑 | 提交时统一增加后端校验，如处于审批中，则toast提示：该单据处于“审批中”状态，提交失败 |
| 计划回款关联子列表 | 入口：合同订单等全部上游对象详情页，关联tab列表1\. “审批中”、“审批拒绝”、“待提交”、“已撤回”状态，计划回款显示「审批状态」标签2\. “审批中”、“审批拒绝”、“待提交”、“已撤回”状态，隐藏【添加回款记录】入口3\. “审批中”状态，隐藏计划回款【编辑】【删除】入口 |
| 关联对象选择列表 | 场景：比如实际回款，会在模板中配置**关联计划回款**字段（系统字段），此时计划回款的关联对象选择列表，只展示【审批通过】状态的单据数据注意如是自定义的关联对象字段，则还是按照关联对象设置的过滤条件为准 |

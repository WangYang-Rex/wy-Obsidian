# 任务动态负责人等优化

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20220523 | v1.0 | 新建 |
|  |  |  |

## 1、需求背景
1. 固定任务分配规则，往往是需要自动分配给对应项目负责人、某个角色等，而目前只能分配给指定的人，不满足业务需求
2. 任务很容易误操作完成，但目前无法再修改

## 2、业务流程/架构

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 任务设置 | 任务负责人支持动态人员 |  |
| 任务详情 | 任务重启 |  |
| 文案 | 任务状态未完成文案优化 |  |
| 日志 | 增加行为日志 |  |

## 4、需求详情

### 4.1、负责人支持动态人员



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeobgXva0Wlpj/img/1c661009-ac11-4159-81aa-bc8e1e173a6d.png?Expires=1774937197&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=kBRxS7PnO%2FSdM2caQO9IRWZxgQk%3D "")

| **模块** | **说明** |
|----------|----------|
| 任务负责人设置 | 增加支持动态人员选择，可选择【负责人】【创建人】，对应项目单据的人员。只能单选 |
| 特殊说明 | 设置动态人员，任务生成后，如对应项目负责人等发生变更，则任务状态为「未开始」或「进行中」的任务，系统将同步变更任务负责人为最新人员\+ 如原任务配置被删除，则不进行变更\+ 如原任务配置被修改为其他动态人员，会更新最新配置人员 |

### 4.2、任务支持重启



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeobgXva0Wlpj/img/4605d32d-0bfa-4be6-a58e-a4951cd717a8.png?Expires=1774937197&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=eqyXxfZjEotHmka%2B7zwV8QmQgHI%3D "")

| **模块** | **说明** |
|----------|----------|
| 任务详情（pc\+mb） | 1\. 任务状态为「已完成」且审批状态为「已通过」的任务，详情页展示【重启任务】操作按钮2\. 按钮权限：仅任务负责人、crm管理员可见3\. 点击后，将任务状态变更为「进行中」 |
| 消息提醒 | 任务重启后，系统即时发送消息推送通知1、文案：**任务重启**\{操作人\}重启了任务：\{任务名称\}，请知悉消息跳转：任务详情页2、推送人：任务协作人、任务负责人 |

### 4.3、任务状态优化

任务状态“未完成”文案改为“未开始”。历史数据统一变更

### 4.4、任务日志补充

完成任务、重启任务增加日志记录

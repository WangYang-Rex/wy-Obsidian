| 企业授权 | 一、交互1、未勾选同意开通协议，【限时免费开通】按钮不可操作（置灰）2、点击任何页面返回至有成CRM对应入口页；二、完成企业授权 且 该企业有第一个授权的用户后1、为开通的企业，创建1个小程序\+ 套餐：2023-9-30之前开通，默认为高级版 1年；之后开通，默认为标准版 永久免费2、为开通的企业，创建1张地图\+ 地图名称默认为‘企业客户地图’3、为开通的企业，创建2个文件夹\+ 文件夹名称分别为‘客户’、‘公海’ |
| 个人授权 | 一、交互1、未勾选同意开通协议，【立即使用】按钮不可操作（置灰）2、绑定手机号页面\+ 手机号、验证码存在未填写的，【立即绑定】按钮不可操作（置灰）\+ 【获取短信验证码】时，须进行手机号合法性校验，不满足条件的则在操作后toast提示‘发送短信失败，请检查您的手机号是否正确’\+ 手机号和验证码不匹配的，【立即绑定】时toast提示‘验证码与手机不匹配’二、员工进行个人授权成功后1、人员处理\+ 将授权人员加入小程序成员（第一个授权用户为小程序超级管理员）\+ 将授权人员加入地图成员（第一个授权用户为地图超级管理员）- 若该用户在有成crm中的数据权限为‘全公司’，则该成员在2个文件夹中，权限为‘可管理’；其余用户，2个文件夹权限为‘无权限’2、数据同步处理（将有成客户表中数据---\>地图小程序点位）\+ 数据范围- \{作废状态\}≠‘已通过’ and \{审批状态\}=‘已通过’ and 地区定位有值（有经纬度）\+ 数据处理- 根据数据在有成中对客户/公海的分类，对应同步到地图中的两个文件夹- 创建企业地图小程序时，同时将企业客户/公海的数据全量同步至小程序；- 地图中，点位的负责人处理\* 有成公海数据：公海中成员/管理员包含此用户的，对应点位负责人增加该用户\* 有成客户中数据：我负责的\+我协作的\+部门权限下可见的（若有部门权限）点位负责人均加上该用户\+ 异常情况- 点位超出版本限制，超出限制的数据同步失败，并在有成侧记录失败原因‘点位超出版本限制’3、运营后台系统表中将用户填写的手机号存储 |
| 地图小程序 | 1、筛选器：文件夹、负责人、状态2、点位详情展示：点位名称、距当前位置距离、点位地址、负责人、描述 |

### 4.2 其他操作产生的影响（限高德地图小程序使用中时须处理）

备注：红字部分，为若不考虑部门权限人员，则可不用处理（即部门权限人员也只能看自己负责的数据）

| **模块** | **操作/任务** | **处理** | **备注** |
|----------|-----------------|----------|----------|
| 客户/公海 | 新增/导入 | 新增同步至地图，点位负责人处理：\+ 公海：将公海中成员/管理员添加至对应点位负责人\+ 客户：将该数据在有成的负责人、协作人、有该部门权限的人添加至对应点位负责人 | 1、同步条件：其他条件：小程序未到期\+点位未超出限制2、数据过滤：\{作废状态\}≠‘已通过’ and \{审批状态\}=‘已通过’ and 地区定位有值（有经纬度） |
|  | 编辑/公海客户编辑/覆盖导入（含批量操作） | 更新点位信息（根据客户ID匹配，后续同） |  |
|  | 删除/作废（含批量操作） | 删除点位信息 |  |
|  | 还原 | 同公海新增数据处理 |  |
|  | 领取/分配（含批量操作） | 1、点位文件夹变更为：客户2、点位负责人变更为：将该数据在有成的负责人、协作人、有该部门权限的人更新为对应点位负责人 |  |
|  | 更改公海（含批量操作） | 点位负责人变更为：将公海中成员/管理员更新为对应点位负责人 |  |
|  | 回收/自动回收/退回 | 1、点位文件夹变更为：公海2、点位负责人变更为：将公海中成员/管理员更行至对应点位负责人 |  |
|  | 客户合并 | 1、删除被合并的点位信息；2、留存点位负责人变更为：将该数据在有成的负责人、协作人、有该部门权限的人更新为对应点位负责人 |  |
|  | 添加协作人/移除协作人（含批量操作） | 点位负责人变更：将该数据在有成的负责人、协作人、有该部门权限的人更新为对应点位负责人 |  |
|  | 变更负责人（含批量操作） | 点位负责人变更：将该数据在有成的负责人、协作人、有该部门权限的人更新为对应点位负责人 |  |
| 部门员工 | 组织架构同步/关闭自定义组织架构 | 1）将点位负责人全部移除；2）再按逐个人员授权点位负责人处理； |  |
|  | 人员变更部门 | 点位负责人变更：1、将该人员在点位负责人中全部移除；2、再按人员授权的点位负责人处理； |  |
|  | 人员变更角色 | 1、人员处理：按变更后最大权限更新成员文件夹权限2、点位负责人变更：1）将该人员在点位负责人中全部移除；2）再按人员授权的点位负责人处理； |  |
|  | 离职/取消授权 | 1、对应人员为小程序超管：将超管转让为企业CRM管理员中任一，再将该人员从‘企业客户地图’地图成员中移除；2、不是小程序超管：从‘企业客户地图’地图成员中移除； | 离职前必须做数据交接但是协作人不会被交接 |
|  | 数据交接 | 处理同批量变更负责人 |  |
|  | 在职/授权 | 若已是小程序成员：处理同人员授权 |  |
|  | ~~删除部门~~ |  | 有人员在的部门不能删除 |
| 角色设置 | 数据权限变更 | 人员处理：按变更后最大权限更新成员文件夹权限点位负责人变更：1、将该人员在点位负责人中全部移除；2、再按人员授权的点位负责人处理； |  |
| 设置-线索/公海 | 添加成员/添加管理员 | 将成员/管理员添加至公海对应点位负责人 |  |
|  | 移除成员/移除管理员 | 将成员/管理员从公海对应点位负责人移除 |  |
|  | ~~删除~~ |  | 有数据的公海不能删除 |

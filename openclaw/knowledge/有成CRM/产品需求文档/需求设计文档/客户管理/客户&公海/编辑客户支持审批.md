# 编辑客户支持审批

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20220124 | v1.0 | 新建 |
|  |  |  |

## 1、需求背景

编辑客户时也需要支持审批，目前不支持

## 2、业务流程/架构

无

## 3、需求详情

**1、客户设置**



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/eYVOL5jZdv1pNlpz/img/ef90dca4-14a7-45bb-83b9-d8470330994a.png?Expires=1774934889&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=q0bFtMrCHqTWq0%2F7zqA3A1D0k1Y%3D "")
- 客户的流程属性设置，放开展示【审批后修改】设置项，默认选择【允许】。即去除原来的特殊逻辑，与其他对象审批流保持统一
- 历史数据处理：设置为【允许修改】且选择【修改部分字段进审批】，具体字段设置为空
- 交互优化：仅选择允许修改时，出现【修改后触发审批字段】字段

**2、客户详情**
- 审批通过的客户，编辑入口由【审批后修改】设置项决定是否出现
- 结合【修改后触发审批字段】设置，编辑客户后发起审批时，如满足设置的条件，则会触发审批流程。此时会生成新的审批单，原来的审批单同步保留

**3、其他**
- 编辑后触发审批，最终审批不通过时，不会触发自动删除本条客户数据的逻辑
- 此时审批状态为已拒绝的客户，支持正常全部权限范围内的操作

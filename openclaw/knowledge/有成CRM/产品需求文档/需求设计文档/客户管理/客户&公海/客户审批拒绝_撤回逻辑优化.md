# 客户审批拒绝/撤回逻辑优化

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20220401 | v1.0 | 新建 |
| 202204014 | v1.1 | 补充线索转客户拒绝相关逻辑 |

## 1、需求背景
1. 新建客户审批被拒绝后，客户仍需要查看相关审批记录，希望进行审批拒绝后能修改重新提交。而目前是直接删除客户数据，导致上述需求无法满足，需优化
2. 拒绝与撤回状态的客户数据，相关特殊逻辑需补充优化

## 2、需求详情

### 2.1、新建客户审批拒绝后保留数据

1、新建客户审批拒绝后，不删除客户数据，正常保留，此时审批状态为“已拒绝”

**2、线索转客户逻辑优化**
- **原线索的【转换状态】，改为仅当新建的关联客户状态流转至“审批通过”时，才变更为“已转化”，其他状态时，仍保持未转化状态**
- **线索的转化操作，增加校验，如存在“审批中”的关联转化客户，则出现如下报错提示：**

### 2.2、客户不同审批状态操作限制

| **审批状态** | **详情页操作按钮展示（最全情况，具体将受各类权限控制）** | **大列表批量操作（客户&公海）** |
|----------------|----------------------------------------------------------------------------------|--------------------------------------------|
| 审批中 | 审批详情 | 1、各审批状态仅支持左侧罗列的操作，否则逐一报错提示2、添加协作人、移除协作人操作除外，可支持 |
| 已拒绝 | 编辑、作废、删除 |  |
| 已撤回 | 编辑、作废、删除 |  |
| 待提交 | 编辑、提交审批、作废、删除 |  |
| 已通过 | 编辑、退回（非公海客户）、回收（非公海客户）、领取（公海客户）、分配（公海客户）、更换公海、移交负责人、作废、删除 | ~~批量编辑客户时，也需触发审批~~ |

### 2.3、其他模块

| **模块** | **说明** |
|----------|----------|
| 关联客户弹窗 | 过滤仅展示审批通过状态的数据（已实现） |
| 客户保有上限计算 | 1、各审批状态的客户，全部计入个人客户保有量计算2、作废客户后，将不计入客户保有量计算 |
| 查重 | 所有审批状态都执行查重规则 |
| 自动回收规则 | 1\. 编辑客户跟进行为，将更新自动回收时间及最近跟进时间，此逻辑保持不变，即编辑后提交审批时间即更新跟进时间，与最后审批结果无关2\. 执行回收动作，也是与审批状态无关 |

### 2.4、客户支持草稿功能

客户对象支持草稿功能，对应待提交状态

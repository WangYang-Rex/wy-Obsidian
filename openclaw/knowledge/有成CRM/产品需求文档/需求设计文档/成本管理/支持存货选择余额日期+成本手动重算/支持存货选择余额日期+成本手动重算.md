# 支持存货选择余额日期\+成本手动重算

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2023-410 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

1）经常会有客户，存货余额是历史时间的，后续需要补录当前日期之前的单据；（目前我们存货余额日期取的是完成期初存货初始化的日期）

2）存在成本计算有问题的情况，需要手动执行重算，目前都是技术在执行。

## 2、需求详情

| **需求** | **描述** |
|----------|----------|
| 期初存货 | 库存表为空时，点击完成初始化，弹出‘完成初始化’页：1\. 余额日期选择：不可选择超出当天的日期；2\. 点击【确定】1. 企业完成初始化日期取\{余额日期\}2. 生成的期初余额/期初库存单据的‘业务日期’取\{余额日期\} |
| 成本重算 | 一、增加【手动重算】按钮，弹出‘选择重算维度’页面1\. 字段1. 产品明细：关联对象---产品明细（不校验功能/数据权限）2. 仓库：关联对象---产品明细（不校验功能/数据权限）3. 批次：单行文本4. 生产日期：日期5. 保质期天数：正整数2\. 操作1. 根据所选产品明细1. 产品明细若启用了批次，\{批次\}输入框可编辑；2. 产品明细若启用了保质期，\{生产日期\}、\{保质期天数\}输入框可编辑；2. 保存时校验1. 产品明细/仓库必填2. 若启用了批次，\{批次\}必填，否则toast提示‘保存失败：\{产品明细名称\}启用了批次管理，批次信息必填’3. 若启用了保质期，\{生产日期\}、\{保质期天数\}必填，否则toast提示‘保存失败：\{产品明细名称\}启用了保质期管理，生产日期、保质期天数信息必填’3. 处理：上述校验通过，则生成一条重算任务- 单据类型：-- 单据编号：-- 操作：手动重算（\{产品明细名称\}-\{仓库\}-\{批次\}-\{生产日期\}-\{保质期天数\}）- 操作人：执行手动重算的用户名- 操作时间：执行手动重算的时间- 任务状态/开始时间/结束时间：保持现有逻辑3、列宽调整\+ 单据类型：94.02px（原来是105.02px）\+ 单据编号：250px（原来是300px）\+ 操作：280.7px（原来是60.7px）\+ 操作时间/开始时间：140.5（原来是190.5px）\+ 结束时间：140.56（原来是190.56px） |

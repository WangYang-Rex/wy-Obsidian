# 退货业务和上游单据解绑

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|

## 1、需求背景

1、系统启用前，已经发生了销售/采购业务，启用后发生了之前业务的退货，目前不管是销售还是采购业务都无法直接支撑（例如采购退货，只能先去建其他出库，再建实际付款，但是到财务侧做凭证又有问题）

2、目前销售/采购相关的出入库和上游单据联系太紧密，导致用户使用起来觉得流程很长，觉得系统很难用。

## 2、业务流程/架构

无。

## 3、功能清单

| **模块** | **二级模块** | **功能点** |
|----------|----------------|-------------|
| 入库管理 | 销退入库单 | 可选择不关联的销售退货单； |
| 出库管理 | 采退出库单 | 可选择不关联的采购退货单； |

## 4、需求详情

### 4.1 销退入库单

#### 4.1.1 销退入库单字段需求

| **字段名称** | **是否必填** | **备注** |
|----------------|----------------|----------|
| 关联单据对象 | 默认必填（可更改） | 新旧客户一致 |
| 关联单据记录 | 默认必填（可更改） | 新旧客户一致 |

#### 4.1.2 销退入库单产品字段需求（直接选择产品明细时）

| **字段名称** | **默认填充逻辑** |
|----------------|----------------------|
| 产品明细 | 取所选择的产品明细 |
| 产品单位 | 默认取所选择的产品明细\{产品单位\} |
| 系统编码 | 引用取所选择的产品明细\{系统编码\} |
| 实际单价 | 默认取所选择的产品明细\{产品价格\} |
| 规格型号 | 引用所选择的产品明细\{规格型号\} |
| 货币名称 | 默认取所选择的产品明细\{货币单位\} |
| 条形码 | 引用所选择的产品明细\{条形码\} |
| 待入库数量 | 为0 |
| 产品简介 | 引用所选择的产品明细\{产品简介\} |
| 良品 | 为1 |
| 产品类型 | 引用所选择的产品明细\{产品类型\} |
| 源单明细编码 | 为null |
| 次品 | 为0 |
| 耗损 | 为0 |
| 专项税率 | 为0 |
| 批次保质期 | 一、是否可编辑，根据所选的产品设置的是否开启批次保质期来定（同其他入库）二、字段【保质期天数】：默认根据选中的产品明细中带出同名字段值（前提是对应产品明细\{启用保质期\}=‘是’ |
| 专项税额 | 根据计算公式得出 |
| 小计金额（不含税） | 根据计算公式得出 |
| 小计金额 | 根据计算公式得出 |
| 单位成本 | 根据现有的单位成本的默认取数逻辑现有逻辑为：取产品明细最近一次入库（含期初余额）的单位成本（如果没有最近一次入库单位成本则默认为0）； |
| 成本 | 根据计算公式得出：=单位成本\*（良品\+次品） |
| 备注 | 为null |

#### 4.1.3 其他需求

| **需求** | **描述** |
|----------|----------|
| 新建编辑页 | 一、关联单据记录为空1、选择入库单产品：支持选择所有产品明细（过滤掉可见范围外、下架的产品明细，同合同订单选择产品）2、单据保存提交时，校验‘相同\{源单明细行编码\}的明细行∑（\{良品\}\+\{次品\}\+\{损耗\}）须≤\{待入库数量\}’去除；3、单据提交成功后，包括后续的单据编辑、删除、审核不通过，无须更新销售退货单的任何信息；二、关联单据记录有值：保持现有逻辑备注：变更关联记录后明细行还是保持现有逻辑清空（null 变更为有值也是一样） |

# 阶段变更审批

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 2022-02-09 | v1.0 | 新建 |
| 2022-03-09 | v1.1 | 新增了：项目审批设置中，如设置审批后允许修改，此时变更阶段，也将触发审批 |

## 1、需求背景

当项目离开某个阶段之前，需要审核项目是否都达到要求进入下个阶段的要求。如项目从“项目立项”阶段需要审批通过，才能进入到“正式实施”阶段。

## 2、业务流程/架构

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 1 | 项目审批设置中，如设置审批后允许修改，此时变更阶段，也将触发审批 |  |
| 2 | 项目审批设置中，修改后触发审批字段设置了“阶段”字段后，变更项目的阶段将触发审批 |  |
| 3 | 项目审批设置中，支持将阶段作为触发条件、节点条件 |  |
| 4 | 阶段变更前如触发审批，则提示用户 |  |

## 4、需求详情

### 4.1、阶段变更支持审批
1. 项目审批设置中，如设置审批后允许修改，此时编辑项目，或变更阶段，将触发审批
2. 项目审批设置中，修改后触发审批字段设置了“阶段”字段后，变更项目阶段，将触发审批
3. 审批触发条件支持设置“阶段”字段作为条件（目前“阶段”字段支持选择，但是不能选择具体阶段）
4. 如变更的阶段满足触发审批条件，则弹窗提示，由用户确认是否提交审批；如变更的阶段不满足触发审批条件，则直接变更阶段，无需弹窗确认（PC和MB都需要做这个提示）

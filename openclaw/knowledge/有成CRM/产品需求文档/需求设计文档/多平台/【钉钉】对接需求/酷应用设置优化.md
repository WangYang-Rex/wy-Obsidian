# 酷应用设置优化

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2023-9-12 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景
- 推送时段支持自定义：例如工作日可推送，周末不推送
- 推送内容可自定义：例如单子中不展示客户名称，要展示负责人，不展示创建人
- 推送范围可自定义：例如订单金额=0的不推送
- 实际回款pk情况要展示

## 2、需求详情

| **需求** | **描述** |
|----------|----------|
| 业绩PK(合同订单) 推送 | 名称优化为‘业绩PK(合同订单) 推送’（之前是‘业绩PK(合同订单) 推送：’） |
| 业绩PK（实际回款）推送（新增模块） | 1、推送机制：每日早上9点钟推送业绩PK榜到群中2、取数逻辑\+ 业绩PK榜内容为每月的回款数据排名（取目标PK榜的实际回款数据，时间为当月）\+ 默认为全公司的数据3、点击“查看详情”，打开MB“目标PK榜”页面，查看详细的排名数据 |
| 合同订单推送 | 1、名称优化为‘合同订单推送’（之前是‘合同订单推送：’）2、新增【配置】按钮（mb没有）\+ 推送时间段- 推送天：选相值‘每天’‘仅工作日’（默认为‘每天’）- 提送时间段：00:00:00～23:59:59\+ 字段配置- 范围：合同订单的主对象字段（除附件、图片类型字段，参考详情摘要字段）- 限制：最少2个，最多5个，不满足条件时则在保存配置时toast提示‘字段配置须最少选择2个，最多5个’- 功能上线时，默认配置为‘客户’、‘合同订单金额’\+ 推送数据范围：筛选器同消息设置的‘触发条件’3、推送规则\+ 在之前推送合同订单的规则基础上（新增合同订单审批通过即推送），增加按照各群聊设置的推送时间段、推送数据范围推送对应单据信息 |
| 实际回款推送 | 相关功能同合同订单，区别为：1、对象：实际回款2、字段配置：‘客户名称’‘合同订单金额’ |
| 群吊顶（目标完成度） | 名称优化为‘群吊顶（目标完成度）’（之前是‘群吊顶（目标完成度）：’） |

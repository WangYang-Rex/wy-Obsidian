## 1、需求背景

对于项目的整体收支、利润统计，是对外宣传的一大差异化亮点功能，目前许多客户反馈现有项目概览数据统计缺失报销相关的支出数据，导致实际统计数据不太具备参考价值；尤其对于飞鸟一体客户，需要提供新的解决方案，以增加crm的竞争优势

## 5.有成报销接口文档

### 5.1 获取单据模版

url:/reimburse/order/getOrderTemplate.rjson

参数：

| **字段** | **类型** | **是否必填** | **描述** |
|----------|----------|----------------|----------|
| userId | string | 是 | 用户id |
| orderType | integer | 是 | 单据类型  1:报销单 2:借款单 3:付款单 |

样例：

\{

"userId":"1212121212",

"orderType": 1          //单据类型  1:报销单 2:借款单 3:付款单 

\}

返回结构不变；

### 5.2 获取项目报表

url:/reimburse/project/getProjectForm.rjson

参数:

| **字段** | **类型** | **是否必填** | **描述** |
|----------|----------|----------------|----------|
| beginTime | string | 是 | 获取统计数据开始时间 |
| endTime | String | 是 | 获取统计数据结束时间 |
| projectCode | string | 是 | 项目编码 |

样例：

\{

"beginTime":"2022-07-01 00:00:00",    //获取统计数据开始时间

"endTime":"2022-07-30 00:00:00" //获取统计数据结束时间

"projectCode":"xm.2022-07-26-0001"    //项目编码

\}

返回结构：

| **字段** | **类型** | **描述** |
|----------|----------|----------|
| skhtAmount | bigdecimal | 应收款金额（收款合同） |
| yskhtAmount | bigdecimal | 已收款金额（收款合同） |
| dskhtAmount | bigdecimal | 待收款金额（收款合同） |
| skhtInvoiceAmount | bigdecimal | 收款已开票金额（收款合同） |
| skhtUninvoiceAmount | bigdecimal | 收款未开票金额（收款合同） |
| fkhtAmount | bigdecimal | 应付款金额（付款合同） |
| yfkhtAmount | bigdecimal | 已付款金额（付款合同） |
| dfkhtAmount | bigdecimal | 待付款金额（付款合同） |
| fkhtInvoiceAmount | bigdecimal | 付款已开票金额（付款合同） |
| fkhtUninvoiceAmount | bigdecimal | 付款未开票金额（付款合同） |
| reimburseAmount | bigdecimal | 报销总金额（报销单） |
| reimburseOverAmount | bigdecimal | 报销已支付金额（报销单） |
| reimburseUnoverAmount | bigdecimal | 报销中金额（报销单） |
| payAmount | bigdecimal | 付款总金额（付款单） |
| payOverAmount | bigdecimal | 付款已支付金额（付款单） |
| payUnoverAmount | bigdecimal | 付款支付中金额（付款单） |
| unRelaContractpayOverAmount | bigdecimal | 未关联合同的付款已支付金额（付款单） |
| loanAmount | bigdecimal | 借款总金额（借款单） |
| repayAmount | bigdecimal | 已还款金额（借款单） |
| remainAmount | bigdecimal | 待还款金额（借款单） |
| profitAmount | bigdecimal | 盈利 |

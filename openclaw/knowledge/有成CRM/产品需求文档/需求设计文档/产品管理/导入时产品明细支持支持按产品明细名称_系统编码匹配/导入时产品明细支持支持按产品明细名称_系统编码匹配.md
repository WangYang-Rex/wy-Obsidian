# 导入时产品明细支持支持按产品明细名称/系统编码匹配

| **修订时间** | **版本** | **修订人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 2023-03-29 | v1.0 | 黄艺平 | 新建 |
|  |  |  |  |

## 1、需求背景

问题：

新版导入上线后，根据关联字段匹配唯一值。

由于很多子对象的关联字段配置到了“产品明细”字段，无法实现之前用“系统编码”判断唯一值的需求。



解决方案：

不能一刀切统一切换到只根据‘系统编码’匹配，因为很多客户也是根据‘产品明细名称’匹配即可。

故本期考虑做成支持用户配置。

| **需求** | **描述** |
|----------|----------|
| 设置-业务模板-产品明细 | 增加‘导入时唯一标识’设置，所有企业（新旧）设置均默认为‘产品明细名称’说明中的内容：说明：1）举例合同订单-合同订单产品导入时，导入模板中‘产品明细’应填写产品明细的‘\{唯一标标识字段\}’，否则会导入匹配失败；2）已做过支持自定义关联对象的模块才支持唯一标识字段的切换，其余的对象默认均按照系统编码匹配。 |
| 导入 | 一、已铺开对象中（包含主子对象），关联对象=产品明细的，导入时，根据企业导入时唯一标识字段配置，进行产品明细匹配，举例：企业导入时\{唯一标识字段\}='系统编码'合同订单-订单产品导入，则拿导入文件中‘产品明细’的字段值去匹配系统中产品明细名称-系统编码二、导入文件优化（仅针对产品明细对象）改为：‘示例数据:\{唯一标识字段\}’三、错误数据提示（仅针对产品明细对象）1\. 必填提示1. 改之前：关联对象不存在2. 改之后：关联对象不存在（请填写\{唯一标识字段\}）2\. 非必填提示1. 改之前：关联对象不存在2. 改之后：关联对象不存在（请填写\{唯一标识字段\}）,\[合同订单产品\].\[第1行数据\]:(\[合同订单产品\]产品明细)必须传入,请检查! |

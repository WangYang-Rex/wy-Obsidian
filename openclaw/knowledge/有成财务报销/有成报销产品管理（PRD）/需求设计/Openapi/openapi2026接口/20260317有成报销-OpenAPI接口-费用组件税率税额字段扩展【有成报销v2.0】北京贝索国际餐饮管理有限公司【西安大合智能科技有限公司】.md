## **1\. 需求背景**

### **1.1 需求描述**

[费控OPENAPI](https://yiqbopenapi.superboss.cc/#/share/7YX0lXxR/Zd2qAr2w)

在单据推送接口的`feeComponentList`中放出税率税额组件（taxRate/taxAmount/netAmount）的字段，支持外部系统通过OpenAPI推送完整的增值税发票税务信息，实现凭证进项税分录的自动生成。

### **1.2 需求背景**
- **需求方**：财务管理员、系统对接开发工程师（氚云对接客户）
- **现状问题**：
• `feeComponentList`缺少税率税额组件部分字段，无法存储税率、税额、不含税金额 • 氚云业务单推送至有成单据，无法将单据中消费下的税率税额等推送过去，导致凭证生成时缺少进项税额分录，需要人工补录发票信息
• 无法实现"推送即完成"的全流程自动化对接
- **解决思路**：在`feeComponentList`中新增税率税额组件字段，与现有税率税额组件结构保持一致，支持字段自动计算与校验

**业务流程**


```plain
氚云/宜搭系统创建业务单据（含发票信息-税率税额价税合计）
    ↓
调用有成报销单据推送接口
    ↓
接口解析feeComponentList中的税率税额组件
    ↓
存储费用明细（含税率税额组件值）
    ↓
单据审批流转
    ↓
可抵扣发票配置，会计科目映射，凭证生成
    ↓
完整凭证分录（含进项税分录，价税分离生成税额分录）
```

## **2\. 功能现状**

**功能位置**：有成报销 OpenAPI → 单据推送接口→ 请求参数 → feeList → feeComponentList

**现有字段**： 

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/yBRq1ZPzLogVyOdv/img/de41fe41-b869-49aa-97bf-cd8412010f56.png?Expires=1775557115&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=LK6SRQ%2BsHlw1tkXj6nHJYFJxYUo%3D "")



**现有操作**：
• 外部系统通过`feeComponentList`推送费用明细组件
• 凭证生成时使用`amount`组件值生成费用科目和资金科目分录

**缺失能力**：
• `feeComponentList`中缺少税率税额组件（taxRate/taxAmount/netAmount）
• 无法通过接口推送税率信息，不能自动生成进项税分录
• 无法实现价税分离，金额口径可能不一致

### **3.2 详细功能描述**

#### **3.2.1 feeComponentList税率税额组件扩展**

**功能概述** 在现有`feeList[].feeComponentList`数组中，新增支持税率税额组件，与现有系统内税率税额组件结构保持一致。

**配置入口** 【有成报销 OpenAPI → 单据推送接口→ 请求参数 → feeList → feeComponentList】

**新增组件code定义**

| **组件code** | **bindingVal类型** | **必填** | **描述** | **示例值** |
|--------------------------------------------------------------|--------------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|-------------------------------------------------------------|
| taxRate | string/number | 否 | 税率（百分比数值，如13表示13%） | "13" |
| taxAmount | string/number | 否 | 税额 | "1150.44" |
| netAmount | string/number | 否 | 不含税金额 | "8849.56" |

**默认配置**

| **字段** | **默认状态** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 税率税额组件 | 可选 | 存量兼容，不传时不影响原有逻辑，非必传参数 |
| 税率范围 | 0-100 | 支持小数点后2位 |
| 金额精度 | 2位小数 | 精确到分 |

**配置存储** • 组件code命名规范与现有组件保持一致
• 费用组件值表（`fee_component_value`）已支持任意组件code存储，无需变更
• 配置保存后即时生效

#### **3.2.2 字段自动计算与校验**

**功能概述** 根据传入的税率税额组件自动计算缺失字段，并校验金额勾稽关系。

**计算与校验逻辑**
```text
接收 feeComponentList 数组
    ↓
识别税率税额组件（通过code匹配）
    ↓
┌────────────────────────────────────────────────────────────┐
│ 场景1：全部组件传入（推荐）                                  │
│ 传入：amount=10000, taxRate=13, taxAmount=1150.44, netAmount=8849.56 │
│ 校验：amount = netAmount + taxAmount（误差<0.01元）          │
│ 校验：taxRate ≈ taxAmount / netAmount × 100（误差<0.01%）    │
│ 结果：全部字段有效，标记为"已校验"                           │
├────────────────────────────────────────────────────────────┤
│ 场景2：只传入税率和含税金额                                  │
│ 传入：amount=10000, taxRate=13                              │
│ 计算：netAmount = ROUND(amount / (1 + taxRate/100), 2)      │
│ 计算：taxAmount = amount - netAmount                        │
│ 结果：自动计算并存储netAmount和taxAmount                     │
├────────────────────────────────────────────────────────────┤
│ 场景3：只传入税额和不含税金额                                │
│ 传入：netAmount=8849.56, taxAmount=1150.44                  │
│ 计算：amount = netAmount + taxAmount                        │
│ 计算：taxRate = ROUND(taxAmount / netAmount × 100, 2)       │
│ 结果：自动计算并存储amount和taxRate                          │
├────────────────────────────────────────────────────────────┤
│ 场景4：存量兼容（不传税率税额组件）                          │
│ 处理：taxRate/taxAmount/netAmount为空                        │
│ 结果：只存储amount                      │
│ 兼容：原有逻辑保持不变                                       │
└────────────────────────────────────────────────────────────┘
```



**边界处理**

| **场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 金额勾稽不符 | amount ≠ netAmount \+ taxAmount（误差\>0.01元） | 返回错误码210，提示"费用金额不等于不含税金额与税额之和" |
| 税率计算不符 | taxRate与taxAmount/netAmount计算结果不一致（误差\>0.01%） | 返回错误码211，提示"税率与税额/不含税金额计算结果不一致" |
| 税率超出范围 | taxRate \< 0 或 taxRate \> 100 | 返回错误码212，提示"税率应在0-100之间" |
| 金额精度错误 | 金额字段小数位数\>2位 | 返回错误码213，提示"金额字段最多支持2位小数" |
| 组件code未定义 | 传入的code未在系统中配置 | 返回错误码214，提示"税率税额组件code未在系统中定义" |

**校验错误码**

| **错误码** | **错误信息** | **处理建议** |
|-------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 210 | 费用金额不等于不含税金额与税额之和 | 请检查传入的金额字段计算是否正确 |
| 211 | 税率与税额/不含税金额计算结果不一致 | 请检查税率或重新计算税额 |
| 212 | 税率应在0-100之间 | 请检查taxRate组件值 |
| 213 | 金额字段最多支持2位小数 | 请检查金额组件值精度 |
| 214 | 税率税额组件code未在系统中定义 | 请联系管理员配置组件code |

#### **3.2.3 存量兼容策略**

**功能概述** 新组件为可选字段，完全向后兼容，不影响存量接口调用。

**兼容策略**

| **场景** | **处理方案** |
|----------------------------------------------------------|----------------------------------------------------------------|
| 存量接口调用（不传税率税额组件） | 原有逻辑保持不变，接口完全向后兼容 |
| 部分组件传入 | 系统自动计算缺失字段，按"传入优先"原则存储 |
| 新接入系统 | 费用模板中配置税率税额组件，推送单据时传入完整信息 |
| 税率=0或空 | 原有逻辑保持不变 |

## **4\. 验收标准**

| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 接口组件接收 | 单据推送接口能正常接收`taxRate`、`taxAmount`、`netAmount`组件，不报错 |
| 2 | 字段自动计算 | 只传`amount`\+`taxRate`组件时，自动计算`netAmount`和`taxAmount`，计算误差\<0.01元 |
| 3 | 金额勾稽校验 | 传入金额勾稽错误时（如amount≠netAmount\+taxAmount），返回错误码210和明确提示 |
| 4 | 税率范围校验 | 传入taxRate\<0或\>100时，返回错误码212和明确提示 |
| 5 | 金额精度校验 | 金额字段小数位数\>2位时，返回错误码213和明确提示 |

| 7 | 存量兼容 | 不传税率税额组件时，单据正常保存，原有逻辑与行为一致 |
| 8 | 接口响应时间 | 单据推送接口（含税率税额组件校验计算）响应时间 \< 500ms（P95） |
| 9 | 并发处理 | 保持原有单据推送并发上限，无数据错乱 |
| 10 | 接口文档 | 接口文档已更新，包含税率税额组件code说明、示例、错误码 |

## **附录**

**A. 典型使用场景**

**场景：氚云采购付款单推送**

配置步骤（氚云系统）：
1. 在氚云低代码平台配置采购付款单，包含发票信息字段（发票代码、发票号码、金额、税率、税额）
2. 配置有成报销单据推送接口调用
3. 在feeComponentList中构造税率税额组件

使用步骤（日常推送）：
1. 采购人员在氚云录入采购付款单，关联增值税专用发票
2. 发票信息：金额10,000元，税率13%，税额1,150.44元，不含税金额8,849.56元
3. 氚云调用有成报销单据推送接口，传入完整税率税额组件
4. 有成报销自动校验金额勾稽关系，存储费用明细
5. 财务审核通过后，系统自动生成完整凭证（含进项税分录）

结果：实现全流程自动化，无需人工补录发票信息，凭证分录完整准确。

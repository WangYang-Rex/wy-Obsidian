## 1、需求背景

一、发票类型做了加工订正，导致源数据区分不出来类型

二、需要拿出来字段，在开票申请单中去使用，开票申请单的前置需求

## 2、需求详情

**二、常用字段中新增一个枚举值字段【发票类型】：**

数据对应到系统中目前的发票类型枚举值；

**组件配置说明：**

| **配置项** | **说明** |
|-------------|----------|
| 组件名称 | 组件名称：发票类型，不可编辑 |
| 显示名称 | 1、显示名称：【发票类型】2、可编辑，max=20，必填 |
| 提示信息 | 1、默认：请选择发票类型2、可编辑，max=50，非必填 |
| 属性设置 | 1、必填、隐藏、完成后隐藏：均是默认不选中，可编辑，走系统通用逻辑2、数据范围：当前字段中新增配置，勾选后出现选择框，点击显示所有发票类型选项，支持多选，选中后，在提单页面，只显示勾选后的选项可以选择使用 |
| 默认填充规则 | 无：默认选中上一次选项：选中后，每次新建编辑，都带入上一次的选项，没有则不带入 |

**2、现有发票类型数据的订正**

背景：目前系统内的发票类型跟实际的发票类型有一些出入，对源数据做了一些加工汇总，在对接电票以及做发票查验时，类型区分模糊，因此对发票类型整体做下订正。
- **现在的发票类型：**

未在表中列出的发票类型对本次需求没有影响，也不需要做变动，仍旧保持原值

标红的是目前归类有问题的数据

| **展示的发票类型** | **对应的实际发票** |
|-------------------------|-------------------------|
| 增值税纸质专用发票 | 增值税纸质专用发票 |
| 增值税纸质普通发票 | 增值税纸质普通发票 |
| 增值税电子专用发票 | 增值税电子专用发票、电子发票（增值税专用发票） |
| 增值税电子普通发票 | 增值税电子普通发票 |
| 普通发票 | 电子发票（普通发票）、XX电子普通发票 |
- **更改后的发票类型：**将【电子发票（增值税专用发票）】从【增值税电子专用发票】中拿出来，单独新增一个类别叫【电子发票（专用发票）】原来的类型【普通发票】更名给【电子发票（普通发票）】历史数据：主要是对历史数据归集、单据列表搜索、**配置项目**的影响已存在归类好的历史发票数据能否处理？比如电子发票（增值税专用发票）能否摘出来，更名为新增的类别中去；单据列表搜索：普通发票更名后的发票类型的搜索不能受影响；列表中药支持到新的发票类型枚举值：我的-发票单据管理-发票管理配置项：原来勾选了【增值税电子专用发票的】，需要刷成【增值税电子专用发票】\+【电子发票（专用发票）】：会计科目映射-基础设置-可抵扣发票会计科目映射-科目映射-发票类型：改为支持多选，将原来的已配置的【增值税电子专用发票】，增加新的类型基础设置-全局设置-发票设置-自动同步税率税额、发票格式转换、~~发票抬头校验~~

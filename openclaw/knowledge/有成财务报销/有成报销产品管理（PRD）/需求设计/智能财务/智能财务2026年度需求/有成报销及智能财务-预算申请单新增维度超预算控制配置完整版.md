## 1\. 需求背景

### 1.1 需求描述

在有成报销及智能财务的预算管理基础设置中新增统一配置项，为预算申请单新增的预算维度提供可配置的超预算控制规则（提示警告 / 禁止提交 / 不提示），解决当前新增维度只能固定预警、无法禁止提交导致的管控失控问题。

### 1.2 需求背景
- **需求方**：预算管理员、财务审核人员
- **现状问题**：
• 预算申请单新增的维度，单据占用时系统固定仅作预警提示，不支持禁止提交
• 员工可随意超额占用新增维度预算，管理人员无法前置阻断
• 有成报销和智能财务均存在该问题，需统一解决
- **解决思路**：在预算管理基础设置中新增全局 fallback 配置，统一控制预算申请单新增维度的超预算行为，优先级低于预算方案单独配置和浮动预算配置

### 1.3 业务流程
```
【预算管理员】
   ↓
进入预算管理 → 基础设置 → 预算管控
   ↓
配置「预算申请单新增维度管控」规则
（固定档位100% + 可选自定义档位）
   ↓
保存配置，即时生效
   ↓
────────────────────────────────────
【员工提交单据】（有成报销/智能财务）
   ↓
报销单/付款单/预算申请单提交
   ↓
系统校验预算占用
   ↓
占用的是「预算申请单新增的维度」？
   ├─ 否 → 按现有方案规则/浮动预算执行
   └─ 是 → 该维度在预算方案中有单独配置？
            ├─ 有 → 按方案规则执行
            └─ 无 → 有浮动预算配置？
                     ├─ 有 → 按浮动预算规则执行
                     └─ 无 → 读取基础设置新增维度管控规则
                              ↓
                          执行对应档位处理方式
                              ↓
                          提示警告 / 禁止提交 / 不提示

```

## 2\. 功能现状

### 2.1 预算方案维度超预算控制
- **位置**：预算费控 → 预算管理 → 预算方案 → 选中维度 → 超预算控制
- **当前能力**：可为每个预算维度单独配置超预算规则（超出预算 X% 时 → 提示警告 / 禁止提交 / 不提示）
- **生效范围**：预算方案中直接配置的维度，单据提交时按规则校验
> 本次需求不改动该逻辑，优先级高于新配置。   

### 2.2 预算申请单新增维度的现状
- **位置**：预算费控 → 预算管理 → 预算申请单 → 新增下级维度
- **当前能力**：员工可在预算申请单中新增下级预算维度并配置预算额度，审批通过后生效
- **缺失能力**：新增维度被单据占用时，系统固定仅作「预警提示」，无法配置为「禁止提交」
- **影响范围**：有成报销及智能财务中，所有通过预算申请单新增的维度均受此限制
> 本次需求需突破该逻辑，为预算申请单新增维度增加可配置的超预算控制规则。   

### 2.3 浮动预算管控现状
- **位置**：预算费控 → 预算管理 → 基础设置 → 预算管控 → 开启浮动预算管控
- **当前能力**：开启后，可为维度配置浮动比例，在固定阈值基础上增加浮动容忍空间
- **生效方式**：浮动预算是在维度自身超预算规则基础上的附加判断
> 本次需求不改动浮动预算逻辑，优先级高于新增维度管控规则。

### 3.2 详细功能描述

#### 3.2.1 配置区块：预算申请单新增维度管控

**功能概述**

在预算管理基础设置中新增「预算申请单新增维度管控」配置区块，预算管理员可统一设置预算申请单新增维度的超预算控制规则。有成报销及智能财务共用同一套预算管理基础设置，配置一次，两处生效。

**配置入口**
- **有成报销**：预算费控 → 预算管理 → 基础设置 → 预算管控
- **智能财务**：预算费控 → 预算管理 → 基础设置 → 预算管控

**界面样式**
```text
┌────────────────────────────────────────────────────────────────┐
│  预算申请单新增维度管控                                         │
│                                                                │
│  大于预算额度的  100  % 时   ○ 提示警告 ● 禁止提交  ○ 不提示    │
│                                                                │
│  大于预算额度的 [    ] % 时  ○ 提示警告  ○ 禁止提交  ○ 不提示   │
│                                                                │
└────────────────────────────────────────────────────────────────┘

```
- 固定档位百分比（100）为系统预设，不可编辑，仅可修改处理方式
- 自定义档位百分比为输入框，需手动填写
- 处理方式为单选按钮组：提示警告 / 禁止提交 / 不提示

**配置项说明**

| 档位 | 说明 | 默认值 | 可编辑项 |
|------|------|---------|------------|
| 固定档位 | 大于预算额度的 **100%** 时执行对应处理方式 | 禁止提交 | 处理方式（三选一） |
| 自定义档位 | 大于预算额度的 **【输入框】%** 时执行对应处理方式 | 空（未配置） | 百分比（0-500整数）\+ 处理方式（三选一） |

**配置规则约束**
- 固定档位（100%）为系统预设，始终存在，作为兜底策略，不可删除
- 仅允许配置 1 个自定义档位
- 自定义档位百分比范围：0-500 的整数
- 自定义档位允许为空（不填写百分比则不生效，仅固定档位起作用）
- 不区分年度/月度，仅一组统一规则

**编辑规则**
- **固定档位（100%）**：可修改处理方式（切换单选选项），切换后即时生效
- **自定义档位**：可修改百分比（输入框实时校验：仅允许整数、0-500）；可修改处理方式（切换单选选项），均即时生效
- 配置保存后，无需二次确认，直接生效

**配置存储**
- 配置按企业维度存储
- 配置保存后即时生效，不区分存量/增量维度，全量适用

#### 3.2.2 超预算校验逻辑

**功能概述**

员工在有成报销或智能财务中提交单据时，系统对预算申请单新增的维度执行超预算校验，按优先级链逐层判断。

**优先级链**（从高到低）
```
1. 预算方案中为该维度单独配置的超预算规则
2. 预算维度浮动配置（如开启且该维度配置了浮动比例）
3. 基础设置-预算申请单新增维度管控规则（本次新增，fallback）
4. 默认仅预警禁止（系统最底层兜底）

```

**校验时机与逻辑**
- 触发单据：报销单、付款单、预算申请单等所有涉及预算占用的单据
- 触发条件：占用的是「预算申请单新增的维度」
- 校验流程： 
    1. 判断该维度在预算方案中是否有单独配置 → 有则按方案规则执行
    2. 判断该维度是否有浮动预算配置 → 有则按浮动预算规则执行
    3. 读取基础设置中的「新增维度管控规则」→ 按配置的档位执行
    4. 以上均无 → 执行默认仅预警禁止

**与浮动预算管控的交互关系**
- 浮动预算管控是在维度自身超预算规则基础上的**附加判断**（先判断固定阈值，再判断浮动阈值）
- 新增维度管控是**替换**默认的超预算处理方式，两者作用层面不同，不冲突
- 如果管理员后续在预算方案中为申请单新增的维度单独配置了超标规则，则该维度不再走新增维度管控规则，优先执行方案配置

**生效范围**
- 有成报销：配置保存后，所有通过有成报销提交的、占用预算申请单新增维度的单据均按新规则校验
- 智能财务：配置保存后，所有通过智能财务提交的、占用预算申请单新增维度的单据均按新规则校验
- **不区分存量/增量维度**：配置保存前已新增的维度，同样按新规则执行

#### 3.2.3 批量操作与权限

**批量操作说明**
- **不支持批量修改**：本配置是全局 fallback 规则，不按预算方案区分，因此无需批量操作入口
- **与预算方案批量修改的关系**：预算方案中的「批量修改超标控制」仅修改维度自身的配置，不影响本全局 fallback 规则
- **优先级提醒**：批量修改后，被修改的维度将按方案规则执行，本配置不再作为该维度的 fallback

**权限与可见性**

| 角色 | 可见性 | 操作权限 |
|------|---------|------------|
| 预算管理员（有成报销） | 可见配置区块 | 可查看、可修改 |
| 预算管理员（智能财务） | 可见配置区块 | 可查看、可修改 |
| 普通员工 | 不可见配置页面 | 提交单据时仅感受校验结果 |

**端适配**

| 端 | 有成报销 | 智能财务 |
|---|------------|------------|
| PC端管理后台 | 配置页面新增配置区块 | 配置页面新增配置区块 |
| 移动端 | 不涉及配置，仅单据提交时按规则校验 | 不涉及配置，仅单据提交时按规则校验 |

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 自定义档位为空 | 用户未填写自定义档位的百分比 | 仅固定档位100%生效，作为兜底策略 |
| 百分比输入非法 | 输入小数、负数或超过500 | 输入框实时拦截，仅允许0-500的整数 |
| 百分比输入0 | 用户输入0% | 允许，表示预算为0时即触发对应处理方式 |
| 预算方案补配规则 | 某维度原无方案配置，后续补配了超标规则 | 补配后该维度按方案规则执行，优先级高于基础设置 |
| 基础设置修改后 | 配置保存后，已新增的维度后续被占用 | 已新增维度按最新基础设置规则执行（全量生效） |
| 关闭全面预算 | 企业关闭「开启全面预算」开关 | 新增配置区块置灰或隐藏，新增维度保持默认禁止提交 |
| 未开启预算管控 | 企业未开启预算管控相关功能 | 不显示新增配置区块，新增维度保持默认禁止提交 |
| 单据占用多维度 | 一张单据同时占用预算申请单新增维度和普通方案维度 | 各维度独立按自身优先级链校验 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 预算方案超预算控制 | 优先级高于新增维度管控 | 维度有方案配置时，完全按方案规则执行，不走新增维度管控 |
| 浮动预算管控 | 优先级高于新增维度管控，与方案配置协同 | 维度有浮动配置时，先按方案规则判断，再叠加浮动容忍 |
| 预算方案批量修改 | 独立操作，互不影响 | 批量修改仅影响维度自身配置，不影响全局 fallback 规则 |
| 预算申请单下级维度管理 | 被管控对象 | 本配置控制的是该功能新增的维度的超预算行为 |

## 附录

### A. 典型使用场景

**场景一：首次配置并生效**

配置步骤（预算管理员）：
1. 登录有成报销（或智能财务），进入预算费控 → 预算管理 → 基础设置 → 预算管控
2. 在「预算申请单新增维度管控」区块，确认固定档位100%为「禁止提交」
3. 在自定义档位输入框填写 120%，选择「提示警告」
4. 系统自动保存，配置即时生效

使用步骤（员工）：
1. 员工提交报销单，占用某预算申请单新增的维度
2. 系统校验：该维度在预算方案中无单独配置，无浮动配置
3. 读取基础设置：占用达100% → 禁止提交；占用达120% → 提示警告
4. 若占用110%：阻断提交，提示「预算已超支，禁止提交」
5. 若占用130%：弹窗提示超预算，用户可选择继续提交或取消

结果：预算申请单新增维度实现了分层管控，100%以上完全禁止，120%以上预警提示。

**场景二：预算方案补配规则后切换**

配置步骤：
1. 预算管理员在基础设置中配置了100%=禁止提交
2. 后续为某预算申请单新增的维度A在预算方案中单独配置了「150%禁止提交」

使用步骤：
1. 员工提交单据占用维度A
2. 系统检测到维度A在预算方案中有单独配置
3. 按方案规则执行：占用达150%时禁止提交
4. 基础设置中的100%规则对该维度不再生效

结果：方案规则优先，精细化管理不影响全局 fallback 的稳定性。

## **1\. 需求背景**

### **1.1 需求描述**

在员工基础信息中新增"在编状态"字段（编内/编外），并支持在公式计算中引用该字段，实现表格组件内员工组件与其他组件的数据联动。

### **1.2 需求背景**
- **需求方**：系统管理员、财务人员、业务单据配置人员
- **现状问题**：
    - 员工基础信息缺少"在编状态"字段，无法区分编内/编外人员身份
    - 表格组件内添加的员工组件，其字段信息在公式计算中无法被其他组件引用
    - 无法实现"选择员工→自动带出在编状态→根据在编状态计算其他值"的联动场景
    - 第三方系统无法通过接口获取和推送员工的【在编状态】字段
- **解决思路**：
    - 新增员工基础字段"在编状态"
    - 扩展公式计算能力，支持员工组件【常用组件和自定义组件】的子字段（包括在编状态）在公式计算中被引用
    - 兼容场景：自定义单据【员工组件，文本组件和表格组件】，自定义费用【员工组件，文本组件和表格组件】
    - 扩展有成报销**【获取用户列表】和【批量新增/更新企业人员】接口的【在编状态】字段**

**业务流程**
```plain
HR维护员工在编状态
    ↓
表单设计器配置表格组件（员工组件+文本组件）
    ↓
文本组件配置公式引用员工组件的在编状态
    ↓
用户选择员工后，文本组件自动带出在编状态
```

## **2\. 功能现状**

**功能位置1**：基础设置 → 员工管理 → 员工信息 → 编辑员工

**现有字段**：姓名、工号、手机号、邮箱、角色、直属上级、常驻地、所属部门、默认部门、在职状态

**缺失字段**：在编状态

**功能位置2**：自定义单据 → 表单设计器 → 左侧组件库

**现有组件**：
- 申请人组件
- 员工组件
- 文本、数字、日期等基础组件
- 表格组件

**现有能力限制**：
- 公式计算中，表格组件内添加的员工组件缺少在编状态信息
- 表格组件内文本组件无法通过公式计算引用员工组件的在编状态

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 员工未选择 | 公式引用了员工子字段，但员工组件未选择 | 公式返回空值，不报错 |
| 员工在编状态为空 | 员工已选，但在编状态字段未维护 | 公式返回空值 |
| 导入值为空 | Excel导入时在编状态列未填写 | 保持为空，不默认填充 |
| 导入值错误 | 填写了"编内"、"编外"之外的值 | 校验失败，提示错误信息 |
| 公式语法错误 | 公式表达式格式错误 | 语法检测，提示错误信息 |
| 存量数据 | 历史员工信息无在编状态 | 显示为空（NULL），由客户手动维护 |
| 存量单据 | 历史单据中的公式引用 | 若员工无在编状态，返回空值 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| OpenAPI接口 | 字段扩展 | 员工查询/推送接口返回数据中包含在编状态 |

## **附录**

**A. 典型使用场景**

**场景：事业单位区分编内编外差旅标准**

配置步骤：
1. HR在员工管理维护每位员工的"在编状态"
2. 表单设计器中添加表格组件，配置列：
    - 员工（员工组件-仅单选）
    - 在编状态（文本）
    - 住宿标准

使用步骤：
1. 员工提交差旅报销单
2. 在表格中添加行，选择员工
3. 系统自动带出在编状态并计算对应住宿标准
4. 超额时按对应标准提示或控制

结果：实现了编内编外人员的差异化费用管控

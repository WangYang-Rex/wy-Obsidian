## **1\. 需求背景**

### **1.1 需求描述**

目前员工基础信息缺少"在编状态"字段，导致自定义单据中的人员组件无法通过公式自动带出员工的编内/编外状态，影响业务流程中的身份识别和审批规则配置。

### **1.2 需求背景**
- **需求方**：系统管理员/财务人员
- **现状问题**：
• 员工基础信息中无"在编状态"字段，无法区分编内/编外人员
• 自定义单据的申请人/人员组件公式计算中，无法获取员工的在编状态
• 部分企业需要根据编内/编外身份设置不同的审批流或费用标准
- **解决思路**：在员工基础信息中新增"在编状态"字段，并支持在表单组件公式中引用

**业务流程**
```plain
HR在员工管理维护在编状态
↓
员工提交单据时，系统自动带出在编状态
↓
审批流根据在编状态路由到不同审批人
```

## **2\. 功能现状**

**功能位置1**：基础设置 → 员工管理 → 员工信息 → 编辑员工

**现有申请人基础信息字段**：
- 姓名、工号、手机号、邮箱
- 角色、直属上级、常驻地
- 所属部门、默认部门

**申请人缺失字段**：
- 在编状态（编内/编外）

**员工缺失字段**：
- 姓名、工号、手机号、邮箱
- 角色、直属上级、常驻地
- 所属部门、默认部门
- 在编状态（编内/编外）

**功能位置2**：自定义单据 → 表单设计器 → 申请人/人员组件 → 组件设置 → 默认填充值 → 公式计算

**现申请人有可引用字段**：
- 名称、工号、手机号、邮箱
- 所属部门名称、所属部门编号
- 在职状态、常驻地、直属上级、直属上级工号

**缺失引用**：
- 在编状态

### **3.2 详细功能描述**

#### **3.2.1 员工基础信息增加在编状态字段**

**位置**：基础设置 → 员工管理 → 员工信息 → 新增/编辑员工

**界面调整**

在"基础信息"区域增加"在编状态"字段：
```plain
基础信息
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
* 姓名          [输入框]
  工号          [输入框]
  手机号        [输入框]
  邮箱          [输入框]
  角色          [下拉选择]
  直属上级      [下拉选择]
  常驻地        [输入框]
  在编状态      [下拉选择 ▼]  ← 新增字段
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```



**字段定义**

| **属性** | **值** |
|----------------------------------------------------------|-------------------------------------------------------|
| 字段名 | 在编状态 |
| 字段标识 | employment\_status / in\_service\_status |
| 数据类型 | 枚举（单选） |
| 可选项 | 编内、编外 |
| 默认值 | 空（NULL） |
| 必填 | 否（可选填） |

**交互规则** • 下拉选择，选项为"编内"、"编外"
• 默认不选中（为空）
• 支持清空（不填）
• 保存后即时生效

#### **3.2.2 员工信息导入导出增加在编状态**

**导入模板**

| **序号** | **字段名称** | **必填** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| ... | ... | ... | 现有字段 |
| N | 在编状态 | 否 | 新增字段，填写"编内"或"编外"，为空时保持为空 |

**导出Excel**

在导出文件中增加"在编状态"列，位置建议放在"角色"之后：
- 列名：在编状态
- 数据：编内/编外/空

**数据校验** • 导入时若填写值不在枚举范围内（非"编内"、"编外"），提示"在编状态填写错误，请填写'编内'或'编外'"
• 为空时默认为"编内"

#### **3.2.3 表单组件公式计算支持在编状态**

**位置**：自定义单据 → 表单设计器 → 申请人/人员组件 → 组件设置 → 默认填充值 → 公式计算

**可引用字段扩展**

在"申请人/人员" → "添加字段"弹窗中增加"在编状态"选项：
```plain
申请人
├── 名称
├── 工号
├── 手机号
├── 邮箱
├── 所属部门名称
├── 所属部门编号
├── 在职状态
├── 常驻地
├── 直属上级
├── 直属上级工号
└── 在编状态        ← 新增
```

**使用场景**

**场景1**：单据中自动显示申请人/员工在编状态
```plain
组件：单行文本（在编状态显示）
默认填充值公式：【员工/申请人.在编状态】
效果：员工/申请人选择后，自动填充"编内"或"编外"
```

**场景2**：根据在编状态设置审批流条件
```plain
条件分支：在编状态 = "编内" → 走编制内审批流程
条件分支：在编状态 = "编外" → 走编制外审批流程
```

**场景3**：根据在编状态设置费用标准
```plain
费用标准规则：在编状态 = "编内" → 标准金额 500元
费用标准规则：在编状态 = "编外" → 标准金额 300元
```

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 存量员工无数据 | 历史员工信息中无在编状态 | 显示为空（NULL），由客户手动维护或批量导入补充 |
| 导入值为空 | Excel导入时该列未填写 | 保持为空，不默认填充 |
| 导入值错误 | 填写了"编内"、"编外"之外的值 | 校验失败，提示错误信息 |
| 公式引用为空 | 员工在编状态未填写 | 公式返回空值，不报错 |
| 公式计算类型 | 在编状态作为枚举值参与计算 | 按文本类型处理，支持等于/不等于判断 |

**存量数据处理**

**方案**：保持为空
• 存量员工在编状态为空（NULL）
• 由客户根据实际业务情况手动维护或批量导入补充

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 审批流条件分支 | 支持引用 | 在编状态可作为审批流条件分支的判断字段 |
| 费用标准规则 | 支持引用 | 在编状态可作为费用标准规则的匹配条件 |
| 报表筛选/统计 | 支持引用 | 在编状态可作为报表的筛选条件和统计维度 |
| OpenAPI接口 | 字段扩展 | 员工查询/推送接口返回数据中包含在编状态 |

## **附录**

**A. 典型使用场景**

场景：某事业单位区分编内编外人员，采用不同的差旅标准

配置步骤：
1. HR在员工管理维护每位员工的"在编状态"（编内/编外）
2. 财务设置费用标准规则，按"在编状态"区分标准
    - 在编状态 = "编内" → 住宿标准 500元/晚
    - 在编状态 = "编外" → 住宿标准 300元/晚
1. 表单设计器中，差旅报销单增加"在编状态"显示字段（自动带出）

使用步骤：
1. 员工提交差旅报销单
2. 选择申请人/人员后，系统自动带出"在编状态"
3. 系统根据在编状态匹配对应的费用标准
4. 超额时按对应标准提示或控制

结果：实现了编内编外人员的差异化费用管控

**C. 字段枚举值对照表**

| **存储值（建议）** | **显示值** | **使用场景** |
|-------------------------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| '编内' | 编内 | 正式编制人员 |
| '编外' | 编外 | 外包/临时工/合同工 |
| NULL / '' | 空 | 未设置（默认） |

**C. 与其他系统的集成**

若客户从HR系统同步员工数据：
• 需在同步映射中增加"在编状态"字段映射
• 若HR系统使用其他编码（如1/2、A/B），需在同步时转换

## 1、需求背景

客户：健民

**背景/需求：**

1）我们针对健民，在操作日志-数据对接日志中 增加了单据推送金蝶 的日志信息，推送失败时，需要能够手动重新推送

2）现在在进行入参数据查看时，当内容较多，气泡会展示不全，需要能支持点击查看全部详情（弹窗展示）

## 5、需求详情

1、列表中新增一列操作栏：操作栏中显示“重新推送”操作项；

针对由我们调三方的接口，**主动推送数据**的这一类日志数据，“对接系统”为“XXX推送”时，当状态为“**失败**”时，可点击“**重新推送**”按钮，点击后将该条数据进行重新推送；

2、操作栏**固定在列表尾部**展示，不受列表左右滑动进度条的影响；



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/8K4nyeZXLkQ6jnLb/img/f9762ce7-a2b8-471f-af63-5dd89a1607da.png?Expires=1778767384&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=8XGxNg%2FifhzMR44F7SB3gMyKjOg%3D "")
1. **“调用入参数据”“调用返回值数据”**，两列支持点击查看接口详情。当内容一屏展示不完时，可上下滑动展示；（排版样式根据技术专业样式调一调）

注：当返回为 json 格式时，解析为按照 json 的格式化数据

当返回为非 json 格式时，直接按照文本的形式展示即可

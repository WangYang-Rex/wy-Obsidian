## **第一部分：场景2 外币对外币推凭优化**

### **1\. 需求背景**

**客户**：万里汇**问题**：支付外币、对方收外币时，推凭后借方被错误换算成人民币，自动带出汇兑损益分录

### **2\. 详细业务规则**

#### **2.1 触发条件判定**


| **判定项** | **判定逻辑** | **数据来源** |
|-------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 支付币种 | 单据中【支付金额】组件的币种 | `payment.currency` |
| 收款币种 | 收款信息中的账户币种 | `receipt.account&#95;currency` |
| 本位币 | 企业设置的账套本位币 | `company.base&#95;currency` |

**触发公式**：


```plain
IF payment_currency == receipt_currency AND payment_currency != base_currency:
    走【外币对外币】推凭逻辑
ELSE:
    走其他逻辑
```


#### **2.2 凭证分录生成规则**

**借方分录**：


| **字段** | **取值** | **数据类型** | **备注** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 科目代码 | 费用类型映射的科目 | VARCHAR(50) | 根据费用类型配置 |
| 科目名称 | 费用类型映射的科目名称 | VARCHAR(100) |  |
| 币别 | 支付币种 | VARCHAR(10) | USD/EUR/GBP等 |
| 原币金额 | 支付金额 | DECIMAL(18,2) | 正数 |
| 汇率 | 单据汇率 | DECIMAL(18,6) | 单据维护的汇率 |
| 本位币金额 | 原币金额 × 汇率 | DECIMAL(18,2) | 自动计算 |

**贷方分录**：


| **字段** | **取值** | **数据类型** | **备注** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 科目代码 | 银行账户映射的科目 | VARCHAR(50) | 根据收款账户配置 |
| 科目名称 | 银行账户映射的科目名称 | VARCHAR(100) |  |
| 币别 | 支付币种 | VARCHAR(10) | 与借方一致 |
| 原币金额 | 支付金额 | DECIMAL(18,2) | 负数（贷方） |
| 汇率 | 单据汇率 | DECIMAL(18,6) | 与借方一致 |
| 本位币金额 | 原币金额 × 汇率 | DECIMAL(18,2) | 负数（贷方） |

**汇兑损益**：不生成

## **第二部分：场景3 外币账户人民币支付推凭优化**

### **1\. 需求背景**

**问题**：外币账户支付人民币时，系统按汇率倒推原币，提示借贷不平

### **2\. 详细业务规则**

#### **2.1 触发条件判定**

**表格**


| **判定项** | **判定值** | **数据来源** |
|-------------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 支付币种 | CNY | `payment.currency = 'CNY'` |
| 账户类型 | 外币账户或多币种账户 | `account.type` |
| 账户币种 | 非CNY（如USD） | `account.currency != 'CNY'` |

**触发公式**：

**plain**

复制
```plain
IF payment_currency == 'CNY' AND account_currency != 'CNY':
    走【外币账户人民币支付】推凭逻辑
```


#### **2.2 凭证分录生成规则**

**借方分录**：

**表格**


| **字段** | **取值** | **数据类型** | **备注** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 科目代码 | 费用类型映射的科目 | VARCHAR(50) |  |
| 币别 | CNY | VARCHAR(10) | 固定人民币 |
| 原币金额 | 支付金额 | DECIMAL(18,2) |  |
| 汇率 | 1 | DECIMAL(18,6) | 人民币汇率为1 |
| 本位币金额 | 支付金额 | DECIMAL(18,2) | 等于原币金额 |

**贷方分录**：

**表格**


| **字段** | **取值** | **数据类型** | **备注** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 科目代码 | 银行账户对应的人民币科目 | VARCHAR(50) | 需配置映射 |
| 币别 | CNY | VARCHAR(10) | 固定人民币 |
| 原币金额 | -支付金额 | DECIMAL(18,2) | 负数 |
| 汇率 | 1 | DECIMAL(18,6) |  |
| 本位币金额 | -支付金额 | DECIMAL(18,2) | 负数 |

**关键规则**：不进行汇率换算，不倒推原币

#### **2.3 科目映射配置**

**配置路径**：财务管理 → 基础设置 → 外币账户科目映射

**表格**


| **配置项** | **说明** | **必填** |
|-------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| 外币账户 | 选择外币银行账户 | 是 |
| 外币科目 | 外币核算的银行科目 | 是 |
| 人民币科目 | 用于人民币支付的银行科目 | 是 |
| 默认币种 | CNY/外币 | 是 |

## **第五部分：预算占用释放逻辑优化**

### **1\. 详细流程设计**

**现状流程**：

**plain**

复制
```plain
报销单提交 → 校验剩余预算 → 预算不足 → 提交失败
```


**优化后流程**：

**plain**

复制
```plain
报销单关联申请单 → 预释放申请单预算 → 校验可用额度 → 通过
     ↓
提交审批 → 正式释放申请单预算，占用报销单预算
     ↓
审批通过 → 预算转移完成
     ↓
审批驳回 → 恢复申请单预算
```


### **2\. 预算状态流转**

**表格**


| **阶段** | **申请单预算** | **报销单预算** | **操作** |
|----------------------------------------------------------|-------------------------------------------------------------------|-------------------------------------------------------------------|----------------------------------------------------------|
| 申请单已提交 | 已占用 | - | 申请单占用预算 |
| 报销单关联申请单 | 预释放 | 预占用 | 系统预释放/预占用 |
| 报销单提交审批 | 已释放 | 已占用 | 正式转移 |
| 审批通过 | 已释放 | 已占用 | 保持状态 |
| 审批驳回 | 恢复占用 | 释放 | 回滚状态 |

### **3\. 数据库字段设计**

**预算占用记录表（budget\_occupy\_record）**：

**表格**


| **字段** | **类型** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| `id` | BIGINT | 主键 |
| `budget&#95;rule&#95;id` | BIGINT | 预算规则ID |
| `document&#95;id` | BIGINT | 单据ID |
| `document&#95;type` | VARCHAR(50) | 单据类型 |
| `occupy&#95;type` | TINYINT | 占用类型：1申请单 2报销单 |
| `occupy&#95;status` | TINYINT | 状态：1已占用 2预释放 3已释放 |
| `amount` | DECIMAL(18,2) | 占用金额 |
| `related&#95;document&#95;id` | BIGINT | 关联单据ID（报销单关联申请单） |
| `create&#95;time` | DATETIME | 创建时间 |
| `release&#95;time` | DATETIME | 释放时间 |

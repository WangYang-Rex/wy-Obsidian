# 支持按发票类目计算税额

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20230718 | v1.0 | 新建 |

## 1、需求背景

客户：省财开、同心

根据税费抵扣规定，同一个发票类型的发票，需要根据发票类目进行区分，部分发票类目可以进行税额申报。比如同为增值税普通发票，当发票类目为住宿服务费时，可以进行税额抵扣。当发票类目为餐饮服务或生活服务时却不允许抵扣。因此，客户需要按发票类目去计算每个类目的税额。

## 2、业务流程/架构

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 自定义单据/费用 | 税率税额组件支持设置，需要填写发票类目与可抵扣税额的计算 |  |
| 税额填充 | 根据自定义需要填充的税额的发票类型 |  |
| 可抵扣税额的计算 | 支持在提交单据时即计算可抵扣税额 |  |
| 单据/费用填写 | 税率税额组件支持按发票类目自动带入与填写 |  |

#### 4.1.2 可抵扣税额

**增加勾选项：“可抵扣税额”，默认不勾选**

一、右侧帮助icon，鼠标移入帮助文案：将按照设置的发票类型或发票类目自动计算可抵扣税额

二、**勾选后**，则在提交单据和费用时，税率税额组件中新增“可抵扣税额”的填写，同税额属性，默认为空，非必填，可编辑。
1. UI：单税率税额/多税率税额组件中均增加这个字段的显示，见上面发票类目的两张图
2. 单税率税额则在“税额”字段下，新增可抵扣税额的显示，币种显示随税额字段变化不可编辑，金额可编辑；
3. 多税率税额则新增一列“可抵扣税额”，可编辑，且右上角增加可抵扣税额的合计值统计。
4. 计算规则：
5. 税额组件中，数据只要满足其中一项配置（发票类型或发票类目），就将可抵扣税额带入 ，可抵扣税额=税额；不符合条件则不带入；
6. 多税率税额-可抵扣税额的合计值，为税率税额组件中，各明细行的合计。

三、其他：
7. 分摊中的字段兼容：若税率税额组件中有可抵扣税额，在分摊模版中也增加“可抵扣税额”列，放在税额列后面，分摊中可抵扣税额=税率税额组件中的可抵扣税额合计值\*分摊比例，可编辑；
8. 计算公式：兼容“可抵扣金额”；
9. 单据详情：无可抵扣金额时，该字段显示币种\+0.00，跟目前的“税额”字段一样；

### 4.2、发票上传税额自动填充开关

基础设置-全局设置-发票设置中，新增开关“自动同步税额”



![Picture 5](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jJG8VYbq3p/img/a39ba274-960f-4860-a317-915dc40f5fe7.png?Expires=1778767453&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=URE%2BvNVYD8PSTbYwtE%2B8y56UX8I%3D "")

一、帮助文案：开启后，上传发票时将发票中的税额自动同步到税额字段中；

二、默认开启：默认配置“增值税纸质专用发票、增值税电子专用发票”，可以下拉选择其他发票类型（增值税纸质专用发票、增值税电子专用发票、增值税电子普通发票、增值税纸质普通发票、普通发票、其他发票、通用电子发票），支持多选与删除；

  \*\*开启后，若当前单据或费用明细中有税额组件时，上传对应发票类型的发票时，将指定类型发票中的税额填充到税额组件中\*\*，只有在可抵扣税额字段不存在值时，才自动填充，否则不自动填充；

三、开关关闭：关闭后，发票上传后不自动填充税额。

四、配置后自动保存配置

### 4.3、发票类目与可抵扣金额填写逻辑

有以下入口，当税率税额组件中发票类目字段时，支持填写与自动填充：手工填写税率税额信息；手工创建消费明细与单据时，上传发票后填充税率税额；OCR识票后自动生成消费。

一、手工填写税率税额

发票类目支持模糊搜索，单选；

可抵扣税额根据税率税额组件中的设置，符合条件（发票类型与发票类目）的自动带入；

二、手工填写消费明细或单据，发票上传生成消费时；与OCR识票自动生成消费时，当税率税额组件中有发票类目时，发票类目填充逻辑

**1、单税率税额：**
- 上传一张或多张发票，同一个发票类目，直接带入发票类目字段；
- 上传一张或多张发票，有多个不同发票类目且类目不唯一，发票类目字段不带入；
- 上传一张或多张发票，有多个不同发票类目且类目唯一，取类目的唯一值填入到发票类目字段；
- 税率税额中发票类目已经有值，再次上传发票，发票类目不做自动变更；发票类目中没有值，则将发票类目值填充。

**2、多税率税额：**
- 同一张发票，发票类目都相同，发票类目字段直接带入，生成一条税额记录；
- 同一张发票，有多个不同发票类目，每个发票类目明细，一个发票类目生成一条税额记录，带入发票类目信息；
- 多张发票同时上传，有多个不同发票类目，同一个发票类目合并生成一条税额记录，将发票类目信息带入；
- 税率税额组件已经有数据，再次上传发票，已存在的数据不做变动，针对此次上传的发票再次按上述规则填充税额数据。

三、其他规则：
10. 金额取值：
11. 如果税额组件中设置了“填写发票类目”，税额取发票明细中的税额填充进税额字段；同时带入发票类目信息；
12. **如果组件中有“可抵扣税额”字段，在发票上传自动填充税率税额组件数据时，符合条件（指定的发票类型或发票类目）的，将“可抵扣税额”字段带入，可抵扣税额=税额。**
13. 同一张发票，多个发票科目分别生成到多个消费明细中：场景兼容，将发票中的发票类目与可抵扣税额值填充到消费明细中；
14. 费用拆分：兼容，将被拆分消费明细中的发票类目与可抵扣税额带入新的消费明细中；
15. 发票类目新增：若识别到的发票类目不存在，自动新增发票类目---按以前逻辑
16. 无发票类目的发票：根据发票类型，将发票类型填充到发票类目字段（如出租车、机票行程单、火车票、汽车票、ETC发票）---按以前逻辑

### 4.4、会计科目映射

一、会计科目映射-基础设置-可抵扣发票类型中，同时支持设置发票类目：
- 下拉列表中：分为两个大类“发票类型”“发票类目”，支持多选，模糊搜索支持搜索发票类型与发票类目
- 文案“可抵扣发票类型”更改为“可抵扣发票”
- 税额组件中，数据只要满足其中一项配置（发票类型或发票类目），在生成凭证时，就需要生成税额分录
- 显示区域：页面固定一个最大高度，超出则使用鼠标滚动查看

二、税额科目映射

1、税额科目映射-发票字段中，新增字段“发票类目”的勾选

2、勾选后，可下拉选择的发票类目选项，为基础设置-可抵扣发票类型中设置的发票类目。

3、税额科目的匹配：
- 基础设置中已设置的发票类目，若单据中有对应指定的发票类目数据，则在生成凭证时，必须生成税额分录，若没有匹配到映射关系，则取缺省科目；
- 生成凭证时，增加发票类目的匹配，税额组件中，数据只要满足其中一项配置（发票类型或发票类目），在生成凭证时，就需要生成税额分录若配置了发票类目字段为匹配条件，则支持按税率税额组件中的发票类目去匹配科目；

4、金额的取值：
- **按优先级取值：可抵扣税额\>税额，若单据中有可抵扣税额（是指表单有可抵扣税额字段，就取可抵扣税额），优先取可抵扣税额字段参与计算，没有则取税额；计算规则同原先税额的计算。**
- 分摊：若有分摊，在计算时，需要增加按发票类目去计算可抵扣税额的校验。

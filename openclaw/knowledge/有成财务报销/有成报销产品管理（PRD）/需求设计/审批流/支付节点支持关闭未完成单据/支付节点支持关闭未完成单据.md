## 1、需求背景

客户：武汉科前

员工提交的付款单，审批通过后，存在只需要支付部分金额，后续不需要再支付了，此时需要给到对应出纳权限，对这部分单据进行直接关闭，不再进行支付。

## 2、业务流程/架构

如复杂需求，放置相关流程/架构图

## 3、功能清单

| **模块** | **功能点** |
|----------|-------------|
| 审批流控制 | 增加是否允许关闭的控制 |
| 关闭信息确认 | 关闭时，需要确认一些必要信息：已支付的金额、单据之间的关联登 |
| 审批历史变更 | 审批历史留存关闭的相关操作记录与详情 |
| 单据状态处理 | 新增关闭的单据状态处理逻辑 |
| 单据列表处理 | 单据列表中兼容关闭的单据 |

## 4、需求详情

**只做新版审批流**

**做PC端，不做手机端**

### 4.1、审批流节点控制


![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/3BMqYybN5121AqwZ/img/beb9db6b-c243-4e16-9a6a-3c368fa4e5f6.png?Expires=1774156920&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=c3X%2F3gcfnDCz9yWVZ%2Fat%2FLLWuow%3D "")

自定义审批流-出纳节点中，新增配置“允许关闭未完成的单据”：

一、鼠标移入显示帮助文案，帮助文案：开启后，未审批完成的单据可手动结束流程，结束后该单据无法再继续进行后续的支付审批等所有操作。

二、默认不勾选，现有逻辑，正常可进行支付

三、勾选后：

1、在待办入口，点击查看单据详情时，当前节点的单据详情页，显示关闭按钮；已支付完成的单据不展示这个按钮。

2、关闭按钮：电机打开另一个信息确认页。在信息确认页确认关闭后，需要回到这个页面：

**其中有以下情况不允许关闭**，点击关闭按钮给到对应提示：
- 当前单据有待复核的数据，则提示“当前单据存在待复核任务，请先处理完待复核任务后再关闭”；
- 单据还未进行支付，则提示“单据还未支付，不支持关闭”--相当于只有在多次打款开关打开时，这个按钮才有效；
- 在线支付场景：在线支付时，支付的信息还未收到银行回调信息--支付状态还未明确时，也不允许关闭，点击提示“当前单据还存在支付未完成的任务，请稍候再关闭”
- 付款单关联的有计划地合同时，不允许关闭，点击提示“当前付款单关联了合同，暂不允许关闭”。
- 收款单本次暂不支持关闭---若是收款单将这个关闭按钮隐藏。

### 4.2、关闭后的流程

#### 4.2.1、关闭时的信息确认

一、相关信息；

1、部分信息可修改；以上组件的显示，按照原型图中的先后顺序依次排列显示；
- 点击给到提示语：关闭后，单据将变为已完成状态，无法再进行支付
- 同时提示用户需要确认的信息：应付总额、已支付金额、待支付金额，一直显示

2、涉及到的需要确认组件有：常用字段-金额、常用字段-支付金额、组件-收款信息、组件-费用明细、组件-关联单据
- 以上组件，当前单据中有，则在关闭确认信息页显示，若当前单据中没有对应组件，则在关闭确认的信息页不显示；

| **组件/字段** | **说明** |
|-----------------|----------|
| 应付总额 | 当前单据的应付总额，只显示，不可修改 |
| 已支付金额 | 当前单据已支付的金额不可修改 |
| 待支付金额 | 当前单据已支付的金额不可修改 |
| 收款信息 | 一、组件顶部显示当前账户信息里面的应付总额、已支付金额、和待支付金额：1、应付总额：单据中填写的每个账户填写的金额总和2、已支付金额：出纳已支付的每个账户的金额3、待支付金额：应付总额-已支付金额；二、列表中的字段：1、展示收款信息相关字段，账户信息字段不可修改（收款银行、开户行、收款账号、收款人、备注）；2、应付金额：当前单据每个账户填写的金额，不可编辑；3、已支付金额：出纳支付时当前账户已经支付的金额；不可编辑； |
| 费用明细 | 一、显示当前单据中所有的费用明细，默认展示的列表字段有：序号、费用类型、金额、已支付金额、日期、税额、已支付税额字段：1、金额：费用中填写的常用字段-金额字段；2、税额字段：多个税额字段逗号隔开显示；3、已支付金额：默认等于费用金额，点击记录进去可编辑修改；4、已支付税额：默认等于可抵扣税额字段或税额字段，点击记录进去可编辑修改；二、点击单行记录，打开费用明细编辑页，这个页面跟原有的费用明细编辑页有所不同；将需要修改的组件放在前面，最后面放费用的基础信息。1、已支付金额：默认带入当前费用的金额，可点击，手动修改金额\+ 有币种信息带入“金额”字段的币种信息，币种不可修改；2、税率税额组件：当前费用中税率税额组件才显示，否则不显示；\+ 显示当前费用中已有的税率税额字段：发票类目和可抵扣税额字段在费用中不存在时，这里也不显示；\+ 增加列“已支付税额”：默认带入可抵扣税额或者税额；点击可修改金额有可抵扣税额时优先取可抵扣税额，没有这个字段取税额。\+ 单税率税额组件，在原有组件下新增一个输入项“已支付税额”的填写3、费用分摊：当前费用明细中有分摊时，显示分摊明细组件编辑，没有则不显示；在原有费用明细字段基础上增加“已支付金额”和“已支付税额”的字段；\+ “已支付金额”加在“承担金额”后面，根据承担比例*费用的已支付金额，自动带入；\+ “已支付税额”：根据承担比例*可抵扣税额或者税额，自动带入；\+ 分摊纬度：当前单据中有的分摊纬度就显示。有可抵扣税额时优先取可抵扣税额，没有这个字段取税额。三、校验逻辑：1、费用明细中，每条费用编辑时的校验\+ 费用中的已支付金额不能大于费用的金额；\+ 已支付税额不能大于可抵扣税额；没有可抵扣税额时，已支付税额不能大于税额；\+ 分摊中的已支付金额不能大于每条对应的承担金额；分摊中的支付金额、税额总和需要等于费用的已支付金额/税额；2、整个费用明细组件中的校验：\+ 费用中的已支付金额必须等于整单的已支付金额 |
| 关联单据 | 一、在**关闭报销单**时显示这个组件，同时可以点击编辑主消费中的关联金额，其他单据关闭不显示这个组件2、关联主消费和关联费用固定在右侧，拉动列不影响这两列；3、展示要素见原型中，只有关联主消费可编辑，其他字段不可编辑：\+ 关联主消费显示当前关联的费用类型名称和关联金额；点击打开编辑费用的弹框；\+ **编辑费用弹框：在以前字段的基础上，增加“已支付金额”“原关联金额”“实际关联金额”的显示：已支付金额：显示当前关联的费用中已维护的已支付金额；原关联金额：原来单据中已填写的关联金额；实际关联金额：默认带入单据中已填写的关联金额，可修改；**\+ **校验：关联金额不能大于已支付金额；若关联金额大于已支付金额，则红框显示。\*二、\*付款单**关联了无计划合同和有计划合同情况不同，不同处理：1、若付款单关联了无计划合同，显示这个组件，其他情况不显示：\+ 在原有字段基础上，增加“实际使用金额”字段；\+ 自动带入金额，可编辑；\+ 每一行的实际使用金额需要小于当前行次的“金额”，同时整个组件的实际使用金额的总和必须等于已支付金额。2、若付款单关联了有计划的合同，不显示这个组件；本次需求暂不支持这个场景，后续迭代；未支付完成的单据有关联付款计划，点击关闭提示“当前单据已关联付款计划，暂不支持关闭”。 |
| 确认关闭 | 一、点击进行相应校验：1、费用明细组件中的支付金额必须等于整单的已支付金额；2、关联关联单据组件中的关联主消费-关联金额，不能大于每个关联费用中的已支付金额；二、单据字段信息校验通过后给二次弹框确认，并输入关闭原因后才能关闭。1、弹框文案为：关闭后，单据将变为已完成状态，无法再进行支付，请输入关闭原因再关闭流程。\+ 取消：关闭弹窗；\+ 确认：进行单据关闭操作，同时关闭当前窗口，回到之前打开的单据详情页；三、关闭结果：关闭成功：回到单据详情页；关闭失败：弹框给到报错提示，页面停留在当前页面。 |

**3、在自定义报表中：**

**“已支付金额”字段要取实际支付金额；**

**同时增加“未支付金额”的字段选择**

3.1、明细表：
- 字段选择与汇总字段中，增加“未支付金额”的选项；

3.2、柱状图：
- 纵坐标：增加“未支付金额”的选项

3.3、饼图：
- 统计内容：增加“未支付金额”的选项

#### 4.2.4、核销关系释放

一、报销单：
- 报销单关联申请单，申请单可用金额的释放：会出现关联单据编辑关联主消费组件，根据最后填写的“关联金额”字段，**释放出申请单对应的可用金额与预算；**

二、付款单：
- 付款单：若关联无计划合同，最终合同中生成的计划，以最终支付金额为准生成；
- 关联有计划的合同，本期暂不考虑；关联有计划合同的付款单不允许关闭。
- 若单据中有付款场景付款单中的到票金额，取当前付款单中最终的已支付金额生成。

已支付金额若大于到票金额，则未到票金额=已支付金额-到票金额；

已支付金额若小于等于到票金额，则未到票金额取0；

到票金额始终以填入的数据为准；
- 到票/未到票金额的影响有：项目报表-合同应付款中的已开票金额和未开票金额；自定义报表-明细表、柱状图（纵坐标）、饼图（统计内容）中的已开票金额和未开票金额所有列表字段显示中“已到票、未到票”金额的统计；比如我的-单据-发票核销管理-已到票、未到票统计；单据管理-付款管理中的已到票、未到票统计；财务管理中的待收发票；

三、借款单：
- 借款单的整单待还金额为最终支付金额；

四、费用标准释放---参照编辑单据-编辑费用时的费用标准重算逻辑-

计算规则同现有规则，涉及到的标准类型有以下类型：
- 住宿标准：根据费用明细中最终支付的金额，重新计算当前单据费用标准的占用情况；
- 补贴标准：当补贴金额按范围控制时，按当前费用最终的支付金额，重新更新当前申请人与同行人的费用标准占用情况；
- 其他标准：按当前费用的最终的支付金额去更新费用标准占用情况，客户标准和部门标准中的按次不受影响；

---以上计算包含申请人和同行人；多个同行人关闭后标准等这些如何计算

五、预算相关

1、预算释放与金额处理：
- 对当前单据的影响：取单据中每项最终的支付金额，取重新占用预算。
- 申请单释放预算按表单中填写的关联金额释放；提交单据时，预占的金额，在关闭之后，需要将未支付的金额释放掉，释放的纬度需要按照原先占用的纬度释放；--预算报表查看时需要兼容此逻辑
- 单据详情中-预算详情里面，当前单据占用金额取最终支付金额显示；

### 4.3、流程监督功能

一、流程监督功能需要支持关闭未支付完成的单据；

1、单据列表中将支付状态为支付中的单据显示出来；并只支持单笔操作；

2、批量同意和驳回操作：不支持选中待支付的单据，选中了待支付的单据，则在点击批量操作时，弹窗提示；

二、流程监督中点开单据详情，单据状态为支付中；

1、在原有“流程干预”按钮这种状态下不显示；

2、增加“关闭”按钮，点击之后按上述逻辑进行相应的校验，校验逻辑同4.1中的描述；

3、点击关闭后跳转至信息确认页，成功关闭后，记录操作人的信息，在审批历史中；已关闭的单据不在当前待处理人的待支付列表中，转到已支付列表。

## **1\. 需求背景**

### **1.1 需求描述**

审批流连续直属领导审批功能增强，支持**三个独立功能点**，在保留原有「指定角色不参与审批」逻辑的基础上，新增「指定角色必须实际审批」逻辑：
1. **依次审批** → 「申请人连续直属领导审批」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」
2. **依次审批** → 「通过单据员工连续直属领导匹配」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」
3. **非会签** → 「连续部门主管审批」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」

### **1.2 需求背景**

**需求方**：企业管理员/审批流配置人员

**现状问题**：
1. 「申请人连续直属领导审批」匹配到指定角色时**流程提前结束，被指定角色不参与审批**（旧逻辑）
2. 「通过单据员工连续直属领导匹配」匹配到指定角色时**流程提前结束，被指定角色不参与审批**（旧逻辑）
3. 「连续部门主管审批」（非会签）匹配到指定角色时**流程提前结束，被指定角色不参与审批**（旧逻辑）
4. 部分业务场景需要指定角色**实际参与审批**，而非自动结束

**解决思路**：
1. **保留旧逻辑**：匹配到指定角色时流程结束，该角色不参与审批
2. **新增新逻辑**：匹配到指定角色时，该角色必须实际审批后才能结束
3. 通过界面配置项「指定角色审批方式」让用户自主选择

**业务流程对比**
```plain
【旧逻辑：指定角色不参与审批】
员工提交单据
    ↓
逐级查找：B（一级）→ C（二级）→ D（三级，指定角色）
    ↓
B审批 → C审批 → 流程结束（D不参与）

【新逻辑：指定角色必须审批】
员工提交单据
    ↓
逐级查找：B（一级）→ C（二级）→ D（三级，指定角色）
    ↓
B审批 → C审批 → D审批（D必须审批）→ 流程结束
```

```plain

    ↓

    ↓
```

**业务流程 - 新逻辑**
```plain
申请人提交单据
    ↓
逐级查找直属领导链
    ↓

    ↓
是 → 该领导加入审批链，作为最后一个审批人
    ↓
生成完整审批链，按层级依次审批
    ↓

```

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案 - 旧逻辑** | **处理方案 - 新逻辑** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------------------|----------------------------------------------------------------------------|
| 员工无直属领导 | 单据中员工未设置直属上级 | 根据若该审批人控制配置判断【自动跳过/无法提交审批】 | 根据若该审批人控制配置判断【自动跳过/无法提交审批】 |
| 直属领导链断裂 | 中间某级直属领导不存在/离职 | 指定角色终止不生效，流程按直属领导依次全部审批 | 指定角色终止不生效，流程按直属领导依次全部审批 |
| 指定角色未命中 | 审批链中无匹配指定角色 | 指定角色终止不生效，流程按直属领导依次全部审批 | 指定角色终止不生效，流程按直属领导依次全部审批 |
| 多人组件取第一个 | 员工组件匹配到多人 | 取展示在最前面的员工 | 取展示在最前面的员工 |
| 审批人重复 | 连续直属领导链中有重复人员 | 去重后结束 | 去重处理 |

## **附录**

### **A. 典型使用场景**

#### **场景一：旧逻辑 - 指定角色无需审批（存量兼容）**

**配置**：
- 审批方式：依次审批
- 审批人：申请人连续直属领导审批
- 勾选「审批到指定角色时终止」，选择「总监」
- **指定角色审批方式：当前审批人无需审批**

**流程**：
```plain
员工提交 → B（经理）审批 → C（主管）审批 → 流程结束
                                ↑
                        总监无需审批
```


#### **场景二：新逻辑 - 指定角色需要审批**

**配置**：
- 审批方式：依次审批
- 审批人：申请人连续直属领导审批
- 勾选「审批到指定角色时终止」，选择「VP」
- **指定角色审批方式：当前审批人需要审批**

**流程**：
```plain
员工提交 → B（经理）审批 → C（总监）审批 → D（VP）审批 → 结束
                                            ↑
                                    VP必须审批
```


#### **场景三：按项目成员链，指定角色需要审批**

**配置**：
- 审批方式：依次审批
- 审批人：通过单据员工连续直属领导匹配
- 单据员工匹配：项目成员
- 勾选「审批到指定角色时终止」，选择「部门负责人」
- **指定角色审批方式：当前审批人需要审批**

**流程**：
```plain
员工提交（项目成员=B）
    ↓
查找B的直属领导链：C（组长）→ D（经理）→ E（部门负责人）
    ↓
依次审批：C审批 → D审批 → E（部门负责人）审批 → 结束
                                    ↑
                            部门负责人必须审批
```

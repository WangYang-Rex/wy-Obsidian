## 1\. 需求背景

### 1.1 需求描述

在「基础设置-自定义审批流管理」页面增加「提交时显示审批流确认」开关，支持控制**申请人提单时**是否弹出审批流程设置页；当审批流节点中启用了**现有产品逻辑里需要申请人在提单时进行配置操作的功能**时，豁免开关限制，强制弹窗。

### 1.2 需求背景
- **需求方**：企业管理员/申请人
- **现状问题**：
• 申请人提交单据时，必须弹出审批流程设置页并点击"确定"后才能提交，操作路径较长
• 大部分企业的审批流配置固定，申请人无需在提单前进行任何调整，弹窗成为额外操作负担
• 缺少灵活配置能力，无法根据企业实际管理需求选择简化或保留确认环节
- **解决思路**：
• 在「基础设置-自定义审批流管理」页面右上角【更多】菜单中增加【审批流确认设置】入口，管理员可在弹窗中设置关闭审批流确认弹窗，简化提交流程
• 保留豁免机制：当审批流节点中启用了任何**需要申请人在提单时进行配置操作的功能**时，强制弹窗以确保业务完整性
• 默认保持现状（开启弹窗），避免影响存量用户习惯

**业务流程**
```
申请人点击【提交】
↓
判断：审批流节点是否启用了"需申请人提单时配置"的功能？
├─ 是 → 强制弹出审批流设置页 → 申请人完成必要配置 → 提交单据
└─ 否 → 判断：「提交时显示审批流确认」开关是否开启？
         ├─ 开启 → 弹出审批流设置页 → 申请人确认 → 提交单据
         └─ 关闭 → 直接提交单据（无弹窗）

```

## 2\. 功能现状

**现有功能位置**：
- 配置入口：基础设置 → 自定义审批流管理 → 右上角【更多】菜单 → 审批流确认设置
- 生效端：PC端单据详情页 / MB端单据详情页 → 点击【提交】按钮

**现行逻辑-审批流中与申请人有关的配置（提单时）**：

审批流节点支持以下配置，当开启时，申请人需在**提单时**于审批流确认弹窗中进行操作：

| 配置项 | 字段/条件 | 说明 |
|---------|-------------|------|
| **审批节点-手动选择审批人** | manualSelectSwitch=true | 申请人可在提单时选择/修改该节点的审批人 |
| **抄送节点-手动选择抄送人** | ccManualSelectSwitch=true | 申请人可在提单时选择/修改该节点的抄送人 |
| **出纳节点-手动选择出纳人** | cashierManualSelectSwitch=true | 申请人可在提单时选择/修改该节点的出纳人 |
| **节点字段权限-申请人可编辑** | 字段权限配置中申请人拥有编辑权 | 提单时审批流确认弹窗中展示可编辑字段，供申请人填写/修改 |
| **抄送节点-指定费用的同行人** | 抄送范围配置为"指定费用的同行人" | 申请人需在提单时确认或选择同行人抄送对象 |

**现有提交流程**：
1. 申请人填写完单据信息后点击【提交】
2. 系统匹配审批流，检查节点配置： 
    - 若存在需申请人操作的配置（如手动选审批人、允许修改字段等），弹窗展示对应操作项
    - 若无特殊配置，弹窗仅展示审批节点列表供查看/确认
3. 申请人完成必要操作后，点击【确定】提交单据，点击【取消】返回编辑页

**缺失能力**：
- 无法关闭审批流确认弹窗，简化提交流程
- 无法区分"必须操作"与"仅确认"场景，统一弹窗造成操作负担

**核心逻辑**
```
申请人点击【提交】
↓
系统匹配单据对应的审批流
↓
判断：审批流节点是否启用了"需申请人提单时配置"的功能？
├─ 是 → 【强制弹窗】弹出审批流设置页
│       ↓

│       ↓
│       点击【确定】→ 提交单据
│       点击【取消】→ 返回编辑页
│
└─ 否 → 查询全局设置「提交时显示审批流确认」
        ↓
        判断：开关状态
        ├─ 开启 → 【正常弹窗】弹出审批流设置页
        │       ↓
        │       申请人查看/确认
        │       ↓
        │       点击【确定】→ 提交单据
        │       点击【取消】→ 返回编辑页
        │
        └─ 关闭 → 【直接提交】跳过弹窗，直接提交单据

```

**豁免逻辑说明**

只要审批流节点中启用了**现有产品逻辑里需要申请人在提单时对审批流确认弹窗进行配置操作的功能**，即豁免开关限制，强制弹窗：

| 豁免场景 | 配置条件 |
|------------|------------|
| 审批节点-由申请人手动选择审批人 | `manualSelectSwitch=true` |
| 抄送节点-由申请人手动选择抄送人 | `ccManualSelectSwitch=true` |
| 出纳节点-由申请人手动选择出纳人 | `cashierManualSelectSwitch=true` |
| 节点字段权限-申请人可编辑 | 字段权限配置中申请人拥有编辑权 |
| 其他需申请人在单据提交时审批流确认弹窗可编辑的场景 | 产品逻辑中判定为"需申请人编辑"的配置 |

**豁免检测时机** • 在申请人点击【提交】时实时检测
• 基于当前单据数据匹配审批流后的实际配置
• 若单据触发不同审批流（如条件分支），按实际命中的审批流判断

#### 3.2.3 PC端和MB端同步生效

**核心原则**：同一套全局配置，两端行为一致

**适配说明**

| 端 | 弹窗形式 | 说明 |
|---|------------|------|
| PC端 | 居中模态弹窗 | 保持现有弹窗样式 |
| MB端 | 底部抽屉弹窗 | 保持现有弹窗样式 |

**生效范围** • 所有单据类型：报销单、借款单、申请单、付款单等
• 所有提交入口：新建提交、草稿提交、驳回后重新提交
• 两端配置同源，确保体验一致性

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 豁免场景-需申请人配置 | 审批流节点启用了任何需要申请人在提单时配置的功能 | 强制弹窗，不受开关控制 |
| 开关关闭\+无豁免 | 开关关闭且审批流未配置任何"需申请人操作"的功能 | 直接提交，不弹窗 |
| 开关开启 | 无论是否需要申请人操作 | 正常弹窗（豁免场景需操作，非豁免场景供确认） |
| 审批流变更 | 审批流配置发生变更 | 下次提单时重新检测，按最新配置判断 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 委托授权提单 | 被委托人代提交单据 | 按被委托人实际身份判断开关和豁免逻辑 |
| OpenAPI推送单据 | 外部系统推送单据 | 不受此开关影响，按接口原有逻辑执行 |
| 移动端离线提交 | 离线状态下提交 | 在线后同步时按服务器配置判断 |

## 附录

### A. 典型使用场景

**场景1：简化提交流程（开关关闭）**

适用企业：审批流配置固定，无需申请人干预

配置步骤（首次）：
1. 管理员进入「基础设置 → 自定义审批流管理」
2. 点击页面右上角【更多（⋮）】按钮
3. 选择【审批流确认设置】
4. 关闭「提交时显示审批流确认」开关
5. 点击【确定】保存配置

使用步骤（日常）：
1. 申请人填写单据信息
2. 点击【提交】
3. 系统直接提交单据，无弹窗等待

结果：提交流程简化，申请人操作更高效

---

**场景2：保留确认环节（开关开启）**

适用企业：需申请人了解审批路径，或偶尔需调整

配置步骤（首次）：
1. 保持「提交时显示审批流确认」勾选（默认）

使用步骤（日常）：
1. 申请人填写单据信息
2. 点击【提交】
3. 弹出审批流设置页，申请人查看/确认
4. 点击【确定】提交单据

结果：申请人可确认审批流程，必要时可进行操作

---

**场景3：豁免场景（强制弹窗）**

适用企业：部分审批流需申请人在提单时手动选择审批人

配置步骤：
1. 审批流设置中配置某些节点启用"需申请人操作"的功能（如手动选择审批人、申请人可编辑字段等）
2. 「提交时显示审批流确认」开关可关闭（简化其他无需申请人配置的单据）

使用步骤：
1. 申请人填写命中该审批流的单据
2. 点击【提交】
3. 系统检测到审批流节点启用了需申请人配置的功能
4. 强制弹出审批流设置页
5. 申请人完成必要配置后提交

结果：既简化了大部分单据的提交流程，又保证了需要选择审批人场景的业务完整性

## 1\. 需求背景

### 1.1 需求描述

收款节点补齐币种/汇率设置能力，使收款流程与支付流程对齐：出纳在收款确认时可选择收款币种、设置汇率，系统自动记录原币金额；凭证生成时根据收款币种与单据本位币的差额自动计算汇兑损益，差额计入汇兑损益科目。

### 1.2 需求背景
- **需求方**：企业出纳、会计
- **现状问题**： 
    - 支付节点已支持币种/汇率设置及汇兑损益计算，收款节点完全缺失该能力
    - 客户以美元等外币付款时，出纳只能按人民币记账，无法记录实际收到的外币金额
    - 收款原币与单据本位币存在差额时，会计需手工调整汇兑损益分录，易出错
- **解决思路**：复用支付链路成熟的币种/汇率组件和汇兑损益计算逻辑，在收款节点补齐同等能力

**业务流程**
```
企业配置币种 & 汇率
        ↓
收款单审批流 → 到达出纳收款节点
        ↓
出纳在收款页面：
  - 看到应收金额（本位币）
  - 选择收款币种 + 设置汇率
  - 确认收款
        ↓
收款记录写入（含 receiveCurrencyCode / receiveExchangeRate / receiveOriginAmount）
        ↓
单据完成
        ↓
会计科目映射生成凭证时：
  - 取收款记录中的币种/汇率
  - 根据配置计算汇兑损益 → 生成汇兑损益分录

```

## 2\. 功能现状

### 2.1 支付节点币种/汇率功能

**位置**：审批流 → 出纳支付节点 → 确认支付弹窗

**现有能力**：
- 支付确认界面已集成 `foreignPayInput` 组件，支持选择支付币种、输入汇率
- 支付请求参数 `PayAccountRequest` 包含币种三字段：`payCurrencyCode`、`payExchangeRate`、`payOriginAmount`
- 支付记录表 `reimburse_order_payment_details` 已存储币种三字段
- 复核主表 `reimburse_pay_review` 同步存储币种三字段

**外币限制（当前硬编码）**：
- 线下支付（payMethodFlag=1）：支持外币
- 第三方支付（支付宝/微信/银企直联/云直联）：仅支持 CNY
- 多笔合并复核：不支持外币修改
- 有付款计划的单据：不支持外币修改
> 本次需求复用上述组件、字段结构和限制逻辑。   

### 2.2 汇兑损益计算现状

**位置**：account-mapping 子工程 → `CertificatePreviewProcessor.processExchangeLoss`

**现有能力**：
- 报销单（type=1）、借款单（type=2）、付款单（type=3）支持汇兑损益计算
- 汇率取值策略通过 `CorpAccountMapConfigPO.loanExchangeType` 配置 
    - `0` = `PAY_SYS_EXCHANGE`：使用支付时系统配置的汇率
    - `1` = `PAY_SYS_FILL`：使用支付时出纳填写的汇率
- 汇兑损益分录写入 `CertificateSubjectExtends`，sourceType="8"（EXCHANGE\_LOSS）

**缺失能力**：
- 收款单（type=8）未加入 `processExchangeLoss` 支持范围
- 收款记录无币种字段，无法传入汇兑损益计算
> 本次需求扩展 `processExchangeLoss` 支持收款单，并扩展 `loanExchangeType` 配置语义为收付通用。

### 3.2 详细功能描述

#### 3.2.1 审批流收款节点配置

**功能概述**

审批流设置中，出纳收款节点新增"允许非本位币收款"勾选框，控制该节点是否支持外币收款。

**配置入口**

审批流设置 → 出纳收款节点 → 节点配置

**默认配置**

| 配置项 | 默认状态 | 说明 |
|---------|------------|------|
| 允许非本位币收款 | 不勾选 | 默认保持现状，不展示币种选择组件 |

#### 3.2.2 收款确认界面币种/汇率设置

**功能概述**

出纳在收款确认弹窗中可选择收款币种、设置汇率，系统自动计算并展示原币金额。

**配置入口**

审批流 → 出纳收款节点 → 确认收款弹窗

**配置弹窗/界面**

未满足触发条件时（币种置灰，汇率和原币金额不展示）：
```
┌─────────────────────────────────────────┐
│ 确认收款                            [X] │
├─────────────────────────────────────────┤
│                                         │
│  应收金额（本位币）:  ¥ 100.00           │
│                                         │
│  收款币种:  [ CNY （置灰不可点） ]        │
│                                         │
│  收款方式:  [线下收款 ▼]                │
│  收款账户:  [请选择 ▼]                  │
│                                         │
├─────────────────────────────────────────┤
│                          [取消] [确认]  │
└─────────────────────────────────────────┘

```

满足触发条件时（币种可选，汇率和原币金额展示）：
```
┌─────────────────────────────────────────┐
│ 确认收款                            [X] │
├─────────────────────────────────────────┤
│                                         │
│  应收金额（本位币）:  ¥ 100.00           │
│                                         │
│  收款币种:  [CNY ▼]  →  [USD ▼]         │
│  汇率:      [ 7.250000 ]                │
│  原币金额:  [ 13.79 ] USD               │
│                                         │
│  收款方式:  [线下收款 ▼]                │
│  收款账户:  [请选择 ▼]                  │
│                                         │
│  提示：汇率支持6位小数                  │
│                                         │
├─────────────────────────────────────────┤
│                          [取消] [确认]  │
└─────────────────────────────────────────┘

```

**交互规则**
- 组件触发条件（同时满足）：
    1. 审批流节点设置中勾选了"允许非本位币收款"
    2. 单据本位币 ≠ 收款币种
    3. 收款方式为线下收款（非第三方支付）
    4. 无收款计划
- **未满足触发条件时的界面表现**：
    - 币种区域显示默认本位币币种，置灰不可点击
    - 收款汇率和换算本位币金额不显示
- **满足触发条件时的界面表现**：
    - 币种区域可使用下拉框选择其他币种
    - 收款汇率和换算本位币金额显示，允许用户填写
    - 默认汇率：取系统当前配置的该币种汇率（可修改）
    - 自动计算规则： 
        - 修改汇率或本位币金额 → 重算原币 = 本位币 / 汇率（保留2位小数）
        - 修改原币金额 → 重算汇率 = 本位币 / 原币（保留6位小数）

**默认配置**

| 字段 | 默认状态 | 说明 |
|------|------------|------|
| 收款币种 | 本位币 | 未满足条件时置灰展示；满足条件时默认可下拉选择其他币种 |
| 汇率 | 系统当前汇率 | 满足条件后自动填充，可手动修改 |
| 原币金额 | 自动计算 | 满足条件后根据本位币和汇率计算 |

#### 3.2.3 收款记录存储币种信息

**功能概述**

收款请求及记录增加币种三字段，确保收款币种、汇率、原币金额可追溯。

**处理逻辑**
- 收款请求参数新增字段（与支付参数语义对齐）： 
    - `receiveCurrencyCode`：收款币种编码（如 USD）
    - `receiveExchangeRate`：收款汇率（如 7.25）
    - `receiveOriginAmount`：原币金额
- 收款记录表（如 `reimburse_order_receive_details` 或现有收款记录表）新增字段： 
    - `receive_currency_code`：varchar，收款币种编码
    - `receive_exchange_rate`：decimal，收款汇率
    - `receive_origin_amount`：decimal，原币金额 **配置存储**
- 配置按收款记录存储，与单据号关联
- 收款记录保存后即时生效

#### 3.2.4 收款汇兑损益计算

**功能概述**

扩展会计科目映射的汇兑损益计算逻辑，支持收款单（type=8）根据收款币种/汇率自动生成汇兑损益分录。

**业务流程**
```
用户触发凭证生成
        ↓
processExchangeLoss 执行
        ↓
判断触发条件（5项全部满足）
  ├─ 不满足 → 跳过汇兑损益计算
  └─ 满足 → 继续
        ↓
根据 loanExchangeType 取值策略：
  ├─ 0: 按收款时间从汇率历史表查系统汇率
  └─ 1: 使用收款时填写的 receiveExchangeRate
        ↓
遍历凭证分录：
  - 汇总借方金额 totalBorrowAmount
  - 贷方分录按汇率重算 loanAmount = 原始金额 × lossExchangeRate / receiveExchangeRate
  - 汇总贷方金额 totalLoanAmount
        ↓
汇兑损益 = totalLoanAmount - totalBorrowAmount
  ├─ > 0 → 汇兑收益（借方分录）
  ├─ < 0 → 汇兑损失（贷方分录）
  └─ = 0 → 不生成分录
        ↓
生成汇兑损益分录：
  - sourceType = "8"（EXCHANGE_LOSS）
  - isExchangeLoss = true
  - 科目 = 企业默认汇兑损益科目

```

**效果示例**
```
场景：收款单本位币 CNY 100.00，客户实际支付 USD 15.00
      收款时汇率 7.25，凭证生成时系统汇率 7.20

凭证分录（未计算汇兑损益前）：
┌────┬─────────────┬────────┬────────┐
│ 方向 │ 科目          │ 金额(CNY) │ 说明   │
├────┼─────────────┼────────┼────────┤
│ 借方 │ 银行存款      │ 100.00 │ 本位币 │
│ 贷方 │ 应收账款      │ 100.00 │ 本位币 │
└────┴─────────────┴────────┴────────┘

汇兑损益计算（loanExchangeType=1，使用填写汇率7.25）：
  贷方重算 = 100 × 7.25 / 7.25 = 100（无差额）

汇兑损益计算（loanExchangeType=0，使用系统汇率7.20）：
  贷方重算 = 100 × 7.20 / 7.25 = 99.31
  汇兑收益 = 100 - 99.31 = 0.69

生成汇兑损益分录：
┌────┬─────────────┬────────┬────────┐
│ 方向 │ 科目          │ 金额(CNY) │ 说明   │
├────┼─────────────┼────────┼────────┤
│ 借方 │ 汇兑损益      │  0.69  │ 收益   │
└────┴─────────────┴────────┴────────┘

```

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 第三方支付限制 | 收款方式为支付宝/微信/银企直联/云直联 | 不展示币种选择组件，强制使用 CNY |
| 节点配置未开启 | 审批流节点未勾选"允许非本位币收款" | 币种显示本位币但置灰不可点，汇率和原币金额不展示 |
| 收款计划限制 | 单据有收款计划 | 不展示币种选择组件（复用支付逻辑） |
| 收款币种=账套本位币 | 收款币种与账套本位币一致 | 跳过汇兑损益计算 |
| 币种三字段缺失 | 收款记录缺少任一币种字段 | 跳过汇兑损益计算 |
| 汇率配置为空 | `loanExchangeType` 未配置 | 跳过汇兑损益计算 |
| 汇兑损益为0 | 计算后差额等于0 | 不生成汇兑损益分录 |
| 存量数据兼容 | 存量收款单无币种信息 | 正常收款，不参与汇兑损益计算 |
| 单据原币≠收款币种 | 单据原币 USD，收款选 EUR | 正常支持，收款币种与单据原币解耦 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 支付币种/汇率 | 平行功能 | 复用组件、字段结构和限制逻辑 |
| 会计科目映射 | 下游消费 | 扩展 `processExchangeLoss` 支持收款单 |
| 凭证生成 | 下游消费 | 新增汇兑损益分录自动混入已有凭证 |
| 移动端收款 | 同功能多端 | 同步支持币种/汇率组件 |

## 附录

**A. 典型使用场景**

场景：客户以美元支付人民币应收款，产生汇兑收益

配置步骤（首次）：
1. 管理员在「基础设置 → 币种设置」中启用 USD 币种，配置汇率
2. 管理员在「会计科目映射 → 配置」中确认 `loanExchangeType` 取值策略

使用步骤（日常）：
1. 会计创建收款单，应收金额 ¥100.00（本位币 CNY）
2. 收款单审批到达出纳节点
3. 出纳在确认收款弹窗中选择币种 USD，输入汇率 7.25
4. 系统自动计算原币金额 13.79 USD
5. 出纳确认收款
6. 会计在「会计科目映射」中生成凭证
7. 凭证自动生成汇兑损益分录（如实际汇率变动产生差额）

结果：收款原币金额可追溯，汇兑损益自动入账，无需手工调整。

**B. 字段枚举值对照表**

| 字段 | 枚举值 | 含义 |
|------|---------|------|
| `loanExchangeType` | 0 | 系统汇率（按收/付时间从汇率历史表获取） |
| `loanExchangeType` | 1 | 填写汇率（使用收/付时填写的汇率） |
| `sourceType` | "8" | EXCHANGE\_LOSS（汇兑损益分录标识） |

**C. 核心复用资产**

| 资产 | 代码位置 | 复用方式 |
|------|------------|------------|
| `foreignPayInput` 组件 | `ding-cw-doc/.../payTaskBtn/foreignPayInput.ts` | 前端直接复用 |
| `PayAccountRequest` 结构 | `yiqb-common/.../query/process/payment/PayAccountRequest.java` | 后端参照新增收款字段 |
| `PaymentAmountHelper` | `yiqb-process/.../service/payment/helper/PaymentAmountHelper.java` | 参照实现收款转换 |

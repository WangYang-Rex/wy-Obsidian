# 有成报销-审批流节点选择支付账户记录展示优化

## 1\. 需求背景

### 1.1 需求描述

当前有成报销在审批流中，部分审批节点配置了"选择支付账户"的权限。审批人操作后，支付账户信息仅在后续出纳支付节点或支付完成时才能看到，**审批流程时间线中该节点本身不展示已选择的支付账户记录**。这导致审批流追溯时，无法直观查看哪个节点、什么时候、选择了哪个支付账户。

本次需求在审批节点的审批记录区域，**回显该节点最终选择的支付账户信息**，提升审批流的可追溯性和透明度。

### 1.2 需求背景
- **需求方**：财务人员/审批人/审计人员
- **现状问题**：
• 审批节点选择了支付账户，但审批流时间线中没有留痕
• 回溯单据时，无法直观确认支付账户是在哪个审批节点确定的
• 旧版产品中该信息有展示，新版缺失导致体验倒退
- **解决思路**：
• 当审批节点存在"选择支付账户"操作时，在该节点审批记录下方增加账户信息展示
• 保持与旧版一致的展示格式和位置

**业务流程**
```plain
单据提交 → 审批节点A（选择支付账户）→ 审批节点B → 出纳支付
                ↓
         审批记录区域显示：选择支付账户-银行名称,账户名称,账号

```

## 2\. 功能现状

### 2.1 审批流节点配置现状

当前系统支持在审批流程的特定节点配置"选择支付账户"权限：
- 配置路径：流程管理 → 审批流程 → 节点设置 → 节点属性 → 支付设置 → 选择支付账户
- 配置了该权限的节点，审批人可在审批时从已授权的支付账户中选择

### 2.2 审批记录展示现状

**当前展示内容**：
- 审批人姓名
- 审批时间
- 审批结果（同意/驳回/转交等）
- 审批意见（如有）

**当前缺失内容**：
- 审批人选择的支付账户信息（即使节点配置了该权限且实际已选择）

### 2.3 旧版展示参考

旧版产品在审批节点选择支付账户后，会在该节点记录下方显示：
```
选择支付账户
中国民生银行,广西南宁遇见小万公寓管理有限公司,650304855

```
> 本次需求复用旧版的展示逻辑，但位置调整为"紧跟在审批意见下方"。   

### 2.4 关键业务逻辑

**审批节点配置**
- 在流程设计时，可为特定审批节点启用"支付账户选择"功能
- 配置项：`pay_account_set = true`
- 启用后，审批人在审批时可以从已授权的支付账户中进行选择

**账户选择与存储**
- 审批人选择支付账户后，系统调用 `recordPayAccount()` 方法记录账户信息
- **缓存**：短期存储，24 小时过期
- **数据库表**：长期存储，用于后续节点读取和使用

**出纳节点使用**
- 出纳节点在后续处理时，会读取 `process_node.pay_account_id`
- 自动填充已选中的支付账户信息
- 出纳可直接使用预选的账户进行支付操作

## 3\. 本次变更内容

### 3.1 功能变更清单

| 序号 | 变更类型 | 功能点 | 功能说明 | 优先级 |
|------|------------|---------|------------|---------|
| 1 | 新增 | 审批节点支付账户选择记录展示 | 审批流时间线中，对选择了支付账户的节点回显账户信息 | P0 |
| 2 | 兼容 | 节点权限适配 | 任何配置了"选择支付账户"权限且实际选择了账户的节点均生效 | P0 |

### 3.2 详细功能描述

#### 3.2.1 审批节点支付账户选择记录展示

**位置**：单据详情页 → 审批流程 Tab → 审批时间线

![画布 60.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KYYY2bdqDQ/img/31ff4a44-a42f-4c1b-8605-2ad8216cc881.png?Expires=1776848341&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=5ogsdayv3HB9pLceCHZK1PUW6LE%3D "")

**展示规则**：

| 条件 | 展示内容 |
|------|------------|
| 节点配置了"选择支付账户"权限 \+ 审批人实际选择了账户 | 在该节点审批记录区域，**审批意见下方**展示支付账户信息 |
| 节点未配置权限 / 未选择账户 | 保持现有展示，不增加任何内容 |

**展示格式**：
```
选择支付账户
{银行名称},{账户名称},{账号}

```

**示例**：
```
选择支付账户
中国民生银行,广西南宁遇见小万公寓管理有限公司,650304855

```

**样式要求**：
- 文字左对齐，与审批意见保持相同缩进
- 字号、颜色与审批意见一致或略做区分（建议采用系统统一的辅助信息灰色）
- 不额外增加卡片或边框，作为节点记录的一部分自然嵌入

#### 3.2.2 多次修改的处理

若同一审批节点多次修改支付账户（如审批人先选了A账户，后又改为B账户）：
- 审批流中**仅展示最终选择的支付账户**
- 不展示历史修改记录

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------|----------------|----------------|
| 节点有权限但未选择账户 | 审批人跳过了支付账户选择 | 不展示支付账户信息 |
| 节点无权限但后台有账户数据 | 历史数据或异常数据残留 | 以"是否实际发生了选择操作"为准，未操作则不展示 |
| 同一节点多次修改账户 | 审批人反复更换支付账户 | 仅展示最终选择的账户 |
| 账户信息后续被删除/变更 | 支付账户在基础档案中被修改 | 展示审批时刻的快照信息（即当时选择的账户名称和账号） |
| 逗号分隔符冲突 | 银行名称/账户名称本身包含逗号 | 按原有数据直接拼接展示，不做特殊转义处理 |
| 账号脱敏 | 部分场景下账号需要脱敏展示 | 保持与旧版一致的脱敏规则，本次不新增脱敏逻辑 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------|----------------|----------------|
| 审批流打印 | 打印审批流程页面 | 打印时该支付账户信息正常输出 |
| 审批流导出 | 导出审批记录 | 导出内容中同步包含支付账户选择记录 |
| 移动端审批 | 手机端查看审批流 | 同步展示，适配移动端窄屏布局 |
| 单据复制/再次提交 | 新单据继承旧审批流 | 新单据按新逻辑正常展示，与旧单据无关 |

## 4\. 前后端工作任务

### 4.1 前端负责内容
- 【审批流时间线组件】在节点记录中增加支付账户信息展示区域
- 【样式适配】确保展示位置紧跟审批意见下方，与现有时间线风格一致
- 【多端适配】PC 端和移动端审批流详情页同步生效

### 4.2 后端负责内容
- 【审批记录查询接口】在返回节点审批记录时，带出该节点最终选择的支付账户信息（银行名称、账户名称、账号）
- 【数据过滤】仅当节点实际发生了支付账户选择操作时才返回账户数据
- 【快照处理】返回审批时刻的账户信息快照，不因后续账户档案变更而变化

## 5\. 验收标准

| **序号** | **验收项** | **验收标准** |
|----------|-------------|----------------|
| 1 | 选择账户后展示 | 配置了权限且选择了支付账户的审批节点，时间线中显示"选择支付账户"及账户信息 |
| 2 | 展示位置 | 支付账户信息紧跟在审批意见下方，不跳出节点记录区域 |
| 3 | 展示格式 | 格式为"银行名称,账户名称,账号"，逗号分隔 |
| 4 | 未选择不展示 | 配置了权限但未选择账户的节点，不额外显示支付账户区域 |
| 5 | 无权限不展示 | 未配置选择支付账户权限的节点，即使单据有关联账户也不展示 |
| 6 | 多次修改显示最终 | 同一节点多次修改账户，仅显示最终选择的账户信息 |
| 7 | 历史数据不补 | 存量已审批单据保持原样，不因本次需求新增展示 |
| 8 | 账户删除不影响 | 原选择的支付账户后续被删除，审批流中仍显示当时选择的账户名称和账号 |
| 9 | 移动端同步 | 手机端查看审批流时，同样能看到支付账户选择记录 |

## 附录

**A. 典型使用场景**

**场景**：报销单审批流程中，财务经理节点需选择支付账户
- 员工提交报销单
- 部门主管审批同意
- 财务经理审批时选择支付账户：中国民生银行,XX公司,650304855
- 财务经理节点在审批流中显示： 
    - 审批意见：费用审核无误
    - 选择支付账户：中国民生银行,XX公司,650304855
- 总经理审批
- 出纳按已选账户完成支付

| `account&#95;no` | 支付账户档案 | 展示为账户信息的第 3 段 |

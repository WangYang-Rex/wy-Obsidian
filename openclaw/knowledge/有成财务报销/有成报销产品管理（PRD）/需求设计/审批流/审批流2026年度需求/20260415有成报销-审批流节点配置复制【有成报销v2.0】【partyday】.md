## 1\. 需求背景

### 1.1 需求描述

在自定义审批流设计器中，支持**点击节点上的复制按钮复制该节点配置**，再通过新增节点的悬浮面板**粘贴节点配置**，减少重复配置操作；复制后的节点完全独立，可自由编辑。

### 1.2 需求背景
- **需求方**：企业管理员/审批流配置人员
- **现状问题**：
• 同一条审批流中，多个同类型节点（如多个出纳节点、多个审批节点）的配置高度相似
• 每添加一个新节点都需从头勾选10\+个配置项，操作繁琐且易遗漏
• 节点名称默认重复（如多个"审批"、多个"出纳"），需手动修改区分
- **解决思路**：
• 在审批流设计器的节点卡片上增加"复制"按钮，一键复制该节点全部配置
• 点击"\+"添加节点时，悬浮面板中显示"粘贴节点"选项，快速生成配置相同的节点
• 复制后的节点完全独立，与原节点无关联，可自由修改
• 自动为新节点生成"副本"后缀名称（如"审批-副本"），允许用户手动修改

**业务流程**
```
用户进入审批流设计器
    ↓
将鼠标悬停在目标节点上
    ↓
点击节点卡片右上角的【复制】图标按钮
    ↓
系统缓存节点配置，显示"复制成功"提示
    ↓
用户点击蓝色圆形【+】新增节点按钮
    ↓
悬浮面板以九宫格/网格布局展示节点类型：
    ├─ 审批（人像图标）    ├─ 抄送（飞机图标）    ├─ 签收（对勾图标）
    ├─ AI审批（N+图标）    ├─ 分支条件（树状图）   ├─ 出纳（¥图标）
    ├─ 同步（循环箭头）    ├─ ...                 ├─ 粘贴节点（仅在有缓存时显示）
                                                        ↓ 点击粘贴
        新节点生成，复制源节点配置
        ↓
        用户自由编辑所有配置项

```

## 2\. 功能现状

**现有功能位置**：
- 路径：基础设置 → 自定义审批流
- 操作：鼠标悬停在节点上，节点右上角显示【删除】和【复制】图标按钮；点击蓝色圆形"\+"号按钮弹出九宫格/网格布局的悬浮面板，添加审批节点/抄送节点/出纳节点等

**现行逻辑**：
1. 用户点击蓝色圆形"\+"按钮弹出九宫格/网格布局的悬浮面板，选择节点类型
2. 系统生成空白节点，使用各配置项的默认值： 
    - 节点名称：默认"审批"、"出纳"、"抄送"等（同类型重名）
    - 审批方式：默认"非会签"
    - 审批人设置：默认"手动选择"
    - 其他设置：各开关默认关闭
3. 用户逐项勾选配置
4. 点击【保存】完成节点添加

**缺失能力**：
- 节点卡片上无快速复制配置的入口
- 添加节点时无法粘贴已复制的节点配置
- 同类型节点名称重复，需手动修改区分

#### 3.2.1 节点复制按钮

**触发位置**：审批流设计器中任意节点卡片的右上角（与【删除】按钮并列）

**交互界面**
```text
┌──────────────────────────────────────────┐
│  审批                              [🗑][📋]│  ← 删除按钮右侧增加复制按钮
├──────────────────────────────────────────┤
│  审批方式：非会签                         │
│  审批人：张三                             │
└──────────────────────────────────────────┘

```

**交互规则** • 鼠标悬停在节点卡片上时，节点右上角显示操作按钮区：【删除】和【复制】
• 复制按钮以图标形式呈现（建议图标：复制/剪贴板图标）
• 点击【复制】后，系统将该节点的全部配置缓存到当前页面会话中

**复制后的交互效果** • **Toast提示**：节点卡片附近弹出轻量提示："已复制'审批'的配置"（提示2秒后自动消失）
• **缓存有效期**：复制缓存仅在当前审批流设计器页面有效，刷新页面或切换审批流后自动清空

#### 3.2.2 粘贴节点

**触发位置**：审批流设计器 → 点击蓝色圆形【\+】新增节点按钮 → 弹出九宫格/网格布局悬浮面板

**交互界面**
```text
           ┌───────────────────────────────────────────┐
           │  [👤]审批      [✈️]抄送      [✓]签收      │
           │  [N+]AI审批    [🌿]分支条件   [¥]出纳      │
           │  [↻]同步       ...          [📋]粘贴节点  │  ← 有复制缓存时显示
           │                             （复制自：审批）│
           └───────────────────────────────────────────┘

```

**交互规则** • 点击蓝色圆形【\+】按钮后，弹出九宫格/网格布局的悬浮面板展示节点类型
• 面板中包含的节点类型：审批（人像图标）、抄送（飞机图标）、签收（对勾图标）、AI审批（N\+图标）、分支条件（树状图）、出纳（¥图标）、同步（循环箭头）等
• 当系统存在复制缓存时，在网格中增加"粘贴节点"格子，与常规节点类型一同以网格形式展示
• 选项文案可显示复制源节点名称，如"粘贴节点（复制自：审批）"
• 当**无复制缓存**时，"粘贴节点"格子**完全不显示**，面板保持现有网格样式
• 点击"粘贴节点"后，在点击【\+】的位置插入一个新节点，并加载复制的配置

**粘贴行为** • 粘贴生成的节点类型与复制源节点类型一致
• 粘贴后的节点与原节点**完全独立**，无任何关联关系
• 修改原节点配置，**不会影响**已粘贴的节点
• 粘贴节点的所有配置项均可自由编辑，无锁定限制

#### 3.2.3 配置复制与独立编辑

**复制范围（按节点类型）**

| 节点类型 | 可复制的配置项 | 不复制的字段 |
|------------|---------------------|------------------|
| **审批节点** | 节点基础设置（节点位置、字段权限）、审批人设置、审批方式（会签/或签/依次）、操作设置（允许转交/加签/退回/撤回/催办）、审批意见设置（必填/非必填）、超时设置（超时时间/自动处理/提醒方式）、按钮权限设置（同意/拒绝/转交/加签按钮显隐）、消息通知设置（待办/已办/抄送消息推送） | 节点名称（自动加"副本"后缀） |
| **抄送节点** | 抄送人设置、抄送方式（单人/多人/角色）、抄送范围（本部门/全公司/自定义）、消息通知设置、字段权限（抄送人可见字段配置） | 节点名称（自动加"副本"后缀） |
| **出纳节点** | 节点基础设置、出纳人设置、审批方式、付款设置（付款方式/付款条件/付款限额）、操作设置（允许转交/退回）、消息通知设置、按钮权限设置 | 节点名称（自动加"副本"后缀） |
| **发起人节点** | 发起人设置、发起范围（部门/角色/人员限制）、表单字段权限（发起人可编辑字段）、发起条件设置、消息通知设置 | 节点名称（自动加"副本"后缀） |

**各节点详细配置项**

**1\. 审批节点配置项**
- **节点基础设置**：节点名称、节点位置（前/后插入）、字段权限（查看/编辑权限）
- **审批人设置**：审批人选择方式（指定人员/上级/角色/表单字段）、审批人范围、多人审批处理方式
- **审批方式**：会签（全部同意）、或签（一人同意）、依次审批（按顺序）
- **操作设置**： 
    - 允许转交：开启/关闭
    - 允许加签：开启/关闭
    - 允许退回：开启/关闭（退回方式：退回上一步/退回发起人/自由选择）
    - 允许撤回：开启/关闭
    - 允许催办：开启/关闭
- **审批意见设置**： 
    - 同意意见：必填/非必填、默认意见内容
    - 拒绝意见：必填/非必填、默认意见内容
    - 转交意见：必填/非必填
- **超时设置**： 
    - 超时时间：小时/天
    - 超时处理：自动同意/自动拒绝/自动转交/提醒不处理
    - 提醒方式：短信/邮件/应用内推送
    - 提醒频率：单次/循环提醒
- **按钮权限设置**：同意按钮、拒绝按钮、转交按钮、加签按钮的显示/隐藏
- **消息通知设置**：待办通知、已办通知、抄送通知的推送方式

**2\. 抄送节点配置项**
- **节点基础设置**：节点名称、字段权限
- **抄送人设置**：抄送人选择方式（指定人员/角色/部门/表单字段）
- **抄送方式**：单人抄送、多人抄送、按角色抄送
- **抄送范围**：本部门人员、全公司、自定义范围
- **抄送时机**：审批前抄送、审批后抄送、审批中抄送
- **消息通知设置**：抄送消息推送方式、是否允许抄送人评论
- **字段权限**：抄送人可见字段配置（敏感字段隐藏）

**3\. 出纳节点配置项**
- **节点基础设置**：节点名称、节点位置、字段权限
- **出纳人设置**：出纳人选择方式（指定人员/角色/部门）、多人出纳处理方式
- **审批方式**：会签/或签/依次
- **付款设置**： 
    - 付款方式：对公付款/对私付款/混合付款
    - 付款条件：达到条件自动付款/手动确认付款
    - 付款限额：单笔限额/日累计限额
    - 付款账户：默认付款账户选择
- **操作设置**：允许转交、允许退回、允许撤回
- **审批意见设置**：同意/拒绝/转交意见的必填设置
- **按钮权限设置**：确认付款、拒绝、转交按钮的显隐
- **消息通知设置**：待办/已办通知、付款完成通知

**4\. 发起人节点配置项**
- **发起人设置**：发起人范围限制（全员/指定部门/指定角色/指定人员）
- **发起条件**：满足条件才可发起（金额限制/部门限制/费用类型限制）
- **表单字段权限**：发起人可编辑字段、必填字段、隐藏字段
- **附件设置**：是否必须上传附件、附件类型限制、附件大小限制
- **消息通知设置**：发起成功通知、被驳回通知

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 无复制缓存 | 用户未点击复制按钮 | 新增节点悬浮面板网格中不显示"粘贴节点"格子 |
| 复制源节点被删除 | 用户删除了已被复制的节点 | 缓存中的复制配置仍保留，可继续粘贴；粘贴后节点独立存在 |
| 删除提示效果 | 用户删除已被复制的节点 | 保持原状不变，不因为该节点被复制而增加额外提示或变更删除确认弹窗 |
| 跨页面复制 | 用户刷新页面或切换审批流 | 复制缓存自动清空，"粘贴节点"选项隐藏 |
| 重名保存 | 用户未修改默认名称导致重名 | 允许保存，不强制校验名称唯一性 |
| 序号溢出 | 同类型节点超过99个 | 序号继续递增（审批-100），界面正常展示 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 审批流版本管理 | 不同版本的审批流节点 | 复制关系仅在当前页面生效，版本间无关联 |
| 审批流复制 | 复制整个审批流到其他应用 | 复制的节点均为独立配置，无复制缓存携带 |
| 条件分支 | 分支内添加节点 | 分支内的节点复制仅在该分支内生效，粘贴也在该分支内 |
| 节点拖拽排序 | 改变节点顺序 | 不影响已粘贴节点的配置，复制缓存不受排序影响 |

## 4\. 验收标准

| 序号 | 验收项 | 验收标准 |
|------|---------|------------|
| 1 | 复制按钮展示 | 鼠标悬停在节点上时，节点右上角显示【复制】按钮 |
| 2 | 复制交互反馈 | 点击【复制】后，显示"复制成功"提示 |
| 3 | 粘贴节点显示 | 存在复制缓存时，点击【\+】的悬浮面板中显示"粘贴节点"选项 |
| 4 | 粘贴节点隐藏 | 无复制缓存时，悬浮面板中不显示"粘贴节点"选项 |
| 5 | 粘贴生成节点 | 点击"粘贴节点"后，在点击位置生成与复制源同类型的新节点 |
| 6 | 配置复制完整性 | 粘贴生成的节点完整复制源节点的所有配置项（除节点名称外） |
| 7 | 复制后独立 | 粘贴后的节点与原节点完全独立，修改互不影响 |
| 8 | 自由编辑 | 粘贴节点的所有配置项均可自由编辑，无锁定限制 |
| 9 | 智能命名 | 粘贴节点时自动添加"副本"后缀（如"审批-副本"），允许用户手动修改 |
| 10 | 缓存失效 | 刷新页面或切换审批流后，复制缓存清空，"粘贴节点"选项隐藏 |

| 12 | 全屏退出 | 点击【退出全屏】按钮，恢复原有布局，审批流配置和状态不丢失 |
| 13 | 全屏刷新 | 刷新页面后，布局恢复为默认非全屏状态 |

## 附录

### A. 典型使用场景

**场景1：连续添加相似审批节点**

背景：一条审批流需要3个审批节点，配置相似（都允许转交、都需填写意见）

操作步骤：
1. 添加第一个审批节点"财务部审批"，手动配置所有选项
2. 将鼠标悬停在"财务部审批"节点上，点击【复制】按钮
3. 提示"已复制'财务部审批'的配置"
4. 点击蓝色圆形【\+】按钮，在网格面板中选择"粘贴节点（复制自：财务部审批）"
5. 系统生成"财务部审批-副本"，配置与源节点一致
6. 再次点击蓝色圆形【\+】按钮，再次在网格面板中选择"粘贴节点"，生成"财务部审批-副本"
7. 根据需要分别修改各节点的审批人

结果：3个审批节点配置快速完成，仅需调整差异项

---

**场景2：间隔添加同类型节点**

背景：审批流结构为 审批 → 抄送 → 审批 → 出纳 → 审批，后两个审批节点需与第一个一致

操作步骤：
1. 第一个"总经理审批"节点配置完成
2. 将鼠标悬停在该节点上，点击【复制】
3. 添加抄送节点和出纳节点（此期间复制缓存一直保留）
4. 在需要添加第二个审批节点的位置点击蓝色圆形【\+】按钮，在网格面板中选择"粘贴节点（复制自：总经理审批）"
5. 系统生成"总经理审批-副本"
6. 在需要添加第三个审批节点的位置再次点击蓝色圆形【\+】按钮，在网格面板中选择"粘贴节点"
7. 系统生成"总经理审批-副本"

结果：间隔节点也能快速复制目标节点配置，缓存不随其他操作丢失

---

**场景3：复制后独立修改**

背景：粘贴生成的节点需要大幅修改，与原节点差异变大

操作步骤：
1. 打开粘贴生成的节点设置页
2. 所有配置项均已预填充复制源的配置
3. 自由修改所有配置
4. 保存后节点独立生效

结果：节点配置完全独立，与原节点无任何关联

### B. 与钉钉/飞书的对比

| 产品 | 复制行为 | 后续关系 |
|------|------------|------------|
| **钉钉** | 点击"复制节点" → 生成新节点 | **完全独立**，修改原节点不影响副本 |
| **飞书** | 点击"复制" → 生成新节点 | **完全独立**，两个节点再无关联 |
| **有成报销** | 点击节点【复制】→ 点击【\+】选择"粘贴节点" → 生成新节点 | **完全独立**，复制后无关联 |

本方案与行业主流产品（钉钉/飞书）保持一致，降低用户学习成本。

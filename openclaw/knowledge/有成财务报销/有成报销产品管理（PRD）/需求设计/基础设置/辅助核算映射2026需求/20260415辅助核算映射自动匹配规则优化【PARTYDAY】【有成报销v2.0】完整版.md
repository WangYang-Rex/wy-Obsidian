## 1\. 需求背景

### 1.1 需求描述

有成报销「基础设置-辅助核算映射」功能用于建立单据数据与财务辅助核算项之间的映射关系。当前规则下，当用户配置了映射关系但未手动维护每一项的一对一映射时，系统默认根据双方**中文名称**进行自动匹配。由于辅助核算项允许名称重复，该机制会导致名称相同但编码不同的辅助核算项被错误关联（例如编码 `0010105` 的数据可能随机匹配到编码 `0010192` 的辅助核算项），进而引发凭证推送报错。

本次需求在辅助核算映射模块的**设置弹窗**中新增「辅助核算匹配方式」配置项，支持选择：
• **辅助核算名称匹配**（默认）：按中文名称自动匹配（现有逻辑）
• **辅助核算编码\+辅助核算名称匹配**：按编码\+名称组合自动匹配（新增）

存量用户默认关闭（即默认使用「名称匹配」），用户可根据需要手动切换。

**本次优化范围限定**：
• 仅针对单据数据源为「辅助核算」的数据类别
• 其他单据内数据类别（如费用类型、部门、项目等）不受本次需求影响，保持原逻辑

### 1.2 需求背景
- **需求方**：财务人员/系统管理员
- **现状问题**：
• 辅助核算映射中，未手动配置行次映射时，系统仅按中文名称自动匹配
• 辅助核算项支持名称重复，系统默认取第一个匹配项，导致编码不同的项被错误关联
• 错误关联后，凭证推送至财务系统时因编码不一致而报错
- **解决思路**：
• 新增可配置的匹配方式，用户可按需选择「名称匹配」或「编码\+名称匹配」
• 选择「编码\+名称匹配」后，自动匹配时同时校验编码和名称
• 手动配置的一对一映射关系仍保持最高优先级
• 存量用户默认关闭，不影响现有业务

**业务流程**
```plain
用户进入基础设置 → 辅助核算映射 → 设置弹窗
    ↓
选择辅助核算匹配方式（名称匹配 / 编码+名称匹配）
    ↓
保存设置
    ↓
单据填写时选择辅助核算数据（如供应商、部门、项目）
    ↓
系统根据配置的匹配方式，查找对应的目标辅助核算项
    ↓
    ├─ 存在手动配置的一对一映射 → 直接使用手动映射结果
    └─ 不存在手动配置 → 按配置的匹配方式自动匹配
       ├─ 名称匹配：仅比较名称（现有逻辑）
       └─ 编码+名称匹配：编码和名称均一致才匹配
    ↓
生成凭证时携带正确的辅助核算编码推送至财务系统

```

## 2\. 功能现状

### 2.1 辅助核算映射现状

**功能位置**：基础设置 → 辅助核算映射

**现有功能**：
• 配置报销数据类别（单据字段/费用字段/辅助核算组件）与辅助类别的对应关系
• 支持「编辑行次」进入弹窗，维护具体数据项与辅助核算项的一对一映射
• 未手动配置映射时，系统自动根据**中文名称**进行匹配关联
• 辅助核算映射结果用于凭证生成时的辅助核算项填充

**当前缺失/问题**：
• 自动匹配规则过于宽松，仅校验名称，不校验编码
• 辅助核算项名称重复时，匹配结果不可预期，容易关联到错误编码
• 无组合匹配（名称\+编码）的精确自动匹配能力
• 无匹配方式配置开关，所有用户强制使用名称匹配

### 3.2 详细功能描述

#### 3.2.1 辅助核算匹配方式配置

**功能概述**

在辅助核算映射模块的设置弹窗中，新增「辅助核算匹配方式」配置项，用户可选择自动匹配时使用的匹配规则。

**配置入口**

基础设置 → 辅助核算映射 → 设置按钮 → 设置弹窗

**设置弹窗内容**
```text
┌─────────────────────────────────────────────────────────┐
│ 辅助核算映射设置                                    [X]  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ 辅助核算匹配方式                                    [?]  │
│                                                         │
│  ○ 辅助核算名称匹配（默认）                             │
│    按中文名称自动匹配关联                               │
│                                                         │
│  ○ 辅助核算编码+辅助核算名称匹配                        │
│    按编码和名称同时一致自动匹配关联                     │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                          [取消] [保存]  │
└─────────────────────────────────────────────────────────┘

[?] 气泡弹窗内容：
┌────────────────────────────────────────────────────┐
│ 辅助核算编码+辅助核算名称匹配功能仅对               │
│ "报销数据类别-辅助核算"生效                         │
└────────────────────────────────────────────────────┘

```

**配置项说明**

| 选项 | 说明 | 适用场景 |
|------|------|------------|
| 辅助核算名称匹配（默认） | 仅按中文名称进行自动匹配 | 辅助核算项名称不重复，或名称重复但编码一致的场景 |
| 辅助核算编码\+辅助核算名称匹配 | 按编码和名称同时一致进行自动匹配 | 辅助核算项存在名称重复但编码不同的场景 |

**功能说明提示**

• 配置项标题右侧显示问号图标按钮（❓）
• 鼠标悬停或点击问号图标，弹出气泡提示框
• 气泡内容：「辅助核算编码\+辅助核算名称匹配功能仅对"报销数据类别-辅助核算"生效」
• 气泡样式：采用系统统一的气泡/Tooltip组件，浅色背景，文案一行或自动换行

**默认状态** • 新增用户/存量用户：默认选中「辅助核算名称匹配」
• 用户可手动切换为「辅助核算编码\+辅助核算名称匹配」
• 切换后立即生效，影响后续单据的自动匹配行为

**生效范围** • 仅对单据数据类别为「辅助核算」的映射生效
• 其他数据类别（费用类型、部门、项目等）不受该配置影响，保持原逻辑

#### 3.2.2 两种匹配方式的匹配规则

**方式一：辅助核算名称匹配（现有逻辑，默认）**

| 规则项 | 说明 |
|---------|------|
| 匹配字段 | 仅比较中文名称 |
| 匹配条件 | 名称相同即视为匹配 |
| 名称重复时 | 取第一个匹配上的辅助核算项 |

**方式二：辅助核算编码\+辅助核算名称匹配（新增）**

**匹配优先级（由高到低）**
```plain
1. 手动配置的一对一映射（编辑行次中明确配置的映射关系）
2. 自动匹配：按"编码+名称"组合同时一致进行匹配
3. 匹配失败：不建立关联

```

**匹配规则详情**

| 规则项 | 说明 |
|---------|------|
| 匹配字段 | 同时比较**中文名称**和**辅助核算编码** |
| 匹配条件 | **名称相同且编码相同**才视为匹配 |
| 名称重复时 | 因编码不同，判定为**不匹配** |
| 匹配失败时 | **不匹配，不建立关联**，单据数据中该辅助核算项为空 |

**匹配逻辑伪代码**
```java
// 方式一：名称匹配（现有逻辑）
for (targetItem : targetAuxiliaryItems) {
    if (sourceItem.name.equals(targetItem.name)) {
        return targetItem;
    }
}

// 方式二：编码+名称匹配（新增）
for (targetItem : targetAuxiliaryItems) {
    if (sourceItem.name.equals(targetItem.name) 
        && sourceItem.code.equals(targetItem.code)) {
        return targetItem;
    }
}
// 未匹配上则返回 null，不建立关联

```

#### 3.2.3 映射数据取值说明

**单据端数据（Source）**

| 字段 | 说明 | 示例 |
|------|------|------|
| 辅助核算编码 | 单据组件中该辅助核算数据的唯一编码 | 0010105 |
| 辅助核算名称 | 单据组件中该辅助核算数据的显示名称 | 临沂童风玩具有限公司 |

**目标端数据（Target）**

| 字段 | 说明 | 示例 |
|------|------|------|
| 辅助核算编码 | 辅助核算项在系统中的唯一编码 | 0010105 |
| 辅助核算名称 | 辅助核算项在系统中的显示名称 | 临沂童风玩具有限公司 |

**匹配示例**

**场景1：名称\+编码均一致（编码\+名称匹配方式下匹配成功）**
```plain
单据端：编码=0010105，名称=临沂童风玩具有限公司
目标端：编码=0010105，名称=临沂童风玩具有限公司
结果：✅ 匹配成功，建立关联

```

**场景2：名称相同但编码不同（编码\+名称匹配方式下匹配失败）**
```plain
单据端：编码=0010105，名称=临沂童风玩具有限公司
目标端存在两条：
  - 编码=0010105，名称=临沂童风玩具有限公司
  - 编码=0010192，名称=临沂童风玩具有限公司

名称匹配方式结果：⚠️ 可能匹配到 0010192
编码+名称匹配方式结果：✅ 精确匹配到 0010105；若编码不存在，则不匹配

```

**场景3：手动配置优先于自动匹配**
```plain
单据端：编码=0010105，名称=临沂童风玩具有限公司
目标端：编码=0010105，名称=临沂童风玩具有限公司
手动配置：单据端 0010105 → 目标端 009_0010123
结果：✅ 优先使用手动配置结果 009_0010123（不受匹配方式配置影响）

```

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------|----------------|----------------|
| 手动映射已配置 | 编辑行次中已明确配置某条数据的映射关系 | **优先使用手动映射**，不受匹配方式配置影响 |
| 名称匹配方式下名称重复 | 目标端存在同名不同编码的辅助核算项 | 按现有逻辑取第一个匹配项 |
| 编码\+名称匹配方式下名称重复 | 目标端存在同名不同编码的辅助核算项 | 按编码精确匹配；编码不存在则不匹配 |
| 编码\+名称匹配方式下编码相同名称不同 | 理论上因编码唯一不应出现；若出现，按组合判定为不匹配 | 不匹配，不建立关联 |
| 单据端编码为空 | 单据组件中该辅助核算数据未维护编码 | 名称匹配：正常按名称匹配；编码\+名称匹配：因编码不一致，判定为**不匹配** |
| 目标端无对应记录 | 按当前配置的匹配方式在目标辅助核算中找不到匹配项 | 不匹配，该辅助核算项在单据/凭证中为空 |
| 存量数据 | 历史已配置的手动映射关系 | **完全保留**，不受本次规则变更影响 |
| 切换匹配方式 | 用户从名称匹配切换为编码\+名称匹配 | 立即生效，影响后续新单据的自动匹配；已提交单据不受影响 |
| 非辅助核算数据类别 | 数据类别为费用类型、部门、项目等 | 不受「辅助核算匹配方式」配置影响，保持原逻辑 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------|----------------|----------------|
| 辅助核算映射弹窗 | 展示和配置入口 | 弹窗展示格式保持「编码 名称」，不调整交互 |
| 凭证生成 | 消费映射结果 | 匹配成功则凭证带正确编码；匹配失败则辅助核算项为空 |
| 辅助核算同步（财务→报销） | 数据来源 | 辅助核算编码来自财务系统同步，保持唯一性 |
| 历史凭证 | 已生成凭证 | 已生成的历史凭证不受本次变更影响 |
| 其他数据类别映射 | 费用类型、部门、项目等 | 不受「辅助核算匹配方式」配置影响 |

## 附录

**A. 典型使用场景**

**场景1：企业启用编码\+名称匹配**
1. 系统管理员进入基础设置 → 辅助核算映射 → 设置
2. 设置弹窗中「辅助核算匹配方式」默认选中「名称匹配」
3. 管理员切换为「辅助核算编码\+辅助核算名称匹配」，点击保存
4. 员工提交报销单时选择了供应商「0010105 临沂童风玩具有限公司」
5. 系统按编码\+名称精确匹配，关联到编码为0010105的辅助核算项
6. 凭证推送时编码一致，成功过账

**场景2：名称不重复的企业保持名称匹配**
1. 企业辅助核算项名称均不重复，无需启用编码\+名称匹配
2. 保持默认「名称匹配」即可
3. 系统按现有逻辑正常运作，不受影响

**B. 字段说明**

| 字段 | 数据来源 | 用途 |
|------|------------|------|
| 辅助核算匹配方式 | 用户配置（设置弹窗） | 控制自动匹配时使用的规则 |
| 辅助核算编码 | 单据组件/辅助核算档案 | 唯一标识辅助核算项，用于精确匹配 |
| 辅助核算名称 | 单据组件/辅助核算档案 | 辅助核算项的显示名称 |

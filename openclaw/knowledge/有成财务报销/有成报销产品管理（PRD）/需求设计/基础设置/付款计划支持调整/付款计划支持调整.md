# 付款计划支持调整

| **修订时间** | **版本** | **修订说明** |

## 1、需求背景

客户：武汉科前

员工发起付款单，相关领导进行审批时，会根据当前业务情况， 调整当前单据的付款计划。比如先付30%，后续再付剩余的70%。到出纳节点时，根据领导的规划，作为付款参考，进行付款操作。

因此，在系统中增加收款付款的计划概念，这个跟合同的收付款计划不同，这个仅仅是资金收付时刻的规划，不做其他业务关联逻辑。

## 4、需求详情

以下描述基本都基于付款场景：报销单、借款单、付款单，本期只做付款流程

### 4.1、自定义单据-多次打款多次收款

基础设置-自定义单据中，增加组件“付款计划”，只在“报销单、借款单、付款单”中出现；可以添加进单据中

二、组件在表单中：

1、自定义单据-表单中的显示，参照收款信息一样，具体显示见下面的图：是否必填：默认非必填，可修改；

**在组件名称后面加一个帮助icon，这个icon只在付款计划组件中出现，鼠标移入给提示文案“暂不支持银企直联的支付方式”；**

2、在审批流-表单操作权限中，新增付款计划和收款计划组件的勾选配置项，现有逻辑有

3、每次填写/编辑单据，若单据中有打款计划，则根据单据信息自动生成打款计划；

### 4.3、提单/审批/支付

#### 4.3.1、提单/审批（编辑单据）

若单据模版中开启了多次打款，且审批流开始节点，收付款计划组件隐藏未勾选，则在**提单界面**增加收付款明细的组件显示；若勾选了可编辑，则可编辑：

一、将付款数据按批次规划好，根据表单上的数据自动生成付款记录与数据校验，其中对应的字段有：

| **字段** | **需求说明** |
|----------|----------------|
| 添加批次 | 在表格外显示，点击添加批次在表格中新增一行，批次号依次增加； |
| 自动生成 | 1、当前组件内没有数据时，点击自动将单据中的数据同步生成到该组件中，具体生成逻辑见“收款账户”和“支付金额”的说明；2、当前组件中有数据时，点击需要二次弹框提示“已有的数据将被覆盖，是否确认自动生成”：是：根据逻辑自动生成--生成逻辑见“收款账户”和“支付金额”的说明否：取消自动生成 |
| 批次号 | 新增/删除行次后，后面的行次依次递增递减，自动计数； |
| 操作 | \+ 新增：在当前行次下新增一行，新增只在当前批次号下新增，不做新增批次号的操作；---批次的新增只通过“添加批次”的按钮\+ 删除：点击即可删除，必填的时候，至少需要保留一行记录，当删除到只剩一行时，删除按钮不可点击；非必填的时候，允许全部删除 |
| 收款账户 | 一、单据中有收款信息组件：将单据中选择的收款信息自动生成到表格中，自动到一个批次中，导入选择的收款账户和支付金额信息。1、只有一个收款账户：\+ 将收款信息直接带入；\+ 操作列（新增/删除）不显示2、有多个收款账户：\+ 根据填写的收款信息账户，依次生成\+ 点击可调起选择收款信息弹框选择收款账户，可选的收款账户范围为收款信息组件中已选择的收款信息，点击区域为这个收款账户区域3、无收款信息：\+ 为空二、显示：显示银行Logo\+银行账户名称\+账号---若开启了收款信息加密，此处也需要加密 |
| 支付金额 | 字段可编辑一、有收款账户将收款信息对应的金额字段直接带入，有币种信息要一起带入二、无收款账户：将当前表单的应付总额带入生成一行记录，有币种信息要一起带入 |
| 顶部合计值 | 一、编辑状态下，显示合计值应付总额和待计划金额：\+ 应付总额：当前单据中已统计到的应付总额；\+ 待计划金额：应付总额-组件中已填写的支付金额总和在每一行输入完，鼠标移开金额栏后，自动更新计算待计划金额。 |
| 校验 | 一、校验通过条件：\+ 待计划金额=0，单据中的金额都已分配完\+ 每个收款账户的支付金额合计值=表单收款信息中填写的支付金额合计值；二、若以上条件未校验通过，在编辑-保存或提交审批时（在调起审批流前进行校验），给到具体提示：\+ 付款明细中杭州光云的支付金额不等于单据中填写的应付金额，请重新分配。---需要将具体不对的收款账户账户名称显示出来。三、已新增的行次必须要填入对应值，才能保存，未填写的，在保存时弹框\+红框提示。 |
| 其他说明 | 可以手工填写，只要最后的校验结果通过即可 |

二、新建单据和编辑单据都按以上的校验逻辑，涉及到PC端和手机端。

#### 4.3.1、出纳支付操作

一、出纳支付节点，收款和支付的弹框整个UI，按照新的样式重新做；

1、原来的“支付金额”文案更改为“应付总额”，与单据详情中的文案保持一致；

2、付款的表格增加表头“支付明细”

3、“支付宝打款使用帮助”超链接，只有当选择的支付账户类型为“支付宝”时，才显示在弹框中，否则不显示。

![Picture 6](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KbWAzyKqDQ/img/0ed85a33-c24c-488b-a9c6-b6ec3ec042e4.png?Expires=1778767181&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=8PU07bnhquWpWcbVYRvhO6CnMOQ%3D "")

二、若当前单据有打款计划组件，且组件中有批次数据，则点击支付，进入支付操作时：没有打款计划时，交互跟原来一样1、自动带入最前面的（序号最小的）一次批次中未支付完成的付款计划记录；若一个批次中有部分成功部分失败了，只需要带入待支付的。

2、带入的打款计划，收款账户和金额不可编辑；

本次支付金额也不可编辑，直接支付明细中金额自动汇总得出；

3、点击右侧新增，显示付款计划弹框选择，支持选择多个批次：
- 只显示待支付的记录，已支付成功的不显示
- 支持选择任意批次里面的付款记录，前面已经有的记录，进入这个弹框时自动勾选上，可以取消勾选
- 右上角统计项，在这里增加一个统计字段“已选择金额”，勾选后实时自动更新汇总后的金额。
- 点击确定后，回到前面弹框，将选中的计划重新带入支付明细中，之前已选中的支付账户仍保留，；
- 调整支付计划按钮：当出纳节点付款计划组件支持可编辑时，当前弹框显示这个按钮，点击进入编辑单据页面；同时定位到付款计划组件处。



![Picture 7](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KbWAzyKqDQ/img/f7e4178d-95cd-4ebd-b4d8-3afa9b09787f.png?Expires=1778767181&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=fJcaPBAvq6V8%2BM05C0WO%2FfN2v0I%3D "")

**4、发起支付：支付逻辑未动，跟现有的一样；**

**一、允许部分成功部分失败**
- 支付成功：同步更新单据中-付款计划中的支付状态，支付状态更新为“支付成功”；若后续查到回单了，同时需要在付款计划中更新回单文件，支持直接点击预览下载（预览下载操作同审批历史）；
- 支付失败：现有逻辑，提示失败原因，付款计划中支付状态仍为支付失败

二、批量支付，默认付当前单据所有剩余待支付的金额。



![Picture 8](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KbWAzyKqDQ/img/8384fab2-e332-4fbd-baab-058fc497c53d.png?Expires=1778767181&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=kWxOiFDPxZYNz3tDfv%2BEqms8ZRM%3D "")

三、已支付过的单据：

1、已发起支付的批次信息不允许再修改，这一行记录只做显示，点击不可修改；

2、编辑单据：编辑单据时，金额不能小于已支付金额---现有逻辑

### 4.4、其他相关影响：

一、原支付逻辑影响：同一个账户的支付金额可以拆分为多次支付

二、审批历史：现有逻辑，每个支付记录都有一条记录在

三、打印：暂不加这个字段，不受影响，先过滤掉这个组件不展示

四、单据导出：单据列表的暂不导出这个收付款计划；

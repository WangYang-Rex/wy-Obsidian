### 4.3、配置效果

1）当前单据是“需要检验的单据”配置的单据才会执行校验，需要校验的历史单据状态，指已发起审批的单据，包含"待审批、审批中、待支付、支付中、复核中、审批完成"2）根据配置的字段/组件，在单据和行程组件中进行日期范围校验
- 若行程组件中含多个常用日期字段，则都生效，时间取多个日期字段的并集3）获取当前单据中日期范围、人员（申请人、同行人），和历史单据进行重叠比对 -申请人：根据当前单据的申请人字段，找申请单名下单据中的对应日期字段进行重叠校验 -同行人：根据当前单据中的同行人字段中的每个人，找每个人名下单据的对应日期字段进行重叠校验4）“多单据校验类型”：
- 选择的是相同单据类型，只获取当前单据类型的日期字段；
- 如果是选择的是所有单据类型，将找配置的所有类型的单据的日期字段；

### 4.4、日期重叠校验规则

\- 如果日期是单个日期，不校验\- 如果日期是起止日期，新提交的单据与历史单据时间存在重叠，即进行控制；首尾连接也需要进行校验，比如以下场景：

| **类型** | **示意图** | **规则** | **案例** |
|----------|-------------|----------|----------|
| 在历史日期前 |  | 当前日期的开始时间\<= 历史日期的结束时间 | 历史：2024-10-11 ～ 2024-10-12当前：2024-10-01 ～ 2024-10-11 |
| 在历史日期后 |  | 当前日期的结束时间\<= 历史日期的开始时间 | 历史：2024-10-11 ～ 2024-10-12当前：2024-10-12 ～ 2024-10-14 |
| 当天多段 |  | 精度为天、上下午时：历史日期结束天 = 当前日期的开始天=当前日期的结束天历史日期开始天 = 当前日期的开始天=当前日期的结束天 | 历史：2024-10-11 ～ 2024-10-11当前：2024-10-11 ～ 2024-10-11         2024-10-11 ～ 2024-10-13历史：2024-10-11 上午 ～ 2024-10-11  上午当前：2024-10-11 上午 ～ 2024-10-11 上午         2024-10-11 上午 ～ 2024-10-11 下午 |
| 日期直接重叠 |  | 历史日期的开始时间-结束时间包含当前单据时间 | 历史：2024-10-11 ～ 2024-10-13当前：2024-10-12 ～ 2024-10-14 |

### 4.4、报错交互

1、『可疑费用校验』、『重复费用校验』的预警规则选择“禁止提交”时，如果命中规则，提交按钮将禁用，填单页面显示警示信息。

2、行程重叠报错如下：

【字段名称】存在重复时间段，详情：

在警告提示下：报错提示页面点击下一步调起审批流

4、如果是行程组件，可以看到每段行程的重叠情况：
- 组成结构为「行程序号」与 「单据号」「单据中重叠的行程号」重叠
- 如果与当前单据重叠，将提示：「行程序号」与 当前单据 「行程号」重叠
- 如果行程与多个单据重叠，将分多行展示
- 如果行程与同一单据中多个行程重叠，将展示多个行程号
- 点击单据编号，在当前页面可以打开单据详情的弹框，关闭单据详情弹框，回到当前页面

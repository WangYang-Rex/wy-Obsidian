## **1\. 需求背景**

### **1.1 需求描述**

在现有里程补贴功能基础上，新增全局配置实现里程补贴行程超时自动关闭，解决员工忘记手动结束行程导致的补贴计算问题，确保行程及时结算。

### **1.2 需求背景**
- **需求方**：企业财务管理员、使用MB端里程补贴的员工
- **现状问题**：
• 员工创建行程后忘记手动结束，行程长期处于"进行中"状态
• 无自动兜底机制，影响补贴准确计算和后续报销流程
• 员工忘记结束行程时无感知提醒
- **解决思路**：
• 新增全局配置控制自动关闭策略
• 超时后系统自动关闭行程并按规则计算补贴
• 关闭前1小时推送提醒通知

**业务流程**
```plain
员工创建里程补贴行程
        ↓
    开始行程（自动定位始发位置）
        ↓
    ┌─────────────────────────────────────┐
    │       员工手动打卡记录位置           │
    └─────────────────────────────────────┘
        ↓
    到达自动关闭时间-1小时
        ↓
    系统推送提醒（如有打卡记录）
        ↓
    ┌─────────────────────────────────────┐
    │              分支判断                │
    ├─ 员工手动结束 → 走正常流程生成消费   │
    └─ 超时未结束 → 系统自动关闭行程      │
            ↓
        判断是否有打卡记录：
        • 有打卡：计算距离 → 生成消费
        • 无打卡：仅关闭状态，无影响
```

### **3.2 详细功能描述**

#### **3.2.1 全局设置-自动关闭行程配置**

**位置**：基础设置 → 全局设置 → 其他设置 → 功能启用

**新增配置项**（放置于"合同自动结束"下方）
```plain
┌─────────────────────────────────────────────────────┐
│  功能启用                                            │
│  ...                                                │
│  合同自动结束              ○ 开启  ● 关闭           │
│                                                      │
│  里程补贴自动关闭行程      ○ 开启  ● 关闭           │
│                                                      │
│  自动关闭时间（小时）                                 │
│  ┌─────────┐                                         │
│  │   24    │  小时                                    │
│  └─────────┘                                         │
│  说明：行程开始超过设定时间后，系统将自动关闭该行程   │
│                                                      │
│  ☑ 关闭前提醒（提前1小时推送）                        │
│  说明：自动关闭前1小时向员工推送提醒通知              │
└─────────────────────────────────────────────────────┘
```


**交互规则** • 未勾选「里程补贴自动关闭行程」时，「自动关闭时间」和「关闭前提醒」不展示
• 首次启用时，自动关闭时间默认值为24小时
• 时间输入范围为1-168整数（最大7天）
• 配置保存后即时生效，对公司下所有员工行程生效

**默认配置**

| **字段** | **默认状态** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 里程补贴自动关闭行程 | 不勾选 | 默认关闭，保持原有逻辑 |
| 自动关闭时间 | 24 | 首次启用后必填，范围1-168 |
| 关闭前提醒 | 勾选 | 默认开启提醒 |

**配置存储** • 配置按全局维度存储
• 配置保存后即时生效
• 存量数据默认关闭，不受影响

#### **3.2.2 定时任务-自动关闭行程**

**触发条件**：全局配置开启「里程补贴自动关闭行程」

**执行频率**：每5分钟执行一次

**行为分支**

| **条件** | **系统行为** | **结果** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 全局开关关闭 | 任务直接结束 | 无操作 |
| 行程无打卡记录 | 仅修改状态为"已自动关闭" | 无消费、无提醒 |
| 行程有打卡记录 | 计算距离→生成消费→修改状态 | 自动生成消费 |

**核心逻辑说明** • 自动关闭功能**仅对至少有一次打卡记录的行程生效** • 无打卡记录的行程：时间到了仅自动关闭状态，不计算距离、不生成消费、不发送提醒
• 有打卡记录的行程：按「始发位置→打卡点1→...→最后打卡点」计算距离，自动生成消费
• 手动结束始终优先于自动关闭

**执行流程**
```plain
定时任务启动
        ↓
    读取全局配置
        ↓
    开关是否开启？
    ├─ 否 → 任务结束
    └─ 是 → 查询进行中行程
                ↓
            遍历每条行程
                ↓
            计算经过时间
                ↓
            是否到达提醒时间？
            ├─ 是 → 有打卡记录？
            │       ├─ 是 → 推送提醒
            │       └─ 否 → 跳过
            └─ 否 → 继续
                ↓
            是否到达关闭时间？
            ├─ 否 → 跳过
            └─ 是 → 有打卡记录？
                    ├─ 是 → 计算距离→生成消费→关闭行程
                    └─ 否 → 仅关闭行程
```

#### **3.2.3 消息推送-关闭前提醒**

**触发条件**：距离自动关闭还有1小时且行程有打卡记录

**推送规则**

| **条件** | **推送行为** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 关闭前提醒开关关闭 | 不推送 | 全局配置控制 |
| 行程无打卡记录 | 不推送 | 仅对有打卡记录行程推送 |
| 已推送过提醒 | 不推送 | 幂等控制，每个行程仅提醒一次 |
| 满足所有条件 | 推送提醒 | 标题\+内容形式 |

**推送内容** • 标题：里程补贴行程即将自动关闭
• 内容：您的【\{出发地\}-\{目的地\}】行程将于1小时后自动关闭，请及时确认或手动结束行程
• 跳转：MB端里程补贴行程详情页

**技术说明** • 使用有成报销**现有系统通知模块**推送
• 通知渠道（App/企微/钉钉）由技术侧控制，不在本次需求范围内
• 只需开发消息模板，调用现有通知接口

#### **3.2.4 MB端行程管理-自动关闭状态展示**

**核心原则**：明确区分"手动结束"和"自动关闭"两种状态，已自动关闭行程不可编辑

**展示规则**

| **场景** | **状态标签** | **操作按钮** | **特殊标识** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 进行中 | 进行中 | 结束行程、继续打卡 | 无 |
| 手动结束 | 已完成 | 查看详情 | 无 |
| 自动关闭-有消费 | 已自动关闭 | 查看详情 | 消费已生成 |
| 自动关闭-无消费 | 已自动关闭 | 查看详情 | 无打卡记录 |

**行程列表展示**
```plain
├─ 里程补贴    进行中        2026-04-09
│
├─ 里程补贴    已完成        2026-04-08
│
├─ 里程补贴    已自动关闭    2026-04-07 [消费已生成]
│              （系统自动关闭，补贴金额按实际记录距离计算）
│
└─ 里程补贴    已自动关闭    2026-04-06 [无打卡记录]
               （行程已关闭，无打卡记录无法报销）
```


**行程详情页** • 已自动关闭行程：
- 隐藏「结束行程」按钮
    - 隐藏「继续打卡」按钮
    - 展示「系统自动关闭」标识及关闭时间
    - 展示关闭原因说明（有消费/无打卡记录）
    - 保留「查看打卡记录」功能
    - 消费生成失败时显示「重新生成消费」按钮

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 自动关闭前手动结束 | 员工在自动关闭前手动点击结束 | 走正常结束流程，手动优先于自动 |
| 无打卡记录自动关闭 | 行程超时但无任何打卡记录 | 仅关闭状态，不计算距离、不生成消费、不推送提醒 |
| 全局配置关闭 | 管理员关闭自动关闭功能 | 所有进行中行程不再执行自动关闭检查，即时生效 |
| 修改自动关闭时间 | 管理员修改时间配置 | 进行中行程按新配置执行，即时生效 |
| 定时任务执行异常 | 系统异常或超时 | 记录异常日志，下次任务继续处理 |
| 员工删除行程 | 行程被标记为已删除 | 已删除行程不参与自动关闭检查 |
| 跨天行程 | 行程持续时间超过自然日 | 按实际经过小时数计算，不考虑日历日边界 |
| 消费生成失败 | 自动关闭时消费生成异常 | 记录失败日志，MB端显示失败提示，支持手动重试 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 现有手动结束流程 | 互斥操作 | 手动结束优先于自动关闭，已手动结束的不执行自动关闭 |
| 里程补贴标准配置 | 独立配置 | 自动关闭配置在全局设置，与里程补贴标准配置独立 |
| 消费生成模块 | 依赖调用 | 自动关闭后调用现有消费生成接口，复用现有逻辑 |
| 系统通知模块 | 依赖调用 | 调用现有通知接口推送提醒，不单独开发渠道配置 |

## **附录**

**A. 典型使用场景**

**场景1：正常自动关闭**

配置步骤（首次）：
1. 管理员进入「基础设置→全局设置→其他设置→功能启用」
2. 勾选「里程补贴自动关闭行程」
3. 设置自动关闭时间为24小时
4. 勾选「关闭前提醒」
5. 保存配置

使用步骤（日常）：
1. 员工上午9:00创建里程补贴行程并开始打卡
2. 次日上午8:00，系统推送提醒：行程将于1小时后自动关闭
3. 员工忘记处理
4. 次日上午9:00，系统自动关闭行程，计算距离和补贴，生成消费记录
5. 员工在消费列表查看已生成的里程补贴消费

结果：员工无需手动结束行程，系统自动兜底处理

---

**场景2：无打卡记录自动关闭**

使用步骤：
1. 员工创建里程补贴行程（仅设置始发位置）
2. 行程期间未进行任何打卡（未开自动定位也未手动打卡）
3. 到达自动关闭时间
4. 系统仅将行程状态修改为"已自动关闭"
5. 不计算距离、不生成消费、不推送提醒
6. MB端显示"已自动关闭（无打卡记录）"

结果：员工无法报销该行程，可选择删除或保留记录

---

**场景3：手动结束优先**

使用步骤：
1. 员工创建行程并开始打卡
2. 行程开始20小时后，员工手动点击「结束行程」
3. 员工确认后行程状态变为"已完成"
4. 24小时后定时任务检查，因状态已不是"进行中"，跳过处理

结果：手动结束流程不受自动关闭影响

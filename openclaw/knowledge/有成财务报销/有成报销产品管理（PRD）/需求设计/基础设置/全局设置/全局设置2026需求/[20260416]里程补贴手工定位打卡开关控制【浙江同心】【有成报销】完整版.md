## 1\. 需求背景

### 1.1 需求描述

在全局设置中新增「里程补贴手工定位打卡」功能开关，管理员可自由控制员工在MB端随手记-里程补贴中是否看到「手工点位记录」入口，满足部分企业对里程记录方式统一管控的需求。

### 1.2 需求背景
- **需求方**：企业管理员、财务人员
- **现状问题**：
• 部分企业希望统一使用高德导航自动记录行程，避免员工手工定位带来的数据误差
• 现有MB端始终展示「手工点位记录」入口，管理员无法关闭
• 缺乏统一的功能级入口控制能力
- **解决思路**：
• 在全局设置-功能启用中新增「里程补贴手工定位打卡」开关
• 开启时显示入口，关闭时隐藏入口
• 默认开启，确保存量用户不受影响

**业务流程**
```text
管理员进入全局设置
    ↓
配置「里程补贴手工定位打卡」开关
    ↓
    ├─ 开启 → MB端显示「手工点位记录」入口
    └─ 关闭 → MB端隐藏「手工点位记录」入口

```

## 2\. 功能现状

### 2.1 MB端里程补贴现状

**位置**：MB端 → 随手记 → 里程补贴

**现有入口**：
• 手工点位记录（页面下方左侧按钮）
• 记录新里程(推荐)（页面下方右侧蓝色按钮，跳转高德小程序）

**现有流程**：
1. 手工定位打卡 → 多次打卡 → 生成行程信息 → 生成里程补贴消费 → 提里程补贴报销
2. 记录新里程（高德小程序）→ 高德导航定位记录行程 → 生成行程信息 → 生成里程补贴消费 → 提里程补贴报销

### 2.2 全局设置现状

**位置**：基础设置 → 全局设置 → 其他设置 → 功能启用

当前已支持功能启用类配置项，但缺少针对「里程补贴手工定位打卡」的独立开关控制。

## 3\. 本次变更内容

### 3.1 功能变更清单

### 3.2 详细功能描述

#### 3.2.1 全局设置-功能启用新增开关

**功能概述**

在全局设置的功能启用区域新增一个开关，管理员通过该开关统一控制公司下所有员工是否能在MB端使用手工定位打卡功能。

**配置入口**

基础设置 → 全局设置 → 其他设置 → 功能启用

**配置界面**
```text
┌─────────────────────────────────────────────────────┐
│  功能启用                                            │
│  ...                                                │
│                                                      │
│  里程补贴手工定位打卡      ○ 开启  ● 关闭           │
│  ⓘ 关闭后，MB端随手记-里程补贴页面将隐藏「手工      │
│    点位记录」入口，员工仅可使用「记录新里程」        │
│    进行行程记录                                      │
└─────────────────────────────────────────────────────┘

```

**交互规则** • 默认状态：**开启** • 线上存量用户默认开启，保持原有使用习惯
• 配置保存后即时生效，对公司下所有员工生效
• 该开关**仅控制入口显隐**，不影响已通过手工定位方式生成的历史行程和进行中的行程

**默认配置**

| 字段 | 默认状态 | 说明 |
|------|------------|------|
| 里程补贴手工定位打卡 | 开启 | 默认开启，兼容存量用户 |

**配置存储** • 配置按公司/租户维度存储
• 配置保存后即时生效
• 建议复用现有全局配置表结构

#### 3.2.2 MB端入口显隐控制

**功能概述**

MB端里程补贴页面根据全局配置状态，动态控制「手工点位记录」按钮及相关提示文案的显示与隐藏。

**效果示例**

开关开启时：
```text
┌─────────────────────────────────────┐
│                                     │
│  [手工点位记录]  [记录新里程(推荐)] │
│                                     │
│  如遇隧道、高架等复杂路况...        │
│                                     │
└─────────────────────────────────────┘

```

开关关闭时：
```text
┌─────────────────────────────────────┐
│                                     │
│         [记录新里程(推荐)]          │
│                                     │
│  高德地图 | 地图小程序              │
│                                     │
└─────────────────────────────────────┘

```

**页面布局适配** • 开关关闭时移除「手工点位记录」按钮
• 下方提示文案（"如遇隧道、高架等复杂路况..."）同步隐藏
• 「记录新里程(推荐)」按钮保持原位置或自动居中，避免页面布局错乱

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 开关关闭时已有进行中的手工行程 | 员工在开关关闭前已通过手工定位创建了行程 | 进行中行程继续按原有逻辑执行，不受入口隐藏影响 |
| 开关频繁切换 | 管理员反复开启/关闭 | 每次保存后即时生效，MB端下次进入页面时读取最新状态 |
| 缓存导致状态不同步 | MB端读取到旧配置缓存 | 页面进入时实时读取配置，或按现有全局配置缓存刷新策略处理 |
| 无权限查看全局设置 | 普通员工进入基础设置 | 保持现有权限控制，仅管理员可修改 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 高德导航记录行程 | 独立功能 | 不受本开关影响，始终可用 |
| 里程补贴消费生成 | 依赖行程数据 | 仅控制入口显隐，不干扰消费生成逻辑 |
| 里程补贴报销 | 依赖消费数据 | 报销流程不受任何影响 |
| 全局设置其他开关 | 平行配置 | 互相独立，无联动关系 |

## 5\. 验收标准

| 序号 | 验收项 | 验收标准 |
|------|---------|------------|
| 1 | 开关配置 | 管理员可在「基础设置→全局设置→其他设置→功能启用」中配置该开关 |
| 2 | 默认开启 | 新注册企业及存量企业默认均为开启状态 |
| 3 | 开启显示 | 开关开启时，MB端里程补贴页面正常显示「手工点位记录」按钮 |
| 4 | 关闭隐藏 | 开关关闭时，MB端里程补贴页面隐藏「手工点位记录」按钮及相关提示文案 |
| 5 | 布局正常 | 开关关闭时页面布局无错乱，「记录新里程」按钮正常展示 |
| 6 | 高德入口不受影响 | 无论开关状态如何，「记录新里程(推荐)」按钮始终可用 |
| 7 | 历史数据不受影响 | 开关关闭不影响已有进行中的行程和历史消费报销 |
| 8 | 即时生效 | 开关保存后，MB端再次进入里程补贴页面即按最新状态展示 |
| 9 | 权限控制 | 仅管理员可修改该开关，普通员工无此配置权限 |

## 附录

**A. 典型使用场景**

**场景1：企业统一使用高德导航记录**

配置步骤：
1. 管理员进入「基础设置→全局设置→其他设置→功能启用」
2. 将「里程补贴手工定位打卡」切换为「关闭」
3. 保存配置

使用结果：
• 员工在MB端随手记-里程补贴页面仅看到「记录新里程(推荐)」按钮
• 无法通过「手工点位记录」入口创建行程
• 员工统一使用高德导航自动记录轨迹

**场景2：恢复手工定位打卡能力**

配置步骤：
1. 管理员进入同一配置页面
2. 将「里程补贴手工定位打卡」切换为「开启」
3. 保存配置

使用结果：
• 员工重新看到「手工点位记录」入口
• 可自由选择手工打卡或高德导航两种方式

## 1\. 需求背景

### 1.1 需求描述

企微渠道新增待办推送范围配置，钉钉/飞书/企微三个渠道统一在待办推送的人员范围下方增加单据模板维度的筛选条件。

### 1.2 需求背景
- **需求方**：有成报销企业管理员及普通员工
- **现状问题**：
• 企微渠道：全局设置-其他设置页面内"消息推送"区域**没有**企微待办推送范围配置，企微用户完全无法控待办消息推送
• 钉钉/飞书渠道：已有待办推送范围配置（全部推送/部分推送/不推送），但仅支持按人员维度控制，无法控制特定单据模板的消息推送
• 员工待办消息过多，大量非关注单据（如借款单、付款单）的消息干扰日常工作
• 企业希望按业务场景精细化管理，例如仅将费用报销类单据推送至待办
- **解决思路**：
• 企微渠道：在"消息推送"区域内新增"企微待办推送范围"配置行（全部推送/部分推送/不推送\+人员选择\+单据模板范围）
• 钉钉/飞书/企微：统一在人员范围下方增加单据模板范围配置行

**业务流程**
```
管理员进入全局设置-其他设置-消息推送区域
→ 配置各渠道待办推送范围（全部推送/部分推送/不推送）
→ 部分推送时配置人员范围
→ 配置单据模板范围
→ 保存配置
→ 消息发送时：校验单据模板范围 → 校验人员范围 → 通过则推送到渠道待办

```

| **企微** | ❌ 无 | "消息推送"区域内**没有**企微待办推送范围配置行。仅企微渠道企业可见（本次新增） |

### 2.2 单据模板禁用与删除逻辑
- **单据模板有两个关键状态**： 
    - 是否已删除：true=可用，false=已删除（软删除）
    - 启用/停用：true=启用，false=停用
- **删除保护**：已被使用过的模板（已创建过单据）不允许删除
- **显示规则**： 
    - 员工新建单据时的下拉：已删除和已停用的模板均不显示
    - 管理页面（如单据模板管理页）：已停用的模板显示（带"已停用"标记，供管理员查看和管理），已删除的不显示
- **核心结论**：禁用即不显示（员工场景）；已删除的模板所有下拉场景都不显示

### 2.3 消息推送链路现状
- **配置存储**：企业通知配置，包含推送范围类型、人员接收范围、黑名单
- **发送链路**：消息在聚合分发阶段按企业注册渠道分发到各平台
- **单据信息**：消息通知中携带单据模板ID，当前链路未对单据模板做过滤
> 本次需求：企微渠道新增待办推送范围配置行；钉钉/飞书/企微三个渠道统一在人员范围下方增加单据模板范围配置行。各渠道配置行仅对对应渠道企业可见。

#### 3.2.2 钉钉/飞书渠道-待办推送范围增加单据模板筛选

**功能概述**

在钉钉/飞书渠道现有的"待办推送范围"配置行中，于人员范围下方新增"单据模板范围"配置行。

**配置入口**

有成管理后台 → 基础设置 → 全局设置 → 其他设置 → 页面内"消息推送"区域

**变更内容**
- 现有"钉钉/飞书待办推送范围"三单选（全部推送/部分推送/不推送）保持不变
- 现有"部分推送范围"人员选择组件保持不变
- 新增"单据模板范围"配置行，位于人员范围下方，逻辑与企微渠道完全一致：

#### 3.2.3 消息发送链路-单据模板校验

**功能概述**

在消息聚合分发阶段增加单据模板范围校验，被拦截的消息不推送到渠道待办，但站内信正常发送。

**业务流程**
```text
业务事件触发（审批/支付/驳回等）
↓
构造消息通知（含单据模板ID）
↓
异步处理
↓
构造消息体
↓
消息聚合分发
  ├─【新增】获取企业该渠道通知配置中的单据模板范围
  ├─【新增】判断单据模板范围是否为空
  │    ├─ 为空 → 通过，继续人员范围校验
  │    └─ 非空 → 判断单据模板ID是否在范围内
  │         ├─ 在范围内 → 通过，继续人员范围校验
  │         └─ 不在范围内 → 拦截，跳过渠道分发
  │              （注意：站内信在此之前已发送，不受影响）
  ↓
通过校验后按渠道分发（钉钉/飞书/企微）

```

**效果示例**

假设企业配置：
- 钉钉渠道：推送范围=全部推送，单据模板范围=仅费用报销

| 触发场景 | 单据模板 | 单据模板校验 | 人员范围校验 | 钉钉待办 | 站内信 |
|------------|------------|------------------|------------------|------------|---------|
| 费用报销待审批 | 费用报销 | ✅ 通过 | ✅ 通过 | ✅ 推送 | ✅ 发送 |
| 借款单待审批 | 借款单 | ❌ 拦截 | — | ❌ 不推送 | ✅ 发送 |
| 差旅报销待支付 | 差旅报销 | ❌ 拦截 | — | ❌ 不推送 | ✅ 发送 |

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 企微存量兼容 | 企微渠道原先无此配置 | 首次进入页面时自动创建默认配置（全部推送\+全部单据模板） |
| 钉钉/飞书存量兼容 | 钉钉/飞书渠道单据模板范围配置为空 | 按全部单据模板处理，保持原有推送行为 |
| 新企业默认 | 新企业首次创建配置 | 单据模板范围为空，即全部单据模板推送 |
| 仅配置人员范围 | 管理员只改了人员范围，未触及单据模板范围 | 单据模板范围按原有值/默认值处理 |
| 单据模板被禁用 | 已配置的单据模板后续被禁用或删除 | 该模板仍在范围内保留，校验时若模板不存在则按不存在处理（即不推送）；管理员可在设置页面看到"已停用"标记并手动移除 |
| 消息无单据模板ID | 非单据类消息（如系统通知）不携带单据模板ID | 直接通过单据模板校验，继续人员范围校验 |
| 配置即时生效 | 管理员保存配置后 | 即时生效，无需重启，后续消息按新配置执行 |
| 双维度 AND 关系 | 人员范围通过但单据模板范围未通过 | 不推送到渠道待办；反之亦然 |
| 全部推送/不推送状态下的单据模板范围 | 推送范围选择"全部推送"或"不推送" | 单据模板范围配置行不展示，仅"部分推送"时展示 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 用户级消息开关 | 独立控制 | 用户开关控制短信/模板消息接收，单据模板范围控制渠道待办推送，两者独立 |
| 差旅单强制短信 | 特殊逻辑 | 差旅单强制发短信逻辑不受单据模板范围影响（短信与渠道待办是不同渠道） |
| 催票任务强制短信 | 特殊逻辑 | 同上，不受单据模板范围影响 |
| 站内信 | 始终发送 | 单据模板范围仅拦截渠道待办，站内信始终发送 |

| 8 | AND关系验证 | 人员范围含用户A\+单据模板含费用报销，用户A提交费用报销 → 对应渠道待办推送 |
| 9 | AND关系-人员不满足 | 人员范围不含用户B\+单据模板为全部，用户B提交费用报销 → 不推送对应渠道待办 |

| 13 | 钉钉/飞书存量兼容 | 钉钉/飞书渠道单据模板范围为空，视同全部推送，行为与线上一致 |
| 14 | 三渠道配置相互隔离 | 各渠道配置独立存储，修改某一渠道不影响其他渠道 |
| 15 | 配置即时生效 | 修改配置保存后，下一条消息按新配置执行 |
| 16 | 渠道配置行隔离 | 钉钉渠道企业仅展示"钉钉待办推送范围"，不展示飞书/企微配置行；飞书/企微同理 |

| 18 | 单据模板被删除 | 因删除保护机制，已被使用的模板不允许删除；若配置中出现已删除的模板ID，按不在范围内处理（即不推送） |

## 附录

**A. 典型使用场景**

场景：某企微渠道企业只想将费用报销和差旅报销的待办消息推送到企微待办，其他单据（借款单、付款单等）仅保留站内信通知。

配置步骤（首次）：
1. 管理员进入 基础设置 → 全局设置 → 其他设置 → 消息推送区域
2. "企微待办推送范围"切换为"部分推送"
3. 配置人员范围（如需限定人员）
4. 单据模板范围切换为"部分单据模板"

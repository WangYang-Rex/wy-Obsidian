# 辅助核算编码自定义

## 1、需求背景

客户：上海快卜

辅助核算的编码需要系统按照自定义的规则自动生成，不能员工随便输入，导致辅助核算的编码错乱了

| 飞鸟等三方对接 | 导入三方对接 |

| 历史数据影响 | 一、历史类别的配置项   配置项还是原来的配置，不显示自定义规则选项（减少编码变来变去的几率，尽量引导用户新的类别使用新的规则）；--留下历史数据可刷配置的口子   二、填充规则切换，对编辑的影响：1、无--\>自定义规则：\+ 历史已存在的数据，编码不受新设置的规则影响，历史数据的编码不变\+ 新建辅助核算，根据设定的规则，取当前已存在的数据，符合规则的编码数据后重新计数加一，默认填入；2、自定义规则--\>无：\+ 历史已存在的数据，编码不变；\+ 重新新建辅助核算的编码，现有逻辑，为空，可以直接填入； |

### 4.2、新建辅助核算

以下说明都基于当前辅助核算模版填充规则开启了自定义规则：



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/meonarbP5BX7JqXx/img/5eec80f8-51c0-42cc-a33e-6e3e2fc4d140.png?Expires=1778767213&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=3urV44rxYLpsG%2FHA8uM4IbzuLL4%3D "")

一、新建入口包含：基础档案-辅助核算中新增、新建单据/费用中新增（PC端\+MB端）

二、编码字段填入规则：根据设置的规则自动生成带入，是否可编辑根据“编辑”的设置项来；

1、**在层级下新增第一个数据：**取上级的编码，增加当前级次的编码长度，从1开始计数
- 比如第一层级的客户编码为KH01，编码规则为KH 2-4，在当前层级下新增客户张三，编码默认为KH010001

2、**当前层级已有数据，新增新的数据**，取当前层级中，符合规则尾数最大的\+1
- 当前层级下已有客户张三（编码为KH010001），再次新增客户李四，编码默认为KH010002；
- 当前层级下已有客户张三（编码为KH010001），有客户李四（编码为KH010002），有客户光云（KH0100DS），再次新增客户易企云，编码默认为KH010003；

3、**编码长度增大，比如3--\>4**，按新的长度重新开始编号
- 当前层级下已有客户张三（编码为KH01678），新增客户李四，编码默认为（KH010001）

4、**编码长度减少，比如4--\>3**，按新的长度重新开始编号
- 当前层级下已有客户张三（编码为KH019788），新增客户李四，编码默认为（KH01001）

5、**长度已达限度**，比如长度为4，数据达到9999，按超限规则处理
- 满则随机编码，再次新增时，保持前缀不变，前缀后的编码自动随机生成，比如默认为（KH989089）
- 满则无法新增，再次新增，直接在编码输入框下提示：编码额度已使用完，请联系管理员增加长度后使用

6、**层级超限制**，比如长度为4，在第四层级下再次新增数据，按超限规则处理
- 点击新建则提示：能新增的最大层级为4级，请修改层级级次后再新增

6、~~断号补充：~~若中间出现断号，需要优先取断号补充（后续可能会做成配置）~~
- ~~比如已有客户张三（编码为KH01~~~~0001~），客户李四（编码为KH01~~0003）~~，再次新增一个客户，此时默认给的编码是（编码为KH01~0002）~~

7、是否可编辑：
- 可编辑:自动带入，用户可编辑，编辑后按照规则和长度进行校验
- 不可编辑：自动带入，置灰显示

三、校验：

1、已设定的编码需要符合自定义规则设置的格式，包括前缀、级次长度以及每个层级的长度限制；
- 输入后不符合规则限制的话，光标移开或保存时，直接标红输入框，并在输入框下给到具体格式提示，提示信息格式为：编码格式示例格式为KHHJ010001,点击一键编码，确定后将按照新编码保存
- 格式与模版中的预览样式一致
- “点击一键编码”为文本按钮，点击直接生成新的编码到编码输入框，生成的编码规则参照上面第二点

2、编码需要保持唯一，不能重复；

3、编码长度超限提示。

### 4.4、其他影响点

一、辅助核算导入

导入的辅助核算编码规则需要符合设定的编码规则

二、飞鸟

通过接口新增/同步的辅助核算编码，不符合设定的规则，需要在接口上进行限制报错

三、报销财务一体化
- 无影响，正常同步，财务侧没有做编码格式限制
- 反向同步：财务到报销的辅助核算，需要按照编码规则进行限制

四、三方对接，调用有成提供的接口新增辅助数据

不做编码规则限制

五、垃圾数据处理：个人级别（非企业级）的数据，创建时间超过24小时，且没有关联单据信息，且将此类数据清除掉

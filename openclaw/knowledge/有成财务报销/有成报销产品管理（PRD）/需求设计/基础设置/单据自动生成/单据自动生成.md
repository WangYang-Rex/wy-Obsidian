## 1、需求背景

客户名称：吉布鲁、浙江省财开等多个大客户常见需求

需求背景：除了系统内的收付单据，客户线下会有一些交易是不会用报销系统，但是需要去做账，客户希望可以通过交易明细去生成凭证；系统生成凭证需要有单据做基础，所以可以通过交易明细生成单据，再通过单据生产凭证可满足

### 4.2、新建规则

1、新建规则时，会从右侧抽屉展开设置内容，顶部为设置主要信息：
- 规则名称：必填、文本格式、最多100字
- 数据源对象：必填、下拉选择，可选「资金明细-支付」、「资金明细-收入」
- 生成单据名称：如果数据有是「资金明显-支付」，可选择以下类型的所有单据：付款单、借款单、申请单如果数据有是「资金明显-收入」，可选择以下类型的所有单据：收款单、申请单



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/r4mlQ5bzJVBPklxo/img/d1117aa7-690f-4f3e-9d98-5c36279ab04e.png?Expires=1775017292&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=S5RQrSGN57rsBlWonO4fv3cCR4Q%3D "")

2、「单据字段设置」中根据「生成单据类型」选择的单据模板中，获取字段和组件，字段和组件的必填性和单据模板中一致，（单据模板中字段是配置必填，在这里也是必填）

字段可设置的「自定义取值规则」：

5、字段规则：

| **可设置字段类型** | **文本类型：多选取值字读****金额类型：单选取值字读****数字类型：单选取值字读****部门：单选取值字读** | **文本类型包含：标题、文本、描述****金额包含：常用金额、支付金额****数字包含：税率、税额、数字、常用数字** |
|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------|
| 不可设置字段类型 | 成本中心、城市、辅助核算、成本中心、下拉选择、角色、费用类型、人员、同行人、附件、图片、月份范围、子表单、分割线、文本说明、链接、手机号 | 填充「不可设置」禁用 |
|  |  |  |
- 文本类型字段：字段均可选择
- 申请人：默认制单人，禁用状态
- 部门类型字段：可选「制单人所在部门」、「制单人默认部门」
- 支付金额：默认「本位币金额」，禁用状态
- 其他金额：可选「原币金额」、「本位币金额」
- 数字类型字段：可选「原币金额」、「本位币金额」、「汇率」、「交易后余额」
- 日期类型字段：可单选「交易时间」、「制单时间」

6、组件规则：
- 收款信息：默认为『对方账户』禁用
- 收款账号：默认『公司支付账户』，禁用
- 发票抬头：不可设置「自定义取值规则」
- 费用明细：不可设置「自定义取值规则」
- 发票：不可设置「自定义取值规则」
- 项目：不可设置「自定义取值规则」
- 关联单据：不可设置「自定义取值规则」
- 付款场景：不可设置「自定义取值规则」
- 税率税额：不可设置「自定义取值规则」
- 行程：不可设置「自定义取值规则」
- 产品明细：不可设置「自定义取值规则」

7、保存时，需要校验：
- 当前单据中，是否包含不可设置取值规则的字段或者组建，否则将报错提示“单据包含xxx组件、xxx组件、xxx字段，不可设置自动生成规则”
- 数据源对象只能有一个单据，否则将报错“相同数据源对象只能配置一个单据”

8、每次切换单据名称、点击编辑时，会从选种的单据模板中获取最新字段和组件

### 4.2、资金明细生成单据



![Picture 3](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/r4mlQ5bzJVBPklxo/img/62809755-7adb-48fa-b539-04afe6daa921.png?Expires=1775017292&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=CPZf4lImNcEGEWNloG7gqaClszA%3D "")

**一、列表调整**

1、列表增加「制单状态」字段，可以支持在表头筛选，选项可以支持多选。
- 未生成：手动关联单据的第三方明细、非第三方明细
- 生成中：通过流水生成中的明细
- 已生成：通过流水生成完成的明细
- 生成失败：通过流水生成失败的明细

**二、手动生成**

1、「资金明细」增加「生成单据」按钮，点击后需要校验是否勾选数据没有关联将toast提示：请选择数据

2、选择完数据后，点击「生成单据」需要校验资金明细是否已经关联了单据，勾选了后将toast提示：已选明显中存在已经生成的单据，请筛选未关联的单据

3、点击「生成单据」后，会提示：

已选择x条流水，y条支出流水生成「付款单」，z条收入流水生成「收款单」

x为已经勾选的流水数量，y为支付流水数量，z为收入流水数量



![Picture 4](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/r4mlQ5bzJVBPklxo/img/df9537bc-d5c7-443a-b209-0b31671af827.png?Expires=1775017292&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=9D%2Fs8wwhAiUn7jBhVoUq5SaR7s0%3D "")

4、点击「确定生成」后，会进行异步生成，此时流水生成状态将为“生成中”，并提示：

生成期间，您可关闭窗口操作其他功能；稍后您可在 生成记录 中查看生成记录

5、点击「生成记录」会将弹框关闭并打开生成记录弹框，点击「知道了」会关闭弹框，不影响生成过程

三、自动生成设置



![Picture 5](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/r4mlQ5bzJVBPklxo/img/99f9d357-4218-4cdd-bbee-63ec872aee26.png?Expires=1775017292&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=owMMVfCVCMkAFfWwEuQeg5JlSys%3D "")

1、点击「自动生成单据」会有弹框弹出设置生成规则和生成记录

2、设置生成规则，：

增加备注：每晚00:30开始根据以下条件生成单据，没有设置条件默认所有没有关联单据的流水均可生成
- 状态：停、启用 
- 指定公司账号：可多选所有有效的公司账户，最多50个，必填字段点击后会进入「公司账户选择弹框」可以多选公司账户，选择完后并确定后，需要校验至少选择一项，否则将报错“至少选择一个账户”，设置成功后按钮会展示「已选择x账户」，取消还显示「设置账户」   
- 指定单据申请人：可单选所有有效员工，默认当前操作人，必填字段
- 生成有回单的流水
- 生成支出流水
- 生成收入流水
- 生成前x天数据：根据执行日期的前x计算

例如：生成前 1 天数据， 执行日期为 4号凌晨 00:30，将会把 3号 00:01 - 23：59 的单据生成

3、保存后需要校验，当状态是开启状态下，账户和申请人必填，否则将报错“公司账户、单据申请人必须设置”

### 4.4、生成单据逻辑

1、校验规则：

点击关联单据，校验已经勾选的明细中，不能包含非第三方和生成中、已经生成；否则将报错：

非第三方支付、已生成和生成中的明细不支持关联单据

点击生成单据，校验已经勾选的明细中，不能包含非第三方和生成中、已经生成、已经关联单号

非第三方支付、生成中和已经关联单据号的明细不支持生成单据



2、生成单据时，单据字段的取值根据会根据最新的规则生成，如果字段配置了多个字段，字段与字段间通过“-”隔开，配置生成单据后，流水将会关联对应生成的单据编号，流水生成状态变为“已生成”

3、生成单据时，如果单据有审批节点，会生成草稿；如果单据没有审批节点，单据将直接为审批完成的状态

4、收付款组件字段取值规则：

| **场景** | **单据组件字段** | **取值逻辑** |
|----------|----------------------|----------------|
| 交易记录是支出方向 | 收款信息 | 根据流水的“户名”“银行账户”在“首款信息”中匹配，并将此账户id填充到单据中，如果匹配到多个，按照创建时间取最新创建的一条 |
|  | 支付账户 | 取交易明细中，顶部关联的公司账户 |
| 交易记录是收入方向 | 收款账号 | 取交易明细中，顶部关联的公司账户 |
|  | 收款信息 | 根据流水的“户名”“银行账户”在“首款信息”中匹配，并将此账户id填充到单据中，如果匹配到多个，按照创建时间取最新创建的一条 |

5、收付款信息填充规则：

| **场景** | **单据组件字段** | **取值逻辑** |
|----------|----------------------|----------------|
| 交易记录是支出方向 | 收款信息 | 根据流水的“户名”“银行账户”在“首款信息”中匹配，并将此账户id填充到单据中，如果匹配到多个，按照创建时间取最新创建的一条 |
|  | 支付账户 | 取交易明细中，顶部关联的公司账户 |
|  | 支付时间 | 交易明显中的交易时间 |
| 交易记录是收入方向 | 收款账号 | 取交易明细中，顶部关联的公司账户 |
|  | 收款信息 | 根据流水的“户名”“银行账户”在“首款信息”中匹配，并将此账户id填充到单据中，如果匹配到多个，按照创建时间取最新创建的一条 |
|  | 支付时间 | 交易明显中的交易时间 |

6、生成单据时，会根据预算配置情况占用预算；如果是借款单，会占用借款额度

7、生产单据时，如果制单人有多个所属部门，按照维护顺序取第一个维护的所属部门

8、如果单据中只有支付金额，则无法生成，需要增加金额字段，并且金额字段值和支付金额一致时

9、支付金额字段默认为本位币，填充值也是本位币，如果交易明细没有本位币，则填充原币

10、金额字段币种默认为本位币，按照配置取绝对值

11、如果生成的单据删除了，需要将单据和这个资金明细解绑，资金明细状态改为是未生成，关联单据清空，资金明细数据保留，还可以继续生成。

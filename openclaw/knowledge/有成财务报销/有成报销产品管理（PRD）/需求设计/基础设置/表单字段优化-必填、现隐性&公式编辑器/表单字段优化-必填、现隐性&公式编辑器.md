## 1、需求背景

目前字段中必填项、显隐性都是固定的开关，但是企业经常会有一定情况下时，某个字段非必填、不显示等情况，这类情况会非常多，条件规则也近乎列举不全

## 4、需求详情

### 4.1、表单字段调整

1、设置顶部显示为组件名称

2、下方为显示名称、提示信息、字数限制、文本类型等设置，

3、每个字段增加「属性设置」，每个字段可能会不以一样
- 必填，勾选后会展开必填条件设置，此设置默认不勾选此，勾选后下方条件必填，下方条件点击后为规则编辑器，通过编辑器内容来判断是否必填
- 隐藏，勾选后会展开隐藏条件设置，此设置默认不勾选此，勾选后下方条件必填，下方条件点击后为规则编辑器，通过编辑器内容来判断是否隐藏设置字段隐藏时，制单人不可见，不做删除处理，如果有默认值，同时设置隐藏时，字段和内容保留，仅不可见，审批流、打印等依然可以引用此字段值来使用审批时默认所有人可见，可通过流程节点中的权限设置是否针对「申请人或者审批人」设置是否显示隐藏的字段（见文档4.3）（开发备注：当前是默认所有人可见，默认所有人不可见是否需要变更原逻辑）
- 可编辑，默认勾选，由原来「是否可编辑」字段来，没有勾选为只读
- 流程完成后隐藏，默认不勾选
- 多选，此字段是否可以多选
- 展示“无”选项，字段是否可以选择“无”这个选项

4、每个字段增加「默认填充规则」，不同字段内容不一样
- 无，不填充任何值
- 固定值，勾选后会出现输入或者选项值，提单时默认此值
- 公式计算，点击后会显示计算公式的入口，点击后会进入公式编辑器，会通过编辑器内容赋值
- 公式计算只在 文本、数字、金额类型的字段显示

7、编辑器运用场景：
- 作为条件编辑器：运用在必填性、显隐性上，内容为条件规则。当满足编辑器规则时，执行显示、隐藏规则
- 作为赋值编辑器：运用在默认填充规则上，根据不同字段类型，赋不同类型的字段：

数字类型字段：数字、常用数字、外部同行人数、手机号、发票张数、人数

文本类型字段：标题、描述、文本、链接、目的地、出发地、费用事由

金额类型字段：金额、支付金额、税额、不计税金额、民航发展基金

8、编辑器运算特殊说明：

1）编辑器执行运算结果时，如果字段被隐藏，依然可以获取此字段的值来进行计算

2）如果运算时，一个公式中，存在一处计算规则或者函数规则计算因为设置的值不对导致算不出来，则会停止运算，不返回任何值，也不报错。例如：IF ( 申请人名称 == 日期,1,2)

3）新建单据时，会根据单据默认字段计算一遍结果，当字段失焦时会重新计算一遍结果

4）用户输入运算符时，默认是文本格式，可使用「TAB」键将光标前的第一个运算符转化为系统运算符

5）当用户通过「其他单据引入」功能添加费用时，此费用不会触发此费用中公式计算，当编辑此费用时，也可正常触发公式计算

### 4.5、校验规则

|  | **校验规则** | **报错** |
|---|----------------|----------|
| 运算符 | 算数运算符（加减乘除，取余）左右两侧只能是数字格式、括号等号、不等号左右两边可以是 字符串或者数字 | 算数运算符左右两侧只能为数字字段 |
|  | 逻辑运算符（或且）左右两侧是boolean类型或者是表达式 | &&和 |
|  | 比较运算符（大于小于等于）左右两侧是数字或者表达式 | 比较运算符左右两侧必须为数字字段或者逻辑表达式 |
|  | 分组运算符（就是括号）左右两侧是数字、表达式、文本、运算符、函数 | -- |
| 函数 | 函数中的参数字段数量需要满足函数结构要求 | 「函数名称」参数不满足结构要求 |
|  | 函数中字段类型（数字、文本、日期等）是否满足要求 | 「函数名称」中的不支持添加「字段名称」 |
|  | 函数中的参数字段数量超过最大数量 | 「函数名称」中的参数数量不得超过200个 |
| 字符串字段 | 字符串两边只能是运算符 | 「字符串内容」两边只能是运算符 |
| 数字字段 | 数字字段两边只能是数字、算数运算符、分组运算符（括号） | 「数字内容」两边只能是数字或者运算符 |
| 系统插入字段 | 插入字段两边只能是算数运算符、比较运算符、分组运算符（括号） | 「插入字段名称」两边只能是运算符 |
|  |  |  |

### 4.6、空置规则

1、表单字段增加「其他字段」其他字段可选择「空值」，仅可配置在IF函数的 2、3参数位置中。


![Picture 12](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZ8AzDoAqze/img/834d96fb-f831-4b04-8e48-0376ae3ca168.png?Expires=1774157041&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=e%2B1zIynlEELPpRZnTKK%2BRB6zzYw%3D "")

2、空值计算规则：

1）当SUM、MIX、MIN等聚合函数中，有空值字段或者字段为空值，计算时会跳过此值（不参与计算）

2）提单计算时，如果参与计算的字段/组件不存在，则当null处理，计算结果为空值

3）提单计算时，如果参与计算的字段/组件存在，但是没有值，当空值处理（金额、数字为空值视为0），计算结果为0

### 7.2 null、空值特殊运算场景

定义：

null：字段不存在时为null

空值：字段存在但是没有值

1）数值计算时，null、空值的计算效果当0处理，即，1\+null = 1； null \+ null = 0；1\*null = 0 ；1/null = ∞

2）当计算结果时正负∞时，计算结果为null 

3）当null参与大小比较时，效果也当0处理， 即 null = 0 为ture ，null \>null 为false

4）ABS、MIN、MAX、ROUND、ROUND\_UP、ROUND\_DOWN、SUM函数中，如果有null、空值则会跳过，不参与计算

5）INCLUDE函数，参数2为null、空值时，均返回 false

6）IS\_NULL函数，参数为null、空值时，均返返回ture，如果是数组类型，数组长度为0时返回ture（即，取费用明细某个字段是否为空，当所有费用明细字段不存在返回ture，后面待优化项。）


### 7.3 金额计算逻辑

1、参数都是数字，计算结果为数字，给金额填充币值；币种根据是否开启币种判断：

\- 没有开启币种，无币种\- 开启币种，无本位币，默认CNY\- 开启币种，有本位币，默认本位币

2、参数有一个金额和多个数字，计算结果为金额币值与数字计算，币种取金额字段

3、参数有金额与金额计算，并币种相同时，计算结果为币值计算，币种取金额字段币种

4、参数有金额与金额计算，并币种不相同时，计算时需要转位本位币计算，币种取本位币

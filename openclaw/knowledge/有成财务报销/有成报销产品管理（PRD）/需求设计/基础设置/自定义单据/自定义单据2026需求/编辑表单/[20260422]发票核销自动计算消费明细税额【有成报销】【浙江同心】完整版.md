## 1\. 需求背景

### 1.1 需求描述

发票核销申请单在选择要核销的单据后，系统支持将发票明细信息自动带入到申请单费用明细中。当前产品存在两种业务场景：
- **场景一（原逻辑）**：发票识别出的发票明细分别全部带入到申请单内每个费用下，适用于需要完整发票明细信息的场景；
- **场景二（新需求）**：合并开票场景下，一张发票关联多笔费用，需要按每笔费用的实际核销金额独立计算税额，确保税额数据准确。

本次需求在保留原功能逻辑的基础上，新增「费用核销金额带入」模式，支持企业根据实际业务场景选择适合的带入方式。

### 1.2 需求背景

**业务痛点**：
1. **合并开票场景税额计算不准确**：当前原逻辑下，使用合并开票时上传一张发票为多笔费用填写核销金额，该发票的全部发票明细会分别带入到申请单的费用明细下的税率税额组件中。这导致每个费用都包含发票的全部明细信息，在按费用独立核算税额时数据不准确。
2. **缺乏灵活性**：原功能「发票上传自动带入费用明细」为单一模式，无法满足不同企业/场景对发票明细带入方式的差异化需求。
3. **存量兼容压力**：直接修改原逻辑会影响存量用户的既有使用习惯，需要提供平滑过渡方案。

**用户场景**：
- **财务核销人员**：在合并开票场景下，需要将一张发票的税额按各费用的实际核销金额比例分摊，确保每笔费用的税额计算准确，便于后续入账和税务申报。
- **财务管理员**：需要根据企业实际业务特点，灵活配置发票明细的带入方式（全部带入或按核销金额带入）。

### 1.3 业务流程
```
【发票核销申请单创建流程】

1. 员工创建发票核销申请单
   ↓
2. 选择要核销的单据（付款单/借款单等）
   ↓
3. 上传发票
   ↓
4. 系统根据组件设置中的「带入方式」执行不同逻辑：
   ├─ 选项A（发票明细全部带入）：
   │   └─ 发票的全部发票明细分别带入到每个费用下
   │   └─ 税率优先取发票识别税率，无则取默认税率
   │   └─ 税额按原逻辑计算
   │
   └─ 选项B（费用核销金额带入）：
       └─ 按每笔费用的实际核销金额独立计算税额
       └─ 税率：发票唯一税率时取发票税率；多税率/无税率时取默认税率
       └─ 税额 = 核销金额 - 核销金额/(1+税率)
   ↓
5. 用户可手动调整各费用明细的税率/税额（选项B下仅影响当前费用）
   ↓
6. 核销确认前校验：所有费用税额之和 vs 发票总税额
   ├─ 相等 → 允许确认
   └─ 不等 → 提示差异，用户确认后允许提交
   ↓
7. 核销确认，税额锁定，进入审批流

```

## 2\. 功能现状

### 2.1 发票上传自动带入费用明细

**现有功能入口**：基础设置 → 自定义单据 → 表单编辑 → 发票核销组件 → 组件设置 → 属性设置

**现有功能逻辑**：
- 「发票上传自动带入费用明细」开关：开启后，发票核销申请单选择要核销的单据，将要核销的单据内费用明细信息带入申请单费用明细中
- 使用合并开票时，上传一张发票为多笔费用填写核销金额，该发票的全部发票明细会分别带入到申请单的费用明细下的税率税额组件中
- 若开启了发票查重，会触发查重逻辑

**现有税率取值逻辑**：
- 优先取发票识别后的税率
- 若发票明细没识别出税率，取税率税额组件中默认税率（来自费用表单模板配置）
- 支持用户手动修改税率

**现有税额计算**：根据费用不含税金额计算税额

### 2.2 税率税额组件（）

有成报销已存在税率税额组件（聚合组件），核心字段：

| 字段 | 说明 |
|------|------|
| `amount` | 含税金额 |
| `taxRate` | 税率（如 "13%"） |
| `taxValue` | 税额 |
| `deductibleTaxValue` | 可抵扣税额 |
| `subInvoiceTypeName` | 发票类型名称 |
| `invoiceCategory` | 发票类目列表 |

**现有能力**：
- 组件支持新旧版兼容落库（统一落 `newTaxRateAmount`）
- 已有 `shareTax` 费用分摊税额逻辑：按 `shareRate` 分摊，最后一条差额补齐
- 税额已参与预算联动（`feeTaxAmountMap` 费用级 / `orderTaxHolder` 单据级）
- 公式引擎已暴露 `taxAmountTotal`（税额总额）、含税金额总额等字段

### 3.2 详细功能描述

#### 3.2.1 发票核销组件设置新增带入方式选项

**位置**：基础设置 → 自定义单据 → 表单编辑 → 发票核销组件 → 组件设置 → 属性设置

**新增配置项**：

| 配置项 | 类型 | 默认值 | 说明 |
|---------|------|---------|------|
| 发票上传自动带入费用明细 | 开关 | 开启 | 现有功能，保持不变 |
| **带入方式** | **单选** | **A.发票明细全部带入** | **新增选项，仅当上方开关开启时显示** |

**带入方式选项**：

| 选项 | 说明 |
|------|------|
| **A. 发票明细全部带入** | **原功能逻辑**。发票识别出的发票明细分别全部带入到发票核销申请单内每个费用下。每个被关联费用都包含发票的全部明细信息。 |
| **B. 费用核销金额带入** | **新逻辑**。发票上传后，系统按每笔费用的实际核销金额独立计算税额，并带入到对应费用下。 |
> A和B为**单选**，仅支持选一种。选A时走原逻辑，选B时走本次迭代的新逻辑。   

**交互规则**：
- 「发票上传自动带入费用明细」开关关闭时，「带入方式」选项隐藏
- 开关开启时，「带入方式」选项显示，默认选中A
- 存量用户升级后，默认保持A（原逻辑），不受影响
- 用户可随时从A切换到B或从B切换到A

#### 3.2.2 选项A：发票明细全部带入（原逻辑保留）

**逻辑说明**：
- 与现有功能完全一致，保持不变
- 发票核销申请单选择要核销的单据后，发票识别出的发票明细分别**全部带入**到申请单内**每个费用**下
- 使用合并开票时，被关联的每个费用都包含发票的**全部明细信息**
- 若开启了发票查重，触发查重逻辑

**税率取值逻辑（选项A）**：

| 优先级 | 税率来源 | 说明 |
|---------|------------|------|
| 1 | 发票识别税率 | 优先取发票识别后的税率 |
| 2 | 税率税额组件默认税率 | 发票明细没识别出税率时，取费用表单模板中配置的默认税率 |
| 3 | 用户手动修改 | 支持用户在核销页面手动修改税率 |

**税额计算（选项A）**：
- 根据费用不含税金额计算税额
- 保持原有计算逻辑不变

#### 3.2.3 选项B：费用核销金额带入（新逻辑）

**触发时机**：发票上传完成后，系统自动触发税额计算

**计算规则（各消费明细独立计算）**：
```
不含税金额 = 实际核销金额（含税）/ (1 + 税率)
税额 = 实际核销金额（含税）- 不含税金额

```

| 参数 | 说明 |
|------|------|
| 实际核销金额 | 员工设置的发票在该费用上的核销金额（**含税金额**） |
| 税率 | 见下方「选项B税率取值逻辑」 |
| 不含税金额 | 由上述公式计算得出 |
| 税额 | 由上述公式计算得出 |

**选项B税率取值逻辑**：

| 优先级 | 税率来源 | 触发条件 |
|---------|------------|------------|
| 1 | 发票识别税率 | 发票识别出**唯一税率**时，取该税率 |
| 2 | 税率税额组件默认税率 | 发票**有多个税率**或**没有识别出税率**时，取费用表单模板中配置的默认税率 |
| 3 | 用户手动修改 | 支持用户在核销页面手动修改税率 |
> 与选项A的区别：选项A优先取发票识别税率（只要识别出就取），选项B仅在发票识别出**唯一税率**时才取，多税率或无税率时取默认税率。   

**示例**：

发票总额 16,000（税额 1,440），员工设置在各费用上的实际核销金额：

| 消费明细 | 费用类型 | 实际核销金额（含税） | 发票识别税率 | 默认税率 | 最终取值 | 不含税金额 | 税额（taxValue） |
|------------|------------|------------------------------|------------------|------------|------------|---------------|--------------------|
| 明细1 | 办公用品 | 10,000 | 13% | 13% | **13%**（唯一） | 8,849.56 | 1,150.44 |
| 明细2 | 差旅费 | 5,000 | 6% | 6% | **6%**（唯一） | 4,716.98 | 283.02 |
| 明细3 | 招待费 | 1,000 | 无/多税率 | 6% | **6%**（fallback） | 943.40 | 56.60 |
| **合计** | — | **16,000** | — | — | — | **14,509.94** | **1,490.06** |
> 上表示例中，按各费用独立计算后，合计税额 1,490.06 ≠ 发票票面税额 1,440。系统提示差异，由用户确认是否调整。   

**税率税额组件内部平衡（选项B）**：

消费明细的税率税额组件（`newTaxRateAmount`）内部若包含多条发票明细记录，按以下规则平衡：
```
组件内所有记录 amount 之和 = 该费用实际核销金额
组件内所有记录 taxValue 之和 = 该费用计算出的税额
单条记录 taxValue = 该记录 amount × (该费用税额 / 该费用核销金额)
最后一条记录兜底 = 该费用税额 - 前面已分摊之和

```
> 组件内部平衡复用现有 `shareTax` 的「最后一条差额补齐」机制。   

**交互规则（选项B）**：
- 发票上传后，系统按选项B逻辑自动计算各消费明细的税额和不含税金额
- 计算结果回填到税率税额组件的 `taxRate`、`taxValue`、`amount` 等字段
- 用户可手动修改任意消费明细的税率或税额
- **修改仅影响当前消费明细，不联动其他明细**
- 核销确认前，系统校验：所有消费明细税额之和 vs 发票总税额 
    - 相等 → 允许确认
    - 不等 → 提示「消费明细税额合计与发票税额不一致，请确认是否继续」
- 核销确认后，税额锁定，不可再修改

#### 3.2.4 手动调整税额（单条独立，仅选项B）

**位置**：发票核销页面 → 消费明细列表

**规则**：
- 自动计算后，每条消费明细的税率、税额字段可独立编辑
- **用户修改某条明细的税率或税额后，仅影响当前消费明细**： 
    - 修改税率 → 按新税率重新计算该明细的不含税金额和税额
    - 修改税额 → 同步调整不含税金额（不含税金额 = 核销金额 - 税额）
- **不联动其他消费明细**，其他明细保持原值不变
- 修改后实时更新该明细税率税额组件内部的记录平衡（组件内最后一条兜底）
- 核销确认时，系统校验所有明细税额之和是否等于发票总税额，不等则提示确认

### 3.3 边界处理

| 序号 | 边界场景 | 处理规则 |
|------|------------|------------|
| 1 | 选项B下发票税额为0 | 各费用明细税额均自动填为0 |
| 2 | 选项B下实际核销金额为0 | 该费用税额为0 |
| 3 | 选项B下所有费用实际核销金额均为0 | 不触发计算，提示「核销金额为空，无法计算税额」 |
| 4 | 选项B下费用类型无默认税率 | 税率字段为空，提示用户手动填写 |
| 5 | 选项B下各费用税额之和等于发票税额 | 核销确认正常通过 |
| 6 | 选项B下各费用税额之和不等于发票税额 | 核销确认时提示差异，需用户确认是否继续 |
| 7 | 选项B下修改税率后组件内部平衡 | 组件内多条记录按新税率重新分摊，最后一条兜底 |
| 8 | 发票查重开启\+选项B | 正常触发查重，不影响选项B计算逻辑 |
| 9 | 存量用户升级 | 默认选中选项A，原逻辑不变，不受影响 |
| 10 | 选项切换（A↔B） | 切换后重新上传发票时按新选项逻辑计算，历史已保存单据不受影响 |
| 11 | 发票被删除后重新上传 | 按当前选中的选项A/B重新计算 |
| 12 | 关联费用被移除后重新添加 | 按当前选中的选项A/B重新计算 |

| 5 | 选项B-多条费用独立计算 | 组件设置选B，上传发票关联3条费用 | 各费用按自身税率独立计算税额，合计可能≠发票税额，提示差异 |
| 6 | 选项B-组件内部尾差兜底 | 组件设置选B，费用A核销金额10,000，组件内2条发票明细（6,000/4,000） | 明细1=690.26，明细2=460.18（最后一条兜底） |
| 7 | 选项B-手动修改税率（单条独立） | 组件设置选B，将费用A税率从13%改为9% | 仅费用A重新计算，费用B和C不变 |
| 8 | 选项切换A→B | 从选项A切换到选项B，重新上传发票 | 按选项B逻辑重新计算 |
| 9 | 选项切换B→A | 从选项B切换到选项A，重新上传发票 | 按选项A原逻辑全部带入 |
| 10 | 存量用户升级 | 升级后查看组件设置 | 默认选中选项A，原逻辑不变 |
| 11 | 开关关闭时选项隐藏 | 关闭「发票上传自动带入费用明细」开关 | 「带入方式」选项隐藏 |

## 附录

### A. 选项A vs 选项B 对比

| 对比项 | 选项A（原逻辑） | 选项B（新逻辑） |
|---------|----------------------|----------------------|
| 明细带入 | 每个费用带入发票**全部明细** | 按费用**核销金额**独立计算 |
| 税率取值 | 优先取发票识别税率，无则取默认税率 | 发票**唯一税率**时取发票税率；**多税率/无税率**时取默认税率 |
| 税额计算 | 按费用不含税金额计算（原逻辑） | 按费用核销金额和税率独立计算 |
| 费用联动 | 各费用都包含全部发票明细 | 各费用独立计算，修改一条不影响其他 |
| 适用场景 | 需完整发票明细信息的场景 | 合并开票、需按费用独立核算税额的场景 |

### B. 税率税额组件字段说明

| 字段 | 来源 | 用途 |
|------|------|------|
| `amount` | `newTaxRateAmount` 组件 | 含税金额，作为税额分摊的计算基数 |
| `taxRate` | `newTaxRateAmount` 组件 | 税率展示，用于校验发票税率与明细税率是否匹配 |
| `taxValue` | `newTaxRateAmount` 组件 | 税额，自动计算并回填的字段 |
| `deductibleTaxValue` | `newTaxRateAmount` 组件 | 可抵扣税额，入账时参与可抵扣税额汇总 |
| `subInvoiceTypeName` | `newTaxRateAmount` 组件 | 发票类型，用于预算联动时的可抵扣白名单过滤 |

### C. 预算与入账联动

发票核销回填的 `taxValue` 参与现有预算联动逻辑：

| 联动点 | 说明 |
|---------|------|
| 预算占用 | 税额根据企业预算配置（`taxNotInBudget`）决定是否占预算 |
| 可抵扣过滤 | 通过 `subInvoiceTypeName` × `invoiceCategory` × 费用类型三维度白名单过滤 |
| 费用级汇总 | `feeId != null` 时，`taxValue` 累计到 `feeTaxAmountMap` |
| 单据级汇总 | 无 `feeId` 时，`taxValue` 累计到 `orderTaxHolder` |
| 公式引擎 | 已暴露 `taxAmountTotal`（`taxValue` 求和）、含税金额总额等公式字段 |

### D. 典型使用场景

**场景一：合并开票按费用核算税额**
> 企业采购办公用品和差旅服务，供应商合并开一张发票（总额16,000，税额1,440）。财务需要在发票核销时按各费用实际核销金额分别核算税额。   

配置：组件设置选 **B.费用核销金额带入**

操作：
1. 创建发票核销申请单，关联付款单
2. 上传合并发票
3. 设置费用A（办公用品）核销金额10,000，费用B（差旅费）核销金额5,000，费用C（招待费）核销金额1,000
4. 系统自动计算各费用税额：A=1,150.44，B=283.02，C=56.60
5. 合计税额1,490.06 ≠ 发票税额1,440，系统提示差异
6. 财务确认后提交

**场景二：需要完整发票明细信息**
> 企业审计要求每笔费用都保留完整的发票明细信息备查。   

配置：组件设置选 **A.发票明细全部带入**

操作：
1. 创建发票核销申请单
2. 上传发票
3. 每个费用都自动带入发票的全部发票明细
4. 税额按原逻辑计算
5. 确认提交

## 1\. 需求背景

### 1.1 需求描述

在自定义单据「费用明细」组件的「动态可选范围」配置中，将规则里「字段值」的选择方式从"单选"升级为"多选"，使管理员可以在一条规则内同时配置多个字段值映射到同一组费用类型，减少重复规则配置。

### 1.2 需求背景
- **需求方**：企业系统管理员
- **现状问题**：
• 当多个部门（如财务部、技术部、行政部）都需要使用相同的费用类型范围（如办公费、差旅费）时，管理员需要为每个部门分别创建一条规则
• 规则数量随部门和费用类型的组合爆炸式增长，配置效率低
• 后续维护成本高，修改费用类型范围时需要逐条规则修改
- **解决思路**：
• 将规则中「字段值」从单选改为多选
• 一条规则可配置多个字段值，这些字段值共同映射到同一组可选费用类型
• 保持现有 OR 匹配逻辑不变，任一字段值匹配即生效

**业务流程**
```text
管理员进入自定义单据 → 费用明细组件 → 可选范围设置 → 动态可选范围
    ↓
选择联动组件（部门/辅助核算/员工/成本中心/项目/常用下拉框/下拉选择）
    ↓
新增规则 → 字段值支持多选（如同时勾选「财务部」「技术部」「行政部」）
    ↓
选择该规则对应的可选费用类型（如办公费、差旅费）
    ↓
保存 → 仅生成1条规则，替代原先的3条规则

```

## 2\. 功能现状

### 2.1 动态可选范围现有能力

**位置**：自定义单据 → 表单编辑器 → 费用明细组件 → 组件设置 → 可选范围设置 → 动态可选范围 → 设置规则

**现有能力**：
• 三种可选范围模式：不限制 / 静态可选范围 / 动态可选范围
• 动态可选范围支持选择联动组件：部门、下拉选择、辅助核算、成本中心、项目（共5种）
• 每条规则包含：字段值（单选）→ 可选费用类型（多选）
• 支持新增、复制、删除规则
• 多条规则之间为 OR 关系

**现有缺失**：
• 字段值仅支持单选，无法在一个规则内配置多个字段值
• 当 N 个字段值对应 M 个相同费用类型时，需要配置 N 条重复规则

### 2.2 后端现有逻辑

**数据模型**：`ConfigureOrder.expense` 存储在 `CorpOrderTemplate.configure` JSON 中

**匹配逻辑**：`matchConfig()` 中多条规则为 OR 关系——任一规则匹配即包含对应费用类型。

**运行时过滤**：`getCodesTreeByOrderLimit()` 根据单据中联动组件的实际值，遍历规则进行匹配。
> 本次需求仅改动前端规则配置的字段值选择方式（单选→多选），后端匹配逻辑（OR 关系）保持不变，兼容存量数据。

### 3.2 详细功能描述

#### 3.2.1 动态可选范围-字段值多选

**功能概述**

在动态可选范围配置弹窗中，每条规则的「字段值」列支持多选，管理员可在一条规则内同时选择多个字段值，这些字段值共同映射到该规则的可选费用类型。

**配置入口**

自定义单据 → 表单编辑器 → 费用明细组件 → 组件设置 → 可选范围设置 → 动态可选范围 → 设置规则

**字段值多选交互规则**

• **选择方式**：点击字段值区域弹出下拉选择器，支持勾选多个值（复选框形式）
• **展示形式**：多选后，字段值列以标签形式展示已选值（如「财务部」「技术部」「行政部」），标签可逐个删除
• **多选范围**：同一规则内选择的多个字段值，必须属于同一联动组件（如都是部门）
• **值之间关系**：OR 关系，即单据中联动组件的值匹配任一已选字段值，该规则即生效
• **费用类型**：每条规则的可选费用类型仍为多选，从费用类型树中选择

**字段值列与可选费用类型列的对应关系**

| 规则行 | 字段值（多选） | 可选费用类型（多选） | 含义 |
|---------|---------------------|------------------------------|------|
| 规则1 | 财务部、技术部、行政部 | 办公费、差旅费 | 财务/技术/行政部的人，费用明细中可选办公费和差旅费 |
| 规则2 | 销售部、市场部 | 招待费、交通费 | 销售/市场部的人，费用明细中可选招待费和交通费 |

#### 3.2.2 联动组件类型扩展

**现有支持**：部门、下拉选择、辅助核算、成本中心、项目

**本次新增**：员工组件、常用下拉框

**完整联动组件列表**：

| 组件类型 | 说明 |
|------------|------|
| 部门 | 单据中的部门组件值 |
| 辅助核算 | 单据中的辅助核算组件值 |
| 员工 | 单据中的员工组件值 |
| 成本中心 | 单据中的成本中心组件值 |
| 项目 | 单据中的辅助核算（项目）组件值 |
| 常用下拉框 | 单据中的常用下拉框组件值 |
| 下拉选择 | 单据中的下拉选择组件值 |
> 注：联动组件下拉仅支持单选，即一次规则配置只能基于一个联动组件。切换联动组件时，当前规则列表清空（见 3.2.3）。   

#### 3.2.3 切换联动组件-规则刷新

**交互规则** • 联动组件下拉为单选，切换时触发确认提示
• 提示内容：「切换联动组件将清空当前已配置的所有规则，是否继续？」
• 用户点击「确定」后，清空当前规则列表，字段值选项更新为新联动组件的取值范围
• 用户点击「取消」后，保持当前联动组件和规则不变
• 点击弹窗「确定」保存后，新配置覆盖旧配置

**存量数据兼容** • 存量规则中字段值为单选的，正常展示和运行
• 新配置中字段值为多选的，按多选逻辑运行
• 后端存储格式兼容：单选值为字符串，多选值为字符串数组（或逗号分隔字符串），后端解析时统一按数组处理

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------|----------------|----------------|
| 字段值全部取消选择 | 规则中字段值一个都不选 | 该规则无效，保存时校验提示「字段值不能为空」 |
| 不同规则间字段值重复 | 规则1选「财务部」，规则2也选「财务部」 | 允许重复配置，运行时多条规则为 OR 关系，结果一致，但建议管理员避免重复 |
| 联动组件值为空 | 单据中联动组件未填写 | 无法匹配任何规则，费用明细显示空列表（与现有逻辑一致） |
| 联动组件值不在规则中 | 单据中部门为「研发部」，但规则中无研发部 | 无法匹配，费用明细显示空列表（与现有逻辑一致） |
| 单条规则字段值过多 | 一条规则选了50个字段值 | 前端展示折叠或滚动，后端正常处理 |
| 切换联动组件后取消 | 切换联动组件后点击「取消」 | 不保存任何变更，保持原联动组件和规则 |
| 存量单选规则编辑 | 打开存量规则进行编辑 | 字段值列展示为单标签，编辑时支持追加多选 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------|----------------|----------------|
| 静态可选范围 | 与动态可选范围互斥 | 保持现有互斥逻辑不变 |
| 费用类型树 | 可选费用类型从费用类型树多选 | 保持现有选择逻辑不变 |
| 规则复制 | 复制一条规则（含多选字段值） | 复制后字段值和费用类型均保留，可在此基础上修改 |
| 单据运行时过滤 | `getCodesTreeByOrderLimit()` 过滤 | 后端匹配逻辑不变，单值/多值统一按「包含」判断 |

## 附录

**A. 典型使用场景**

**场景：多个部门共用相同费用类型**

配置前（3条规则）：
- 规则1：部门=财务部 → 办公费、差旅费
- 规则2：部门=技术部 → 办公费、差旅费
- 规则3：部门=行政部 → 办公费、差旅费

配置后（1条规则）：
- 规则1：部门=\[财务部、技术部、行政部\] → 办公费、差旅费

效果：管理员配置工作量减少67%，后续新增费用类型时只需修改1条规则。

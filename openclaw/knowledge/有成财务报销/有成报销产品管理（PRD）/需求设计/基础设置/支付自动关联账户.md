# **1\. 需求背景**

单据支付的时候根据辅助核算公司主体自动关联支付账户，避免人工选择错误

# **2\. 业务流程/架构**

# **3\. 功能清单**

| 模块 | 功能点 | 备注 |
|------|---------|------|
| 表单配置 | 新增“公司账户”组件。可以下拉单选公司账户值 |  |
| 单据支付 | 支付时根据提前维护好的核算单位与公司账户关系，自动获取公司账号，并提供用户进行二次确认 |  |
|  |  |  |

1. 表单编辑器中新增组件“公司账户”，下拉多选框，枚举值“公司账户板块的所有账户信息”

注：该组件仅在辅助核算表单中可用，其余表单配置时隐藏
2. 右侧组件设置板块内容：
    1. 组件名称：固定为“公司账户”
    2. 显示名称：必填项，默认取值为“公司账户”，会回显编辑修改后内容【10个字符上限】
    3. 提示信息：底色提示语，非必填。【20个字符上限】
    4. 属性设置：必填、隐藏、可编辑、完成后隐藏（默认选可编辑\+必填）
    5. 默认填充规则：无、计算公式（默认选择无）

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/34d5068c-ac5f-4b22-988c-d9c9551dd593.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=ewRtsH2l41AMCIcFYoW6qh0XmWk%3D "")
3. 辅助核算表单编辑补充“公司账户”字段，根据维护时选择的是否必填来控制是否一定要维护。若要维护则选择基础档案-公司账户中的数据，多选。

#### **4.2.1单条支付**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/31edc3cd-3389-48a2-815f-3f82e438ba66.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=6ErqwzIiQZtYYsBmPkTMpds0ToY%3D "")
1. 自动赋值支付账户逻辑1:选择“记住已选账户”【优先级最低】

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/4815b71f-1997-47ab-a33d-c43c1157e55b.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=0WU%2FDf6agYfPPofyUJTPycYQA0k%3D "")

因为记录的逻辑是根据当前用户上次选择的支付账户来选择的，这个不准确性非常大，所以该情况下的自动赋值优先级最低

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/2fad64f6-22dd-4404-b1b4-edee9fa9ee95.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=8MpDU%2FhcDKqKJfwlrPVyoLAfNY8%3D "")
2. 自动赋值支付账户逻辑2：发起的单据存在辅助核算值，并且对应的辅助核算维护了公司账户
    1. 该单据只有一个辅助核算值，并且辅助核算只绑定了一个公司账户，则支付账户自动获取辅助核算维护的公司账户
    2. 该单据只有一个辅助核算只，但是辅助核算绑定了多个公司账户，则支付账户不会自动获取该公司账户，而是会过滤掉原本可选的支付账户，仅保留本单据所有辅助核算维护的公司账号值
    3. 该单据有多个辅助核算值，但多个辅助核算维护的公司账户是同一个，则支付账户自动获取该公司账户
    4. 该单据有多个辅助核算值，并且多个辅助核算维护的公司账户是多个，则支付账户不会自动获取该公司账户，而是会过滤掉原本可选的支付账户，仅保留本单据所有辅助核算维护的公司账号值
    5. 若辅助核算未维护公司账户，则不会自动赋值



故：优先级按逻辑2  〉逻辑1的先后顺序来处理。若都没有能自动赋值的则还是由用户手动赋值

注：赋值节点为“表单进入到待支付环节时，系统会自动查询单据并根据维护好的绑定关系获取当前核算主体的公司账户，并自动展示在单据页面上。（用户也可手动修改）”

3、审批流节点开启选择支付账户，优先级最高



#### **4.2.2批量支付**

点击批量支付，系统判断所有勾选的单子中，支付账户是否为同一值，若相同则自动赋值。若不同则为空，用户手动选择：
1. 当存在辅助核算时，判断（逻辑同上）每一笔支付单中辅助核算绑定的公司账户，作为支付账户来选择
2. 当存在辅助核算，若所有支付单均为维护辅助核算或维护了但是未绑定公司账户，则判断是否存在“记住已选账户”的逻辑
    1. 若存在则根据该逻辑选择支付账户
    2. 若不存在则支付账户视为未被过滤，则获取“基础档案-公司账户”中的数据

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/77cffc8a-6ea7-4516-a3bc-f3e54d51f5e3.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=HAkAVg2smgmiu45lnGOZXBkDPbc%3D "")



注：无论是单条支付，还是批量支付。点击支付并确认后页面展示二次确认弹窗

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/KM7qeob2RV1Nblpj/img/7aaf8b11-6c07-4aa7-b362-f389c31bd3a1.png?Expires=1775017205&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=bLKpVAZbI%2FCJqN3XDZctpis9LE0%3D "")

点击确认后正常按照后续流程发起支付

#### **4.3.3带入逻辑**

1、没匹配到公司账户：不赋值，可选全部

2、匹配到唯一公司账户：赋值，可选只有这一个

3、匹配到多个公司账户：不赋值，可选范围就是匹配到的结果集



问答：
1. 批量支付的时候，可选范围是汇总所有单据内的辅助核算的关联公司账户。此时如果只有一个单据内的辅助核算维护了公司账户，那多选的数条单据一起支付时，支付账户就只有这一个了。想选别的也选不到。除非是分开单条支付。确认

### 2、现有的公司账号和收款信息都是弹窗分页，确认用弹窗

3、记住已选择账户：问号提示，当辅助核算关联了公司账户时，优先记住关联账户。

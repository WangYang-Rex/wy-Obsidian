### 4.1、收款信息

设置功能改用弹窗的样式，同时账户共享、账户创建、必填校验的样式都放在弹窗中，同时对样式进行修

![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/MAeqxebXy53wAO8j/img/9b74915b-ea09-47ba-ab61-e393d3ef54a7.png?Expires=1778767414&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=AqCq8CalN9MWs6oBioD5CKvgYkg%3D "")

一、允许员工共享：
- 使用开关形式，开关后给提示文案，文案内容：开关开启，则允许员工将收款信息共享给他人；
- 历史用户原来是勾选的，这里默认打开，原来是未勾选，这里不打开；

二、账户创建：对应原来的“允许员工创建账户信息”，使用开关形式：
- 开关打开，则显示选项选择框，使用下拉选择，支持多选，下拉选项为原来功能中的账户类型；
- 开关关闭：选项框不显示，但重新打开后，原来的配置信息还在；
- 历史用户原来是勾选的，这里默认打开，同时将已勾选的配置项同步显示在选项中‘

原来是未勾选，这里不打开，配置项无。

三、必填校验：对应原来的“必填设置”，使用开关形式：
- 开关打开，则显示选项选择框，使用下拉选择，支持多选，下拉选项为原来功能中的必填字段项；
- 开关关闭：选项框不显示，但重新打开后，原来的配置信息还在；
- 历史用户原来是勾选的，这里默认打开，同时将已勾选的配置项同步显示在选项中‘

原来是未勾选，这里不打开，配置项无。

**四、****信息加密：本次新增选项，默认不打开开关，提示文案内容：开关开启，仅白名单人员可查看完整账户信息（如户名、账号）；**

1、开关关闭：
- 默认关闭，关闭走现有逻辑，正常显示信息；
- 关闭时白名单显示信息区域不显示，再次打开可恢复原来的数据；

2、开关打开： 打开之后，同时显示白名单选择项；
- 开关打开之后，收款信息加密显示，本次加密的字段有户名和银行账号；
- 本次加密的字段有：户名、银行账号：户名：留下最后一个字符，前面的统一用两个星号显示，如\*\*子；不论账户名称原来的字样多少，若开启了加密，则使用以这个样式显示；银行账号：留下最后四个字符显示，前面统一用四个星号显示，如**\*6214；若账号未满4位，则展示账户的原有的位数，同时保持星号，如**\*214；
- 加密的入口：当开启了信息加密时，以下入口：**单据详情（PC&MB）、审批历史（PC&MB）、单据详情中的历史版本中收款信息相关的显示、单据列表（待办、单据管理、项目与合同、财务管理）中收款信息字段的显示、单据列表的导出、打印；**在对应入口，按照上述形式加密，见描述下放的图；审批历史-支付节点中的显示样式优化一下：收款信息和支付信息换行显示。
- **白名单：**点击调用选人组件进行配置，跟系统内部分可见的弹框一样，支持部门、角色、员工的选择，非必填；白名单的人员查看单据时，以上入口中，收款信息不加密；

### 开发备注

新建表 corp\_account\_secrect 企业收款信息加密配置

统一配置查询接口，更新接口

allowShareBankcard bool允许员工共享

bankcardAdminFlag bool 允许员工创建账户信息

allowAccountType  string\[\] 含义同 accountType接口 1，2，3，4，14，15

setMandatory string\[\] 含义同setMandatory 接口

encryptionOfBankInfo bool 银行信息加密

visibleUser string\[\] 可见人员

visibleRole String\[\] 可见角色

visibleDepartment string\[\] 可见部门

CREATE TABLE isv\_corp\_extend\_account\_encryption\_config(  \`id\` bigint(20) unsigned NOT NULL AUTO\_INCREMENT,  \`corp\_id\` varchar(64) CHARACTER SET utf8mb4 NOT NULL DEFAULT '' COMMENT '企业id',  \`encryption\_of\_bank\_info\` tinyint(1) DEFAULT '0' COMMENT '是否开启银行信息脱敏，0:未开启 1:开启 默认0',  \`enc\_white\_visible\_user\` varchar(255) DEFAULT '' COMMENT '脱敏白名单可见用户',  \`enc\_white\_visible\_role\` varchar(255) DEFAULT '' COMMENT '脱敏白名单可见角色',  \`enc\_white\_visible\_dept\` varchar(255) DEFAULT '' COMMENT '脱敏白名单可见部门',  \`created\` datetime COMMENT '创建时间',  \`modified\` datetime COMMENT '编辑时间',  \`modified\_user\` varchar(50) COMMENT '编辑用户',  PRIMARY KEY (\`id\`) USING BTREE,  UNIQUE KEY \`unq\_idx\_corp\_id\` (\`corp\_id\`)) ENGINE=InnoDB DEFAULT CHARSET=utf8 COMMENT='企业收款信息脱敏配置';

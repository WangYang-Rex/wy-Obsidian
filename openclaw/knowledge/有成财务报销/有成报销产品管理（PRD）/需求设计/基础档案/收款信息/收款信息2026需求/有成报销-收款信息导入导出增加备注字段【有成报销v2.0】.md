## **1\. 需求背景**

### **1.1 需求描述**

财务人员在核对银行信息时需要用到备注字段，当前收款信息的导入模板和导出功能均不支持备注字段，导致数据不完整。

### **1.2 需求背景**
- **需求方**：财务人员
- **现状问题**：
• 新建收款账户时系统已支持填写备注信息
• 但导出收款信息时，Excel文件中不包含备注字段
• 批量导入收款信息时，导入模板中没有备注字段列
- **解决思路**：在收款信息的导入模板和导出功能中统一增加备注字段

**业务流程**
```plain
财务人员导出收款信息 → 核对银行数据 → 发现缺少备注字段
↓
财务需补充备注信息 → 只能逐条手动编辑 → 效率低下
```

#### **3.2.1 收款信息导出增加备注字段**

**位置**：基础档案 → 收款信息 → 导出

**功能说明**

**字段位置**

|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 1 | 序号 | 现有字段 |
| 2 | 账户类型 | 现有字段 |
| 3 | 账户名称 | 现有字段 |
| 4 | 银行账号 | 现有字段 |
| 5 | 银行名称 | 现有字段 |
| 6 | 开户行所在省 | 现有字段 |
| 7 | 开户行所在市 | 现有字段 |
| 8 | 开户行 | 现有字段 |
| 9 | 全员可见 | 现有字段 |
| 10 | 可见员工工号 | 现有字段 |
| 11 | 可见部门 | 现有字段 |
| 12 | 可见角色 | 现有字段 |
| 13 | 联行号 | 现有字段 |
| 14 | **备注** | **新增字段，位置在最后** |

**字段格式**
- 字段名：备注
- 数据类型：文本
- 内容：取收款账户的备注字段值
- 为空时：单元格留空

#### **3.2.2 收款信息导入模板增加备注字段**

**位置**：基础档案 → 收款信息 → 导入 → 下载模板

**功能说明**

导入模板中增加"备注"列，支持批量导入时携带备注信息。

**字段位置**

| **序号** | **字段名称** | **必填** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| 1 | 序号 | 是 | 现有字段 |
| 2 | 账户类型 | 是 | 现有字段 |
| 3 | 账户名称 | 否 | 现有字段 |
| 4 | 银行账号 | 否 | 现有字段 |
| 5 | 银行名称 | 否 | 现有字段 |
| 6 | 开户行所在省 | 否 | 现有字段 |
| 7 | 开户行所在市 | 否 | 现有字段 |
| 8 | 开户行 | 否 | 现有字段 |
| 9 | 全员可见 | 否 | 现有字段 |
| 10 | 可见员工工号 | 否 | 现有字段 |
| 11 | 可见部门 | 否 | 现有字段 |
| 12 | 可见角色 | 否 | 现有字段 |
| 13 | **备注** | **否** | **新增字段，位置在最后** |

**字段格式**
- 字段名：备注
- 数据类型：文本
- 最大长度：500字符（与前端填写限制保持一致）
- 必填：否
- 校验规则：
• 超过500字符时，截取前500字符导入
• 支持任意文本内容，包括中文、英文、数字、特殊符号

**导入逻辑**
- 备注字段为空时，不更新备注（保持原有值或为空）
- 备注字段有值时，覆盖原有备注
- 导入校验不通过时，在错误提示中标识具体行号的备注字段问题

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 存量数据导出 | 历史收款账户已有备注信息 | 正常导出，显示实际备注内容 |
| 空备注导出 | 收款账户备注为空 | Excel单元格留空，不显示null或undefined |
| 长文本导出 | 备注内容超过Excel单元格显示长度 | 正常导出完整内容，Excel自动处理显示 |
| 特殊字符导入 | 备注包含换行符、制表符等特殊字符 | 保留特殊字符，正常导入存储 |
| 超长文本导入 | 备注超过500字符 | 自动截取前500字符，给出警告提示tips“导入成功，备注超过500上限”但不阻断导入 |
| 模板列缺失 | 使用旧版模板（无备注列）导入 | 兼容处理，备注字段按空值处理，其他字段正常导入 |
| 存量覆盖导入 | 更新已有收款账户且备注字段为空 | 保持原有备注不变，不覆盖为空 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 收款信息列表展示 | 独立功能 | 列表页不展示备注字段（保持现状），编辑弹窗可查看/编辑备注 |
| 单据选择收款信息 | 独立功能 | 选择收款信息时不展示备注，保持现有交互 |
| 收款信息API接口 | 关联功能 | 接口已支持remark字段，无需修改 |

## **附录**

**A. 典型使用场景**

场景：财务批量维护收款账户并添加备注信息

配置步骤（首次）：
1. 进入基础档案 → 收款信息
2. 点击"导入" → "下载模板"
3. 在模板中填写收款账户信息，包括备注列
4. 上传模板，完成批量导入

使用步骤（日常导出核对）：
1. 进入收款信息页面
2. 点击"导出"按钮
3. 打开导出文件，查看包含备注信息的完整数据
4. 与银行数据进行核对

结果：财务人员可在导出文件中直接查看备注信息，无需逐条进入系统查看

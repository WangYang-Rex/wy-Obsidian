## 1\. 需求背景

### 1.1 需求描述

统一消息中心与单据管理的单据详情展示行为：消息中心查看已完成的单据详情时，备注字段按「完成后隐藏」配置正确隐藏。

### 1.2 需求背景
- **需求方**：审批人员、抄送人员
- **现状问题**：
• 单据表单编辑中，备注组件支持勾选「完成后隐藏」属性，配置后单据审批完成时该字段不在单据上展示
• 单据管理 → 已完成的单据详情 → 备注字段已按「完成后隐藏」配置正确隐藏
• 消息中心 → 审批单据 → 查看已完成的单据详情 → 备注字段**未按配置隐藏**，仍可查看
- **解决思路**：消息中心单据详情页的字段展示逻辑与单据管理保持一致，读取同一套「完成后隐藏」配置并生效

**业务流程**：
```
管理员配置：基础设置 → 自定义单据 → 单据表单编辑 → 备注组件 → 勾选「完成后隐藏」
  ↓
员工提交单据 → 单据审批完成（状态变为「已完成」）
  ↓
用户查看单据详情：
  ├─ 单据管理 → 已完成单据详情 → 备注字段已隐藏 ✅
  └─ 消息中心 → 审批单据 → 已完成单据详情 → 备注字段未隐藏 ❌（本次修复）

```

## 2\. 功能现状

### 2.1 备注组件「完成后隐藏」配置
- **位置**：基础设置 → 自定义单据 → 单据表单编辑 → 备注组件 → 属性设置
- **配置项**：「完成后隐藏」复选框（见图1）
- **作用**：勾选后，单据状态为「已完成」时，该备注字段在单据详情中不展示
- **当前生效范围**：单据管理-已完成单据详情页

### 2.2 消息中心审批单据详情
- **位置**：消息中心 → 审批单据 → 点击单据 → 查看单据详情弹窗
- **当前行为**：展示单据所有字段，不判断「完成后隐藏」配置
- **缺失能力**：未与单据管理的字段展示逻辑对齐

### 3.2 详细功能描述

#### 3.2.1 消息中心已完成单据字段展示逻辑对齐

**功能概述**

消息中心审批单据详情页中，字段的展示逻辑与单据管理详情页保持一致：当单据状态为「已完成」时，读取各字段的「完成后隐藏」配置，已勾选的字段不展示。

**配置入口**

不涉及新增配置入口，复用现有「完成后隐藏」配置：基础设置 → 自定义单据 → 单据表单编辑 → 各字段组件 → 属性设置 → 完成后隐藏

**业务处理逻辑**
```
用户点击消息中心审批单据 → 打开单据详情弹窗
  ↓
系统读取单据状态
  ├─ 状态 ≠ 已完成 → 正常展示所有字段（含备注）
  └─ 状态 = 已完成 → 读取该单据类型各字段的「完成后隐藏」配置
                        ↓
                      字段配置了「完成后隐藏」？
                        ├─ 是 → 该字段不展示
                        └─ 否 → 该字段正常展示

```

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 单据未完成 | 单据状态为审批中/待提交/已驳回等 | 不判断「完成后隐藏」，所有字段正常展示 |
| 备注字段为空 | 备注字段无内容 | 无论是否配置「完成后隐藏」，空备注均不展示（保持现有逻辑） |
| 多个备注组件 | 单据中有多个备注字段，部分配置隐藏、部分未配置 | 每个备注组件独立按自身配置生效 |
| 其他字段也配置了「完成后隐藏」 | 非备注字段（如描述、附件说明等）也勾选了「完成后隐藏」 | 同步生效，消息中心中该字段也按配置展示/隐藏 |
| 单据状态回退 | 已完成单据被撤回/驳回，状态变回非已完成 | 备注字段恢复展示 |
| 存量已完成单据 | 历史已完成的单据 | 按配置正确展示/隐藏（本次修复后生效） |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| 单据管理-已完成单据详情 | 同一套展示逻辑 | 消息中心与单据管理共用同一套字段展示过滤逻辑 |
| 打印模板 | 独立功能 | 打印模板的字段展示不受「完成后隐藏」影响，保持现有逻辑 |
| 移动端消息中心 | 如有该入口 | PC端修复后，移动端同步生效 |

## 5\. 验收标准

| 序号 | 验收项 | 验收标准 |
|------|---------|------------|
| 1 | 备注已配置「完成后隐藏」的已完成单据 | 消息中心 → 审批单据 → 打开已完成的单据详情，备注字段不展示 |
| 2 | 备注未配置「完成后隐藏」的已完成单据 | 消息中心 → 审批单据 → 打开已完成的单据详情，备注字段正常展示 |
| 3 | 未完成的单据 | 消息中心 → 审批单据 → 打开审批中的单据详情，备注字段正常展示（不受「完成后隐藏」影响） |
| 4 | 单据管理对比验证 | 同一单据，分别在单据管理和消息中心查看，两者备注展示行为完全一致 |
| 5 | 包含多个备注组件的单据 | 每个备注组件独立按自身「完成后隐藏」配置生效 |
| 6 | 其他字段「完成后隐藏」同步生效 | 非备注字段配置了「完成后隐藏」的，消息中心中该字段也正确隐藏 |
| 7 | 单据状态回退 | 已完成单据被驳回/撤回后，消息中心中查看该单据，备注字段恢复展示 |
| 8 | 历史已完成单据 | 历史已完成的单据，消息中心中查看时备注字段按配置正确展示/隐藏 |

## 6\. 附录

### A. 典型使用场景

**场景**：管理员配置备注「完成后隐藏」，员工查看已完成单据

配置步骤（首次）：
1. 管理员进入：基础设置 → 自定义单据 → 单据表单编辑
2. 选中备注组件 → 属性设置 → 勾选「完成后隐藏」
3. 保存单据表单

使用步骤（日常）：
1. 员工提交报销单，填写备注「2月和3月备用金申请」
2. 单据审批完成，状态变为「已完成」
3. 审批人员/抄送人员在消息中心 → 审批单据中查看该已完成单据

结果：备注字段按「完成后隐藏」配置正确隐藏，不展示在单据详情中

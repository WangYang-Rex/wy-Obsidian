## 1\. 需求背景

### 1.1 需求描述

在预算报表的**调整额报表**列表中，当统计调整筛选仅勾选"预算申请"时，调整额字段列支持点击下探，在新页面展示关联的预算申请单明细列表。列表支持点击单条明细查看预算申请单详情，支持导出Excel。

### 1.2 需求背景

**需求方**：财务/预算管理员

**现状问题**：
- 预算报表展示调整金额，但无法追溯金额调整来源
- 用户需手动去预算申请模块查询，操作路径长
- 缺乏从报表数据到原始单据的穿透能力

**解决思路**：特定筛选条件下，调整额字段值变为可点击链接，一键下探查看关联预算申请单，并可进一步查看单据详情。

### 1.3 业务流程
```
用户进入预算报表
  ↓
选择"调整额报表" → 列表展示调整额字段列
  ↓
仅勾选"预算申请" → 调整额变为蓝色链接
  ↓
点击调整额 → 新标签页打开预算申请单明细页
  ↓
查看14字段列表 / 点击明细查看详情 / 导出Excel

```

## 2\. 功能现状

**功能所在页面路径**：报表 → 预算报表 → 调整额报表

**当前能力**：
- 统计调整多选：手工调整、上期结转、预算申请、动态预算
- 调整额字段：纯文本展示，无交互
- 表格支持图表/表格视图切换
- 报表场景下管理员过滤条件放宽（isFilter = false），可查看全量数据

### 3.2 详细功能描述

#### 3.2.1 调整额点击下探

**位置**：报表 → 预算报表 → 调整额报表 → 调整额字段列

**交互规则**：

| 统计调整勾选状态 | 调整额样式 | 是否可点击 | 说明 |
|------------------------|---------------|---------------|------|
| 仅勾选"预算申请" | 蓝色链接，带下划线，hover加深 | ✅ 可点击 | 点击下钻展示关联预算申请单 |
| 勾选多个（含预算申请\+其他） | 黑色纯文本 | ❌ 不可点击 | 混合数据无法精准关联 |
| 仅勾选其他（手工调整/上期结转/动态预算） | 黑色纯文本 | ❌ 不可点击 | 不支持下钻 |
| 未勾选任何项 | 黑色纯文本 | ❌ 不可点击 | 不支持下钻 |

**点击行为**：
- 点击后新标签页打开预算申请单明细页
- 原预算报表页面保持当前筛选状态不变
- URL携带筛选参数（年度、预算维度等）

#### 3.2.2 预算申请单明细页（新页面）

**页面路径**：报表 → 预算报表 → 调整金额明细

**页面标题**：预算申请明细-\[预算维度名称\]

**列表字段**（对齐导出模板 `调整金额预算申请单导出模版.xlsx`）：

| 序号 | 字段名 | 字段说明 | 数据来源 |
|------|---------|------------|------------|
| 1 | 单据类型 | 预算调整单 | budget\_apply.type |
| 2 | 单号 | 预算申请单号 | budget\_apply.bill\_no |
| 3 | 单据状态 | 完成/审批中 | budget\_apply.status |
| 4 | 标题 | 申请标题 | budget\_apply.title |
| 5 | 描述 | 申请描述 | budget\_apply.description |
| 6 | 申请人 | 申请人姓名 | user.name |
| 7 | 部门 | 费用承担部门/成本中心 | dept.name |
| 8 | 辅助核算 | 关联辅助核算 | auxiliary.name |
| 9 | 项目 | 关联项目 | project.name |
| 10 | 提交时间 | 单据提交时间 | budget\_apply.submit\_time |
| 11 | 审批完成时间 | 最终审批完成时间 | budget\_apply.approve\_time |
| 12 | 预算名称 | 预算方案名称 | budget.name |
| 13 | 预算单元 | 预算维度（部门/成本中心/项目等） | budget\_unit.name |
| 14 | 预算详情 | 调整期间及金额 | budget\_detail.period \+ budget\_detail.amount |

**筛选条件**：
- 自动带入原预算报表的筛选维度（年度、预算维度等）
- 自动筛选与该调整金额关联的预算申请单
- 新页面支持独立筛选，不影响原报表页

**点击明细查看详情**：
- 点击列表中任意一条明细，弹出预算申请单详情查看页
- 详情页为只读模式，不可编辑
- 详情页展示该预算申请单的完整信息（同预算申请模块的详情页）

**分页**：默认20条/页，数据量大时分页展示

#### 3.2.3 导出功能

**位置**：预算申请单明细页右上角

**导出规则**：
- 点击"导出"按钮，导出当前列表数据为Excel
- 导出字段与列表字段完全一致（14个字段）
- 导出模板沿用 `调整金额预算申请单导出模版.xlsx` 格式
- 导出数据量上限：单次最多导出10000条

#### 3.2.4 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 全部功能适配，蓝色链接、新标签页打开、导出Excel |
| 移动端 | 下钻逻辑同PC端，列表改为卡片式堆叠展示，支持导出 |

### 3.3 边界处理

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 调整金额为0 | 该维度无预算申请调整 | 显示"0.00"，黑色纯文本，不可点击 |
| 无关联预算申请单 | 数据异常或筛选条件过于严格 | 新页面展示空列表，按原产品空信息逻辑展示 |
| 新页面修改筛选 | 用户在新页面修改筛选条件 | 允许独立筛选，不影响原报表页 |
| 一对多数据量大 | 关联单据超过100条 | 分页展示，默认20条/页，单次导出上限10000条 |
| 详情页权限 | 点击明细查看详情 | 沿用报表场景权限逻辑（isFilter = false），管理员可查看全量单据详情 |

## 6\. 附录

### 6.1 典型使用场景

**场景：追溯预算调整来源**

财务查看2026年某部门预算报表，选择"调整额报表"后发现调整金额异常。仅勾选"预算申请"后点击调整额，新页面展示该部门所有关联预算申请单。财务点击其中一条明细查看详情，快速定位是哪笔申请导致了调整，并导出Excel存档。

### 6.2 相关数据模型

**调整金额与预算申请单关系**：
- 一对多：一个调整金额可能对应多笔预算申请单
- 关联维度：年度 \+ 预算维度（部门/成本中心/项目等）\+ 调整类型 = 预算申请

**关键字段映射**：

| 前端字段 | 后端字段 | 说明 |
|------------|------------|------|
| 单据类型 | budget\_apply.type | 预算调整单类型 |
| 单号 | budget\_apply.bill\_no | 预算申请单号 |
| 单据状态 | budget\_apply.status | 完成/审批中 |
| 标题 | budget\_apply.title | 申请标题 |
| 描述 | budget\_apply.description | 申请描述 |
| 申请人 | user.name | 关联用户表 |
| 部门 | dept.name | 关联部门表 |
| 辅助核算 | auxiliary.name | 关联辅助核算表 |
| 项目 | project.name | 关联项目表 |
| 提交时间 | budget\_apply.submit\_time | 单据提交时间 |
| 审批完成时间 | budget\_apply.approve\_time | 最终审批完成时间 |
| 预算名称 | budget.name | 关联预算方案表 |
| 预算单元 | budget\_unit.name | 关联预算维度表 |
| 预算详情 | budget\_detail.period \+ amount | 调整期间及金额 |

### 6.3 接口参考

| 接口名称 | 请求方式 | 说明 |
|------------|------------|------|
| /api/budget/report/adjustment/detail | GET | 预算申请单明细列表查询 |
| /api/budget/report/adjustment/export | POST | 预算申请单列表导出 |
| /api/budget/apply/detail/\{id\} | GET | 预算申请单详情查询（复用现有接口） |

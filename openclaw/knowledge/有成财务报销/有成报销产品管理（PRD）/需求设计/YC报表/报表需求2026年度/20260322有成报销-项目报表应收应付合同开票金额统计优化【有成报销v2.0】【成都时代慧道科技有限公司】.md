## **1\. 需求背景**

### **1.1 需求描述**

优化项目报表中应收合同/应付合同维度的开票金额统计逻辑，将关联合同计划的收付款单中发票核销金额纳入统计，确保开票金额计算公式完整准确：开票金额 = 合同内计划上传的发票核销金额 \+ 关联单据发票核销合同计划金额。

### **1.2 需求背景**
- **需求方**：财务
- **现状问题**：
• 客户查看项目报表-应收合同/应付合同报表时，开票金额仅统计了合同自身上传的发票金额
• 关联合同计划的收付款单（报销单、付款单、收款单等）中发票核销合同计划的金额未被纳入统计
• 导致项目报表展示的合同开票金额与实际业务发生不符，影响财务对账和分析
• 财务人员需要跨模块手动核对合同计划和关联单据的发票核销记录，工作效率低且易出错
- **解决思路**：
• 统一开票金额计算公式：开票金额 = 合同内发票 \+ 关联单据发票核销金额
• 关联单据包括：报销单、付款单、收款单等涉及合同计划发票核销的业务单据 

**业务流程**
```plain
合同创建并维护合同计划
  ↓
收付款单/报销单关联合同计划并提交
  ↓
发票关联单据费用并核销合同计划金额
  ↓
合同开票金额 = 合同内计划上传的发票核销金额 + 关联单据发票核销合同计划金额
  ↓
项目报表展示完整开票金额
```

### **3.2 详细功能描述**

#### **3.2.1 开票金额计算公式统一定义**

**计算公式**
```plain
开票金额 = 合同内计划上传的发票核销金额 + 关联单据发票核销合同计划金额

其中：
• 合同内计划上传的发票核销金额 = 合同计划上直接上传的发票的核销金额合计
• 关联单据发票核销合同计划金额 = 关联合同计划的收付款单/报销单中，发票核销该合同计划的核金额合计
```



**关联单据范围**

| **单据类型** | **适用场景** | **统计规则** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 收款单 | 应收合同 | 发票核销合同计划的金额计入开票金额 |
| 付款单 | 应付合同 | 发票核销合同计划的金额计入开票金额 |
| 报销单 | 应收/应付合同 | 费用关联的发票核销合同计划的金额计入开票金额 |
| 借款单 | 应付合同 | 核销时关联的发票金额计入开票金额 |

#### **3.2.2 应收合同开票金额统计优化**

**功能概述** 优化应收合同报表中开票金额的统计逻辑，将关联合同计划的收款单、报销单中发票核销金额纳入统计。

**统计逻辑调整**

**原逻辑**：
```plain
开票金额 = 合同计划直接上传发票核销金额合计
```

**新逻辑**：
```plain
开票金额 = 合同计划直接上传发票核销金额合计 + 关联单据发票核销金额合计
```

#### **3.2.3 应付合同开票金额统计优化**

**功能概述** 优化应付合同报表中开票金额的统计逻辑，将关联合同计划的付款单、报销单、借款单中发票核销金额纳入统计。

**统计逻辑调整**

**新逻辑**：
```plain
开票金额 = 合同计划直接上传发票核销金额合计 + 关联单据发票核销金额合计
```



### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 部分核销 | 发票金额大于实际核销金额 | 按实际核销金额计入开票金额 |
| 多笔核销 | 一个合同计划被多笔单据分次核销 | 各笔核销金额累加计入 |
| 核销撤回 | 已核销金额被撤回 | 从开票金额中扣除对应金额 |
| 发票红冲 | 已统计发票发生红冲 | 红冲金额从开票金额中扣除 |
| 跨合同核销 | 一张发票核销多个合同计划 | 按各合同计划实际核销金额分别计入 |
| 单据驳回 | 关联单据审批被驳回 | 驳回后核销金额不生效，不计入开票金额 |
| 单据删除 | 关联单据被物理删除 | 删除后对应核销金额从开票金额中扣除 |
| 发票作废 | 已核销发票被作废 | 作废发票金额从开票金额中扣除 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 合同计划管理 | 数据来源 | 合同金额作为未开票金额计算基数 |
| 收付款单 | 数据来源 | 单据审批通过后，发票核销金额才计入合同开票金额 |
| 发票管理 | 数据来源 | 发票状态影响统计：已验真且未作废/红冲的发票才计入 |
| 报表导出 | 数据展示 | 导出数据与界面展示一致 |
| 自定义报表 | 数据源 | 自定义报表数据源同步更新开票金额计算逻辑 |

**存量数据处理** • 存量数据按新公式重新计算开票金额
• 历史关联单据的发票核销记录纳入统计

## **附录**

**A. 典型使用场景**

场景：财务查看项目报表，核对采购合同的开票情况

使用步骤：
1. 财务登录系统，进入【报表中心-项目报表-应付合同】
2. 查看合同列表，开票金额列展示完整金额（合同内发票核销金额\+关联单据内发票核销合同金额）
3. 点击【项目报表-应收/应付合同报销-开票金额】，查看相关发票：
    - 合同内核销发票：¥50,000
    - 关联单据发票核销：¥30,000
    - 开票金额合计：¥80,000
4. 导出报表用于供应商对账

结果：财务可准确掌握合同开票全貌，开票金额统计口径完整

## 4、需求详情

### 4.1、需求说明

API接口地址：

[<u>https://open.yonyouup.com/apiCenter/index</u>](https://open.yonyouup.com/apiCenter/index)

一、获取token

[<u>https://open.yonyouup.com/apiCenter/token\_get</u>](https://open.yonyouup.com/apiCenter/token_get)

二、获取科目信息：

[<u>https://open.yonyouup.com/apiCenter/code\_batch\_get</u>](https://open.yonyouup.com/apiCenter/code_batch_get)
- 批量查询，存储科目编码、科目名称、外币名称、辅助核算（个人/客户/供应商/部门/项目往来核算）
- 按账套分别存储

三、获取辅助核算信息

1、客户：
- 客户分类：客户分类信息同步到“客户”类别下

[<u>https://open.yonyouup.com/apiCenter/customerclass\_batch\_get</u>](https://open.yonyouup.com/apiCenter/customerclass_batch_get)
- 客户信息：批量查询同步到”客户“类别下，存储客户编码、客户名称根据所属分类码自动放到对应类别下，按树排列--分类码具体调接口再看下

[<u>https://open.yonyouup.com/apiCenter/customer\_batch\_get</u>](https://open.yonyouup.com/apiCenter/customer_batch_get)

2、供应商：
- 供应商分类：供应商分类信息同步到“供应商”类别下

[<u>https://open.yonyouup.com/apiCenter/vendorclass\_batch\_get</u>](https://open.yonyouup.com/apiCenter/vendorclass_batch_get)
- 供应商信息：批量查询同步到”供应商“类别下，存储供应商编码、供应商名称根据所属分类码自动放到对应类别下，按树排列--分类码具体调接口再看下

[<u>https://open.yonyouup.com/apiCenter/vendor\_batch\_get</u>](https://open.yonyouup.com/apiCenter/vendor_batch_get)

3、部门：同步到“部门”类别下，存储部门编码、部门名称

根据编码级次，判断部门层级，按树存储

[<u>https://open.yonyouup.com/apiCenter/department\_batch\_get</u>](https://open.yonyouup.com/apiCenter/department_batch_get)

四、生成凭证推送过去

[<u>https://open.yonyouup.com/apiCenter/voucher\_add</u>](https://open.yonyouup.com/apiCenter/voucher_add)

| **字段** | **说明** |
|----------|----------|
| voucher\_type | 凭证类别：凭证字 |
| fiscal\_year | 凭证所属的会计年度：凭证日期的年份 |
| accounting\_period | 凭证所属的会计年度：凭证日期的月份 |
| voucher\_id | 凭证号 |
| date | 制单日期 |
| enter | 制单人名称 |
| entry\_id | 分录号 |
| account\_code | 科目编码 |
| abstract | 摘要 |
| currency | 币种 |
| exchange\_rate2 | 汇率 |
| primary\_debit\_amount | 借方原币/贷方原币 |
| natural\_debit\_currency | 借方本位币金额/贷方本位币金额 |
| dept\_id | 部门：根据科目上挂的辅助核算，有则填入 |
| cust\_id | 客户：根据科目上挂的辅助核算，有则填入 |
| supplier\_id | 供应商：根据科目上挂的辅助核算，有则填入 |

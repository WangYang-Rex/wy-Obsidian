#### 一、应用中心-日常管理中新增模块【钉钉日程】：

1、文案介绍：申请单同步至钉钉日程，可在钉钉中查看所有日程安排，重要事情一目了然

2、开通按钮：未开通时显示，点击可直接开通与日程的对接

3、开通后：



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/8oLl952Ee3ao0lap/img/da0d24c1-f2f9-4dc5-9d9c-3b07c3ce336b.png?Expires=1775016881&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=BsMLro5drZmmj0iSHXU%2B%2BOAnl6g%3D "")
- 显示“开通中”状态；
- 同时显示“管理配置”和“停用”的文本操作按钮；管理配置：点击打开一个弹窗，配置同步信息停用：需要二次弹框确认，“停用后，申请单将不会同步到钉钉日程，确定停用吗？”；确定停用后，状态变更为未开通状态；

#### 二、管理配置页面：

点击管理配置，打开弹窗，配置需要同步的单据：
- 只显示申请单类型可选；至少需要指定一个单据类型
- 可选多个；



![Picture 3](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/8oLl952Ee3ao0lap/img/a31bd436-7368-4da5-ba1f-09b530c4180d.png?Expires=1775016881&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=b3CGc3GVtFtH5rC2yaT1rjvgGtY%3D "")

##### 1、创建日程

[<u>https://open.dingtalk.com/document/isvapp/create-schedule</u>](https://open.dingtalk.com/document/isvapp/create-schedule)

一、根据管理配置中配置的单据，**在单据完成之后**，调用创建日程接口，同行人和申请人中创建一条日程提醒；

**BODY参数**

| **钉钉侧字段** | **有成字段** |
|-------------------|----------------|
| 日程标题 summary | 常用字段：标题 |
| 日程描述 description | 常用字段：描述 |
| 日程开始时间  start | \+ 日程开始时间和日程结束时间都使用**常用字段中的【日期】**字段；**补充的需求内容：****优先取常用字段-日期，没有再查单据固定的日期（申请日期），这种没有日期类型的，统统走日期类型=日期这个逻辑**\+ **取单据字段-日期以及行程中的日期字段；**如果常用字段同时存在于单据和行程中，优先取行程中的时间；如果行程组件中有多段行程，需要取到所有行程，找到日期字段后，将连续的时间拼接起来，如果涉及到多段行程，则需要多次调用接口，创建多段日程：比如行程一时间为：2024-11-09至2024-11-12行程二时间为：2024-11-14至2024-11-16那么创建时间时，是创建了两段日程，一个申请单绑定了两个日程ID，后续若申请单删除，也需要删除这两个日程 |
| 日程开始时间  date | 根据日期组件做不同的处理：1、日期类型=日期，格式为时分秒或上午/下午，此字段留空2、日期类型=日期，格式为年月日；填入日期字段3、日期类型=起止日期，取开始日期填入 |
| 日程开始时间  datetime | 格式为ISO-8601的date-time格式1、日期类型=日期，格式为时分秒或上午/下午，填入时间：\+ 时分秒：直接将日期\+时分秒填入\+ 上午/下午：将上下午转换为时分秒格式后填入，取起始时间填入，比如00：00：00、12：00：00上午为：00：00：00-11：59：59下午为：12：00：00-23：59：592、日期类型=日期，格式为年月日；此字段为空3、日期类型=起止日期，此字段为空4、传入过去的时间值需要按照ISO-8601的date-time格式，加上相对于 UTC 的时区偏移量，下面的字段timezone是时区，根据时区要处理一下：比如中国是东八区，传过去的格式如：2020-01-01T10:15:30\+08:00 |
| 日程开始时间  timezone | 指时区   TZ database name格式，传入时区字段，比如：\+ **Asia/Shanghai**：中国标准时间（CST） |
| 日程结束时间  end |  |
| 日程结束时间  date | 根据日期组件做不同的处理：1、日期类型=日期，格式为时分秒或上午/下午，此字段留空2、日期类型=日期，格式为年月日；填入日期字段3、日期类型=起止日期，取结束日期填入 |
| 日程结束时间  datetime | 格式为ISO-8601的date-time格式1、日期类型=日期，格式为时分秒或上午/下午，填入时间：\+ 时分秒：将日期\+时分秒填入\+ 上午/下午：将上下午转换为时分秒格式后填入，取结束时间填入，比如11：59：59、23：59：59上午为：00：00：00-11：59：59下午为：12：00：00-23：59：592、日期类型=日期，格式为年月日；此字段为空3、日期类型=起止日期，此字段为空4、传入过去的时间值需要按照ISO-8601的date-time格式，加上相对于 UTC 的时区偏移量，下面的字段timezone是时区，根据时区要处理一下：比如中国是东八区，传过去的格式如：2020-01-01T10:15:30\+08:00 |
| 日程结束时间  timezone | 指时区   TZ database name格式，传入时区字段，比如：\+ **Asia/Shanghai**：中国标准时间（CST） |
| ~~是否全天日程   isallday~~ |  |
| 日程参与人列表 attendees | 传入【申请人】字段只取单据字段中的【同行人】字段，将同行人一起传入。费用中若有同行人字段暂不取。 |
| 日程地点  location |  |
| 日程提醒  reminders |  |
| 提醒方式   method | 传入：**dingtalk**: 钉钉内提醒时间字段【minutes】不传 |
| 通知  extra | \+ noPushNotification：创建日程时是否向关联人员发送钉钉推送通知。默认传入：false--发送 |

返回参数中，将申请单与“日程ID”关联起来，后续申请单删除需要用到此字段

#### 四、操作日志

系统设置-操作日志-数据对接日志中增加日程同步的日志记录：
- 对接系统：钉钉日程
- 接口描述：创建日程、删除日程
- 其他字段自行填入
- 推送失败的支持重新推送（在操作栏中）

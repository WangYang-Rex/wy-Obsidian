## 1\. 需求背景

### 1.1 需求描述

有成报销飞书版本接入飞书官方分享能力，补齐飞书端与钉钉端在单据分享功能上的差距，使用户能够在飞书生态内将单据以消息卡片形式快速分享给联系人或群组。

### 1.2 需求背景

当前有成报销同时支持钉钉和飞书两大平台。钉钉版本已实现完整的单据分享功能：用户在单据详情页点击分享，唤起钉钉联系人选择器，选择会话后发送消息卡片。而飞书版本目前分享按钮被隐藏未渲染，用户无法直接在飞书内分享单据，沟通成本较高。

飞书端技术基础已具备：
- 飞书 JS-SDK 已在前端加载（登录流程完成）
- 后端飞书消息发送能力已实现（`FeiShuCardMsgHandler`）
- 飞书平台 API 定义已存在（`PlatformType` 包含飞书标识）

### 1.3 业务流程
```
用户进入飞书端有成报销 → 打开单据详情页
    ↓
点击「分享」按钮（飞书平台判断后展示）
    ↓
唤起飞书 chooseContact 选择器
    ↓
用户选择联系人/群组
    ↓
确认发送 → 后端构建消息卡片 → 调用飞书消息API
    ↓
目标飞书会话收到单据消息卡片

```

## 2\. 功能现状

### 2.1 钉钉版分享功能（已有）

钉钉版本已实现完整的单据分享流程：
- **前端**：分享按钮组件 → 唤起分享选择器 → 调用钉钉 `dd.biz.chat.chooseConversation` API
- **后端**：`OrderInfoController.shareOrder()` → 消息业务逻辑 → `DingTalkService.sendConversationMsg()` → 钉钉API底层调用
- **消息卡片**：`TaskCardContentBuild` 构建消息卡片内容，包含单据标题、金额、申请人、状态等字段

### 2.2 飞书版现状
- 飞书 JS-SDK 已引入，前端已加载
- 飞书平台 API 定义已存在（`PlatformType` 包含 FeiShu）
- **分享按钮未渲染**：现有逻辑通过 `Platform.includes(dingtalk)` 判断，飞书不满足条件，分享按钮不展示
- **后端能力已具备**：`FeiShuCardMsgHandler` 已实现飞书消息卡片发送能力

### 2.3 现有痛点

| 痛点 | 影响场景 | 当前 workaround |
|------|------------|-----------------|
| 飞书端无法分享单据 | 财务人员需要在飞书里把单据发给同事确认 | 截图或复制链接手动发送，体验差 |
| 飞书端分享入口缺失 | 审批人员习惯用分享快速转发单据 | 无 workaround |

## 3\. 本次变更内容

### 3.1 功能变更清单

| 序号 | 功能点 | 变更类型 | 影响范围 | 优先级 |
|------|---------|------------|------------|---------|
| 1 | 飞书端分享按钮渲染 | 新增 | 单据详情页（飞书端） | P0 |
| 2 | 飞书 chooseContact 选择器唤起 | 新增 | 单据详情页（飞书端） | P0 |
| 3 | 飞书消息卡片发送 | 新增 | 后端消息发送逻辑 | P0 |

### 3.2 详细功能描述

#### 3.2.1 分享入口：单据详情页分享按钮

**界面变化**：
- 单据详情页操作栏增加「分享」按钮，位置与钉钉版保持一致
- 按钮渲染通过平台判断控制：`Platform.includes` 适配飞书平台标识

**平台判断逻辑**：
- 钉钉平台：`Platform.includes('dingtalk')` 为 true，展示分享按钮，走钉钉分享流程
- 飞书平台：`Platform.includes('feishu')` 为 true（或对应飞书标识），展示分享按钮，走飞书分享流程
- 其他平台：不满足上述条件，不展示分享按钮

#### 3.2.2 分享流程：选择会话 → 发送消息卡片

**前端流程**：
1. 用户点击「分享」按钮
2. 前端调用飞书 JS-SDK `chooseContact` API，唤起飞书联系人/群组选择器
3. 用户选择目标会话（单选：一个联系人或一个群组）
4. 用户点击确认，前端将选择的会话信息传递给后端
5. 后端构建消息卡片并调用飞书消息发送接口

**消息卡片内容**：
- 复用钉钉版现有消息卡片模板结构
- 卡片字段：单据标题、单据类型、金额、申请人、申请日期、审批状态等关键信息
- 卡片底部携带「查看详情」跳转链接
- **后续根据飞书消息卡片接口实际字段结构做适配调整**

#### 3.2.3 后端消息发送路径

**后端改动**：
- 在 `OrderInfoController.shareOrder()` 或同类分享入口中，增加飞书消息发送分支
- 复用已有的消息构建逻辑（`TaskCardContentBuild`）
- 调用已有的飞书消息发送能力（`FeiShuCardMsgHandler`）
- 发送路径：分享入口 → 消息业务逻辑 → 飞书消息发送分支 → 飞书开放平台 API

### 3.3 边界处理

| 场景 | 处理方式 |
|------|------------|
| 用户未选择会话点击取消 | 关闭选择器，不发送消息，停留在单据详情页 |
| 用户未选择会话点击确认 | 确认按钮置灰不可点 |
| 发送过程中网络异常 | 提示发送失败，不重复发送 |
| 飞书 JS-SDK 版本不兼容 | 给出友好提示，引导用户更新飞书客户端 |
| 用户无单据查看权限 | 分享按钮不展示或点击后提示无权限 |
| 同一单据多次分享 | 每次分享独立发送，支持分享到多个不同会话 |
| 分享到群组 | 群组成员均能看到消息卡片 |

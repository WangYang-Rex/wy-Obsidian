### **1\. 需求背景**

#### **1.1 需求描述**

审批流评论区@某人时，企业微信工作通知及"待办-我参与的"入口，均根据被@人实时审批权限动态展示审批操作。

#### **1.2 需求背景**
- **需求方：** 审批人/被@员工
- **现状问题：**
    - 被@人收到企业微信工作通知，点击后进入有成报销H5单据详情
    - 被@人可在"待办-我参与的"看到@他的单据
    - 两个入口均只展示单据详情，即使被@人有审批权限也无法操作
    - 需返回待办列表找到单据才能审批，路径冗余
- **解决思路：** 企业微信员工点击被@单据工作通知打开页面时，实时读取该账号在审批流中的权限，根据权限展示带审批操作的单据详情；"我参与的"页面同步按此逻辑改造

**业务流程：**
```plain
被@人收到通知/进入我参与的列表
  ↓
点击单据进入详情
  ↓
实时读取该员工在审批流的权限
  ↓
权限判定
  ├─ 当前节点有权限 → 展示单据详情 + 审批操作
  └─ 无权限 → 仅展示单据详情
```

### **2\. 功能现状**

**功能所在页面路径：**
- 企业微信：工作通知 → 有成报销@提醒 → 单据详情（H5）
- 有成APP：待办 → 我参与的 → 单据详情

**当前能力：**
- 评论区@某人，被@人收到工作通知
- 被@人可在"我参与的"列表查看单据
- 单据详情展示完整单据信息

**当前缺失的能力：**
- 未根据被@人实时审批权限动态展示审批操作入口

### **3\. 本次变更内容**

#### **3.1 功能变更清单**

| **变更类型** | **功能点** | **功能说明** | **优先级** |
|----------------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|-------------------------------------------------------------|
| 修改 | H5单据详情权限渲染 | 企业微信通知H5页面实时读取权限，动态展示审批操作 | P0 |
| 修改 | 我参与的页面权限渲染 | 我参与的入口实时读取权限，动态展示审批操作 | P0 |

#### **3.2 详细功能描述**

##### **3.2.1 实时权限读取与渲染**

**核心逻辑** 进入单据详情页时，实时调用权限查询接口，根据返回结果渲染页面。

**权限判定规则**

| **场景** | **实时权限状态** | **页面展示** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 场景1 | 当前审批节点处理人 | 单据详情 \+ 审批操作区 | 有审批权限 |
| 场景2 | 非当前节点（提前@） | 仅单据详情 | 无审批权限 |
| 场景3 | 已审批完成/流程结束 | 仅单据详情 | 无审批权限 |

**权限读取时机：** 进入单据详情页时实时查询，非缓存

##### **3.2.2 企业微信H5单据详情**

**入口：** 工作通知 → @提醒 → 点击打开H5

**有权限时：**
```plain
┌─────────────────────────┐
│ 单据详情               │
│                         │
│ [审批操作区]            │
│ [同意] [驳回] [转交]    │
│                         │
│ 单据信息...             │
└─────────────────────────┘
```


**无权限时：**
```plain
┌─────────────────────────┐
│ 单据详情               │
│                         │
│ （无审批操作区）        │
│                         │
│ 单据信息...             │
└─────────────────────────┘
```


##### **3.2.3 我参与的页面迭代**

**页面路径：** 待办 → 我参与的

**列表页变更：**

| **字段** | **展示规则** |
|----------------------------------------------------------|----------------------------------------------------------------|
| 单据标题 | 展示 |
| 单据状态 | 展示 |
| @来源标识 | 展示 |
| 审批快捷入口 | **新增**，有权限时展示\[审批\]按钮 |

**详情页变更：**
- 进入详情时实时读取权限
- 有权限：展示审批操作区（同H5）
- 无权限：仅展示单据详情

---

### **3.3 逻辑互斥与边界处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 权限实时变更 | 进入页面时有权限，操作前被转交 | 提交时二次校验，提示"当前节点已处理" |
| 并发操作 | 多人同时点击审批 | 提交时按现有并发控制，后提交者提示"已被处理" |
| 网络异常 | 权限查询接口失败 | 降级为仅展示详情，提示"权限获取失败，请重试" |
| 节点已跳过 | 查看时审批流已流转 | 按无权限处理，仅展示详情 |
| 单据已作废 | 查看时单据已作废 | 展示作废状态，无审批操作 |

- 【我参与的-列表】增加审批快捷入口，点击进入详情
- 【我参与的-详情】进入页面时调用权限查询，根据返回渲染审批操作区
- 【审批组件】复用现有审批操作，适配H5及APP样式


### **5\. 验收标准**

| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 权限查询 | 进入单据详情时实时读取权限，非缓存 |
| 2 | 有权限展示 | 当前节点处理人看到审批操作按钮（同意/驳回/转交） |
| 3 | 无权限展示 | 非当前节点仅看到单据详情，无操作按钮 |
| 4 | 企业微信H5 | 工作通知点开的H5页面按权限动态渲染 |
| 5 | 我参与的列表 | 有权限的单据展示\[审批\]快捷入口 |
| 6 | 我参与的详情 | 进入详情页按权限动态渲染 |
| 7 | 权限变更 | 节点流转后刷新页面，隐藏审批操作 |
| 8 | 并发处理 | 重复提交给出明确提示 |
| 9 | 异常降级 | 权限查询失败时仅展示详情，不报错退出 |

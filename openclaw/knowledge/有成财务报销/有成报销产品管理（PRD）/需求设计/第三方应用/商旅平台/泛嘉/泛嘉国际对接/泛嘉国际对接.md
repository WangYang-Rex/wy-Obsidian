当员工在泛嘉侧触发超标时，泛嘉推送超标审批单给有成，有成侧自动生成一张草稿状态泛嘉超标申请单；

同时推送一条消息提醒，仅提醒一次：

:::info标题：您有一笔单据待提交

副标题：您的单据\{超标申请单名称\}

标题：单据标题

单号：

金额：

完成时间：

查看更多详情：

:::

生成的审批单中同步过来的数据不允许修改。

有成侧将审批结果推送给泛嘉，审批通过则与员工可预订超标的行程；审批不通过则员工无法预定超标的行程。

| **接口名称** | **请求参数** | **说明** |
|----------------|----------------|----------|
| 审批结果通知[<u>https://openplatform.fanjiaxing.com/api/services/app/approval/notify</u>](https://openplatform.fanjiaxing.com/api/services/app/approval/notify) |  |  |
|  | clientId 必填对接时分配的Client ID |  |
|  | productType   必填产品类型（flight：机票；酒店：hotel；火车票：train） |  |
|  | notifyType 必填通知类型（book：预订；change：改签：refund：退票） | 类型默认为 book--预定 |
|  | orderId 必填订单 ID |  |
|  | approvalResult 必填审批结果（0：驳回；1：通过） |  |
|  | timestamp 必填 |  |
|  | sign 必填 |  |

注：

1、 如果第一次超标审批单在有成侧审批被驳回，此时会将驳回结果传给商旅，商旅侧便不可超标购买，此时如果再有成侧重新提交已被驳回的审批单，审批通过，结果不会再回传给商旅，因为商旅不会再修改超标审批的状态，需要员工重新发起行程预定申请

2、签名算法：参考副本审批及审批结果通知说明.docx (附件: attachments/副本审批及审批结果通知说明\_20250701\_142438.docx)

#### 系统设置--角色权限--职级

泛嘉侧配置差旅标准是根据职级来，职级只是一个文本字段，需要提前在泛嘉侧维护。在报销侧角色权限手动维护【商旅职级】，同步人事信息时将商旅职级下的角色同步为人员的职级信息

【该角色的名称必须要泛嘉管理后台维护一致】

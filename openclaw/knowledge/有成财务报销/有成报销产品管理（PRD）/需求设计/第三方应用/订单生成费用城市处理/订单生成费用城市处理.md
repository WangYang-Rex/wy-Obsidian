## 4、需求详情

### 4.1、订单城市和费用城市匹配

1、订单生成费用时，订单的城市填充到费用的城市将需要转化为现有的格式（国家,省,市,区）

地区编码精确匹配：

1）城市表中“行政编码”字段，与订单中“城市行政编码”来匹配

2）接口字段如下：

| **平台** | **订单类型** | **字段名称** |
|----------|----------------|----------------|
| 阿里商旅 | 飞机 | 无，采用名称匹配 |
|  | 火车 |  |
|  | 酒店 |  |
|  | 用车 |  |
| 携程商旅 | 飞机 | DepartureDistrictCode出发城市行政编码ArrivalDistrictCode  到达城市行政编码 |
|  | 火车 | DepartureDistrictCode出发城市行政编码ArrivalDistrictCode  到达城市行政编码 |
|  | 酒店 | DistrictCode城市行政编码 |
|  | 用车 | DepartureDistrictCode出发城市行政编码ArrivalDistrictCode  到达城市行政编码 |
| 泛嘉国际 | 飞机 | 无，采用名称匹配 |
|  | 火车 |  |
|  | 酒店 |  |
|  | 用车 |  |
| 高德打车 | 用车 | 无，采用名称匹配 |

2、名称匹配逻辑：

订单上的城市和我们库中的市级别名称匹配，匹配上后取对应的地区名称（国家,省,市,）

如果市级别名称匹配不到则和区名称匹配，匹配上后取对应的地区名称（国家,省,市,区）

如果市、区均匹配不到，则取订单城市名称填充

3、涉及到的费用字段有：消费城市、出发城市、目的城市

### 4.2、城市名称处理

背景：目前城市列表中，最近使用和热门城市会显示 xxx市区，用户会误认为是下面的某个区，实际选中后，字段显示还是xxx市，没有区字

需求：MB端和PC端，热门城市和最近使用中，将xxx市区，改为 xxx市 ；取消区字

涉及字段：

单据：费用城市、出发城市、目的城市、城市

行程：出发城市、目的城市

费用：费用城市、出发城市、目的城市

# 新建单据时消费明细支持带入

| **修订时间** | **版本** | **修订说明** | **作者** |
|----------------|----------|----------------|----------|
| 20221019 | v1.0 | 新建 | 昭觉 |

## 1、涉及系统

有成报销 & 有成财务 PC & MB

## 2、需求背景

1、付款单--消费明细--由其他单据引入：限制了应付合同的展示

2、关联单据后，编辑了消费，不能把选择的消费自动带入到消费明细

## 3、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 付款单--消费明细--由其他单据引入 | 放开对应付合同的限制 |  |
| 新建单据--关联单据--关联消费后 | 消费明细自动带入 |  |
|  |  |  |

## 4、需求详情

### 4.1、需求说明

1、付款单--单据中消费明细组件--其他单据引入：放开对应付合同的限制，支持选择应付合同

2、新建/编辑单据时，关联单据后，编辑消费，将选择的消费自动带入到消费明细组件

2.1、消费明细的金额为编辑消费选择的金额，可以编辑，消费明细的金额和选择的消费金额不做校验

2.2、删除关联单据后，消费明细的金额不删除

2.3、关联单据删除后再次关联，关联消费后，消费明细组件会新增一条消费数据，无需合并

2.4、关联单据--编辑消费后，再次点击【编辑】，确定后不重复带入数据

3、关联单据编辑了消费后，带入到消费明细，消费明细金额可编辑，总额需要等于关联的金额，既当期支付计划的金额

3.1、此时编辑消费明细带入的数据，不影响合同中消费明细的数据

# **1\. 需求背景**

目前线上我们支持发票在一次拆分动作里全部拆分完，但对于发票部分分批使用的场景，在拆分时并不清楚剩余发票归属哪个费用项目，需要在后续报销时才确认。例如发票中包含质保金，需要分批打款。目前我们线上的功能是不支持的，发票只能一次拆分完，剩余金额如果没有拆分也就不能再使用。

客户：云谷、物贸大本营

## **基础设置-全局设置**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/e887fba7-f511-42c6-bc0f-2a5156316e59.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=FWnZ1p9YqvpG0qzwqkURITyx07o%3D "")

全局设置-其他设置，增加“发票部分使用”开关，开启后功能无法关闭。二次弹框提醒用户：
> ⚠️发票部分使用功能功能启用后将无法关闭，是否确认？   
>    
> 该功能开启后：   
> 1、每张发票会记录“可使用金额”，默认为发票金额。随着关联费用被报销，将扣减发票的可使用金额。   
> 2、发票生成消费每次只能对应生成一类费用，如果需要一次生成多类费用，需要操作多次。   
> 3、关联了发票的消费不支持拆分，如果需要分开报销，需要手工调整。   

用户确定开启后，发票支持部分使用。发票增加“可使用金额”字段并默认等于发票金额。

## **我的-发票**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/9a28d2c0-d392-4c95-a17d-06f3e7523ac1.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=UxPL9LjKHVcsbqArSDlMsUnD%2B6c%3D "")

当开启了“发票部分使用”后：
1. 发票列表增加“可使用金额”字段列。顺序在金额列后，支持表头排序、筛选。该字段值默认等于发票金额。注意发票OCR、手工录入、三方同步、有成电票同步、票价同步等新增入口。
2. 用户可以通过发票生成消费、选择消费上传发票扣减可使用金额。当发票的可使用金额为0时，发票状态更新为“已使用”。
3. **当发票上关联的消费删除时，或当消费删除上传的发票时（后面不再描述加回去的场景）：**
    1. 若原发票为“未使用”，更新可使用金额，把被删除的消费上的发票使用金额加回去。功能启用前的发票按照发票金额加回去。
    2. 若原发票已经为“已使用”，则更新发票状态更新为“未使用”，更新可使用金额（之前标记为已使用的部分也加回去）。
4. 下方已选中发票小计增加“可使用金额小计”。
5. 新增“标记为已使用”功能，可批量把发票可使用金额更新为0，发票状态更新为“已使用”。
    3. 但当选中发票包含发票金额=可使用金额的发票时（即发票还没有被任一消费使用时），阻断并提示“未生成过消费的发票可直接删除”
    4. 未选中发票点击时提示“标记为已使用”，提示“请选择发票”
6. 导出列表中增加“可使用金额”字段。
7. 发票转交时，可使用金额不变。

## 发票生成消费，扣减可使用金额

影响入口：
- 我的-发票-生成消费（生成消费弹框）
- 我的-消费-发票生成（选择发票弹框、生成消费弹框）
- 新建单据-消费明细组件-其他方式添加-发票生成（选择发票弹框、生成消费弹框）

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/026ea008-9054-4251-b497-a101cdd38a6a.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=xdlRmDbvTzOESkjgmtvQKqGzTig%3D "")

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/07ba9fae-b10c-4577-98e5-b13dde5b3b7c.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=bqqre0%2BpclY32xzdxA15%2Bg57P0w%3D "")

当开启了“发票部分使用”后，
1. 选择发票弹框，在选择发票时展示发票的“可使用金额”字段。
2. 生成消费弹框，一次操作只能支持多张发票生成到一类费用中。
    1. 当选择了“合并生成消费”时，多笔发票生成为一笔消费。
    2. 当选择了“逐条生成消费”时，多笔发票分别生成多笔消费。
    3. 用户可以单选可见范围内的费用类型。
    4. 下方展示本次操作需要生成消费的发票列表，字段有：

| 字段 | 说明 |
|------|------|
| 发票类型 | 仅展示 |
| 发票号码 | 仅展示 |
| 开票日期 | 仅展示，开票日期yyyy-mm-dd |
| 发票金额 | 仅展示 |
| 可使用金额 | 仅展示 |
| 本次使用金额 | 默认等于可使用金额，用户可编辑。<br>前端控制每行发票的本次使用金额绝对值不能大于可使用金额绝对值<br>可使用金额为正数的，本次使用金额不能小于0。<br>可使用金额为负数的，本次使用金额不能大于0。<br>下方动态统计本次使用金额合计。 |
    5. 点击确定，将这批发票的可使用金额中扣减本次使用金额，按照本次使用金额生成消费金额。
        1. 若扣减后剩余可使用金额为0，则该张发票更新状态为“已使用”。
        2. 若扣减后剩余可使用金额不为0，则该张发票依然为“未使用”。

## 先有消费-选择发票扣减可使用金额

影响入口：
- 我的-消费-添加消费-上传发票/发票选择（选择发票弹框、费用明细弹框）
- 随手记-上传发票/发票选择（选择发票弹框、费用明细弹框）
- 新增或编辑单据-消费明细-添加消费（选择发票弹框、费用明细弹框）

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/60a7ecec-9817-4696-9385-745430a8452a.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=H186K4xrMDQbXgr%2FOg%2FER9Gb3i0%3D "")

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ABmOoWbaJZM3ZOaw/img/65db9868-f5ef-4203-899a-355073360ef2.png?Expires=1775017483&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=EoiINnMacmfT5WsZHDoNFhfmM0g%3D "")

当开启了“发票部分使用”后，
1. 选择发票弹框，在选择发票时展示发票的“可使用金额”字段。
2. 费用详情弹框，每张发票新增“可使用金额”、“本次使用金额”金额字段。
    1. 可使用金额展示在原发票金额的位置，发票金额只能点开发票后查看。
    2. 选择发票带入到费用中时，发票的本次使用金额默认等于发票的可使用金额。用户可以修改本次使用金额。
        1. 前端控制每行发票的本次使用金额绝对值不能大于可使用金额绝对值
        2. 可使用金额为正数的，本次使用金额不能小于0。
        3. 可使用金额为负数的，本次使用金额不能大于0。
    3. 原发票合计金额，用本次使用金额的合计替换。在点击费用详情的确定按钮时，根据基础设置-全局设置-发票设置的金额校验开关，取本次使用金额和费用金额进行对比。若费用金额的绝对值大于本次使用金额，提示“费用的金额超出发票的本次使用金额禁止提交”。
    4. 点击确定，将这批发票的可使用金额中扣减本次使用金额。
        4. 若扣减后剩余可使用金额为0，则该张发票更新状态为“已使用”。
        5. 若扣减后剩余可使用金额不为0，则该张发票依然为“未使用”。
        6. 当一张单据中多次消费选择了同一张发票，发票的可使用金额按照多次消费中使用金额合计扣减。

## **其他功能影响**

### **我的-消费，拆分消费时增加限制**

当开启了“发票部分使用”后，若拆分的消费中包含发票附件，则不允许拆分。阻断并提示“已开启发票部分使用，不支持关联了发票的消费拆分，请手工修改消费金额、发票可使用金额”

### **新增或编辑单据-发票核销，发票核销自动生成消费按照核销金额自动填写可使用金额**

当开启了“发票部分使用”后，发票核销自动生成消费时， 带入发票的可使用金额默认为本次核销金额。

### **发票按照类目自动生成消费、三方过来的自动生成消费的发票**

当开启了“发票部分使用”后，发票生成消费时， 消费上的本次使用金额=发票可使用金额=发票金额。发票自动更新为已使用状态。

### **单据管理-发票管理**

发票管理列表本身已经支持了关联多笔费用和单据，注意回归即可。

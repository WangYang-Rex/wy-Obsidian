# 报销单-核销借款组件优化，增加本次核销金额

## 1、需求背景

在实际报销流程中，可能会涉及一次报销时需要核销多笔借款，对于核销金额在多笔借款中的分配，用户希望能够手工进行编辑调整。

目前线上逻辑是先进先出，即按照选择的借款单/备用金申请提交时间，将一张单子上的剩余可核销金额核完，再使用下一张，不支持用户自己编辑本次核销金额。

### 4.1 报销单单张新增、编辑界面



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75ka0vdZBlAK/img/cb5ca090-d9d8-439d-a300-7677ceb1d340.png?Expires=1775017545&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=DTGcTlL1s%2BH3%2Fw2ANmHKaIe29Vo%3D "")
1. 核销借款组件增加“本次核销金额”列。线上逻辑：有数据权限的用户，即在全局设置-借款核销权限配置的用户-新增&编辑场景、在审批流程中配置了编辑权限的用户-编辑场景，可以查看具体核销核销单据。线上逻辑：核销借款组件中的“可核销借款”计算逻辑=借款单金额-之前已经该单已经核销的金额。线上逻辑：当用户在全局设置-其他设置中禁止手工输入核销金额时，系统自动根据本次报销金额全额分配到所选借款上。按照所选借款单可核销借款逐笔分配，直至本次报销金额被分配完或可核销借款额被分配完。本次新增：**分配的核销金额展示在“本次核销金额”列中。**当用户在全局设置-其他设置中启用手工输入核销金额时，选择借款明细后弹出弹框填写本次需要核销的金额，系统自动将本次核销金额分配给已选借款明细。本次新增：**分配的核销金额展示在“本次核销金额”列中，支持有数据权限的用户再次编辑。编辑后前端动态计算核销金额合计。****前端默认带出金额=系统按照先进先出自动分配的金额。用户可以调整本次核销金额，前端限制范围 0 \<=本次核销金额\<=可核销金额。**
2. **输入框失焦、编辑消费明细，核销借款明细增删改之后需要进行前端校验，校验本次核销金额合计是否超过了报销金额合计，若超过前端提示“本次核销金额（xxx）不能超过报销金额（xxx），请调整。”**
3. **单据保存、提交时后端校验，本次核销金额不得超过报销金额、本次核销金额不得超借款单可核销金额。**
4. 线上逻辑：实际借款核销金额改用本次核销金额扣减。
5. 在预算实际占用/释放时，均改用本次核销金额。

# 单据支持自定义被关联次数

| **修订时间** | **版本** | **编写人** | **修订说明** |
|----------------|----------|-------------|----------------|
| 20230315 | v1.0 | 小乔 | 新建 |

## 1、需求背景

目前申请单被关联次数只支持设置 仅关联一次和不限制，不支持自定义设置关联次数。

客户场景：差旅报销单关联出差申请单，员工不一定一次性提单进行报销，企业会对申请单设定具体的最多被关联次数，员工分多次报销时，根据设定的被关联次数限制报销提单次数

## 3、功能清单

根据个人需要补充

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 自定义单据 | 单据（申请单、收款单、付款单、报销单）被关联次数支持用户自定义 |  |
| 关联单据 | \+ 当单据被关联次数已用完，关联单据选择弹窗不再显示该单据（已有逻辑）\+ 当关联单据删除或在关联单据组件中删除被关联单据数据后，释放关联单据次数 |  |
| 申请单停用 | 当申请单关联次数用完，申请单自动停用 |  |

### 4.2、自定义单据

**自定义单据-被关联次数支持自定义次数（如下图）**
1. 涉及的单据类型包括：申请单、收款单、付款单、报销单
2. 自定义次数：默认 1；取值范围：正整数
3. 样式交互：只有选中“仅可关联X次”时，次数才可编辑
4. 逻辑：不限制——不限制被关联次数；仅可关联X次——单据被关联次数最多X次，被关联次数达到X次后，该单据不再允许关联

### 4.3、关联单据逻辑
5. 关联次数占用：单据A关联单据B，提交成功后，即占用一次单据B的被关联次数
6. 关联次数释放：单据A关联单据B，只有在关联单据组件中取消关联单据B，或删除单据A时。释放一次单据B的被关联次数
7. 单据允许关联次数更新：若在自定义单据中修改了单据允许关联的次数，单据按照最新的次数进行限制。

### 4.4、申请单停用

当申请单允许被关联次数=0时，申请单自动停用，并在单据详情-审批历史中展示停用详情（如下图）

PS：其他几个自动停用文案目前没有，这次一起补上

| **停用原因** | **停用文案** |
|----------------|----------------|
| 申请单可用金额=0 | 单据自动停用  原因：单据可以金额已用完 |
| 已过有效期 | 单据自动停用  原因：单据已过有效期 |
| 被关联次数=0 | 单据自动停用  原因：允许被关联次数已用完 |

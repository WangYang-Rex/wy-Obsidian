# **1\. 需求背景**
1. 目标用户：财务
2. 需求来源：客户
3. 背景：
1. 收款退款单（付款单模版）选到多个应收合同内收款计划的收款单，会导致在应收合同计划详情里面每个收款计划都能显示出来。真实的计划收款退款金额是在收款退款单完成后才计算出来的，也就是退款完成后这里可以显示出真实的退款金额（现在还没有显示），但是在收款退款审批中无法得到真实的退款金额 。
2. 付款退款单（收款单模版）选到多个付款合同内付款计划的付款单。会导致在应付合同计划详情里面每个付款计划都显示出来。真实的计划付款退款金额是在付款退款单完成后才计算出来的，也就是退款完成后这里可以显示出真实的退款金额（现在还没有显示），但是在付款退款审批中无法得到真实的退款金额 。
3. 有成报销v2pc端和移动端都有显示优化需求
4. 核心痛点：
1. 收款退款单关联的多个收款单分别来源于多个应收合同的收款计划，退款流程审批中时，这些应收合同中被关联的收款计划显示“退款状态：退款中”“退款金额：收款退款单内填写的对应的收款计划关联收款单的退款金额”；退款流程完成后，这些应收合同中被关联的收款计划显示“退款状态：已退款”“退款金额：收款退款单内填写的对应收款计划关联收款单的退款金额”
2. 付款退款单关联的多个付款单分别来源于多个应付合同的付款计划，退款流程审批中时，这些应付合同中被关联的付款计划显示“退款状态：退款中”“退款金额：付款退款单内填写的对应的付款计划关联付款单的退款金额”；退款流程完成后，这些应付合同中被关联的付款计划显示“退款状态：已退款”“退款金额：付款退款单内填写的对应付款计划关联付款单的退款金额”

## **4.2 核心功能**

### **4.2.1 退款状态与金额显示**

**收款计划/付款计划列表字段：**


| **字段** | **说明** | **显示逻辑** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|
| 退款状态 | 当前计划的退款状态 | 无退款：不显示；退款中：显示"退款中"；退款完成：显示"已退款" |
| 退款金额 | 当前计划的退款金额 | 退款中/完成时显示具体金额；支持点击退款详情 |

**退款状态流转：**


```plain
无退款 → 退款中 → 退款完成
  ↑        ↓         ↓
  └────────┴──── 驳回/撤回/删除 → 无退款
```


### **4.2.2 多合同多计划场景处理**

**场景说明：**
- 一个收款退款单关联3个收款单，分别来自3个不同的应收合同
- 每个收款单对应各自合同中的一个收款计划

**处理逻辑：**


```plain
收款退款单（单号：SKTK20260303001）
├── 关联收款单A（合同A-计划1）→ 退款金额：10,000
├── 关联收款单B（合同B-计划2）→ 退款金额：20,000
└── 关联收款单C（合同C-计划3）→ 退款金额：15,000

合同A-计划1：退款状态=退款中/已退款，退款金额=10,000
合同B-计划2：退款状态=退款中/已退款，退款金额=20,000
合同C-计划3：退款状态=退款中/已退款，退款金额=15,000
```

| 关联单据类型 | 收款单/付款单 |

| 退款金额 | 该计划关联的收付款单的退款金额 |

### **4.2.5 多笔退款场景**

**场景：** 同一合同计划通过收款/付款单被多个退款单关联

**展示逻辑：**
- 退款状态：合同详情收款/付款计划明细内显示各自的退款状态
- 退款金额：合同详情收款/付款计划明细内显示各自的退款金额
- 鼠标移动至退款状态：弹出退款单号，点击打开退款详情


## **4.3 默认策略**


| **场景** | **默认行为** |
|----------------------------------------------------------|----------------------------------------------------------------|
| 存量合同计划 | 无退款时，不显示退款状态和金额字段 |
| 新增退款关联 | 默认显示"退款中"状态和，退款金额 |
| 退款完成 | 状态更新为"退款完成"，金额保持不变 |
| 退款驳回/删除 | 清空退款状态和金额 |
| 退款单修改 | 更新关联的退款金额 |

## **4.4 存储方案（建议）以现有技术设计方案为准**

**1\. 退款单与合同计划关联表（新增）**
- 记录退款单与合同计划的关联关系
- 字段：退款单ID、关联单据ID、合同ID、计划ID、退款金额、状态等

**2\. 合同计划表（扩展）**
- 增加字段：退款状态、退款金额（累计）
- 或保持现有结构，通过关联表实时计算

**3\. 退款单表（扩展）**
- 记录各关联单据的退款金额明细（已存在退款核销组件数据中）

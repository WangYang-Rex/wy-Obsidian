# **1\. 需求背景**

现状：

1、财务日记账会存在附件的情况只有两种：1️⃣手动添加的日记账可以自主上传附件 2️⃣通过有成报销同步过来的日记账，会根据报销的附件传过来展示

2、用户期望可以针对于对应日记账下的附件，分消费来统计附件数量，更清晰

# **2\. 业务流程/架构**

# **3\. 功能清单**

| 模块 | 功能点 | 备注 |
|------|---------|------|
| 流水账 | 日记账板块需要支持控制附件明细查看的开启或关闭 |  |
|  |  |  |

需要考虑一笔单据生成多流水，那么全量附件展示

包括流水和费用的关系是否有记录。

# **4\. 需求详情**

### **4.1日记账**

流水账--日记账板块，在更多功能中新增“开启附件明细”功能
1. 上线后初始化展示“开启附件明细”按钮，按钮常亮
2. 点击视为进行附件明细展示，按钮变更为“关闭附件明细”同样常亮
3. 开启附件明细后，页面样式如下：
    1. 附件列表还是默认展示汇总附件数量
    2. 点击附件列表进入后，页面按照如下表格来展示，拆分不同费用类型下的附件数量和附件
    3. 如果存在相同的费用类型名称也无所谓，正常展示即可
    4. 若不存在任何费用类型，则就算开启了“附件明细”也不做如下表格样式展示，保持原交互
    5. 若一部分存在费用类型，一部分不存在，则表格专门有一行的费用类型列名为：无类型
    6. 页面存在“更新同步”按钮。按钮常亮，点击即可重新同步一遍报销系统中相应的单据下的附件类型和数量。重新生成表格的内容【页面loading处理】

4. 点击“关闭附件明细”，视为关闭附件明细展示，按钮重新变为“开启附件明细”。点击附件进入的页面同原样式一致



注：若是手动上传的附件，由于不存在费用类型，则无需调整，按照现有的展示。导出、打印等无需根据该按钮控制

注：费用类型就是纯文本，会根据推送生成日记账的时候传的费用类型内容来存储，无需随着报销中的费用类型调整而更新。仅会根据更新同步来处理

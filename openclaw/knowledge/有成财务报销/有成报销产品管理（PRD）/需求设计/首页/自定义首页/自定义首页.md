## 1、需求背景

**（1）客户需求**

省财开提出三个关于自定义首页的需求：

a、管理员可以统一设置首页

b、首页能直接查看我发起的、待我审批，且待审批数量有角标提示。

c、普通员工不用打印报销单，首页待打印数字会一直增加。

目前，报销系统首页只支持用户自定义模块的可见性，不支持管理员预设；现在首页已有待我审批元素，但客户认为其交互ui不符合需求，需要能直接点击审批。待打印的统计逻辑可优化为只要有人打印过就视为已打印。

客户预期国庆节前后能上线。

**（2）竞品现状**

调研了汇联易、易快报、每刻纷享销客的首页自定义功能，大致情况如下：

| **产品** | **功能简介** | **优势** | **劣势** |
|----------|----------------|----------|----------|
| 汇联易 | 系统预设管理员首页且不可编辑，用户可自定义个人首页，有管理员权限的用户可以切换管理员首页和个人首页。pc与app各自独立设置 | 方案简单，投入较少，且操作简单易懂 | 覆盖的场景不多，比如无法支持管理员统一预设，不支持一个用户预设多个首页模版 |
| 易快报 | 只支持设置“随手记”和“发票验真”的可见性 | - |  |
| 纷享销客 | 支持设置多个首页模版，每个模版可以共享给其他用户、部门、角色、用户组，模版支持7种页面布局的选择，业务元素通过自定义表单的形式进行添加 | 方案完整且全面，能支持各类首页自定义的场景 | 若应用于报销业务，可能工作量比较大，ROI低 |
| 每刻 | 不支持 |  |  |

经过对比，目前在与汇联易PK时会处于劣势。

## 4、需求详情

功能简述：每个用户可以自定义多个PC和APP的首页模版，系统管理员可以把模版应用给用户、角色、用户组、部门，用户可在pc首页/app个人中心切换有权限的模版。一个首页模版内支持用户通过拖拽的方式自行添加需要的功能模块及其摆放位置。

##### 4.1.2.1 PC端模版列表页/模版信息编辑页

列表页有新建、修改、编辑、复制四种操作，按钮展示逻辑如下：

（1）列表展示应用给用户的所有模版，初始化之后每个用户都有一个默认模版，用户的默认模版之间互不干扰，用户可以对其进行修改、编辑。

非管理员创建模版能选到的应用范围和编辑权限只有自己（创建的时候系统已经自动带入了，设置模版时置灰不可编辑）

系统管理员创建模版可以选择应用给全员，可以选择给其他人编辑权限（系统管理员创建模版的时候，应用范围和编辑权限系统会自动带入该系统管理员，但他可以选择加入其他人）

（2）每个用户均有新增、复制按钮

（3）当用户是某模版的修改人时（前提是该用户也是该模版的应用者，因为如果不是应用者，该模版不展示在该用户模版列表），该模版才展示编辑、修改按钮

点击新建、修改、复制这三个按钮，打开模版信息弹窗页，交互说明如图：

## **1\. 需求背景**

### **1.1 需求描述**

在预算管理基础设置中，支持将"余额按比例手工结转"升级为"余额按比例自动结转"，系统在预算周期到达时自动执行结转，无需人工干预。

### **1.2 需求背景**
- **需求方**：财务管理员、预算管理员（锅圈投资管理上海有限公司）
- **现状问题**：
• 已配置按比例结转规则，但仍需按周期手工触发
• 财务期末工作量大，容易遗忘结转操作
• 忘记结转导致下一周期预算额度不准确
- **解决思路**：新增"余额比例自动结转"选项，系统按预算周期自动执行结转，同时保留手工结转作为兜底

**业务流程**
```plain
预算仅可编辑最末级维度金额已开启，预算结转规则配置
    ↓
选择结转方式（手工/自动）
    ↓
设置结转比例、选择预算维度
    ↓
系统自动按预算周期执行结转
    ↓
生成操作日志和变更记录
    ↓
发送通知给预算管理员
```

## **2\. 功能现状**

**功能位置**：预算费控-预算管理-基础设置-预算结转设置

**现有字段**：
• 结转方式选择（单选）
• 余额全额结转：手工结转上周期剩余全部余额
• 余额比例手工结转：按比例配置结转选择周期的剩余额度
• 结转比例设置（0-100%）
• 预算维度选择

**现有操作**：
• 需先开启预算仅可编辑最末级维度金额

• 用户需手动选择预算期间执行结转 

• 每次结转需人工触发

**缺失能力**：
• 不支持系统自动按周期执行结转
• 无自动结转执行日志记录
• 无自动结转变更记录区分标识

**结转逻辑**

余额自动全部结转：余额全部（全周期），现有逻辑

**配置入口** 【预算费控-预算管理-基础设置-预算结转-结转规则设置】

#### **3.2.2 自动结转执行逻辑**

**功能概述** 系统按预算周期自动执行结转，支持年度/季度/半年度/月度四种周期类型。

**执行时机**

| **预算周期** | **自动结转执行时间** |
|----------------------------------------------------------------|----------------------------------------------------------------------------|
| 年度 | 财务开始年度月份的次年月份的首日凌晨 |
| 半年度 | 财务开始年度月份起前6个月月末后次日凌晨和后6个月月末次日凌晨 |
| 季度 | 财务开始年度月份起每季度末次日凌晨 |
| 月度 | 财务开始年度月份起月末次日凌晨 |

**执行流程**
```text
系统定时扫描
    ↓
读取"余额比例自动结转"规则
    ↓
遍历规则下的预算维度
    ↓
对每个维度执行：
  ├─ 读取预算周期
  ├─ 判断是否为期末次日
  ├─ 检查当前期间余额 > 0
  ├─ 检查目标期间已创建
  ├─ 检查是否存在手工结转中 ← 冲突检查
  │     ├─ 存在 → 跳过，记录冲突，发送失败通知
  │     └─ 不存在 → 继续
  ├─ 计算结转金额 = 剩余预算 × 结转比例
  ├─ 执行结转
  ├─ 更新预算额度
  └─ 生成操作日志和变更记录
    ↓
汇总执行结果
    ↓
发送通知给预算管理员
```



**结转金额计算**
```plain
结转金额 = 当前期间剩余预算 × 结转比例
剩余预算 = 预算额度 - 已占用 - 已使用
```



**多周期混合处理**
```text
规则A（余额比例自动结转，比例50%）
├── 维度X（年度预算）→ 年度末自动执行
├── 维度Y（季度预算）→ 每季度末自动执行
└── 维度Z（月度预算）→ 每月末自动执行

各维度按自身周期独立触发
分开记录执行结果
```



**边界处理**



| **场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 非执行日 | 当前日期不是周期次日 | 跳过，不记录 |
| 余额为零 | 剩余预算 = 0 | 跳过，记录"无需结转" |
| 冲突状态 | 存在手工结转进行中 | 跳过，记录冲突，发送通知 |
| 目标期间不存在 | 下一期间预算未创建 | 跳过，记录并发送通知 |
| 执行失败 | 系统异常 | 记录失败日志，10分钟后重试，最多3次 |

## **5\. 验收标准**

| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 规则配置 | 可正常新增/编辑"余额比例自动结转"规则，保存后即时生效 |
| 2 | 周期匹配 | 年度/季度/半年度/月度预算按各自周期自动执行，执行时间为周期次日凌晨 |
| 3 | 自动执行 | 到达周期末后次日自动执行结转，生成结转变更详情和操作日志 |
| 4 | 比例计算 | 结转金额 = 当前余额 × 配置比例，计算准确，精度到分 |
| 5 | 冲突处理 | 手工结转进行中时，自动结转跳过并记录，发送失败通知给预算管理员 |
| 6 | 通知发送 | 结转完成后正确发送成功/失败通知给预算管理员（固定接收人） |
| 7 | 存量兼容 | 存量"余额比例手工结转"规则不受影响，可正常编辑使用 |
| 8 | 多周期混合 | 同一规则下不同周期维度按各自周期独立执行，分开记录结果 |
| 10 | 日志记录 | 操作日志和变更记录正确区分手工/自动结转类型 |

## **附录**

**A. 典型使用场景**

**场景：锅圈公司季度预算自动结转**

配置步骤（首次）：
1. 财务管理员进入【预算费控-预算管理-基础设置-预算结转设置】
2. 选择预算方案"2026年度预算"
3. 结转方式选择"余额比例自动结转"
4. 设置结转比例为"50%"
5. 选择适用维度：销售部、市场部、运营部
6. 保存配置

使用步骤（日常）：
1. 系统自动在2026年3月31日、6月30日、9月30日、12月31日次日02:00执行结转
2. 系统自动计算各部门剩余预算的50%结转到下一季度
3. 财务管理员收到结转成功通知，查看执行明细
4. 如有失败项，根据通知提示处理（如创建缺失的预算期间）

结果：财务无需每季度末手动执行结转，系统自动完成，减少遗漏风险。

# **1\. 需求背景**

用户的预算结构是部门-费用，实际按照部门强控制预算，指定一些费用维度用来登记预算明细，费用维度仅做弱管控。

此时线上效果是，如果费用上调了预算额度，那么部门预算自动上调对应额度；如果费用下调预算额度，部门预算不会自动调增或调减。用户需要手工去计算调整部门预算，比较麻烦。

调整了月度，年度也会按照这个逻辑调整。

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/v9kqDejME8LE1OVx/img/9b399fd0-f82c-40ef-8ec2-f7cebc952786.png?Expires=1774156674&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=woosCsDq0bMVswWoHA8jMMzohkg%3D "")
1. 目标用户：财务
2. 需求来源：智能财务
3. 背景：
1. 预算管理员配置预算纬度的金额时，每一层级的预算额度都需要手工填写，且父级纬度的预算总金额需要人工先线下统计出子级纬度总预算金额合计，再将合计金额填入该父级纬度的预算总金额，点击提交保存。交互很不方便。
2. 所有操作在【纬度设置】页面完成
3. 示例：按照部门结构维护预算，公司\>事业群\>部门，公司纬度预算组成：公司a总预算金额=公司a自身预算金额\+公司a下级预算汇总金额；事业群纬度预算组成：事业群n总预算金额=事业群n自身预算金额\+事业群n下级预算汇总金额；部门纬度预算组成：部门n总预算金额=部门n自身预算金额\+部门n下级预算汇总金额；公司a下级预算汇总金额=企业a下级所有事业群总预算金额的合计，事业群n下级预算汇总金额=事业群n下级所有部门总预算金额的合计；每次填写完事业群n下的所有部门各自的总预算金额后，都需要用户自己这些部门的总预算金额相加，然后填写至事业群n的总预算金额；
4. 如果事业群n想有自身纬度的预算金额，在填写事业群n的总预算金额时还需要考虑将获得的事业群n下的所有部门各自的总预算金额合计数加上想要的事业群n想有自身纬度的预算金额数；因为当前逻辑：事业群n的总预算金额=事业群n自身预算金额\+事业群n下级预算汇总金额，其中页面内仅显示了事业群n的总预算金额数；
4. 核心痛点：
1. 预算管理员配置预算纬度的金额时，下级纬度的总预算金额自动汇总至父级纬度的下级预算汇总金额，无需预算管理员手工填写；
2. 填写子级预算时，没有父级纬度金额做参照；

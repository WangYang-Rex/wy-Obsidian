# **1\. 需求背景**

## **1.1 目标用户**

财务管理员、预算管理员、普通员工（费用申请人）

## **1.2 需求来源**

有成报销产品内部优化需求

## **1.3 背景**

在预算管理场景中，预算申请单用于调整预算额度（如追加、调减、跨维度调整）。当前系统存在以下问题：
- 预算申请单提交后进入审批流程，审批期间被变更的预算额度**仍处于可用状态**
- 其他用户可在审批期间继续提交占用该预算的单据
- 若审批通过，可能导致预算实际使用超出调整后的额度

## **1.4 核心痛点**

**典型场景示例：**
```plain
场景：某部门A费用剩余金额1000元

时间点1：预算管理员提交预算申请单
         - 将A费用剩余1000元调整到B费用
         - 单据状态：审批中

时间点2（审批未完成）：员工提交A费用的报销申请
         - 报销金额800元，审批通过
         - 此时A费用实际剩余：1000 - 800 = 200元

时间点3：预算申请单审批完成
         - A费用1000元全部调出到B费用
         - A费用实际剩余：0元（但已占用800元）

结果：A费用超额使用800元，预算管控失效
```



**核心问题：**
- 审批期间预算变更金额未冻结，存在超额使用风险
- 预算管控的时效性存在漏洞
- 财务无法准确把控预算实际可用额度

# **4\. 需求详情**

## **4.1 功能入口**

## **4.2 核心功能**

### **4.2.1 冻结金额计算逻辑**

**冻结触发时机：**
- 预算申请单提交时（状态变为"审批中"）
- 审批中撤回后重新提交时

**冻结金额计算规则：**


```plain
对于每个被变更的预算维度和预算期间：

1. 调减场景（减少预算）：
   冻结金额 = 调减金额
   
   示例：
   - A费用原额度：5000元
   - 申请调减：1000元
   - 冻结金额：1000元（防止审批期间这1000元被占用）

2. 调增场景（增加预算）：
   冻结金额 = 0（无需冻结，因为额度是增加的）
   
   示例：
   - B费用原额度：3000元
   - 申请调增：1000元
   - 冻结金额：0元

3. 转移场景（跨维度调整）：
   调出方冻结金额 = 调出金额
   调入方冻结金额 = 0
   
   示例：
   - A费用调出1000元到B费用
   - A费用冻结：1000元
   - B费用冻结：0元
```



**冻结记录数据结构：**

|-------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| id | bigint | 主键 |

### **4.2.2 冻结与预算占用的关系**

**冻结金额在预算报表中的体现：**


```plain
预算报表字段计算逻辑（调整后）：

预算总额 = 原预算总额（不变）
预算已使用金额 = 单据审批完成占用的金额（不变）
预算预占用金额 = 单据审批中占用的金额 + 预算申请冻结金额（新增）
预算剩余金额 = 预算总额 - 预算已使用金额 - 预算预占用金额

其中：
- 单据审批中占用的金额：普通单据（报销单、借款单等）审批中占用的预算
- 预算申请冻结金额：预算申请单审批中冻结的金额（本需求新增）
```



**预算报表展示调整：**

**表格**





| **字段** | **调整前** | **调整后** |
|----------------------------------------------------------|-------------------------------------------------------------|-------------------------------------------------------------|
| 预算预占用金额 | 仅含单据审批中占用 | 单据审批中占用 \+ 预算申请冻结金额 |
| 预算剩余金额 | 总额 - 已使用 - 单据预占用 | 总额 - 已使用 - 单据预占用 - 冻结金额 |

### **4.2.3 单据提交时的预算校验**

**校验时机：**
- 用户提交报销单、借款单等占用预算的单据时

**校验逻辑（调整后）：**

**plain**

复制
```plain
可用额度 = 预算总额 - 预算已使用金额 - 预算预占用金额（含冻结）

if 申请占用金额 > 可用额度:
    校验不通过，提示"预算不足"
else:
    校验通过，正常提交
```

### **4.2.4 冻结解除逻辑**

**解除触发时机：**

**表格**





| **场景** | **触发条件** | **解除后操作** |
|----------------------------------------------------------|----------------------------------------------------------------|-------------------------------------------------------------------|
| 审批通过 | 预算申请单最终审批节点通过 | 解除冻结，执行实际预算变更 |
| 审批驳回 | 任意审批节点驳回 | 解除冻结，预算额度不变 |
| 单据撤回 | 申请人主动撤回 | 解除冻结，预算额度不变 |
| 审批转交 | 转交给其他人审批 | 保持冻结状态 |
| 审批加签 | 增加审批人 | 保持冻结状态 |

**审批通过时的处理流程：**

**plain**

复制
```plain
1. 审批节点通过
   ↓
2. 【冻结服务】解除该申请单的所有冻结记录
   - 将 freeze_status 更新为 2（已解除）
   - 记录 release_time 和 release_reason
   ↓
3. 【预算调整服务】执行实际预算变更
   - 调减：减少预算总额
   - 调增：增加预算总额
   - 转移：调出方减少，调入方增加
   ↓
4. 更新预算报表数据
   ↓
5. 预算申请单状态变为"已完成"
```



**审批驳回/撤回时的处理流程：**

**plain**

复制
```plain
1. 审批驳回或单据撤回
   ↓
2. 【冻结服务】解除该申请单的所有冻结记录
   - 将 freeze_status 更新为 2（已解除）
   - 记录 release_time 和 release_reason
   ↓
3. 预算额度保持不变（变更未生效）
   ↓
4. 预算申请单状态变为"已驳回"或"已撤回"
```



### **4.2.5 多维度变更场景处理**

**场景说明：** 一笔预算申请单可能同时变更多个预算维度（如同时调整多个部门的预算）

**处理逻辑：**

**plain**

复制
```plain
预算申请单包含多个维度变更时：

1. 提交时：
   - 为每个被调减/调出的维度创建独立的冻结记录
   - 各维度冻结金额独立计算

2. 审批过程中：
   - 各维度冻结状态独立维护
   - 任一维度被占用均受冻结金额限制

3. 审批完成时：
   - 全部通过：解除所有冻结，执行所有变更
   - 部分通过（如有此功能）：仅解除通过的维度的冻结并执行变更
   - 全部驳回：解除所有冻结，不变更
```

**存量数据迁移脚本：**

**sql**

复制
```sql
-- 为存量审批中的预算申请单补录冻结记录
INSERT INTO budget_freeze_record (
    corp_id, bill_number, budget_id, budget_rule_code, 
    budget_period_name, freeze_amount, freeze_type, freeze_status, create_time
)
SELECT 
    corp_id, bill_number, budget_id, budget_rule_code,
    budget_period_name, 
    CASE 
        WHEN adjust_amount < 0 THEN ABS(adjust_amount)  -- 调减金额
        ELSE 0
    END as freeze_amount,
    CASE 
        WHEN adjust_amount < 0 THEN 1  -- 调减冻结
        ELSE 2  -- 其他
    END as freeze_type,
    1 as freeze_status,  -- 冻结中
    submit_time as create_time
FROM corp_budget_apply_bill_analysis
WHERE bill_status = '2'  -- 审批中
AND adjust_amount < 0;   -- 调减或调出金额
```

| 预算申请单冻结 | 预算申请单审批流 | 强依赖，审批状态变更触发冻结/解冻 |
| 预算申请单冻结 | 预算报表 | 强依赖，报表展示冻结金额 |
| 预算申请单冻结 | 预算数据权限 | 依赖，仅控制有权限的维度 |

## **5.3 功能影响**

**表格**





| **功能A** | **功能B** | **影响说明** |
|-----------------------------------------------------------|-----------------------------------------------------------|----------------------------------------------------------------|
| 预算申请单冻结 | 单据提交预算校验 | 可用额度计算扣除冻结金额，可能拦截更多单据 |
| 预算申请单冻结 | 预算报表 | 预占用金额增加，剩余金额减少 |
| 预算申请单冻结 | 预算执行分析 | 分析时需考虑冻结金额对执行率的影响 |
| 预算申请单冻结 | 动态预算 | 动态预算维度不参与冻结（由规则自动计算） |

# **6\. 潜在风险与应对**

**表格**





| **风险点** | **影响** | **应对措施** |
|-------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|
| 冻结金额计算错误 | 预算数据不准确 | 增加计算过程日志，支持核对；冻结金额=调减金额，逻辑简单清晰 |
| 审批流程异常中断 | 冻结未解除，预算长期不可用 | 定时任务扫描异常单据，自动解除超期冻结（如审批中超过30天） |
| 多单据同时冻结同一维度 | 冻结金额叠加，导致可用额度为负 | 后端校验，冻结金额总和不能超过预算总额 |
| 冻结与手工占用冲突 | 用户感知可用额度与系统不一致 | 单据提交时明确提示冻结金额来源 |
| 存量数据迁移异常 | 升级后冻结数据不准确 | 迁移前备份，迁移后校验，支持手动修复 |
| 审批通过时解除与执行变更的原子性 | 解除成功但变更失败，数据不一致 | 使用事务保证原子性，失败时回滚 |

# **9\. 验收标准**

## **9.1 功能验收**

**表格**





| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 冻结创建 | 预算申请单提交时，被调减/调出金额正确创建冻结记录 |
| 2 | 冻结计入预占用 | 预算报表中预占用金额包含冻结金额 |
| 3 | 可用额度计算 | 可用额度 = 总额 - 已使用 - 预占用（含冻结） |
| 4 | 单据校验拦截 | 申请金额超过可用额度（扣除冻结后）时正确拦截 |
| 5 | 审批通过解冻 | 审批通过后冻结解除，预算变更生效 |
| 6 | 审批驳回解冻 | 审批驳回后冻结解除，预算不变 |
| 7 | 单据撤回解冻 | 单据撤回后冻结解除，预算不变 |
| 8 | 多维度冻结 | 一笔申请单变更多个维度时，各维度独立冻结 |
| 9 | 冻结明细展示 | 预算维度详情页正确展示冻结明细 |
| 10 | 存量兼容 | 存量审批中单据升级后正确补录冻结记录 |

## **9.2 性能验收**

**表格**





| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 冻结创建性能 | 提交预算申请单时，冻结记录创建 \< 500ms |
| 2 | 预算校验性能 | 单据提交时预算校验（含冻结）\< 300ms |
| 3 | 报表查询性能 | 预算报表查询（含冻结金额）\< 2秒 |
| 4 | 并发性能 | 50个预算申请单同时提交，冻结创建正常 |

## **9.3 异常场景验收**

**表格**





| **序号** | **验收项** | **验收标准** |
|----------------------------------------------------------|-------------------------------------------------------------|----------------------------------------------------------------|
| 1 | 审批流中断 | 审批流异常中断时，冻结状态可查询，支持手动解除 |
| 2 | 重复冻结 | 同一单据重复提交时，不产生重复冻结记录 |
| 3 | 超期冻结 | 审批中超30天自动解除冻结，发送通知 |
| 4 | 数据一致性 | 冻结金额总和不超过预算总额，避免负数可用额度 |

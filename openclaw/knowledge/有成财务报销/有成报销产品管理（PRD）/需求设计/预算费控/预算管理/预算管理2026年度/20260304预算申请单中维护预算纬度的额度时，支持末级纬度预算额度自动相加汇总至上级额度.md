# **1\. 需求背景**
1. 目标用户：财务
2. 需求来源：精品传媒集团客户
3. 背景：
1. 部门主管填写预算申请单，在申请单中维护自己管理的预算纬度的额度时，每一层级的预算纬度都必须手工填写，在提交单据的时候才会校验该层级额度是否超额；申请人需要自己将末级预算纬度的额度相加后，手工填写至父级纬度后提单，然后系统校验维护的预算额度是否符合规则；
2. 员工在使用预算申请单维护预算纬度的额度时，只能对被授予权限的纬度查看和编辑
4. 核心痛点：
1. 预算申请单中编写末级预算纬度的额度，末级预算纬度的额度自动汇总至父级纬度，父级纬度额度组成（该层级纬度自身额度\+下级汇总预算纬度的额度），且需要每次填写额度时需要校验是否触发超额规则，触发需要提醒；父级预算纬度自身额度也可编辑，也需要参与超额校验；
2. 预算申请人只能查看维护被授予权限的预算纬度



# **1\. 需求背景**

**1\. 目标用户**：预算辅助管理员（所有被授予预算数据权限“被授予预算纬度\+开启查看编辑权限”的用户）

**2\. 需求来源**：精品传媒集团客户

**3\. 背景**：
- 预算辅助管理员通过预算申请单维护被授权预算维度的额度
- 预算维度结构：例：总部 \> 事业群 \> 部门 \> 费用科目（3-4级树形结构）
- 当前预算编制流程：预算申请人需手工计算末级维度之和，填入父级维度，提交后才校验超额
- 现有权限控制粒度粗，申请人可见全部维度的预算额度，无法聚焦负责范围
- 不同预算管理员负责不同维度范围，需要精细化权限控制
- 当前痛点导致每月预算编制耗时2-3天，错误率较高

**4\. 核心痛点**：
- 手工汇总繁琐：父级额度需手工计算子级之和后填写，重复劳动、易出错、效率低
- 校验时机滞后：提交时才校验超额，填写过程无感知，反复修改、体验差
- 权限控制粗放：可见所有维度，无法聚焦负责范围，干扰操作、易误改他人数据

```plain
预算管理员进入预算申请单
        ↓
 系统加载有权限的预算维度树（并集）
        ↓
 选择有权限维度 → 填写/查看额度
        ↓
 实时自动汇总至父级
        ↓
 实时校验周期/年度超额规则
        ↓
 校验通过？ ──否──→ 提示警告/禁止
        ↓ 是
 保存/提交单据
```

**校验时机**：

**校验规则**（继承预算提示级别按周期和年度设置：



| **校验维度** | **规则来源** | **校验逻辑** |

| 周期维度 | 预算设置-周期超额限制 当前期间额度 vs 周期预算上限 |  |
| 年度维度 | 预算设置-年度超额限制 | 年度累计额度 vs 年度预算上限 |

**校验结果展示**：



| **结果类型** | **视觉样式** | **提示文案示例** | **交互限制** |

### **4.2.5 自动汇总计算逻辑**

**计算公式**：

**p**
```plain
父级总额度 = 父级自身额度 + 下级汇总额度

其中：
- 父级自身额度：用户手动输入（有权限时，且"仅最末级可编辑"开关关闭）
- 下级汇总额度：自动汇总所有直接子级的总额度
- 末级维度总额度 = 申请额度
```

# **6\. 潜在风险与应对**



| **风险点** | **影响** | **应对措施** |
|-------------|----------|----------------|
| 自动计算性能问题 | 维度过多时（\>100个）计算延迟 | 前端防抖\+后端异步计算，计算时间\<1秒 |
| 实时校验接口并发 | 多用户同时编辑，校验请求过多 | 接口限流\+缓存，单用户最多10次/秒 |
| 权限数据同步延迟 | 权限变更后，用户仍能看到旧数据 | 权限变更时刷新缓存，下次进入页面生效 |
| 网络异常导致计算失败 | 汇总金额不准确 | 网络恢复后自动重试，失败时提示用户刷新 |
| 父子级额度循环依赖 | 计算逻辑错误 | 后端校验，禁止循环依赖的维度配置 |
| 误操作父级自身额度 | 总额度超出预期 | 实时校验\+警告提示，提交前二次确认 |
| 存量数据格式不兼容 | 编辑存量单据时出错 | 数据迁移脚本，自动转换存量数据格式 |
| "仅最末级可编辑"开关与智能汇总同时开启 | 功能冲突 | 系统判断，开启智能汇总时自动关闭"仅最末级"开关或给出提示 |
| 动态预算与手工编辑冲突 | 用户误改动态预算维度 | 动态预算维度置灰不可编辑，避免误操作 |

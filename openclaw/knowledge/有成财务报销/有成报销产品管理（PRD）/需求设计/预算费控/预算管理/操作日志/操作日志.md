# 操作日志

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20240228 | v1.0 | 新建 |

## 1、需求背景

目前变更历史功能无法正常使用，预算一些的变更没有地方查看，遇到问题无法追溯；

## 2、需求详情

### 2.1、需求说明

一、原来的变更历史入口去掉，不显示了；

在右上角顶部“新建预算”后新增一个按钮“操作日志”，点击新开TAB页，展示预算管理所有的操作日志



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75kammyKzlAK/img/13ccfb8b-6b4e-4174-9935-3e9d8d63ab18.png?Expires=1775016968&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=aEBfVhSz8WcNk5mPV%2B7G8VuHCTM%3D "")

二、操作日志页面说明：

记录预算管理内所有的操作记录，包括预算管理内的编制、基础设置；

1、时间默认为当前月份，指操作时间；



![Picture 2](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75kammyKzlAK/img/5e42d32f-8cc5-4e85-89d5-7bcdfa8d94a9.png?Expires=1775016968&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=%2BclFNgNsvDi3%2FukKmpOWzWb0jTo%3D "")

| **字段** | **需求说明** |

| 操作模块 | \+ 按功能模块划分：基础设置、预算管理；\+ 表头支持直接筛选，模块支持勾选 |

| 操作时间 | 精确到秒 |

**3.1  操作类型与操作内容结合说明**

| **操作类型** | **对应操作内容说明** |
|----------------|----------------------------|
| 新建 | 1、包含预算手工新建和复制新建两种方式2、操作内容文案为：名称：预算规则名称 |
| 修改 | 1、包含预算规则-基础设置的修改和基础设置中各种配置项的修改2、基础设置中配置项修改：\+ 格式采用“标题：变更前的配置项-\>变更后的配置项内容”；\+ 管控纬度有多项，则用分号隔开；\+ 设置内容用【】框定，多个设置内容用逗号隔开\+ 每一个配置项都单独生成一条记录，若一次性保存修改了多条配置，则生成多条记录；只记录显示产生变动的配置项，不是显示所有配置项信息3、预算规则-基础设置的修改\+ 格式采用名称：预算规则名称标题：变更前的配置项-\>变更后的配置项内容---多个配置项内容用逗号隔开\+ 同一个预算规则的基础设置变更只生成一条记录；只记录显示产生变动的配置项，不是显示所有配置项信息 |
| 预算调整 | 一、包含预算规则-纬度设置中的变动信息记录；二、格式采用：名称：预算规则名称三、列表中的预算规则名称可点击，点击后弹框显示当前纬度设置中变动信息，以预算纬度为准，当前预算纬度只要出现变化，就可以在弹框中查看到这个纬度的信息；\+ 展示的数据格式跟预算规则中一样，只展示有变动的预算纬度；\+ 变化指：删除预算纬度、手工修改预算金额、预算申请单产生的变更四、变动详情里面，显示当前预算规则中，有产生变动的纬度的所有数据：1、纬度：只显示当前纬度有变动的数据，没有变动的不展示；层级展示也同理，下级变动导致上级变了，则上级也显示，上级没变则不显示。\+ 手工编辑产生的预算金额变动\+ 动态预算产生的预算金额变动\+ 已删除的纬度：在纬度后面显示已删除字样，标红，括号括起来\+ 有层级的纬度 用/连接起来。2、期间：显示当前预算规则中的所有期间，显示变更前和变更后的数据；同时已用金额也显示：\+ 新增的预算纬度，变更前数据显示为0；\+ 删除的预算纬度，变更后数据显示为0；\+ 如果有汇总的期间，汇总的期间也需要显示；3、金额：产生了变动的标记出来。 |
| 新增预算纬度 | 一、预算规则-预算纬度新增的记录；二、格式为：名称：预算名称预算纬度：预算纬度名称----如果有层级，需要展示层级，用/号连接起来 |
| 预算申请 | \+ 预算申请单审批完成后记录这个动作，显示预算名称和具体的单据；\+ 格式为：名称：预算名称单据：显示单号----单号可点击，点击后直接打开单据详情， |
| 预算结转 | 交互参照预算调整一样的，只是单独拆出一个操作类型做区分：\+ 如果是手动结转，操作人记录具体的人\+ 如果是系统结转，操作人记录为“有成报销” |
| 预算重算 | \+ 显示重算的预算名称，暂不支持查看详情---后续考虑记录重算前数据和重算后数据，评估实现成本\+ 格式为：名称：预算规则名称 |
| 导入 | \+ 显示重算的预算名称，点击预算名称，打开弹框，显示调整数据，弹框采用“预算调整”打开的弹框一样\+ 如果是新增的纬度，则在弹框中标记出来；\+ 格式为：名称：预算规则名称 |
| 导出 | \+ 显示重算的预算名称，不支持查看详情\+ 格式为：名称：预算规则名称 |

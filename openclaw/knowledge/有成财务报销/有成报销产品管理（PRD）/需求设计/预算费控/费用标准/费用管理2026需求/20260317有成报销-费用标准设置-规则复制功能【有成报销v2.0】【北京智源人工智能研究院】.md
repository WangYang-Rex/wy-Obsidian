## **1\. 需求背景**

### **1.1 需求描述**

费用标准设置新增【规则复制】功能，允许用户对已存在的费用规则进行复制，快速生成一条字段值相同的新规则，仅需修改差异化字段即可完成配置。

### **1.2 需求背景**

**需求方**：有成报销企业客户（财务管理员）

**现状问题**：
- 同一人群在多个城市的标准，仅城市级别不同，其他字段（适用范围、往返日、日期类型、金额）完全相同
- 每条规则需重复填写相同字段，配置效率低
- ~~**原方案（继承字段）痛点**~~~~：需要前置配置继承字段，且只能继承最后一行，不够灵活~~

**解决思路变更**：
- ~~~~原方案：全局配置继承字段，新增时自动继承上一行~~~~
- **新方案**：提供【复制】操作，用户可主动选择复制任意已有规则，复制后插入到指定位置，所有字段完全复制，用户自行修改差异化字段

**业务流程**
```plain
创建规则1：VP及以上 + 一线城市 + 1000元
↓
点击规则1的【复制】按钮
↓
在规则1下方插入新规则，字段值完全相同
↓
修改城市级别为二线城市
↓
保存完成
```

## **2\. 功能现状**

费用标准管理位于【预算费控-费用标准-编辑标准-标准设置】。

**规则表格字段**：适用范围（多选）、往返日（多选）、日期类型（单选）、城市级别（单选）、坐席（单选，交通类）、金额。

**规则操作**：新增规则（空行）、删除、上移、下移。**无复制功能**。

### **3.2 详细功能描述**

#### **3.2.1 规则复制功能**

**功能概述** 用户可对任意已存在的费用规则进行复制，系统生成一条字段值完全相同的新规则，插入到被复制规则的下一行，用户仅需修改差异化字段。

**复制入口** 规则列表每行新增【复制】按钮，位于【操作】顺序最后。
```plain
规则列表操作栏：
┌────┬─────────────┬───────────┬──────────┬──────────┬────────┬─────────────────┐
│ #  │ 适用范围    │ 往返日    │日期类型  │ 城市级别 │ 金额   │ 操作            │
├────┼─────────────┼───────────┼──────────┼──────────┼────────┼─────────────────┤
│ 1  │ VP及以上    │出发/返回  │ 节假日   │ 一线城市 │ 800   │ [删除][上移][下移][复制]│
├────┼─────────────┼───────────┼──────────┼──────────┼────────┼─────────────────┤
│ 2  │ VP及以上    │出发/返回  │ 节假日   │ 二线城市 │ 800   │ [删除][上移][下移][复制]│ ← 复制生成
└────┴─────────────┴───────────┴──────────┴──────────┴────────┴─────────────────┘
```

**复制逻辑**
```plain
用户点击某行的【复制】按钮
↓
系统读取被复制行的所有字段值
↓
在被复制行下方插入新规则行
↓
新规则所有字段值 = 被复制行对应字段值
↓
光标定位到新规则的第一个可编辑字段（适用范围）
↓
用户修改差异化字段后保存
```

**复制效果示例**

复制前：
```plain
┌────┬─────────────┬───────────┬──────────┬──────────┬────────┐
│ #  │ 适用范围    │ 往返日    │日期类型  │ 城市级别 │ 金额   │
├────┼─────────────┼───────────┼──────────┼──────────┼────────┤
│ 1  │ VP及以上    │出发/返回  │ 节假日   │ 一线城市 │ 1000  │
└────┴─────────────┴───────────┴──────────┴──────────┴────────┘
```

用户点击第1行【复制】后：
```plain

│ #  │ 适用范围    │ 往返日    │日期类型  │ 城市级别 │ 金额   │
├────┼─────────────┼───────────┼──────────┼──────────┼────────┤
│ 1  │ VP及以上    │出发/返回  │ 节假日   │ 一线城市 │ 1000  │
├────┼─────────────┼───────────┼──────────┼──────────┼────────┤
│ 2  │ VP及以上    │出发/返回  │ 节假日   │ 一线城市 │ 1000  │ ← 复制生成（字段完全相同）
│    │ （可复制）  │ （可复制）│（可复制）│ 【修改为│（可复制）│
```

#### **3.2.2 复制后编辑**

**光标定位** 复制生成新规则后，光标自动定位到新规则的【适用范围】字段，用户可按Tab键或点击切换字段进行编辑。

**字段编辑**
- 所有字段均可编辑，与正常新增规则一致
- 用户可保留部分字段不变，修改差异化字段
- 支持一次复制后不修改直接保存（完全复制）

**连续复制** 用户可复制刚生成的新规则，实现连续快速复制：
```plain
规则1：一线城市标准
↓ 复制
规则2：二线城市标准（修改城市级别）
↓ 复制规则2
规则3：三四线城市标准（修改城市级别）
```


### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 复制空值字段 | 被复制行某字段为空 | 新规则该字段也为空 |
| 复制后未修改直接保存 | 用户复制后未做任何修改 | 允许保存，生成两条完全相同的规则（用户自行管理） |
| 复制后取消 | 用户复制后点击取消或不保存 | 新规则行丢弃，不保存到数据库 |
| 规则数量上限 | 复制后超出系统规则数量限制 | 提示"规则数量已达上限，无法复制" |
| 复制时原规则被删除 | 用户A复制规则，用户B删除该规则 | 复制时校验，若原规则已不存在则提示"该规则已被删除，请刷新后重试" |
| 复制后上移/下移 | 复制生成的规则支持上移下移 | 与现有上移下移逻辑一致 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 上移/下移 | 复制生成的规则支持上移下移 | 与现有逻辑一致 |
| 删除 | 复制生成的规则可单独删除 | 与现有逻辑一致，不影响被复制的原规则 |
| 保存 | 复制生成的规则需保存后才生效 | 与新增规则一致，点击【保存】按钮统一保存 |
| 新增规则（原有） | 保留原【新增规则】按钮 | 【新增规则】仍添加空行，【复制】基于已有规则生成 |

## **6\. 附录**

### **6.1 方案变更对比**

| **对比项** | **原方案（继承字段）** | **新方案（复制功能）** |
|-------------------------------------------------------------|-------------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 使用门槛 | 需先配置继承字段 | 无需配置，直接使用 |
| 复制来源 | 只能继承最后一行 | 可复制任意行 |
| 复制时机 | 新增规则时自动继承 | 用户主动点击复制 |
| 灵活性 | 低（需配置\+固定继承最后一行） | 高（随时复制任意行） |
| 字段控制 | 可配置哪些字段继承 | 所有字段完全复制 |
| 技术复杂度 | 需新增配置表\+修改新增逻辑 | 纯前端实现，无后端改动 |

### **6.2 典型使用场景**

**场景**：VP及以上在三个城市享受不同住宿标准

**操作步骤**：
1. 创建规则1：VP及以上 \+ 一线城市 \+ 1000元（填写全部字段）
2. 点击规则1的【复制】按钮，生成规则2（字段完全相同）
3. 修改规则2的城市级别为"二线城市"，保存
4. 点击规则2的【复制】按钮，生成规则3
5. 修改规则3的城市级别为"三四线城市"，保存

**结果**：3步完成3个城市标准配置，无需前置配置，随时可复制任意规则。

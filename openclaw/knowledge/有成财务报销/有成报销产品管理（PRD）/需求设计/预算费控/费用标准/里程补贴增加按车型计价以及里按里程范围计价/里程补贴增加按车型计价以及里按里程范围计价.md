#### 1.2 名词解释

本需求会增加三个名词，分别对应里程补贴的三种计价模式，定义如下：

| **模式** | **规则** | **示例** |  |  |  |
|----------|----------|----------|---|---|---|
|  |  | **设置示例** | **150公里** | **250公里** | **350公里** |
| 单一价格 | 不管多少公里均按一个价格计算，即现有的计算方式 | 设置2元/公里 | 150\*2 | 250\*2 | 350\*2 |
| 按范围单一价格 | 以里程所在范围的价格做为全程的单价进行计算 | 设置小于200公里，按2元/公里计算，200至300公里，按1.5元/公里计算，超过300公里按1元/公里计算 | 150\*2 | 250\*1.5 | 350\*1 |
| 阶梯价格 | 按照设置把一段里程拆分成多段，每段按照其里程范围的单价分别计算后再汇总 | 设置小于200公里，按2元/公里计算，超过200不超过300公里的部分按1.5元/公里计算，超过300公里的部分按1元/公里计算 | 150\*2 | 200*2\+50*1.5 | 200*2\+100*1.5\+50\*1 |

#### 1.3 竞品情况

汇联易：有单一计价和阶梯计价（交互不易理解），没有车型计价和按范围单一计价

每刻：有单一计价和阶梯计价（交互不错），有车型计价（系统预设了里程补贴用户组），选人时从用户组选，没有按范围单一计价

#### 1.4 产品现状

目前里程补贴只能设置单一价格进行补贴计算（即单一价格模式），本次需要新增按照车型计价以及按照范围单一计价，后续可能会增加阶梯计价。

## 4、需求详情

#### 4.1 里程补贴增加控制方式

（1）在规则列表上方增加了控制方式（按车型统一计算、按车型分别计算），历史数据都是“按车型统一计算”，并且功能逻辑保持不变。

（2）问号的提示文案为：“如需按车型分别计算补贴，则需要做如下两条配置：1、在【基础设置/辅助核算】中增加车型辅助核算；2、在里程补贴的费用中增加辅助核算字段并设置为车型辅助核算类型，作为必填项。费用中设置的辅助核算类型必须与此处选择的保持一致，否则生成费用时无法正常展示车型。”

（3）页面红色字体文案更新为：“规则将对出发地、目的地字段生效，若选择按车型分别计算补贴，则费用中需要有辅助核算字段，请确保需要控制的费用类型中包含对应字段”

###### 4.1.2.3 PC端-员工手动新增里程补贴费用（本次不开发）

员工在PC端新增里程补贴费用有两个入口：报销单增加费用明细和直接在消费出新增费用。对于设置按车型计算补贴的企业，员工新增费用时交互分别如下

（1）若员工没有配置过车型，则提示补充“我的车型”信息

提示文案“本次选择的车型仅用于本次里程补贴的计算且生成费用后不可更改。若您有新增车型，可先至APP【里程补贴-我的车型】进行补充。”

（2）若员工配置过车型且只配置了一个车型，则直接在费用编辑页展示车型信息，车型不可编辑（若没有配置好费用的车型字段，则不展示）

（3）若员工配置过车型且配置了多个车型，则直接在切换到里程补贴费用时，弹窗提示需要选择“本次车型”，选择后。费用编辑页展示车型信息，车型不可编辑（若没有配置好费用的车型字段，则不展示。）

###### 4.1.2.5 APP端-员工记录里程

员工在移动端点击【随手记-里程补贴】时，交互如下：

（1）若员工没有配置过车型，则提示补充“我的车型”信息。

提示文案“本次选择的车型仅用于本次里程补贴的计算且开始本次行程后不可更改。若您有新增车型，可先至APP【里程补贴-我的车型】进行补充。”

（2）若员工配置过车型，则直接打开里程补贴页面。

a、若员工配置了一个车型，在里程补贴页面点击“手工点位记录”或“记录新里程”时，则弹窗提示本次使用车型

b、若员工配置了多个车型，在里程补贴页面点击“手工点位记录”或“记录新里程”时，直接打开对应功能页面即可（即保持现有逻辑），该里程在生成消费时直接按照该车型计算补贴。

（3）行程结束，生成消费前，不再展示单价，改为展示车型

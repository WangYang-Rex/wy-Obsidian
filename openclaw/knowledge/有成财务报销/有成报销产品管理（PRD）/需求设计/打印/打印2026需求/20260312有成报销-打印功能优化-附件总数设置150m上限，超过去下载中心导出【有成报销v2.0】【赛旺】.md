# **1\. 需求背景**

## **1.1 目标用户**
- **主要用户**：财务
- **使用场景**：日常单据打印、月末对账打印、审计材料准备

## **1.2 需求来源**
- **客户名称**：赛旺客户
- **需求类型**：功能优化（体验提升）

## **1.3 背景**

### **1.3.1 现有功能说明**


```plain
有成报销打印功能架构
├── 打印入口
│   ├── 单据详情页打印按钮
│   └── 单据列表批量打印按钮
├── 打印设置弹窗
│   ├── 打印范围
│   │   ├── 选中单据范围（列表勾选时）
│   │   └── 全部单据（列表页支持时）
│   └── 打印内容
│       ├── 单据（面单，必选项）
│       ├── 图片
│       ├── 纸质发票
│       ├── 电子发票
│       ├── 关联单据
│       └── 附件电子回单
└── 打印执行
    ├── 直接预览打印（浏览器弹窗）
    └── 下载中心导出（异步任务）
```



### **1.3.2 现有打印逻辑现状**

当前系统存在两条打印路径：

**路径A：直接预览打印**
- 触发条件：仅勾选【单据】\+ 选中数量≤50张（见20260306需求）
- 执行方式：浏览器直接调起打印预览窗口
- 适用场景：纯面单打印，无附件

**路径B：下载中心导出**
- 触发条件：不满足路径A的所有情况
- 执行方式：生成PDF/ZIP文件，用户去下载中心领取
- 适用场景：含附件打印、或数量\>50张

**现存问题**：
- 当用户勾选了附件（如图片、发票等），但附件总大小并不大（如只有几张图片），仍被强制走下载中心导出，操作链路长
- 财务反馈：小额附件场景下，直接打印体验更流畅

### **1.3.3 关联需求说明**



| **关联需求** | **文档位置** | **核心逻辑** | **与本需求关系** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------------|
| 批量打印单据面单支持50张以下直接预览打印 | [我的附件] | 仅勾选【单据】且≤50张时直接预览打印 | 互斥场景（本需求处理"含附件"场景，关联需求处理"纯面单"场景） |

## **1.4 核心痛点**

### **痛点1：附件大小不敏感，一律走导出流程**

**现状**：
- 用户勾选了【单据】\+【图片】，共3张单据，附件图片总计5MB
- 当前逻辑：因为勾选了【图片】（非纯面单），强制跳转下载中心导出
- 用户需等待异步生成 → 去下载中心 → 下载文件 → 本地打开打印

**影响**：
- 操作链路长（4步 vs 1步）
- 小额附件场景用户体验差
- 下载中心任务堆积（本可直接打印的小任务占用队列资源）

## **2.2 附件总大小计算逻辑**


```plain
附件总大小计算流程
    ↓
获取【打印设置】中勾选的附件类型集合
    ↓
遍历选中单据列表
    ↓
对每个单据，获取其勾选类型的附件文件
    ↓
累加文件大小（单位：字节 → 转换为MB）
    ↓
IF 总大小 ≤ 150MB THEN
    允许直接打印
ELSE
    提示用户去下载中心导出
END IF
```



**计算公式**：


```plain
附件总大小(MB) = Σ(单据i的勾选附件类型文件大小) / 1024 / 1024

其中：
- 单据i ∈ 选中单据集合
- 勾选附件类型 ∈ {图片, 纸质发票, 电子发票, 关联单据, 附件电子回单}
- 仅计算实际存在的文件大小（空文件/已删除文件按0计算）
```

## **4.2 核心功能**

### **4.2.1 附件总大小打印设置确认后开始计算**

**功能说明**：
用户在打印设置弹窗中设置选项后，点击确定系统计算当前选择的附件总大小。



| **打印内容选项** | **是否计入附件大小** | **说明** |
|----------------------------------------------------------------------|----------------------------------------------------------------------------|----------------------------------------------------------|
| 单据（面单） | 否 | 面单为HTML渲染，不计入文件大小 |
| 图片 | 是 | 单据中上传的所有图片附件（如有） |
| 纸质发票 | 是 | 发票影像文件（如有） |
| 电子发票 | 是 | PDF/OFD格式发票文件（如有） |
| 关联单据 | 是 | 关联单据的附件文件（如有） |
| 附件 | 是 | 附件支持的格式有：JPG、JPEG、PNG、PDF（如有） |
| 电子回单 | 是 | 银行回单PDF文件（如有） |

**计算时机**：
- 打印设置弹窗打开时（初始化计算）
- 用户点击确定按钮时（开始计算）



**计算性能要求**：
- 单次计算响应时间 \< 500ms
- 并发场景下不影响弹窗交互流畅性

## **5.3 逻辑优先级（重要）**

当用户选择同时满足以下条件时，优先级如下：
1. **最高优先级**：仅勾选【单据】\+ 数量≤50 → 直接预览打印（小于等于50张审批单批量打印优化需求）
2. **次优先级**：勾选了附件 \+ 附件总大小≤150MB → 直接打印（本需求）
3. **兜底逻辑**：其他所有情况 → 下载中心导出

**说明**：
- 若用户仅勾选【单据】，无论附件大小如何（此时无附件），优先走小于等于50张审批单批量打印优化需求逻辑
- 若用户勾选了【单据】\+【图片】，走本需求的附件大小判断逻辑

| **风险点** | **影响等级** | **影响描述** | **应对措施** |
|-------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|

## **9.3 阈值设定依据**

**150MB的设定考虑**：
1. **浏览器性能**：现代浏览器处理150MB PDF渲染无明显卡顿
2. **网络传输**：150MB在常规办公网络（10Mbps\+）下加载时间约2分钟，可接受
3. **打印机性能**：主流激光打印机处理150MB打印任务无压力
4. **用户调研**：财务用户反馈日常批量打印附件通常在50-100MB范围

**后续可调优**：上线后根据埋点数据和用户反馈，可动态调整阈值（如调整至100MB或200MB）。

## **1\. 需求背景**

### **1.1 需求描述**

优化单据审批单打印模板，当单据内待支付金额币种非本位币时，费用分摊详情区域同时显示待支付金额币种金额和系统本位币金额，解决当前仅显示本位币（人民币）金额导致的财务对账困难问题。

### **1.2 需求背景**
- **需求方**：财务管理员
- **现状问题**：
• 系统本位币为人民币
• 单据内待支付金额支持多币种选择（USD/EUR/GBP等）
• 外币单据审批通过后，打印预览中费用分摊详情仅显示人民币金额
• 财务对账时需根据待支付金额币种手动反推外币分摊金额，效率低且易出错
• 无法满足跨国企业多币种报销的审计要求
- **解决思路**：打印模板读取单据待支付金额币种，非本位币时自动展示双币种金额（待支付币种\+本位币）

**业务流程**
```plain
单据打印模版内开启“打印费用分摊详情”且单据内待支付金额选择外币币种(USD)
    ↓
单据审批完成
    ↓
点击【打印】或【打印预览】
    ↓
系统检测单据待支付金额币种 ≠ 本位币(CNY)
    ↓
费用分摊详情区域展示：
  - 原币金额(USD)：按待支付金额币种显示
  - 本币金额(CNY)：按系统本位币显示
    ↓
财务对账时可同时查看原币和本币金额
```

### **3.2 详细功能描述**

#### **3.2.1 双币种金额展示**

**功能概述** 打印模板读取单据内"支付金额"组件的币种，当该币种不等于系统本位币（CNY）时，费用分摊详情表格同时显示"支付金额"组件的币种和本位币金额。

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/54Lq35o54oJ89l7E/img/8283406e-e396-4c4c-8c1d-76b25d730a8d.png?Expires=1774156263&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=lITFBRr6SEWkaFd2WhALii%2BxyEo%3D "")


**币种判断逻辑**
```text
IF 单据.“支付金额”组件选择的币种 == 系统本位币(CNY) THEN
    费用分摊详情显示：分摊金额(CNY)
ELSE
    费用分摊详情显示：原币金额(单据币种) + 本币金额(CNY)
END IF
```


**配置入口** 【单据管理-报销单/付款申请单-打印/打印预览-费用分摊详情区域】

**打印效果变更**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/54Lq35o54oJ89l7E/img/80d9b385-3f66-42ab-ae19-056202d0dec0.png?Expires=1774156263&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Sd6caKfQGO7%2Fls2IBgwlB2nf7tg%3D "")


**计算逻辑**
```plain
原币分摊金额 = 本币分摊金额 / 单据汇率
本币分摊金额 = 单据分摊比例 × 单据本币支付金额
```


**字段说明**

| **字段** | **来源** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------|
| 原币金额 | 计算得出 | 待支付金额币种的分摊金额 |
| 本币金额 | 数据库 | 系统本位币（CNY）的分摊金额 |
| 币种代码 | 单据.支付金额组件币种 | 显示在表头括号内，如(USD)、(EUR) |


### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 支付金额组件币种 | 单据未选择币种 | 默认按本位币(CNY)展示单币种 |
| 汇率为空 | 外币单据但汇率字段为空 | 原币金额显示"-"，本币金额正常显示 |
| 多币种批量打印 | 批量打印包含CNY和USD单据 | 每张单据根据各自支付金额组件币种独立渲染 |
| 尾差处理 | 原币金额计算产生尾差 | 最后一行分摊维度调整尾差，确保原币合计匹配待支付金额 |
| 精度溢出 | 计算结果小数位超过2位 | 四舍五入保留2位小数 |
| 汇率异常 | 汇率≤0或\>1000 | 原币金额显示"汇率异常"，本币金额正常显示 |
| 存量单据 | 历史外币单据 | 按新逻辑展示双币种，汇率取单据提交时汇率 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 批量打印 | 依赖 | 批量打印时每张单据独立读取支付金额组件币种并渲染 |
| 打印模板配置 | 独立 | 标准模板升级，自定义模板需手动调整 |
| 费用分摊规则 | 依赖 | 读取已有分摊比例和本币金额，计算原币金额 |
| 汇率管理 | 依赖 | 取单据支付时汇率，非实时汇率 |
| 导出PDF | 关联 | PDF导出同步应用双币种展示逻辑 |
| 多币种支付 | 关联 | 单据内多笔支付记录时，取第一笔支付币种作为展示币种 |

**存量兼容策略**


| **场景** | **处理方案** |
|----------------------------------------------------------|----------------------------------------------------------------|
| 本位币单据（支付金额组件币种=CNY） | 保持现有单币种展示，不增加原币列 |
| 外币单据（历史） | 按新逻辑展示双币种，币种取待支付金额币种 |
| 无费用分摊单据 | 不展示费用分摊详情区域，无影响 |
| 自定义打印模板 | 标准模板自动升级，自定义模板需手动调整 |


## **4\. 前后端工作任务**

### **4.1 前端负责内容**

• 【打印模板】费用分摊详情表格动态列渲染（根据单据.待支付金额币种判断）
• 【打印模板】原币金额、本币金额两列样式调整
• 【打印模板】表头币种代码动态显示（取自单据.待支付金额币种）
• 【打印模板】底部汇总信息双币种展示适配
• 【批量打印】批量打印时支持混合币种单据分别渲染

### **4.2 后端负责内容**

• 【接口】GET /api/print/expense-allocation - 费用分摊详情查询接口
- 新增返回字段：originalCurrencyAmount（原币金额）、currencyCode（币种代码）
    - 计算逻辑：原币金额 = 本币分摊金额 / 单据汇率
    - 币种来源：单据.待支付金额币种
• 【数据表】expense\_report（报销单表）- 确认字段：payment\_currency（待支付金额币种）、exchange\_rate（汇率）
• 【数据表】payment\_request（付款申请单表）- 确认字段：payment\_currency（待支付金额币种）、exchange\_rate（汇率）
• 【计算逻辑】尾差调整：最后一行原币金额 = 待支付金额 - 前面各行原币金额累加
• 【接口】GET /api/print/batch - 批量打印接口支持各单据独立币种判断

## **附录**

**A. 典型使用场景**

**场景：外企财务打印美元报销单**

背景：
• 系统本位币：人民币（CNY）
• 员工报销美元差旅费，单据待支付金额\$1,400（币种USD）
• 费用需分摊到项目A（60%）和项目B（40%）

使用步骤：
1. 员工提交报销单，支付金额组件选择币种USD，金额\$1,400
2. 审批流程走完，单据状态变为"审批通过"
3. 财务进入单据详情页，点击【打印预览】
4. 打印预览页面费用分摊详情显示：


```plain
项目A：$840.00 (USD) = ￥6,000.00 (CNY)
项目B：$560.00 (USD) = ￥4,000.00 (CNY)
```


5. 点击【打印】或【导出PDF】，文件同步展示双币种金额
6. 财务将打印件作为审计凭证归档

结果：财务无需手动换算，审计时可同时查看原币（与待支付金额一致）和本位币金额，满足跨国企业合规要求。

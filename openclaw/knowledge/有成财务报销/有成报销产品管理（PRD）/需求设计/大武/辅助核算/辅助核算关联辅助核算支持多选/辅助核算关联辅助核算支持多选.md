# 辅助核算关联辅助核算支持多选

| **修订时间** | **版本** | **作者** | **修订说明** |
|----------------|----------|----------|----------------|
| 20221213 | v1.0 | 大武 | 新建 |

## 1、需求背景

客户关联场景为分公司关联供应商，一个分公司会关联多个供应商，目前仅支持选择一个；

## 2、业务流程/架构

## 3、功能清单

|----------|-------------|----------|
| 辅助核算 | 辅助核算关联辅助核算 |  |
| 新建单据 | 关联辅助自动带出 |  |
|  |  |  |

## 4、需求详情
1. 基础设置-辅助核算中的类别如果关联了辅助核算，则在辅助的选项中可以关联到具体的对应的辅助核算值，目前辅助核算选项关联时仅可关联一个，本次修改关联为多选；如下图所示；



![Picture 1](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZ8yAXWAqze/img/a24ae295-1323-45e1-bc0c-f4587b41b31f.png?Expires=1778766759&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=6NhFXjUZ8f7w9ntH7Oz8ewPeQYI%3D "")
2. 新建单据或者消费的时候选择辅助核算的范围需要根据当前表单内的关联内容自动带出（现有逻辑）；当修改为多选了之后，需要判断关联选项是否选择了多个，如果是多个则取消自动带出逻辑，需要员工手工填写；如果只关联了一个，则自动带出关联的内容；例如：供应商关联客户，供应商1关联了客户1和客户2；则员工选择了供应商1，则不自动带出客户的内容，需要员工自己选择；如果供应商1关联了客户1，则选择供应商1，自动带出客户1；切换供应商则原本带出的客户删除，根据新的供应商带出新的关联客户；
3. 增加选项限制：如果用户在类别中设置了关联辅助核酸，则限制如果仅能选择关联的内容；例如：供应商关联客户，供应商1关联了客户1和客户2，单据中先选择了供应商1，则客户字段仅能选择到客户1和客户2；不能选择其他内容；如果供应商没有选择，先点击选择客户字段，则不限制可选项；选择了供应商1之后，则需要判断现有的客户字段是否是供应商1的关联客户，不是则清除之后再带出关联字段，是则保留；
4. API接口中辅助核算的查询和新增的关联关系需要修改为支持多选；

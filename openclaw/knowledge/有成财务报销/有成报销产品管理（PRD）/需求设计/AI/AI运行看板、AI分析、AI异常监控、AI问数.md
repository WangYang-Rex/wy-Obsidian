# **1\. 需求背景**

从老板角色出发，针对老板的查询、统计、分析场景以及核心的降本提效诉求，设计本期功能。

# **2\. 业务流程/架构**

利用现有系统内数据接口，预置或用户导入给定的分析结构，结合ai的数据处理、分析能力，输出结构化的报告、看板。

其中，目前数据仅限于报销系统内部数据，未来规划支持外部数据源。

    1. 根据所选的单据类型、单据时间重新异步拉取数据，新拉取到的数据按照最新的异常检查规则匹配并标记是否异常。
2. 用户可以点击重新检查历史数据，将数据明细中的所有数据按照当前最新的异常检测规则重新检查一次，并标记是否异常。

### **数据明细**
1. 需要细到费用明细（含分摊）的数据维度，仅统计单据审批状态=审批完成数据。
2. 字段

| 字段 | 说明 |
|------|------|
| 是否存在异常 | 这条明细数据根据异常规则分析后判断是否存在异常<br>正常、异常 |
| 首次提交时间 | 明细数据对应的单据的首次提交时间。<br>按照需要监控的单据时间展示，过早的数据不再展示。 |
| 单号 | 明细数据对应的单据号。 |
| 标题 | 明细数据对应的单据标题 |
| 费用明细 | 分行展示单据内消费明细中的费用名称（根据全局设置中选项控制是否展示全路径）<br>若费用存在分摊情况的，则需要按照分摊分行展示。 |
| 费用金额 | 分行展示单据内消费明细中的费用金额；<br>若费用存在分摊情况的，则需要按照分摊分行展示分摊金额。 |
| 申请人 | 明细数据对应的单据申请人 |
| 报销部门 | 明细数据对应的单据所属部门 |
| 所属项目 | 取数优先级：<br>若费用存在分摊情况的，则需要根据分摊的项目展示。<br>若费用不存在分摊情况但费用上有项目字段的，取费用上的项目；<br>若费用不存在分摊情况且费用上也没遗鸥项目字段，取单据上的项目。 |
| 所属部门/成本中心/辅助核算 | 辅助核算根据监控设置中的“关注辅助核算类型”展示。<br>这些细分维度数据取值逻辑同“所属项目” |
3. 导出，支持导出列表中的全部数据，挂起任务到下载中心。

### **AI异常监控面板**
1. 本期仅支持一套通用的面板展示，后期规划支持企业定制自己的面板。
2. 监控面板中点击的图表卡片支持穿透联查异常数据明细。
3. 元素

| 元素 | 说明 |
|------|------|
| 数据更新时间 | 统计数据按照每一小时进行更新。<br>展示格式：yyyy-mm-dd hh:mm:ss |
| 异常费用数卡片 | 近【设置中的检测时间】扫描出异常费用【异常费用数，数据明细中的一条数据为1】<br>较上【设置中的检测时间】，【上升/下降】【（本周期-上周期）/上周期\*100%】% |
| 异常单据数卡片 | 统计异常费用对应的单据数，按照单据号去重。<br>其他逻辑同上。 |
| 异常费用金额卡片 | 统计上述异常费用的费用金额（按照分摊拆分后的金额）。其他逻辑同上。<br>统计全部费用金额，异常金额占比%。 |
| 异常问题总结 | 让AI根据当前看板内容分析总结三个重点问题。提示词示例：<br>当前需要关注的三个重点问题：（1）部门异常费用金额超额（举个例子）（2）xxx\\xxx\\xxx项目利润里偏低（3）xxx部门预算执行情况不理想 |
| 员工异常费用金额统计柱状图 | 展示员工异常费用金额、费用金额统计柱状图，默认展示异常金额最高的10位员工。 |
| 员工异常费用金额排名条形图 | 展示异常费用最高的前十位员工条形图。红色字体统计前十异常金额合计值，黑色字体展示全部异常费用金额合计，计算前十员工异常金额合计/异常费用合计百分比。 |
| 部门/所属项目/成本中心/辅助核算类别异常费用金额统计柱状图 | 每个维度分别展示统计卡片。<br>展示该维度异常费用金额、费用金额统计柱状图，默认展示异常金额最高的10个。 |
| 部门/所属项目/成本中心/辅助核算类别异常费用金额排名饼状图 | 每个维度分别展示统计卡片。<br>展示异常费用最高的前三个饼状图。红色字体统计前三异常金额合计值，黑色字体展示全部异常费用金额合计，计算前三个异常金额合计/异常费用合计百分比。 |

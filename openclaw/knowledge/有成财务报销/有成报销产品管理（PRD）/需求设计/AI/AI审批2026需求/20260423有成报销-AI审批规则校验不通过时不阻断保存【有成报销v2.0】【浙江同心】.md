## 1\. 需求背景

### 1.1 需求描述

调整AI审批规则校验的交互逻辑：校验不通过时，从「阻断保存」改为「弹窗确认后允许保存」，降低AI规则校验对配置流程的阻断，提升审批管理员配置效率。

### 1.2 需求背景
- **需求方**：审批管理员
- **现状问题**：
• AI审批助手支持配置AI审批规则，填写具体规则后点击「确定」，系统调用AI对规则进行语义校验
• 校验不通过时，系统返回错误原因并**阻断规则保存**，配置人员无法保存规则
• AI校验对规则表述较为严格，轻微语义问题即触发校验失败，导致配置效率低、反复修改
- **解决思路**：校验不通过时仍返回错误原因，但不再阻断保存，改为弹窗提示用户「规则未通过AI校验，是否仍要保存？」，用户确认后允许保存

**业务流程**
```
审批管理员进入：AI应用 → AI审批 → AI审批助手 → AI审批规则
  ↓
填写具体规则内容 → 点击「确定」
  ↓
系统调用AI规则校验
  ├─ 校验通过 → 规则保存成功
  └─ 校验不通过 → 展示错误原因 + 弹出确认提示弹窗
                              ↓
                        用户点击「确定」→ 规则保存成功
                        用户点击「取消」→ 关闭弹窗，返回编辑页

```

## 2\. 功能现状
- **位置**：AI应用 → AI审批 → AI审批助手 → AI审批规则
- **现有操作**：填写具体规则 → 点击「确定」→ 系统进行AI规则校验
- **当前能力**：
• 校验通过 → 调用保存接口 → 保存成功
• 校验不通过 → 前端拦截，不调用保存接口 → 保存失败
- **缺失能力**：
• 校验不通过时无「强制保存」路径
• 无二次确认机制，用户无法在知情前提下保存未通过校验的规则

### 2.1 AI规则校验与保存流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 前端收集规则文本 | 用户填写的AI审批规则内容 |
| 2 | 调用AI规则校验接口 | 系统将规则文本提交给AI进行语义校验 |
| 3 | AI返回校验结果 | 通过/不通过 \+ 错误原因 |
| 4 | 校验通过 | 调用保存接口，保存成功 |
| 5 | 校验不通过 | 前端拦截，返回错误原因，保存失败 |
> 本次需求需突破步骤5的阻断逻辑，改为提示确认后允许保存。

## 3\. 本次变更内容

### 3.1 功能变更清单

| 变更类型 | 功能点 | 功能说明 | 优先级 |
|------------|---------|------------|---------|
| 修改 | AI审批规则校验交互 | 校验不通过时，从阻断保存改为弹窗确认后允许保存 | P0 |
| 修改 | 规则保存接口 | 保存接口新增 `skipAiValidation` 参数，用于强制保存场景 | P0 |

### 3.2 详细功能描述

#### 3.2.1 AI审批规则校验交互调整

**功能概述**

AI规则校验不通过时，系统不再阻断保存，而是弹出确认提示弹窗，展示AI返回的错误原因，由用户决定是否强制保存。

**配置入口**

不涉及新增配置入口，复用现有AI审批规则配置页面：AI应用 → AI审批 → AI审批助手 → AI审批规则

**配置弹窗/界面**
```text
┌─────────────────────────────────────────┐
│ 提示                                [X]  │
├─────────────────────────────────────────┤
│                                         │
│  填写的规则未通过AI校验                 │
│                                         │
│  【错误原因】                           │
│  xxxxxx（AI返回的具体错误描述）         │
│                                         │
│  是否仍要保存该规则？                   │
│                                         │
├─────────────────────────────────────────┤
│                    [取消]    [确定]     │
└─────────────────────────────────────────┘

```

**默认配置**

| 字段 | 默认状态 | 说明 |
|------|------------|------|
| 弹窗标题 | 「提示」 | 固定文案 |
| 错误原因 | AI返回内容 | 直接展示AI校验返回的错误描述 |
| 确认文案 | 「是否仍要保存该规则？」 | 固定文案 |

**配置存储** • 弹窗为前端临时展示，不存储配置
• 规则保存后按现有逻辑存储

**交互规则**
- 弹窗内容需展示 AI 返回的具体错误原因
- 点击「取消」：关闭弹窗，停留在规则编辑页，已填写内容保留
- 点击「确定」：调用保存接口（携带 `skipAiValidation=true`），规则保存成功
- 校验通过时，不弹窗，直接保存成功（保持现有逻辑）

#### 3.2.2 规则保存接口变更

**功能概述**

保存接口新增可选参数 `skipAiValidation`，用于用户确认后的强制保存场景。

**业务流程**
```text
用户点击「确定」
  ↓
前端调用AI规则校验接口
  ├─ 校验通过 → 直接调用保存接口（不传skipAiValidation）→ 保存成功
  └─ 校验不通过 → 展示错误原因 + 弹出确认弹窗
                          ↓
                    用户点击「确定」→ 调用保存接口（传skipAiValidation=true）→ 保存成功
                    用户点击「取消」→ 关闭弹窗，停留在编辑页

```

**效果示例**

| 场景 | 请求参数 | 结果 |
|------|------------|------|
| 校验通过保存 | 正常保存参数 | 保存成功 |
| 强制保存 | 正常保存参数 \+ skipAiValidation=true | 跳过校验，保存成功 |

**接口变更**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `skipAiValidation` | Boolean | 否 | 传 `true` 时，后端跳过AI校验直接保存规则；不传或传 `false` 时，按现有逻辑执行 |
> 注：`skipAiValidation` 仅用于用户确认后的强制保存场景，正常保存流程不传该参数。   

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| 边界场景 | 触发条件 | 处理方案 |
|------------|------------|------------|
| 校验通过 | AI返回校验通过 | 不弹窗，直接保存成功（保持现有逻辑） |
| 校验不通过-用户确认 | 用户点击弹窗「确定」 | 调用保存接口（skipAiValidation=true），保存成功 |
| 校验不通过-用户取消 | 用户点击弹窗「取消」 | 关闭弹窗，停留在规则编辑页，已填写内容保留 |
| 错误原因为空 | AI返回错误原因为空字符串 | 弹窗中错误原因区域展示「校验未通过」兜底文案 |
| AI校验服务异常 | 校验接口超时或报错 | 按现有异常处理逻辑，不阻断保存或提示网络异常 |
| 快速重复点击 | 用户连续点击弹窗「确定」 | 按钮防重复点击，避免重复调用保存接口 |
| 编辑已有规则 | 修改已保存规则后校验不通过 | 确认保存后更新原规则 |

**与其他功能的兼容性**

| 相关功能 | 交互关系 | 处理方案 |
|------------|------------|------------|
| AI审批规则执行 | 独立功能 | 规则保存后，AI审批执行时仍按规则内容正常执行，与校验是否通过无关 |
| 规则历史版本 | 独立功能 | 强制保存的规则同样生成历史版本记录 |

| 序号 | 验收项 | 验收标准 |
|------|---------|------------|

| 2 | 校验不通过-确认保存 | 填写规则，AI校验不通过，点击「确定」→ 弹窗中点击「确定」→ 规则保存成功，toast提示「保存成功」 |
| 3 | 校验不通过-取消保存 | 填写规则，AI校验不通过，点击「确定」→ 弹窗中点击「取消」→ 关闭弹窗，停留在编辑页，已填写内容保留 |

| 6 | 错误原因为空 | 模拟AI返回错误原因为空 → 弹窗中展示「校验未通过」兜底文案 |
| 7 | 防重复点击 | 弹窗弹出后，快速连续点击「确定」→ 仅调用一次保存接口 |

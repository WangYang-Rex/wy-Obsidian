## 2、需求背景

目前关联单据是需要关联具体的消费的，但是关联完单据后没有将选择的消费带入到消费明细，用户还需要再次选择消费明细

## 3、业务流程/架构

## 4、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 关联单据 | 编辑的消费明细能带入到消费明细组件 |  |
| 其他单据引入 | 放开对应付合同的限制 |  |
|  |  |  |

## 5、需求详情

### 5.1、需求说明

1、新建/编辑单据时，消费明细组件--其他单据引入：放开对应付合同的限制，可以从应付合同中引入消费信息

2、新建/编辑单据时，关联单据组件--关联单据后编辑了消费后，如果单据中有消费明细组件，则将编辑的消费带入消费明细组件，金额为关联的金额

  带入的消费可编辑、可删除，不影响关联单据组件的消费关联

# 出纳节点支持多账户选择

| **修订时间** | **版本** | **修订说明** | **作者** |
|----------------|----------|----------------|----------|
| 20221012 | v1.0 | 新建 | 昭觉 |

## 1、涉及系统

有成财务 & 有成报销 PC& MB

## 2、需求背景

目前出纳节点只能选择一个支付账户，出纳在支付节点遇到多个支付账户时，只能一笔一笔去操作

## 3、业务流程/架构

## 4、功能清单

| **模块** | **功能点** | **备注** |
|----------|-------------|----------|
| 审批节点 | 1、审批节点支持多账户选择2、审批节点支持选择支付账户功能迁移 |  |
| 出纳节点 | 1、支持选择多支付账户2、出纳节点选择了记住已选支付账户时：如果已选支付账户或者单据的收款账户只有一个，则在后续节点带出记住的支付账户；如果出纳节点选择了多账户&记住已选支付账户，当收款账户为多账户时，则只带出选择的第一个支付账户 |  |
| 复核节点 | 1、单笔复核同意时支持带出出纳节点选择的支付账户2、批量复核同意时不带出出纳节点选择的支付账户，支持选择多支付账户3、不展示【记住已选账户】选项 |  |
| 单据详情页 | 审批历史展示，一次展示账户名\+支付金额 |  |
| 凭证 | 1、多账户支付后，贷方会出现多条分录借  费用     1000贷  A银行    300贷  B银行    300贷  C银行    4002、如果是分次支付，那么未付的会挂中转借  费用     1000贷  A银行    300贷  B银行    200贷 中转科目   5003、后续支付时借方为中转科目借  中转科目    500贷  A银行    300贷  B银行    200 |  |
| 历史数据 | 1、记住支付账户历史数据多选兼容2、审批节点支持支付账户选择功能迁移后历史数据处理 |  |

## 4、需求详情

### 4.1、需求说明

1、审批节点选择支付账户:  审批节点选择支付账户时允许选择多账户,无需展示招商云直连、支付宝、招商CBS和银企直连的选项,展示逻辑同现有,账户支持多选

1.1 批量审批时不支持审批节点选择支付账户

1.2 批量支付时不支持出纳节点选择多账户

2、出纳节点选择了多账户:

  2.1 如果多账户都是招商云直连/招商CBS,则展示 招商云直连/招商CBS的选项可勾选,根据每笔支付金额批量发送支付数据给招商云直连/招商CBS;如果选择的支付账户都是银企直连/支付宝,则不展示银企直连/支付宝的选项; 2.2 如果选择的账户开通了招商云直连、招商CBS,则同时展示 招商云直连和招商CBS,但是只能勾选一个,点击确定时,根据每笔支付金额批量发送支付数据给招商云直连/招商CBS如果选择的账户同时开通了招商云直连和招商CBS/银企直连,则不展示银企直连的选项2.3 如果选择账户包含了银企直连、支付宝、招商云直连、招商CBS的任意两种以上的组合,则不展示招商云直连、招商CBS、支付宝、银企直连的选项

### 多次支付

1、单收款账户时:

带出收款账户信息,金额默认为剩余可支付金额可新增一条记录选择支付账户,收款账户不允许选择,支付金额需手动输入,总金额必须等于支付金额只有一条支付数据时不允许删除

2、多收款账户时:

 带出收款账户信息,金额默认带出,可修改 可新增记录选择多支付账户,收款账户的选择范围为单据中选择的支付账户,金额手动调整,总金额必须等于支付金额 只有一条数据时不允许删除

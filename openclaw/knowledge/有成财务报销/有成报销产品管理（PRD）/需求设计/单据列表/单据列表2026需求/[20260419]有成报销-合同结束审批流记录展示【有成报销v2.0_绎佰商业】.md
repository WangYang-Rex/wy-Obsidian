## 1\. 需求背景

### 1.1 需求描述

有收款计划的应收合同和有付款计划的应付合同，在审批完成后通过「结束合同」功能终止收/付款操作时，需在PC端和MB端单据详情的审批流区展示结束操作记录，展示样式参照申请单「停用」后的审批流记录。

### 1.2 需求背景
- **需求方**：企业财务人员、合同管理员
- **现状问题**：
• 申请单被「停用」后，PC端和MB端单据详情的审批流区会展示「开始 员工名称 手动停用了单据」的操作记录
• 合同的「结束合同」功能与申请单的「停用」功能作用类似（均终止单据后续操作），但合同结束后审批流区无操作记录留存
• 用户无法直观看到合同是何时、被谁结束的，不利于审计追溯
- **解决思路**：
• 合同执行「结束合同」操作后，在审批流区追加一条结束操作记录
• 展示样式完全参照申请单「停用」的审批流记录格式
• 同步支持PC端和MB端

## 2\. 功能现状

### 2.1 申请单停用审批流记录现状

**位置**：单据详情 → 审批流程

**现有展示样式（PC端）**：
• 审批流最底部追加一条记录
• 格式：「开始 京竹-李亮亮 手动停用了单据」
• 时间戳：操作执行时间（如 2026-04-19 21:29:28）

**现有展示样式（MB端）**：
• 审批流程tab → 历史操作记录区域
• 格式与PC端一致：「开始 京竹-李亮亮 手动停用了单据」
• 时间戳：操作执行时间（如 04-19 21:29）

### 2.2 合同结束功能现状

**位置**：单据管理 → 应收合同/应付合同

**现有能力**：
• 有收款计划的应收合同、有付款计划的应付合同，审批完成后支持「结束合同」
• 「结束合同」后合同终止收/付款操作，不允许再执行收款/付款
• 合同结束后，审批流区**不展示**结束操作记录

**现有缺失**：
• 合同结束无操作记录留存，用户无法追溯结束人和结束时间

### 3.2 详细功能描述

#### 3.2.1 PC端合同结束审批流记录展示

**功能概述**

有收款计划的应收合同和有付款计划的应付合同，在执行「结束合同」操作后，PC端单据详情的审批流程tab中追加展示一条结束操作记录。

**配置入口**

单据管理 → 应收合同/应付合同 → 打开已审批完成的合同详情 → 审批流程tab

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/ce562153-1190-4df3-9f6d-76479d9dd9af.png?Expires=1776848473&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=sdrprND74VL3gfdYDJrQkGR1ySI%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/bd64f6b8-f1fc-4854-8b24-ee6e8170ae46.png?Expires=1776848473&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=hx0W1eOyNoJiQMLjxcNoP15qUsc%3D "")

**展示内容**

| 字段 | 说明 |
|------|------|
| 节点类型 | 「开始」 |
| 操作人 | 执行「结束合同」操作的员工名称（格式：公司-姓名，如「京竹-李亮亮」） |
| 动作描述 | 「手动结束了合同」 |
| 操作时间 | 「结束合同」操作的执行时间，精确到秒（格式：YYYY-MM-DD HH:mm:ss） |

**展示位置**

审批流时间轴的最底部（所有正常审批节点之后），作为最后一条记录追加。

**展示样式**

完全参照申请单「停用」的审批流记录样式：
• 左侧节点图标：与「开始」节点样式一致
• 时间戳在上方，操作描述在下方
• 无操作按钮（不同意/驳回/转交等）

**效果示例**

```
│  开始  京竹-李亮亮  手动结束了合同  ← 新增                   │
│  2026-04-19 21:35:12                                         │
└─────────────────────────────────────────────────────────────┘

```

#### 3.2.2 MB端合同结束审批流记录展示

**功能概述**

与PC端一致，MB端单据详情的审批流程tab中同样追加展示合同结束操作记录。

**配置入口**

MB端 → 单据详情 → 审批流程tab → 历史操作记录

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/de6eb794-2cd5-4230-8de9-b0461517b4e1.png?Expires=1776848473&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=L66oQy6PASRYO12bud7ZDx4mLDs%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/d3021560-6589-4122-8f51-d6567d97ddb3.png?Expires=1776848473&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=6HPqGmm9gI47PYoBuLBk7jZEaNk%3D "")

**展示内容**

与PC端完全一致：
• 节点类型：「开始」
• 操作人：公司-姓名
• 动作描述：「手动结束了合同」
• 操作时间：MM-DD HH:mm（MB端简短时间格式）

**展示位置**

历史操作记录列表的最底部，所有正常审批记录之后。

**展示样式**

完全参照申请单「停用」的MB端样式：
• 左侧圆形头像/默认图标
• 时间戳在上方灰色小字
• 操作描述在主区域

### 3.3 逻辑互斥与边界处理

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------|----------------|----------------|
| 重复结束合同 | 合同已结束，再次点击「结束合同」 | 按钮置灰或隐藏，不允许重复操作 |
| 无收款/付款计划的合同 | 普通应收/应付合同，无收款/付款计划 | 无「结束合同」功能，不涉及 |
| 审批未完成的合同 | 合同仍在审批中 | 「结束合同」按钮不可见/不可操作 |
| 结束合同操作人离职 | 操作员工已离职，名称显示问题 | 显示当时操作人的名称（快照），不因离职变化 |
| 审批流数据为空 | 特殊情况下审批流记录为空 | 结束记录作为唯一记录展示 |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------|----------------|----------------|
| 申请单停用 | 展示样式参照对象 | 合同结束记录与申请单停用记录格式完全一致，仅文案不同 |
| 合同收款/付款 | 结束合同后禁止操作 | 结束合同后，收款/付款按钮置灰，已有计划不可执行 |
| 合同编辑 | 结束合同后是否允许编辑 | 按现有逻辑，结束合同后不允许编辑 |
| 合同删除 | 结束合同后是否允许删除 | 按现有权限逻辑处理 |

## 4\. 验收标准

| **序号** | **验收项** | **验收标准** |
|----------|-------------|----------------|
| 1 | PC端展示-应收合同 | 有收款计划的应收合同结束后，PC端审批流最底部展示「开始 操作人 手动结束了合同」及操作时间 |
| 2 | PC端展示-应付合同 | 有付款计划的应付合同结束后，PC端审批流最底部展示「开始 操作人 手动结束了合同」及操作时间 |
| 3 | MB端展示-应收合同 | 有收款计划的应收合同结束后，MB端审批流程tab历史操作记录最底部展示结束记录 |
| 4 | MB端展示-应付合同 | 有付款计划的应付合同结束后，MB端审批流程tab历史操作记录最底部展示结束记录 |
| 5 | 展示格式一致性 | 合同结束记录的展示样式（字体、颜色、图标、间距）与申请单停用记录完全一致 |
| 6 | 时间准确性 | 展示的操作时间与实际执行「结束合同」操作的时间一致 |
| 7 | 操作人准确性 | 展示的操作人名称为实际执行「结束合同」操作的员工 |
| 8 | 无计划合同不展示 | 无收款/付款计划的普通合同，不展示结束记录（因无结束合同功能） |
| 9 | 审批中合同不展示 | 审批未完成的合同，审批流区不展示结束记录 |

## 附录

**A. 典型使用场景**

**场景：财务结束已完成的应收合同**
1. 财务人员在应收合同列表找到一条已审批完成的合同
2. 点击「结束合同」按钮，确认结束
3. 合同状态变为「已结束」，收款计划终止
4. 打开合同详情 → 审批流程tab，最底部显示：「开始 京竹-李亮亮 手动结束了合同 2026-04-19 21:35:12」
5. 财务可清晰追溯该合同是何时被谁结束的

| 字段 | 类型 | 说明 |
|------|------|------|
| end\_contract\_time | DateTime | 结束合同操作时间 |
| end\_contract\_operator\_id | Long | 结束合同操作人ID |
| end\_contract\_operator\_name | String | 结束合同操作人名称（快照） |

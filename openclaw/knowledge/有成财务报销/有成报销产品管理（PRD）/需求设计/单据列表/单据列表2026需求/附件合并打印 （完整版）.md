## 1\. 需求背景

### 1.1 需求描述

基于有成报销单据现有打印功能，在打印内容选择中增加「审批流附件」选项，支持将单据内附件与审批流评论区的附件合并打包到下载中心，供用户自行下载后打印。

### 1.2 需求背景

当前有成报销在附件打印方面存在能力缺口：
- **单据附件**：已支持16种文件类型的预览、单文件下载和批量ZIP下载
- **评论附件**：审批流评论区上传的附件仅支持单个下载，**不支持批量打印**
- **合并打印**：单据附件与评论附件**无法一起打印**，用户需要分别操作
- **附件打印**：现有打印功能**仅支持单据打印**，不支持附件打印

财务人员在实际工作中经常需要把单据附件和审批意见里的截图、合同等附件一起归档或打印，目前只能逐个下载，操作繁琐。

### 1.3 业务流程
```
用户进入单据详情页/列表页
    ↓
在现有打印功能入口选择「导出全部附件（含评论）」
    ↓
后端查询单据内附件 + 审批流评论附件
    ↓
合并打包为ZIP（同一文件夹，不分子文件夹）
    ↓
ZIP进入下载中心，用户收到通知
    ↓
用户下载ZIP → 解压 → 自行打印

```

## 2\. 功能现状

### 2.1 单据附件（已有）
- **支持格式**：16种文件类型（DOC、DOCX、GIF、JPEG、JPG、MP3、MP4、PDF、PNG、PPT、PPTX、RAR、XLS、XLSX、ZIP）
- **单文件下载**：已支持单个附件下载
- **批量下载**：已支持单据组件附件批量下载（ZIP打包 → 下载中心）
- **附件大小限制**：默认 `MAX_DOWNLOAD_SIZE` 限制
- **前端预览**：图片、PDF 直接预览；Excel、Word 等通过 OSS 预览

### 2.2 评论附件（已有）
- **存储方式**：`OrderOperationInfo` 表中 `attachment` 字段，JSON 数组格式存储附件信息
- **单文件下载**：支持单个评论附件下载
- **批量导出**：**不支持**
- **前端展示**：评论列表中附件以 PopOver 弹出附件列表展示，可逐个点击预览或下载

### 2.3 打印功能（已有）
- **单据打印**：已支持单据打印（生成单据 PDF）
- **附件打印**：**不支持**
- **打印入口**：单据详情页/列表页的打印按钮

### 2.4 现有痛点

| 痛点 | 影响场景 | 当前 workaround |
|------|------------|-----------------|
| 评论附件无法批量打印 | 审批流中有多个评论附件时需要逐个下载 | 逐个点击下载，效率极低 |
| 单据附件与评论附件无法合并打印 | 归档时需要同时打印单据附件和审批截图 | 分别下载两组附件，手动合并 |
| 无附件直接打印能力 | 用户希望直接打印附件内容 | 先下载到本地，再用本地程序打开打印 |

## 3\. 本次变更内容

### 3.1 功能变更清单

| 序号 | 功能点 | 变更类型 | 影响范围 | 优先级 |
|------|---------|------------|------------|---------|
| 1 | 单据附件\+评论附件合并打印ZIP | 新增 | 附件打印逻辑（全模块） | P0 |
| 2 | 打印入口：打印选项弹窗增加选项 | 新增 | 单据详情页/列表页打印入口 | P0 |
| 3 | 下载中心ZIP通知 | 新增 | 下载中心 | P0 |

### 3.2 详细功能描述

#### 3.2.1 打印入口：打印选项弹窗增加「审批流附件」选项

**界面变化**：
- 在「打印选项」弹窗的「打印内容选择」区域，新增「审批流附件」复选框选项
- 位置：现有「附件」选项之后，或同一行末尾
- **默认状态**：默认**不勾选**
- 勾选后，打印/导出的内容包含审批流评论区的附件

**打印内容选择区域示意**：
```
打印内容选择：☑ 单据  ?  ☑ 图片  ☑ 纸质发票  ☑ 电子发票
             ☑ 关联单据  ☑ 附件  ?  ☐ 审批流附件（新增，默认不勾选）
             ☑ 电子回单

```

**适用单据类型**：
- 报销单、借款单、付款单、合同、收款单等所有已有打印功能的单据类型
- 各单据类型统一支持，不区分差异

#### 3.2.2 导出范围：根据打印内容勾选选项决定

**导出内容**：
- **单据内附件**：当「附件」选项勾选时，包含单据详情页中上传的所有附件（全部16种文件类型）
- **审批流评论附件**：当「审批流附件」选项勾选时，包含该单据审批流程中所有审批人在评论区上传的附件
- 两种附件根据用户勾选情况合并到一个ZIP包中

**勾选组合场景**：

| 附件 | 审批流附件 | 导出内容 |
|------|---------------|------------|
| ☑ | ☐ | 仅单据内附件 |
| ☐ | ☑ | 仅审批流评论附件 |
| ☑ | ☑ | 单据内附件 \+ 审批流评论附件 |
| ☐ | ☐ | 不导出任何附件 |

**ZIP包内部结构**：
- ZIP包内所有附件放在**同一文件夹**，不分子文件夹
- 文件名重复处理：同一文件名自动追加序号（如 `invoice.pdf` → `invoice_1.pdf`）
- 保持原始文件名，不做额外重命名规则

#### 3.2.3 打印流程：选择 → 查询 → 打包 → 下载中心

**用户操作流程**：
1. 用户点击「打印」按钮，打开「打印选项」弹窗
2. 用户在「打印内容选择」区域勾选需要的选项（含新增的「审批流附件」）
3. 点击「确定」提交打印请求

**后端处理流程**：
1. 接收打印请求，解析勾选项
2. 若勾选「附件」，查询单据内所有附件URL（`AttachmentHelper` 或同类组件）
3. 若勾选「审批流附件」，查询审批流评论附件URL（解析 `OrderOperationInfo.attachment` JSON 字段）
4. 下载所有勾选附件并合并打包为ZIP（复用现有 `batchDownloadByOrderDetail()` 逻辑并扩展）
5. ZIP生成完成后推送到下载中心
6. 有成报销后台界面右上角「下载中心」入口出现红点提示

**附件大小限制**：
- 复用现有批量下载大小限制逻辑（默认 `MAX_DOWNLOAD_SIZE`）
- 若附件总大小超过限制，提示「附件总大小超过限制，请分批导出」，终止导出

**无附件场景**：
- 若用户勾选了「审批流附件」但单据无评论附件，该部分为空，不影响其他勾选项导出
- 若用户未勾选任何附件类选项，仅导出单据等其他勾选内容

#### 3.2.4 下载中心

**下载中心红点提示**：
- ZIP生成完成后，有成报销后台界面右上角「下载中心」入口出现红点提示
- 用户点击「下载中心」进入下载中心页面，可在列表中看到附件合并打印的ZIP包
- 下载中心列表展示：单据编号/名称、导出时间、附件数量、ZIP文件大小

**下载中心记录**：
- 用户可在下载中心查看导出记录，包含：单据编号/名称、导出时间、附件数量、ZIP文件大小
- 支持点击下载ZIP文件
- ZIP文件保留时长复用下载中心现有策略

### 3.3 边界处理

| 场景 | 处理方式 |
|------|------------|
| 评论附件URL失效 | 某个评论附件URL已过期，其他附件正常导出，失效附件跳过并在导出记录/通知中提示「X个附件已失效，未包含在ZIP中」 |
| 附件总大小超过限制 | 提示「附件总大小超过限制，请分批导出」，终止本次导出 |
| 文件名重复 | 自动追加序号（如 `file.pdf`、`file&#95;1.pdf`） |
| 用户无单据权限 | 导出入口不展示 |
| 网络异常 | 打包过程中网络断开，提示导出失败，不生成损坏ZIP |
| 并发导出 | 同一用户同时导出多个单据附件，各ZIP独立生成互不影响 |
| 特殊字符文件名 | ZIP正常打包，解压后文件名完整保留 |

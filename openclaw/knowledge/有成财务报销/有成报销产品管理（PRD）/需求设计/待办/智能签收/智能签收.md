## 1、需求背景

#### 1.1 客户痛点

财务签收时，需要人工找到线上和线下单据的对应关系，再一一比对发票是否一致，工作量大、枯燥且易出错。

#### 1.2 售前状况

竞品有，我们处于竞劣势，会因此丢单，已有案例。另外，已有签约客户需要本功能

## 2、业务流程/架构

#### 解决方案简述：

现有的签收功能不做任何改变。

在审批流配置的签收节点增加智能签收的功能开关和规则配置。员工在费控系统上提单，涉及纸质发票时，线上（单据审批）和线下（打印单据，平铺贴票在A4纸上，寄送）各自走流程，到达财务签收节点后，财务可以使用智能签收（智能比对）功能，先通过扫码枪扫描单据的二维码快速查到单据，再通过扫描仪\+ocr扫描这些单据的纸票，系统自动比对单据中的发票与收到的纸票是否一致，给出比对结果。

##### （5）发票类型及其主键

发票在进行线上线下匹配时，需要根据发票的主键字段来匹配，只有主键完全一致的才能认定为线上线下的这两个发票文件是同一张，进而审核发票的其他字段信息是否一致。不同发票类型的主键有差异，具体如下：

注：实际场景下，电子发票不需要打印出来做智能签收，但考虑到有可能有客户会进行打印并进行智能签收，所以系统对于电子的发票仍然需要保留按照其主键字段进行匹配后比对发票其他信息的这段逻辑。

| **发票类型** | **主键字段** | **其他说明** |
|----------------|----------------|----------------|
| 增值税电子普通发票 | 发票代码\+发票号码\+开票日期\+发票总额 |  |
| 增值税电子专用发票 | 发票代码\+发票号码\+开票日期\+发票总额 |  |
| 增值税纸质普通发票 | 发票代码\+发票号码\+开票日期\+发票总额 |  |
| 增值税纸质专用发票 | 发票代码\+发票号码\+开票日期\+发票总额 |  |
| 普通发票 | 发票代码\+发票号码\+开票日期\+发票总额 | 全电发票中的普通发票查重依据为：发票代码\+发票号码\+开票日期\+发票总额 |

| 出租车票 | 发票代码\+票号\+日期\+金额 |  |
| 火车票 | 发票类型\+票号\+日期\+金额 |  |
| 定额发票 | 发票代码\+发票号码\+金额 |  |
| 汽车票 | 发票代码\+发票号码\+日期\+金额 |  |

| etc发票 | 发票代码\+发票号码\+日期\+金额 |  |

| 其他发票 | 不查重，无主键 | 智能签收匹配时，使用识别的全部字段进行匹配，全部字段比对成功则匹配成功，否则为未匹配。也就是本类型的发票智能签收只有两种状态“已匹配，比对成功”，“未匹配”。若同时签收的多个单据都有相同的发票（比如定额停车小票），则按 |

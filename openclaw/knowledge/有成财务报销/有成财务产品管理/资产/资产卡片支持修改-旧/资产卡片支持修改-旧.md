# 资产卡片支持修改-旧

| **修订时间** | **版本** | **修订说明** |
|----------------|----------|----------------|
| 20230809 | v1.0 | 新建 |

## 1、需求背景

目前资产卡片信息只允许科目关联登信息，若遇到资产价值变动，需要修改资产价值与折旧值时，无法修改，目前无法支持这种场景；

因此，将资产卡片的信息放开更多可修改的地方。

如复杂需求，放置相关流程/架构图

| 历史版本 | 留存下每个编辑版本的信息 |  |

## 4、需求详情

### 4.1、历史版本

**卡片详情中新增历史版本功能，记录每一个修改之后的卡片信息**

一、版本信息包括原始版本、修改版本以及最终版本，可以下拉筛选切换查看各个版本

1、原始版本：显示文案为“原版本”，首次录入的卡片信息，直接展示

2、修改版本：
- 每次编辑保存下来的版本，记录本次修改后的信息和修改前的信息（标题\+值），放在卡片信息上面显示；
- 版本号从第一个编辑开始记录版本号，例如版本一、版本二；

3、最终版本：显示文案为“卡片信息”，取最后一个保存的版本信息，不显示修改信息模块。

二、点击卡片查看卡片详情，默认显示“卡片信息”，若未发生过编辑修改，则不支持下拉选择

三、清理期间这种系统自动填充的数据，不算修改范围内，不生成新的版本。

### 4.2、本次可修改的字段：

在当前可修改字段的基础上，放开以下字段也可修改，当前卡片已折旧未折旧都允许修改：
- **已清理、已折旧完成的卡片不允许修改**
- 根据折旧方式，修改后对应的规则不一样，**本次只支持平均年限法的修改，当前卡片折旧方式为平均年限法时才允许修改。**

| **字段** | **规则** | **报表影响** |
|----------|----------|----------------|
| 预计使用期限（月） | 一、修改时，预计使用期限不能小于已折旧期间二、修改后，月折旧额根据新的使用期限动态变更：期初净值/（预计使用期限-已折旧期间）=月折旧额其他字段不再自动变更三、修改后，根据预计使用期限和已折旧期间判断是否需要计提：1、预计使用期限\>已折旧期间，系统继续折旧2、预计使用期限=已折旧期间,认定为已经折旧完，不再进行计提折旧，卡片变更为折旧完成状态 |  |
| 原值 | 1、修改后不影响这张卡片之前关联的所有凭证数据2、修改后，这张卡片对应的预计残值、期初累计折旧、月折旧、期初净值自动变更---这个参照新增卡片的逻辑就可3、新的账期进行计提折旧，每次都取卡片当前最新的月折旧额即可 |  |
| 税额 | 已经生成购入凭证，不能修改其他状态可修改 |  |
| 残值率 | 1、修改后不影响这张卡片之前关联的所有凭证数据2、修改后，这张卡片对应的预计残值、期初累计折旧、月折旧、期初净值自动变更---这个参照新增卡片的逻辑就可3、新的账期进行计提折旧，每次都取卡片当前最新的月折旧额即可 |  |
| 期初累计折旧/摊销 | 一、默认显示当前卡片已经计算出的累计折旧额二、这个字段修改后，期初净值和月折旧额随着这个字段进行自动变更期初净值=原值-期初累计折旧/摊销月折旧额=期初净值/（预计使用期限-已折旧期间） |  |
| 月折旧/摊销 | 一、可修改，修改后每期，都取当前卡片中最新的折旧额进行折旧二、计提折旧时的一个特殊校验：1、每一期进行折旧时，需要增加判断，月折旧额大于月初净值，计提折旧时取净值折旧2、折旧期间为最后一期，取当月卡片月初净值进行计提 |  |

当前卡片已进行计提折旧，修改了卡片信息，点击保存提示：

修改卡片可能会影响以后的计提折旧，是否确认保存？

需求现状：
当前会计科目映射-基础设置里有税额分录功能总控开关，开启后规则配置区域按单据范围\+费用类型\+发票类别从上到下匹配，命中即做价税分离。但规则对全部账套统一生效，碧瀚电商这类多账套客户无法按账套差异化配置——比如总公司要求专票价税分离，分公司不需要。本次需求在兼容现有开关和规则逻辑的基础上，仅新增「生效账套」维度。

客户预期：
给税额分录规则增加「生效账套」维度，支持按账套配置规则生效范围。同一笔发票在不同账套下各自独立匹配各自生效的规则，存量规则默认兼容无需重新配置。

|------|---------|------|

| 凭证生成 | 按账套独立匹配税额分录规则 | 同一发票不同账套各自独立匹配 |

## 三、需求详情

### 3.1 规则配置表格新增「生效账套」列

**流程：** 会计科目映射 → 基础设置 → 开启税额分录功能开关 → 规则配置表格配置规则 → 选择生效账套 → 保存

**位置：** 会计科目映射 → 基础设置 → 税额分录-可抵扣发票-价税分离生成税额分录设置开关 → 规则配置表格

### ![画布 56.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/AJdl65ADzx0vmOke/img/8fbd6805-26ea-49bc-8594-c177bb77b145.png?Expires=1779975198&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=pBO%2Fw1u4essmeFMu%2FzAH01IKldY%3D "")![画布 56.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/AJdl65ADzx0vmOke/img/c581faf5-e110-4bf7-85fe-ee7fab5b6008.png?Expires=1779975198&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=FF4ZXwdBty7eusfYBSgVwY4AC5c%3D "")

**配置项：**

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|---------|
| 生效账套 | 下拉多选 | 选项：全部账套、账套A、账套B... | 未选择等价于全部账套；存量规则默认为空 |

**功能描述：**

• 规则配置表格新增「生效账套」列，支持下拉多选
• 「全部账套」与具体账套互斥，选择「全部账套」后自动清空其他已选选项
• 未选择任何账套时，默认等价于「全部账套」
• 存量企业且已启用价税分离生成税额分录功能的：生效账套处默认为空，为空即代表全部账套生效，行为不变 

### 3.2 凭证生成按账套独立匹配

**流程：** 单据生成凭证 → 读取税额分录规则：按单据范围\+费用类型\+发票类别\+生效帐套顺序匹配 → 命中则价税分离，未命中则原有逻辑

**位置：** 凭证生成环节（同一发票在不同账套各自独立匹配）

![画布 13.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/J9LnW6jA6mLVWlvD/img/06f4aba2-55ec-4af3-ab3c-f0e574edba96.png?Expires=1779975198&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=ruHzD0H3PT%2BaOkzWVL1aNDmmwmc%3D "")

**功能描述：**

• 税额分录功能开关关闭时：所有账套按原有逻辑生成凭证（无价税分离）
• 功能开关开启时，根据税额分录生产规则生成凭证：
1. 先筛选「生效账套包含当前账套」的规则（含全部账套）
2. 再按原顺序匹配单据范围、费用类型、发票类别
3. 命中 → 价税分离生成凭证
4. 未命中 → 按原有逻辑生成凭证（无价税分离）
• 匹配维度仍为：单据范围 \+ 费用类型 \+ 发票类别，按顺序从上到下匹配，命中即停

**财务逻辑示例：**

价税分离生成凭证时：
• 借：费用（不含税金额）
• 借：进项税
• 贷：应付/银行

按原有逻辑生成凭证时：
• 借：费用（含税金额）
• 贷：应付/银行

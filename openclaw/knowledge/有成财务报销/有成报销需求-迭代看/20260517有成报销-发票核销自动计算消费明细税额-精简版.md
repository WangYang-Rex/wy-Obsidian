## 〇、现有逻辑说明（供参考）

### 0.1 发票上传自动带入费用明细（现有功能）
- **位置**：基础设置 → 自定义单据 → 表单编辑 → 发票核销组件 → 组件设置 → 属性设置
- **现有功能**：「发票上传自动带入费用明细」开关 
    - 开启后，发票核销申请单选择要核销的单据，将要核销的单据内费用明细信息带入申请单费用明细中
    - 使用合并开票时，上传一张发票为多笔费用填写核销金额，该发票的全部发票明细会分别带入到申请单的费用明细下的税率税额组件中
    - 若开启了发票查重，会触发查重
- **现有税率取值逻辑**：优先取发票识别后的税率；若发票明细没识别出税率，取税率税额组件中默认税率；支持用户手动修改税率
- **现有税额计算**：根据费用不含税金额计算
> 本次需求保留该功能作为「选项A（发票明细全部带入）」，并新增「选项B（费用核销金额带入）」。   

### 0.2 税率税额组件

核心字段：amount（含税金额）、taxRate（税率）、taxValue（税额）、deductibleTaxValue（可抵扣税额）、subInvoiceTypeName（发票类型名称）、invoiceCategory（发票类目列表）

**现有能力**：组件支持新旧版兼容落库；已有 shareTax 费用分摊税额逻辑（最后一条差额补齐）；税额已参与预算联动；公式引擎已暴露 taxAmountTotal 等字段
> 本次需求新增「选项B」时，税额计算由组件按费用核销金额独立计算，组件内部自动平衡。

## 一、需求背景

**需求现状**：
合并开票场景下，原逻辑每个费用都带入发票全部明细，导致税额计算不准确。不同企业需要不同的发票明细带入方式，原功能一刀切。

**客户预期**：
1. 提供A/B两种带入模式，企业按需选择
2. 选项B按费用核销金额独立计算税额，一张发票核销多笔费用时税额准确
3. 分次核销时每次独立计算，最终兜底确保税额合计与发票税额一致

**适用场景**：一张发票核销多笔费用

**触发时机**：发票上传完成后，系统自动触发税额计算

**计算规则（各消费明细独立计算）**：
```
不含税金额 = 实际核销金额（含税）/ (1 + 税率)
税额 = 实际核销金额（含税）- 不含税金额
```

**选项B税率取值逻辑**：

| 优先级 | 税率来源 | 触发条件 |
|---------|------------|------------|
| 1 | 发票识别税率 | 发票识别出唯一税率时 |
| 2 | 税率税额组件默认税率 | 发票有多个税率或没有识别出税率时 |
| 3 | 用户手动修改 | 支持手动修改 |
> 与选项A的区别：选项B仅在发票识别出唯一税率时才取发票税率，多税率或无税率时取默认税率。   

**税率税额组件内部平衡**：
- 组件内所有记录含税金额（amount ）之和 = 该费用实际核销金额
- 组件内所有记录税额（ taxValue ）之和 = 该费用计算出的税额
- 最后一条记录兜底 = 该费用税额 - 前面已分摊之和
- 复用现有 shareTax 的最后一条差额补齐机制

### 3.3 分次核销规则（选项B）
- 一张发票支持多次核销，每次核销独立计算税额，互不影响
- 计算基数为用户填写的本次实际核销金额（含税）
- 发票剩余可核销额度 = 发票总额 - 累计已核销金额

### 3.4 税额一致性兜底策略（选项B）

**触发时机**：发票全部核销完成后（剩余可核销额度 = 0）

**兜底规则**：
- 系统自动检测：所有消费明细已核销税额合计 vs 发票票面税额
- 若有差异，将差额自动补足到最后一条消费明细的税额中
- 兜底后确保：所有消费明细税额合计 = 发票票面税额

**兜底计算公式**：
```
税额差额 = 发票票面税额 - 所有消费明细已计算税额合计
最后一条消费明细税额 = 原计算值 + 税额差额
最后一条消费明细不含税金额 = 该明细核销金额 - 调整后税额
```



### 3.5 核销金额修改触发重算（选项B）
- 用户修改某条消费明细的核销金额后，系统提示：「核销金额已变更，将重新计算该费用税额」
- 提示后自动触发该条明细的税额重算
- 重算规则：按新核销金额和当前税率重新计算不含税金额和税额
- 仅重算当前修改的明细，其他明细保持原值不变；
- 单据提交时校验逻辑按原逻辑处理

### 3.6 手动调整税额（单条独立，仅选项B）
- 每条消费明细的税率、税额字段可独立编辑
- 修改税率 → 按新税率重算该明细不含税金额和税额
- 修改税额 → 同步调整不含税金额（不含税金额 = 核销金额 - 税额）
- 不联动其他消费明细
- 核销确认后税额锁定，不可再修改

### 3.7 配置切换规则
- 选项A↔B的切换仅影响新创建的单据
- 已存在的单据（草稿、审批中、已完成）按创建时的选项执行，不受配置切换影响
- 单据详情页展示该单据创建时使用的选项标识

| 端 | 适配说明 |
|---|------------|
| PC端 | 组件设置新增带入方式单选；核销页面按选项A/B展示 |
| 移动端 | 核销页面同步适配 |

## 四、附录

### 4.1 选项A vs 选项B 对比

| 对比项 | 选项A | 选项B |
|---------|-------|-------|
| 明细带入 | 每个费用带入发票全部明细 | 按费用核销金额独立计算 |
| 税率取值 | 优先取发票识别税率 | 唯一税率取发票税率；多税率/无税率取默认税率 |
| 税额计算 | 按费用不含税金额计算 | 按费用核销金额和税率独立计算 |
| 费用联动 | 各费用都包含全部发票明细 | 各费用独立计算，修改一条不影响其他 |
| 适用场景 | 需完整发票明细信息 | 一张发票核销多笔费用 |

### 4.2 预算与入账联动

| 联动点 | 说明 |
|---------|------|
| 预算占用 | 税额根据企业预算配置决定是否占预算 |
| 可抵扣过滤 | 通过发票类型 × 发票类目 × 费用类型三维度白名单过滤 |
| 公式引擎 | 已暴露 taxAmountTotal、含税金额总额等公式字段 |

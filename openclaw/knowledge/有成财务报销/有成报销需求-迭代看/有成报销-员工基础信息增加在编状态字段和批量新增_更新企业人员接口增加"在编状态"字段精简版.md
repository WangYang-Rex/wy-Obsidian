# 有成报销-员工基础信息增加在编状态字段和批量新增_更新企业人员接口增加"在编状态"字段精简版

### 2.1 员工基础信息增加在编状态字段

**位置**：基础设置 → 员工管理 → 员工信息 → 新增/编辑员工；个人中心（PC/MB）→ 个人信息 → 编辑

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/meonarbw3Rew0qXx/img/3ae0bb8b-7c66-4368-b997-00ac3148f2fd.png?Expires=1776917562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=LdXAfcwsBJZVZFFS1sG%2Fd3H9fMo%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/meonarbw3Rew0qXx/img/6988ee75-b386-4fc4-a2fb-ccfd48f801ce.png?Expires=1776917562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Brw9xg7rKvr4jc2MMZjCFz%2FJzUo%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/meonarbw3Rew0qXx/img/1f089c06-f6ec-4ba0-b00a-2308d9bc6842.png?Expires=1776917562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=MIgiG3wMfXiiGE8HzDX3Gr63F6Y%3D)

**字段定义**：
- **字段名**：在编状态
- **数据类型**：枚举（单选）
- **可选项**：编内、编外
- **默认值**：空（NULL）
- **必填**：否

**交互规则**：
- 下拉选择，支持清空
- 保存后即时生效
- 存量员工数据默认保持为空

### 2.2 员工信息导入导出扩展

**导入模板**：
- 新增"在编状态"列（放在最后），非必填
- 填写值必须为"编内"或"编外"，否则该行导入失败并提示错误
- 为空时保持为空，不自动填充

**导出Excel**：
- 导出文件最后一列增加"在编状态"，数据正确映射

### 2.3 公式计算支持引用在编状态

**位置**：自定义单据/自定义费用 → 表单设计器 → 文本组件 → 默认填充值 → 公式计算

**扩展范围**：
- **员工组件**（常用字段和自定义字段）：支持子字段 `【员工.姓名】`、`【员工.工号】`、`【员工.在编状态】`
- **制单人组件**：支持子字段 `【制单人.在编状态】`
- **申请人组件**：支持子字段 `【申请人.在编状态】`

**引用语法**：
- 表格内/表格外员工组件被引用：`【CONCAT（-，在编状态（文本数组/文本），剩余参数）】`
- 多选时返回文本数组，单选时返回文本

**兼容场景**：
- 自定义单据【员工组件、文本组件、表格组件】
- 自定义费用【员工组件、文本组件、表格组件】

### 2.4 OpenAPI接口扩展

**接口**：
- **【获取用户列表】**：返回数据中增加 `employmentStatus`（在编状态）字段
- **【批量新增/更新企业人员】**：请求参数中增加 `employmentStatus` 字段，支持第三方系统推送

### 2.5 端适配

|---|------------|
| PC端 | 员工管理、个人中心、表单设计器公式编辑器、OpenAPI适配 |
| 移动端 | 个人中心个人信息编辑页适配 |

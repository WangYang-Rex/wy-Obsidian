## **二、需求背景**

### **2.1 现状**
- 功能位置：邮箱绑定-发票自动生成消费（全局设置-费用设置-同一封邮件内的发票和行程单合并为一笔消费）
- 当前逻辑：系统从邮件中获取发票附件时，同一张发票可能包含pdf、ofd、xml三种格式文件。开启"同一封邮件内的发票和行程单合并为一笔消费"后，这三个格式文件的发票仅会合并到一个消费中，且仅保存一个格式的发票文件。消费详情里当前展示的是xml格式附件。
- 问题：xml格式不便于用户直接查看发票票面信息，客户期望按pdf\>ofd\>xml优先级保存发票附件。

### **2.2 目标**

在"同一封邮件内的发票和行程单合并为一笔消费"开启时，系统从邮件获取同一发票的多个格式附件后，按 pdf \> ofd \> xml 优先级选取并保存一份附件到消费中，未开启合并时保持现有逻辑不变。

## **四、功能描述**

### **4.1 邮件发票附件保存优先级优化**

位置 邮件拉取发票服务 → 附件解析与保存逻辑

变更内容
- 系统从邮件中解析同一发票的多个格式附件（pdf、ofd、xml）时，按 pdf \> ofd \> xml 优先级选取一份附件保存到消费中
- 未开启"同一封邮件内的发票和行程单合并为一笔消费"时，保持现有逻辑（三种格式分别生成三个消费，每个消费存储对应格式附件）

处理逻辑
1. 系统从邮件中拉取发票附件，识别同一发票关联的多个格式文件（pdf、ofd、xml）
2. 判断"发票和行程单合并为一笔消费"开关状态：
    - 若开关开启，则按 pdf \> ofd \> xml 优先级，将优先级最高的一份附件关联到生成的消费中
    - 若开关关闭，则保持现有逻辑，各格式附件分别生成独立消费并各自存储
3. 若同一发票仅存在一种格式附件，则直接保存该格式

边界处理

| 场景 | 处理方案 |
|-------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 同一发票仅有xml格式 | 保存xml格式 |
| 同一发票有pdf和xml，无ofd | 保存pdf格式 |
| 同一发票有ofd和xml，无pdf | 保存ofd格式 |
| 同一发票同时有pdf、ofd、xml | 保存pdf格式 |
| 存量消费 | 历史已生成的消费保留原有附件数据，不触发清理 |
| 非邮件拉取的消费 | 手动上传、OCR识别等其他来源生成的消费，保存逻辑不受影响 |

## **五、验收标准**

| 序号 | 验收项 | 验收标准 |
|-------------------------------------------------------------------------|----------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 1 | 保存优先级-pdf优先 | 同一发票包含pdf、ofd、xml三种格式时，消费中保存pdf格式附件 |
| 2 | 保存优先级-ofd优先 | 同一发票仅包含ofd和xml格式时，消费中保存ofd格式附件 |
| 3 | 保存优先级-xml兜底 | 同一发票仅包含xml格式时，消费中保存xml格式附件 |
| 4 | 合并模式限制 | 未开启"同一封邮件内的发票和行程单合并为一笔消费"时，三种格式仍分别生成三个消费，各存对应格式附件 |
| 5 | 存量消费兼容 | 历史已生成的邮件消费，附件数据不被清理，保持原有存储状态 |
| 6 | 消费详情展示 | 新拉取生成的消费，详情页展示按优先级保存的那份附件 |
| 7 | 非邮件消费不受影响 | 手动上传、OCR识别等其他来源生成的消费，保存和展示逻辑均不受影响 |

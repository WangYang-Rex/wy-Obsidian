## 一、快速概览

### 1.1 这个需求要做什么？

**一句话**：辅助核算映射新增可配置的匹配方式，支持选择「名称匹配」或「编码\+名称匹配」，存量默认名称匹配，仅辅助核算数据类别生效。

**具体效果**：
- 辅助核算映射设置弹窗新增「辅助核算匹配方式」配置项
- 两个选项：名称匹配（默认）/ 编码\+名称匹配
- 存量用户默认名称匹配，不影响现有业务
- 用户按需切换为编码\+名称匹配，解决同名不同编码的错配问题
- **限定范围**：仅单据数据类别为「辅助核算」时生效，其他类别保持原逻辑

### 1.2 谁需要这个功能？

| 角色 | 痛点 | 解决方案 |
|------|------|------------|
| 财务人员 | 辅助核算项名称重复时，系统随机匹配错误编码，导致凭证推送报错 | 切换为编码\+名称匹配，精确关联 |
| 系统管理员 | 大量同名辅助核算项需要逐个手动配置映射，维护成本高 | 未手动配置的项按"编码\+名称"自动精确匹配 |

### 1.3 核心价值
- ✅ 可配置开关，不强制升级，存量业务零影响
- ✅ 编码\+名称匹配消除同名辅助核算项的错配风险
- ✅ 手动映射完全兼容，存量配置零影响

## 二、功能需求

### 2.1 辅助核算匹配方式配置

**位置**：基础设置 → 辅助核算映射 → 设置按钮 → 设置弹窗

**配置项**：辅助核算匹配方式（单选），标题右侧带问号图标按钮

| 选项 | 说明 | 适用场景 |
|------|------|------------|
| 辅助核算名称匹配（默认） | 仅按中文名称自动匹配 | 名称不重复，或名称重复但编码一致 |
| 辅助核算编码\+辅助核算名称匹配 | 按编码和名称同时一致自动匹配 | 存在名称重复但编码不同的场景 |

**功能说明提示**：
• 问号图标悬停/点击弹出气泡：「辅助核算编码\+辅助核算名称匹配功能仅对"报销数据类别-辅助核算"生效」

**默认状态**：
• 新增/存量用户默认选中「名称匹配」
• 用户可手动切换为「编码\+名称匹配」
• 切换后立即生效

**生效范围**：
• 仅对单据数据类别为「辅助核算」的映射生效
• 其他数据类别（费用类型、部门、项目等）不受该配置影响

### 2.2 两种匹配方式规则

**方式一：名称匹配（现有逻辑，默认）**

| 规则项 | 说明 |
|---------|------|
| 匹配字段 | 仅比较名称 |
| 匹配条件 | 名称相同即视为匹配 |
| 名称重复时 | 取第一个匹配上的辅助核算项 |

**方式二：编码\+名称匹配（新增）**

**匹配优先级**：
1. 手动配置的一对一映射（最高优先级）
2. 自动匹配：编码\+名称同时一致
3. 匹配失败：不建立关联，生成凭证时报错

| 规则项 | 说明 |
|---------|------|
| 匹配字段 | 同时比较编码和名称 |
| 匹配条件 | 名称相同且编码相同才视为匹配 |
| 名称重复时 | 按编码精确匹配；编码不存在则不匹配 |
| 匹配失败时 | 不匹配，不建立关联，不回退到名称匹配 |

**匹配示例**：
```
单据端：0010105 临沂童风玩具有限公司
目标端：
  - 0010105 临沂童风玩具有限公司
  - 0010192 临沂童风玩具有限公司

名称匹配结果：⚠️ 可能匹配到 0010192
编码+名称匹配结果：✅ 精确匹配到 0010105

```



### 2.3 端适配

| 端 | 说明 |
|---|------|
| PC端 | 设置弹窗新增配置项；单据提交、凭证生成时按配置生效 |
| 移动端 | 移动端提交单据时同样按配置的匹配方式生效 |

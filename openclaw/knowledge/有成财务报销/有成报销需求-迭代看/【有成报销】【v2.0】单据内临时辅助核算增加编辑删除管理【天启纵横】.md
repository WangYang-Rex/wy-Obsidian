## 一、需求背景

### 需求现状：

当前单据内临时新建的辅助核算数据，填错后用户无法在基础档案修改，只能等审批通过后在基础档案-辅助核算成为正式脏数据，后期治理成本高。同时，系统缺少针对"临时数据"的独立权限控制，辅助核算专员被迫看到全部企业层正式数据，不清楚辅助核算里哪些辅助核算项是由员工新增审核通过添加的，也没有时间去排查清理脏数据。

另外，单据内新增辅助核算选择银行账户时，账户类型筛选条件不生效，未授权的银行账户也能被看到，存在数据安全风险。

现有“临时数据”的辅助核算的状态：

| **状态** | **含义** |
|-----------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------|
| 无状态 | 已审核通过，或从未被使用 |
| 审批中 | 使用到的单据正在审批流程中，尚未通过 |
| 已驳回 | 使用到的单据被驳回 |
| 待提交 | 使用到的单据尚未提交或已撤回 |

### 客户预期：

在基础档案-辅助核算模块增加临时数据的管理能力，支持按权限查看、编辑、删除临时数据，让脏数据在入库前就能被治理。同时修复收款账户筛选逻辑。

### **系统设置-角色与权限-辅助核算-数据权限-增加“临时数据”**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/J9LnW6jb20dK8lvD/img/8e725047-280f-48a7-9329-bdb60f050c1c.png?Expires=1781015320&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=%2BrUbFEW%2BifKfmlpMJFqMJtXnBfg%3D "")

**位置：** 系统设置→ 数据权限

**逻辑规则：**
- **所有数据：**可看到基础档案-辅助核算下的所有数据
- **临时数据：**

1.有辅助核算功能权限的用户在基础档案-辅助核算下仅可看到的数据范围：【临时未使用（本次新增）】的辅助核算数据；

2.用户不可对辅助核算类别作任何操作，不可新增辅助核算类别，不可编辑表单；

4.用户可正常使用批量操作：设置可见范围，设置标签，设置发票抬头，设置收款信息，批量删除，取消批量操作，批量停用，批量启用；

5.用户允许导入导出

6.用户在基础档案-辅助核算内新建的辅助核算项都是“临时未使用”的辅助核算；

7.存量数据在本次更新后根据辅助核算的数据权限刷新出对应数据

**位置：** 单据详情 → 辅助核算组件弹窗 → 新增辅助核算 → 收款信息选择

**切换规则/逻辑规则：**

• 应用账户类型筛选条件（全部、个人账户、对公账户、现金账户、承兑汇票、其他账户、虚拟账户）
• 仅返回同时符合权限范围和筛选条件的数据

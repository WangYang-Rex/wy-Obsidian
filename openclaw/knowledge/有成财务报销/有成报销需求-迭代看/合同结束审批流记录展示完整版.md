## 一、快速概览

### 1.1 这个需求要做什么？

**一句话**：合同被「结束」后，在PC端和MB端单据详情的审批流区展示结束操作记录，样式参照申请单「停用」。

**具体效果**：
- 应收/应付合同执行「结束合同」后，审批流最底部追加「开始 操作人 手动结束了合同」记录
- PC端和MB端同步支持
- 展示样式与申请单停用记录完全一致

### 1.2 谁需要这个功能？

| 角色 | 痛点 | 解决方案 |
|------|------|------------|
| 财务人员 | 合同结束后无法追溯是谁、何时结束的，不利于审计 | 审批流区展示结束记录，清晰可追溯 |
| 合同管理员 | 多张合同的状态变更无留痕，难以核对 | 结束操作在审批流中留痕 |

### 1.3 核心价值
- ✅ 合同结束操作可追溯，满足审计要求
- ✅ 与申请单停用保持一致的交互体验

## 二、功能需求

### 2.1 PC端合同结束审批流记录

**位置**：单据管理 → 应收合同/应付合同 → 单据详情 → 审批流程tab

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/ce562153-1190-4df3-9f6d-76479d9dd9af.png?Expires=1778767562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=uVpFLoUrelNMnGBDphLmmGflHKA%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/bd64f6b8-f1fc-4854-8b24-ee6e8170ae46.png?Expires=1778767562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=bBKgcPGKaAkrTcW%2B50D9che4enU%3D "")

**展示内容**：

| 字段 | 内容 |
|------|------|
| 节点类型 | 开始 |
| 操作人 | 姓名（如「京竹-李亮亮」） |
| 动作描述 | 手动结束了合同 |
| 操作时间 | YYYY-MM-DD HH:mm:ss |

**展示位置**：审批流时间轴最底部，所有正常审批节点之后追加。

**展示样式**：完全参照申请单「停用」记录样式（节点图标、字体、颜色、间距一致）。

### 2.2 MB端合同结束审批流记录

**位置**：MB端 → 单据详情 → 审批流程tab → 历史操作记录

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/de6eb794-2cd5-4230-8de9-b0461517b4e1.png?Expires=1778767562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=BH%2B59uCAJBupu5CehhgkDvNwMbw%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jv7QN1Jq3p/img/d3021560-6589-4122-8f51-d6567d97ddb3.png?Expires=1778767562&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Kzg%2BFFhTyeVef1s61aDzPivB1Fc%3D "")

**展示内容**：与PC端一致，时间格式简化为 MM-DD HH:mm。

**展示位置**：历史操作记录列表最底部。

**展示样式**：完全参照申请单「停用」MB端样式。

### 2.3 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 审批流程tab时间轴底部追加结束记录 |
| 移动端 | 审批流程tab历史操作记录列表底部追加 |

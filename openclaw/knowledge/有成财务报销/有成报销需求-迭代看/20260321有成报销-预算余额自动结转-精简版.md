## 〇、现有逻辑说明（供参考）

### 0.1 预算结转设置现有功能
- **位置**：预算费控 → 预算管理 → 基础设置 → 预算结转设置
- **前置条件**：需先开启「预算仅可编辑最末级维度金额」
- **现有结转方式（单选）**： 
    - 余额全额结转：手工结转上周期剩余全部余额
    - 余额比例手工结转：按比例配置结转选择周期的剩余额度
- **现有操作**：用户手动选择预算期间执行结转，每次需人工触发
- **缺失能力**：不支持自动按周期执行、无自动结转日志、无变更记录区分标识
> 本次需求在现有结转方式中新增「自动结转」选项，保留手工结转作为兜底。   

---

## 一、需求背景

**需求现状**：
已配置按比例结转规则，但仍需按周期手工触发，财务期末工作量大，容易遗忘导致下一周期预算额度不准确。

**客户预期**：
配置一次自动结转规则后，系统按预算周期（年度/季度/半年度/月度）自动执行结转，无需人工干预，失败时通知预算管理员处理。

## 三、功能需求

### 3.1 结转规则选项调整

**流程**：进入预算结转设置 → 选择结转方式（4选1）→ 设置比例和维度 → 保存即时生效

**位置**：预算费控 → 预算管理 → 基础设置 → 预算结转 → 结转规则设置

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/NpQlK5jR2mQMDqDv/img/f3a94617-4abd-4bf2-9501-9a6dd25390ee.png?Expires=1779975174&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=7KaNqip8df7ZhdzK8DjBelbxyTw%3D "")

**功能描述**：
- 结转方式从 2 个选项扩展为 4 个（自动结转和手工结转可独立配置）：

| 选项 | 说明 |
|------|------|
| 余额全额结转 | 手工结转上周期剩余全部余额（现有，不变） |
| 余额比例手工结转 | 按比例配置结转，手动触发（原「余额按比例结转」，仅改文案） |
| 余额全额自动结转 | 按周期自动将余额全部结转到下一期间（新增） |
| 余额比例自动结转 | 按周期自动按配置比例结转余额（新增） |
- 选择「比例自动结转」时，需设置结转比例（0-100%）和预算维度
- 选择「全额自动结转」时，需选择预算维度，无需设置比例
- 配置保存后即时生效，自动结转执行时间由预算周期决定，无需单独配置
- 存量「余额比例手工结转」规则保持不变，可编辑切换为自动结转

### 3.2 自动结转执行逻辑

**流程**：系统定时扫描 → 读取自动结转规则 → 遍历预算维度 → 冲突检查 → 计算结转金额 → 执行结转 → 生成日志和变更记录 → 发送通知

**位置**：后台定时任务

**功能描述**：系统按预算周期自动执行结转，支持年度/季度/半年度/月度四种周期类型。

**执行时机**：

| 预算周期 | 自动结转执行时间 |
|------------|------------------------|
| 年度 | 财务开始年度月份的次年月份的首日凌晨 |
| 半年度 | 前6个月月末次日凌晨 \+ 后6个月月末次日凌晨 |
| 季度 | 每季度末次日凌晨 |
| 月度 | 每月末次日凌晨 |

**结转金额计算**：
- 结转金额 = 当前期间剩余预算 × 结转比例
- 剩余预算 = 预算额度 - 已占用 - 已使用

**多周期混合处理**：
- 同一规则下不同周期维度按各自周期独立触发，分开记录执行结果

**边界处理**：

| 场景 | 处理方案 |
|------|------------|
| 剩余预算 = 0 | 跳过，记录「无需结转」 |
| 存在手工结转进行中（冲突） | 跳过，记录冲突，发送失败通知 |
| 目标期间预算未创建 | 跳过，记录并发送通知 |
| 系统异常执行失败 | 记录失败日志，10分钟后重试，最多3次 |
| 非执行日 | 跳过，不记录 |

### 3.4 消息通知

**流程**：自动结转执行完成 → 系统发送通知给预算管理员

**结转成功通知-钉钉消息，飞书消息**
```text
标题：预算自动结转执行成功

内容：
本次共执行【3】条结转规则，成功【3】条，失败【0】条。

结转明细：
     请前往【有成报销-预算管理-操作日志】查看
```



**结转失败告警-钉钉消息，飞书消息**
```text
标题：预算自动结转执行异常

内容：
本次自动结转执行失败，请及时处理。

失败明细：
请前往【有成报销-预算管理-操作日志】查看
```

**功能描述**：
- 接收人固定为预算方案中设置的**预算管理员**，不支持配置
- 成功通知：展示执行规则数、成功/失败条数，引导查看操作日志
- 失败告警：展示失败维度及原因（如「存在进行中的手工结转任务」「目标期间未创建预算」）
- 通过钉钉消息、飞书消息发送

需求现状：
自定义单据「费用明细」组件的「动态可选范围」配置中，规则里「字段值」目前仅支持单选。当多个部门（如财务部、技术部、行政部）需要使用相同费用类型范围时，管理员需要为每个部门分别创建一条规则，规则数量随组合爆炸式增长，配置效率低且维护成本高。

客户预期：
将规则中「字段值」从单选改为多选，一条规则可配置多个字段值共同映射到同一组可选费用类型。6种联动组件全部支持字段值多选。存量单选规则正常兼容，编辑后可升级为多选。

|------|---------|------|
| 自定义单据-费用明细组件 | 动态可选范围规则字段值多选 | 单选下拉改为多选标签，OR关系匹配 |
| 自定义单据-费用明细组件 | 切换联动组件确认提示 | 切换时清空规则列表并提示 |

## 三、需求详情

### 3.1 【动态可选范围规则字段值多选】

**流程：** 自定义单据 → 表单编辑器 → 费用明细组件 → 可选范围设置 → 动态可选范围 → 设置规则 → 字段值多选 → 选择费用类型 → 保存

**位置：** 自定义单据 → 表单编辑器 → 费用明细组件 → 可选范围设置 → 动态可选范围 → 设置规则

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4EZlweZ280YNYqxA/img/759a59f9-0075-47cc-a028-172e4a8e7e6b.png?Expires=1779975184&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Si9Ukw3H2e7pvNLotxYnsVFGIig%3D "")

**功能描述：**

• 规则中「字段值」从单选下拉改为多选标签，支持勾选多个值
• 已选值以标签形式展示，可逐个删除
• 多选字段值之间为 **OR** 关系，单据中联动组件的值匹配任一已选字段值，该规则即生效
• 保存前校验字段值不能为空
• 一条规则内多个字段值共同映射到同一组可选费用类型

### 3.2 【可用联动组件】

**流程：** 动态可选范围设置 → 选择支持的联动组件

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/Q35O851ovVNmwl9V/img/44799854-feaf-4053-bf9a-f79e101eacc6.png?Expires=1779975184&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=vJ%2BdRY2vjCrtc%2BvlUNv%2FXuLhSJc%3D "")

**功能描述：**

• 联动组件6种，完整支持列表：

| 组件类型 | 说明 |
|------------|------|
| 部门 | 单据中的部门组件 |
| 辅助核算 | 单据中的辅助核算组件 |
| 成本中心 | 单据中的成本中心组件 |
| 项目 | 单据中的辅助核算（项目）组件 |
| 常用下拉框 | 单据中的常用下拉框组件 |
| 下拉选择 | 单据中的下拉选择组件 |

• 联动字段下拉为单选，一次规则配置只能基于一个联动组件
• 确认后规则列表清空，字段值选项更新为新联动组件的取值范围
• 取消后保持当前联动组件和规则不变

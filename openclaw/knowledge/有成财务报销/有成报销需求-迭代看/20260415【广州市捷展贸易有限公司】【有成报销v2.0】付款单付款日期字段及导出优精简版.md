## 〇、现有逻辑说明（供参考）

### 0.1 付款日期组件
- 付款单模板中的「付款日期」是**固定组件**，不可删除
- 支持修改组件名称
- 付款日期值为用户提交付款单时选择的日期

### 0.2 付款单导出逻辑
- **导出入口**：我的-单据（勾选付款单）→ 导出；单据管理-付款管理（勾选付款单）→ 导出
- **导出方式**：除「支付格式导出」外的所有导出方式
- **当前问题**：导出的 Excel 中没有「付款日期」列，付款日期数据无法随单据导出
> 本次需求：在导出 Excel 中增加「付款日期」列，不改动付款日期组件本身逻辑。   

---

## 一、需求背景

### 需求现状：

付款单模板中「付款日期」是固定组件，但从我的-单据和单据管理-付款管理导出付款单数据时（支付格式导出除外），Excel 中没有「付款日期」列，无法在导出数据中查看付款日期。

### 客户预期：

付款单导出的 Excel 中增加「付款日期」列，包含两个导出入口：我的-单据和单据管理-付款管理，支付格式导出除外。

## 三、功能需求

### 3.1 我的-单据导出增加付款日期列

**流程**：我的-单据 → 勾选付款单 → 选择导出方式（支付格式除外）→ 导出 Excel 包含「付款日期」列

**位置**：我的 → 单据 → 导出

![画布 85.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/ZWGl05mVvgerBn34/img/cca7ca93-7d16-4e81-a017-511ffbaf7bbd.png?Expires=1781015319&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=hVguKfaZa6BBnp4ZGqnB%2FFwgnpY%3D "")

**功能描述**：
- 勾选付款单使用除「支付格式导出」外的所有方式导出时，Excel 中增加「付款日期」列
- 「付款日期」列在 Excel 中的位置按**单据内组件顺序**排列，保持原逻辑不变
- 表头名称固定显示【付款日期】
- 付款日期为空时，导出单元格留空

### 3.2 单据管理-付款管理导出增加付款日期列

**流程**：单据管理 → 付款管理 → 勾选付款单 → 选择导出方式（支付格式除外）→ 导出 Excel 包含「付款日期」列

**位置**：单据管理 → 付款管理 → 导出 → 导出效果见图3.1

**功能描述**：
- 勾选付款单使用除「支付格式导出」外的所有方式导出时，Excel 中增加「付款日期」列
- 列位置、表头名称规则与 3.1 一致
- 付款日期为空时，导出单元格留空

### 3.3 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 两个导出入口均支持 |
| 移动端 | 不涉及导出操作 |

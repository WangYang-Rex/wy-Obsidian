## 〇、现有逻辑说明（供参考）
> 审批流确认弹窗中每个节点的审批人/签收人/抄送人选择器是否可编辑，由后端返回的字段组合决定。本次需求不改动这些逻辑，但豁免规则依赖这些条件判断。   

### 0.1 审批/签收/支付节点（approve / signReceive / pay）

按优先级从高到低，命中任一条即锁定不可编辑：

| 优先级 | 条件 | 效果 |
|---------|------|------|
| 1 | approveUserString 有值（后端返回固定审批人文本） | disabled，展示固定文本 |
| 2 | manualSelectSwitch=true 且 userSelectType='approver' | disabled，提示「审批人为空，且不允许申请人选择」 |
| 3a | nodeSkipType='noneApproverSkip' | disabled，提示「未匹配到人员，自动跳过」 |
| 3b | nodeSkipType='noneApproverForbid' | disabled，提示「未匹配到人员，无法提交」，同时弹窗确定按钮禁用 |
| 3c | nodeSkipType='selfApproverSkip' | disabled，提示「审批人为发起人，自动跳过」 |
| 4 | type='sync'（同步节点） | disabled，展示 syncSet.showName |
| 5 | userCheckType='systemSelect' 且非会签/非抄送 | disabled，展示「N人会签/或签」文案 |
| — | 以上均未命中 → **可编辑** | userCheckType 为 submitter 或 approver&submitter，且 approveUserString 为空，且未命中跳过/同步策略 |

### 0.2 抄送节点（carbonCopy）

| 条件 | 效果 |
|------|------|
| manualSelectSwitch !== true | 整行不展示编辑控件，直接展示固定抄送人列表（disabled） |
| manualSelectSwitch=true | 可编辑，有限制：固定抄送人（carbonCopyUserId）不可删除（disabledUserIdList）；若开启 limitManualSwitch，只能在固定人员 \+ 限定范围内选择 |

### 0.3 AI审批节点（aiApproval）

不可编辑，展示审批助手名称（disabled Input）。

### 0.4 弹窗底部"抄送给"区域

| 条件 | 效果 |
|------|------|
| manualCarbonCopy=true | 可编辑，在 userRangeArray 范围内自由选择 |
| manualCarbonCopy !== true | 不可编辑 |

### 0.5 小结

审批流确认弹窗里的选项能编辑，核心取决于三个条件同时满足：
1. userCheckType='submitter'（允许申请人选人）
2. approveUserString 为空（后端没锁定审批人）
3. nodeSkipType 未命中任何跳过策略
> 本次需求**不改动**以上任何逻辑。豁免规则（3.3节）基于这些现有条件判断"是否需要申请人在弹窗中操作"。

## 一、需求背景

**需求现状**：
审批流固定的企业，员工每次提交单据都要多点一次审批流确认弹窗，但确认的内容无法修改，属于无效点击。

**客户预期**：
管理员关闭开关后，审批流不需要申请人操作的普通单据，点【提交】直接提交，不再弹确认页；需要申请人操作的审批流（如手动选审批人），不管开关状态，继续强制弹窗。

- 点击「审批流确认设置」，弹出居中模态弹窗
- 弹窗内只有一个开关配置：「提交时显示审批流确认」

### 3.2 提交时显示审批流确认开关逻辑

**流程**：管理员切换开关 → 点击【确定】保存 → 全租户即时生效

**位置**：审批流确认设置弹窗内

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/287c44f8-394a-4ca0-b1a8-f237132de1d5.png?Expires=1781015319&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=slSYn97D%2BkN3LyNtGGOvrsTT3bk%3D "")

**功能描述**：
- 开关默认**开启**（即保持现有行为），存量用户无感知
- 开启时：申请人提交单据弹出审批流确认页
- 关闭时：审批流未配置"需申请人操作"相关功能的单据，点【提交】直接提交，不弹窗
- 配置按**租户级别**存储，变更后即时生效
- 作用于所有单据类型（报销单、借款单、申请单、付款单等）
- 所有提交入口均生效：新建提交、草稿提交、驳回或撤回后重新提交
- 点击【确定】保存生效，【取消】关闭不保存

### 3.3 豁免弹窗规则

**流程**：申请人点【提交】 → 系统匹配审批流并遍历全部节点配置 → 任一节点允许申请人编辑 → 强制弹窗（不受开关控制）

**位置**：单据提交时的审批流确认弹窗

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/00099d8c-430f-47c9-9c7e-46765b7c2c78.png?Expires=1781015319&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=gEY8FgC5JnVVhwmjFcjKu5WwN2o%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/3780505f-dc5e-4e68-8409-cfd5e1593411.png?Expires=1781015319&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Xq%2FQSuLEJe8tB2OrCG2WQuXLuBI%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/J9LnW6jMYDwxVlvD/img/fd49b2a6-70d3-4dd0-a612-5f223a1ab7fb.png?Expires=1781015319&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=JyhM9gESTHiLZOK6xbeZ23M6%2Fd0%3D "")

**功能描述**：
- 审批流中只要存在一个节点需要申请人在确认弹窗中操作，则无论开关状态如何，弹窗必须弹出
- 具体豁免场景（以下任一条件成立即强制弹窗）：

| 豁免场景 |
|------------|
| 审批/签收/支付节点 - 申请人可编辑 |
| 抄送节点 - 申请人可编辑 |
| 审批流确认弹窗底部"抄送给"区域 - 申请人可编辑 |
- 判断逻辑：遍历单据对应审批流全部节点 → 任一节点命中豁免条件 → 强制弹窗 → 均未命中 → 查询开关状态决定是否弹窗

### 3.4 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 弹窗保持现有居中模态弹窗样式 |
| 移动端 | 弹窗保持现有底部抽屉样式 |

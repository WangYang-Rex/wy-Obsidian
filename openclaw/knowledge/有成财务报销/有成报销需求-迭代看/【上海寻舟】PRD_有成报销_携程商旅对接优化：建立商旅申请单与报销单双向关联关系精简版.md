## 一、需求背景

### 需求现状：

携程商旅订单生成的消费信息提报销没有自动带入携程相关的有成申请单

### 客户预期：

选择携程相关消费信息提报销，自动带入商旅申请单；创建报销单选择携程相关有成申请单，自动带入相关的携程消费信息

---

## 二、功能清单

| 模块 | 功能点 | 备注 |
|------|---------|------|
| 我的-消费 | 消费信息记录携程相关有成申请单信息 |  |
| 我的-消费-未使用
我的消费-已使用 | 增加“商旅申请”列 |  |
| 报销单-消费明细 | 增加未使用消费信息去报销带入申请单逻辑 | 申请单随消费带入报销单关联单据中 |
| 报销单-关联单据 | 增加报销单选择关联申请单带入剩余未使用消费逻辑 | 选择商旅申请单时带入剩余未使用消费 |

## 二、功能需求

### 2.1 携程消费信息记录申请单关系

**流程**：携程订单推送 → 生成未使用消费（记录有关申请单信息）

**位置**：我的-消费-未使用

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jyzR3p6q3p/img/d36f5a59-88be-44a7-9cfb-5795e853286f.png?Expires=1778767570&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=N5%2FndvEo%2BAcvH3D6qGh8WKPkBUk%3D "")

**功能描述**：
- 消费信息记录与携程相关有成申请单的关系，注意记录不是绑定申请单；

### 2.2 消费（未使用/已使用）列表增加“商旅申请”列

**位置**：我的-消费（未使用/已使用）

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KY0Xe5GqDQ/img/ee119c6d-b794-4113-a273-cd78bc7785a2.png?Expires=1778767570&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=IFi2%2Bk7h6uUjhtKARGazwl%2B8CLw%3D "")

**功能描述：**
- 未使用和已使用消费都增加“商旅申请”字段列，存放与该消费有关的商旅申请单据号
- 点击可查看申请单详情
- 申请单详情中不显示消费信息
- 新增字段不固定在右侧，放在末尾
- 仅pc端消费列表新增字段列

### 2.3 消费提报销带入申请单逻辑

**流程**：勾选未使用消费 （携程消费）→ 去报销 → 带入申请单

**位置**：报销单详情

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KY0Xe5GqDQ/img/b6a364b2-c516-476f-a632-b4ab72eb9d37.png?Expires=1778767570&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=SVV%2B0%2F4D8yLeRkJnwAiAh9i%2FeJ0%3D "")

**功能描述**：
- 勾选未使用消费信息去报销或创建报销单选择未使用消费，未使用消费信息带入报销单的消费明细中，同时申请单被带入报销单的关联单据中；
- 由于申请单被带入到报销单的关联单据中，与申请单有关系的剩余未使用消费全部被带入该报销单消费明细组件中；
- 允许把报销单中多条消费明细删除剩至一个消费去独立报销，被删除的消费信息释放回我的-消费-未使用；
- 消费使用后，我的-已使用消费的“商旅申请”包含“申请单号”
- pc端和mb端同逻辑处理

### 2.4 单据关联申请单带入商旅消费信息逻辑

**流程**：单据详情→ 关联单据 → 选择申请单（携程相关） → 带入剩余未使用携程消费

**位置**：报销单详情

**功能描述：**
- 创建报销单在关联单据组件选择携程商旅相关的有成申请单关联带入时，找出跟这个申请单有记录关系的所有未使用消费信息带入报销单的消费明细组件中，允许用户删除；
- 若将消费明细组件中消费删除至最后一条消费后，仍然可以正常报销；被删除的消费信息释放回我的-消费-未使用；
- pc端和mb端同逻辑处理

### 2.5 确认报销单或申请单互相关联单据可查

**流程**：单据详情→ 关联单据 → 查看携程商旅相关单据

**位置1**：报销单详情页 → 关联单据区域

**位置2**：商旅申请单详情页 → 关联单据区域

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJ5jyzR3p6q3p/img/d9edafae-6b4e-4c62-a801-cce0c5d469ec.png?Expires=1778767570&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Cr5hrHG2WlUFkLg0FfR2ML%2F7LOw%3D "")

![](https://static.dingtalk.com/media/lQDPD4rTfxZZvY3NAnDNBQCw6WzxttNs9N4J2ecmOjkZAA_1280_624.jpg "")

**功能描述：**
- 支持点击跳转单据详情页
- 存量携程相关的消费信息没有记录与申请单关联关系时，不受到本次需求更新的影响，不会显示在报销单的消费明细中
- pc端和mb端同效果



### 2.6 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 报销单/商旅申请单详情页的关联单据区域增加双向展示 |
| 移动端 | 报销单/商旅申请单详情页的关联单据区域增加双向展示 |

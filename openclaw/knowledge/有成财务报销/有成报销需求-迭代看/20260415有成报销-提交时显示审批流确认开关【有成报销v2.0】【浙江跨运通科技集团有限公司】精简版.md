### 2.1 审批流确认设置入口

**位置**：有成管理后台 → 基础设置 → 自定义审批流管理 → 页面右上角【更多（⋮）】下拉菜单 → 审批流确认设置

**菜单选项**：
- **支付复核设置**（已有）
- **审批流确认设置**（新增）

**交互规则**：
- 点击【更多（⋮）】按钮，下拉菜单展开
- 点击【审批流确认设置】，弹出居中模态弹窗
- 弹窗内只有一个 Toggle Switch 开关：「提交时显示审批流确认」
- 开关切换后，点击【确定】保存生效，【取消】关闭不保存

### 2.2 提交时显示审批流确认开关

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/287c44f8-394a-4ca0-b1a8-f237132de1d5.png?Expires=1776917559&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=EhqMj39LK%2FL4i45R%2BPMFbtt2TYM%3D)

**开关状态**：

| 状态 | 效果 | 默认值 |
|------|------|---------|
| 开启 | 申请人提交单据时，弹出审批流设置页供确认 | 开启 |
| 关闭 | 审批流未配置"需申请人编辑"相关功能时，直接提交不弹窗 | - |

**交互规则**：
- 配置按租户级别存储，变更后即时生效
- 作用于所有单据类型（报销单、借款单、申请单、付款单等）
- 所有提交入口均生效：新建提交、草稿提交、驳回后重新提交

### 2.3 豁免弹窗规则

**规则名称**：只要审批流节点中启用了现有产品逻辑里需要申请人在提单时配置的功能，就强制弹窗，不受开关控制。

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/00099d8c-430f-47c9-9c7e-46765b7c2c78.png?Expires=1776917559&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=jD9RwyHvAyCW5P3x%2FK3mOFpLq6k%3D "")![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/mxPOG5za6xM9QnKa/img/3780505f-dc5e-4e68-8409-cfd5e1593411.png?Expires=1776917559&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=RHU1EXMRm7VV2%2BI%2B%2FRjWy8xseLA%3D)

**具体豁免场景（指不受规则控制）**：

规则：申请人提单时弹出审批流确认弹窗中的节点中只要存在一个节点可编辑，则“提交时审批流确认”开关是否开启和关闭，弹窗必须弹出；

| 豁免场景 | 配置条件 |
|------------|------------|
| 审批节点-由申请人手动选择审批人 | `manualSelectSwitch=true` |
| 抄送节点-由申请人手动选择抄送人 | `ccManualSelectSwitch=true` |
| 出纳节点-由申请人手动选择出纳人 | `cashierManualSelectSwitch=true` |
| 节点字段权限-申请人可编辑 | 字段权限配置中申请人拥有编辑权 |
| 其他需申请人在单据提交时审批流确认弹窗可编辑的场景 | 产品逻辑中判定为"需申请人编辑"的配置 |

**处理逻辑**：
1. 申请人点击【提交】
2. 系统匹配单据对应审批流，遍历全部节点配置
3. 若任一节点启用了"需申请人操作"的功能 → 强制弹窗
4. 若均未启用 → 查询开关状态，按开关决定是否弹窗

### 2.4 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 单据详情页点击【提交】时生效，弹窗保持现有居中模态弹窗样式 |
| 移动端 | 单据详情页点击【提交】时生效，弹窗保持现有底部抽屉样式 |
| H5/小程序 | 同移动端逻辑 |

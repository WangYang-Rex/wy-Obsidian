## **1\. 需求背景**

### **需求说明**

为行程组件内的交通工具下拉选择项增加默认选择项配置能力，支持单选模式下默认选中指定选项，减少用户重复选择操作，提升商旅申请单填写效率。

### **现状痛点（含功能现状）**

**功能现状**：
- 当前功能位置：基础设置-自定义单据-商旅申请单-行程组件
- 现有字段：行程日期（出发/返回日期）、出发地/目的地、交通工具（下拉选择：不限/飞机/火车/酒店/用车等）、舱位等级、行程备注
- 现有操作：单据模板可配置行程组件显示/隐藏；交通工具字段可配置选项内容；可开启多选功能（默认单选）；员工填写时每次添加行程需手动选择交通工具
- 缺失能力：不支持配置交通工具默认选择项；行程组件-行程工具缺少默认填充规则中的「公式计算」项配置

**痛点描述**：
- 企业内部部分角色用户填写商旅申请单时频繁添加多个行程，每次添加行程交通工具选项默认为空，需手动选择"不限"
- 实际业务场景中大部分行程交通工具均选择"不限"，重复选择相同选项增加操作成本，影响填写效率

### **预期结果**

管理员可在单据模板中配置交通工具默认选择项，员工添加行程时系统自动填充默认值，减少手动操作，提升填写效率约30%。

## **3\. 功能描述**

### **3.1 交通工具默认选择项配置**

#### **功能概述**

在单据模板-行程组件-交通工具字段设置中，"默认填充规则"配置下新增"公式规则"，支持预设添加行程时的默认交通工具。用户后续添加行程时，交通工具默认显示"不限"，可后续修改。

#### **位置入口和示例图**
- **页面位置**：【基础设置-自定义单据-商旅申请单-行程组件-交通工具字段设置】
- **示例图**：

![画布 57.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45K56zv9DqDQ/img/b5b94c5a-c5a2-447d-927f-ee77d7ba9487.png?Expires=1778234827&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=PRGwMGKGDWfgabSd6RFZjKba6jM%3D "")

#### **触发条件**

财务管理员进入单据模板编辑页面，对商旅申请单的行程组件-交通工具字段进行配置时触发。

#### **核心逻辑**
```plain
IF 管理员选择"公式规则"
  THEN 展示交通工具下拉选项供选择默认值
  ELSE 保持"不预设默认值"逻辑
END IF

保存模板时：
  将默认填充规则配置写入单据模板配置表
```

|----------------------------------------------------------|----------------------------------------------------------------|
| "不限"选项生效范围 | "不限"选项仅对携程商旅申请单有效，保持原有逻辑 |

### **3.2 添加行程自动填充**

#### **功能概述**

员工填写商旅申请单时，点击"添加行程"按钮，系统自动根据模板配置填充交通工具默认选项。

#### **位置入口和示例图**

**页面位置**：商旅申请单填写页-行程组件区域

**示例图**：

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45K56zv9DqDQ/img/a9cc293b-5554-42f7-aac7-26c9172133d5.png?Expires=1778234827&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=QSz%2FUtdZKTM0ih91KDVUyjX96AY%3D "")

#### **触发条件**

员工在商旅申请单填写页面，点击"添加行程"按钮时触发。

#### **核心逻辑**
```plain
用户点击【添加行程】
    ↓
系统读取单据模板配置
    ↓
判断交通工具字段默认选择项配置
    ↓
IF 默认选择项不为空 THEN
    自动填充默认选项到交通工具字段
ELSE
    交通工具字段为空（保持现有逻辑）
END IF
    ↓
用户可手动修改已填充的选项
```



#### **相关边界（只列本需求涉及的）**

| **场景** | **处理方案** |
|----------------------------------------------------------|----------------------------------------------------------------|
| 模板变更后 | 已添加行程不刷新，仅新添加行程使用新默认项 |
| 正在填写的单据 | 已添加的行程保持原值，新行程使用模板最新配置 |
| 移动端 | 移动端同步支持默认项自动填充逻辑 |

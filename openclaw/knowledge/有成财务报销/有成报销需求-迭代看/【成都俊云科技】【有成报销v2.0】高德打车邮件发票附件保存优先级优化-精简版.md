# 邮件发票附件保存优先级优化
> 版本：v1.1 \| 产品：有成报销v2.0 \| 客户：成都俊云科技   
> 修订记录：v1.1(20260513)精简版格式重构；v1.0(20260414)初始版本   

---

需求现状：
邮箱绑定-发票自动生成消费功能中，系统从高德邮件获取发票附件时，同一张发票可能包含pdf、ofd、xml三种格式文件。当前消费详情里展示的是xml格式附件，xml格式不便于用户直接查看发票票面信息。

客户预期：
在绑定邮箱后，系统从高德邮件获取同一发票的多个格式附件后，按 pdf \> ofd \> xml 优先级选取并保存一份附件到我的-发票。

## 三、需求详情

### 3.1 【邮件发票附件保存优先级优化】

**流程：** 绑定邮箱 → 从高德邮件拉取发票 → 解析同一发票的多个格式附件  → 按优先级保存一份

**位置：** 邮件拉取发票服务 → 附件解析与保存逻辑

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1X3lE5jA1Va6BlJb/img/f05f8486-038d-4ffe-a6fb-3683fadc5afc.png?Expires=1778767571&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=SbQZ%2FYxFX4vKpKxaTZxayJp0pK0%3D "")

**功能描述：**

• 系统从邮件中解析同一发票的多个格式附件（pdf、ofd、xml）时，按 pdf \> ofd \> xml 优先级选取一份附件保存到我的-发票中
• 行程单不受优先级影响，正常保存 

**边界处理：**

| 存量消费 | 历史已获取的发票格式不变，不触发清理 |
|------------|------------------------------------------------------|

### 3.2 【消费详情附件展示】

**流程：** 单据中或消费中打开消费详情页 → 读取关联的发票附件 → pdf和ofd直接展示，xml下载

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1X3lE5jA1Va6BlJb/img/d1f60b26-3387-4bdc-a270-309af8f58740.png?Expires=1778767571&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=m1NJbjIGgcHvxLjVe4OnF7k%2F0ao%3D "")

**功能描述：**

• 消费详情页直接展示该消费下已保存的发票附件
• 存量消费详情页仍展示已保存的附件（如存量为xml则继续展示xml）
• 若消费下无附件，展示区域保持为空

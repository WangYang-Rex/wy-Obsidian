## 〇、现有逻辑说明（供参考）

### 0.1 预算方案维度超预算控制
- **预算管理 → 预算方案**：可为每个预算维度配置超预算控制规则
- **控制选项**：超出预算 X% 时 → 提示警告 / 禁止提交 / 不提示
- **生效范围**：预算方案中直接配置的维度，单据提交时按规则校验
> 本次需求不改动该逻辑，优先级高于新配置。   

### 0.2 预算申请单新增维度的现状
- **预算申请单**：员工可新增下级预算维度并配置预算额度，审批通过后生效
- **当前问题**：预算申请单新增的维度，单据占用时**固定仅预警提示**，无法禁止提交
- **导致结果**：员工可超额占用，管理人员无法精准控制
> 本次需求需突破该逻辑，为预算申请单新增维度增加可配置的超预算控制规则。

## 一、快速概览

### 1.1 这个需求要做什么？

**一句话**：在预算管理基础设置中新增配置项，统一控制预算申请单新增的预算维度超预算时的行为（提示/禁止/不提示）。

**具体效果**：
- 预算管理员可在基础设置-预算管控页面统一配置预算申请单新增维度的超预算规则（不区分年度/月度）
- 员工提交单据占用预算申请单新增维度时，按配置规则执行（可禁止提交）
- 与现有预算方案规则、浮动预算管控并存，优先级清晰

### 1.2 谁需要这个功能？

| 角色 | 痛点 | 解决方案 |
|------|------|------------|
| 预算管理员 | 预算申请单新增的维度无法禁止超额占用，管控失控 | 基础设置中统一配置新增维度的超预算规则 |
| 财务审核人员 | 员工报销单可超额占用新增维度预算，事后才发现 | 超预算时直接阻断提交，前置管控 |

### 1.3 核心价值
- ✅ 预算申请单新增维度也可精准管控，防止超额占用
- ✅ 全局统一配置，无需为每个新增维度单独设置
- ✅ 与现有预算方案规则、浮动预算管控并存，互不影响

## 二、功能需求

### 2.1 新增配置区块：预算申请单新增维度管控-手工调整（包含导入），预算申请可配置

**位置**：预算费控 → 预算管理 → 基础设置 → 预算管控（在「开启浮动预算管控」配置下方）

**界面样式**：参照现有预算方案超预算控制配置样式（见下图），但**不区分年度/月度**，仅展示一组统一规则
```
┌────────────────────────────────────────────────────────────────┐
│  预算申请单新增维度管控                                         │
│                                                                │
│  大于预算额度的  100  % 时   ○ 提示警告 ● 禁止提交  ○ 不提示    │
│                                                                │
│  大于预算额度的 [    ] % 时  ○ 提示警告  ○ 禁止提交  ○ 不提示   │
│                                                                │
└────────────────────────────────────────────────────────────────┘

```

![画布 15.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KR3PxboqDQ/img/0e9e1419-7c50-4a42-accb-0cbd849a0668.png?Expires=1778767560&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=Ov4LNZTRHEFyCynCBYVqgo0JAcM%3D "")

**配置项**（统一规则，不区分年度/月度）：

| 档位 | 说明 | 默认值 |
|------|------|---------|
| 固定档位 | 大于预算额度的 **100%** 时，执行对应处理方式 | 禁止提交 |
| 自定义档位 | 大于预算额度的 **【输入框】%** 时，执行对应处理方式 | 无（需用户填写百分比并选择处理方式） |

**每档阈值包含**：
- 百分比阈值：0-500 的整数
- 处理方式（单选）：提示警告 / 禁止提交 / 不提示

**配置规则约束**：
- 固定档位（100%）为系统预设，始终存在，作为兜底策略
- 仅允许配置 1 个自定义档位
- 自定义档位百分比范围：0-500 的整数
- 自定义档位允许为空（不填写百分比则不生效，仅固定档位起作用）

**编辑规则**：
- **固定档位（100%）**：可修改处理方式（切换单选选项，即时生效）
- **自定义档位**：可修改百分比（直接编辑输入框，实时校验：仅允许整数、0-500）；可修改处理方式（切换单选选项，即时生效）

### 2.2 规则生效逻辑

**优先级**（从高到低）：
```
1. 预算方案中为该维度单独配置的超预算规则
2. 预算维度浮动配置（如开启且该维度配置了浮动比例）
3. 基础设置-预算申请单新增维度管控规则（本次新增，fallback）
4. 默认仅预警禁止（兜底）

```

**与浮动预算管控的交互关系**：
- 浮动预算管控是在维度自身超预算规则基础上的**附加判断**（先判断固定阈值，再判断浮动阈值）
- 新增维度管控是**替换**默认的超预算处理方式，两者作用层面不同，不冲突
- 注意：如果管理员后续在预算方案中为申请单新增的维度单独配置了超标规则，则该维度不再走新增维度管控规则，优先执行方案配置

**生效范围**：
- 配置保存后，对所有预算申请单新增的维度统一适用新规则，**不区分存量/增量**
- 原因：存量维度此前无独立管控规则（走默认禁止），新配置为其提供了明确的 fallback 规则，全量生效可确保管控一致性

**校验时机**：
- 员工提交单据（报销单/付款单/预算申请单等）时，系统校验预算占用
- 若占用的是「预算申请单新增的维度」且该维度在预算方案中无单独配置、无浮动配置： 
    - 读取基础设置中的新增维度管控规则 → 执行对应档位处理方式
    - 若均无配置则默认仅预警禁止

### 2.3 批量操作说明
- **不支持批量修改**：本配置是全局 fallback 规则，不按预算方案区分，因此无需批量操作入口
- **与预算方案批量修改的关系**：预算方案中的「批量修改超标控制」仅修改维度自身的配置，不影响本全局 fallback 规则
- **优先级提醒**：批量修改后，被修改的维度将按方案规则执行，本配置不再作为该维度的 fallback

### 2.4 权限与可见性
- **可见角色**：仅预算管理员（拥有预算管理模块权限）
- **普通员工**：提交单据时仅感受校验结果（提示/禁止），不可见配置页面

### 2.5 端适配

| 端 | 适配说明 |
|---|------------|
| PC端 | 管理后台配置页面新增配置区块 |
| 移动端 | 不涉及配置，仅单据提交时按规则校验 |

### 3.1 功能验收（必测项）

| 序号 | 测试场景 | 操作步骤 | 预期结果 |

| 2 | 默认配置 | 首次打开配置区块 | 固定档位100%默认为「禁止提交」，自定义档位为空待填写 |

| 6 | 浮动预算优先 | 维度B配置了浮动比例（浮动预算管控开启），基础设置默认阈值为「100%禁止」 | 维度B优先按浮动预算规则执行 |
| 7 | fallback生效 | 预算方案中维度C无单独配置、无浮动配置，基础设置默认阈值为「100%禁止提交」 | 维度C占用达100%时禁止提交 |
| 8 | 阈值生效 | 配置100%=禁止；实际占用达130% | 执行「禁止提交」 |
| 9 | 存量维度生效 | 配置保存前已新增的维度D，基础设置为100%禁止，占用超100% | 按基础设置规则执行禁止提交（全量生效） |
| 10 | 单据提交阻断 | 员工提交报销单，占用新增维度达禁止阈值 | 提交被阻断，提示「预算已超支，禁止提交」 |
| 11 | 单据提交提示 | 员工提交报销单，占用达提示阈值但未达禁止阈值 | 弹窗提示超预算，用户可选择继续提交或取消 |
| 12 | 普通员工不可见配置 | 用普通员工账号登录，进入预算管理-基础设置 | 无「预算申请单新增维度管控」配置区块 |

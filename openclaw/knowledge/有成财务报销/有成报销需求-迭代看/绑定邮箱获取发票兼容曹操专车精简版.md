需求现状：
当前有成报销已支持绑定邮箱自动拉取发票生成消费，流程分为三阶段：邮件拉取 → OCR识别 → 消费生成。邮件拉取阶段已支持附件型邮件（滴滴、高德）和正文链接型邮件（诺诺网、阿里发票平台）。正文链接型邮件通过解析HTML获取下载链接，按PDF \> OFD \> XML优先级仅保存一种格式。行程单在合并模式下会被识别为非发票附件并跳过OCR，最终挂载到消费记录。但曹操专车（曹操出行）的发票邮件未被支持，员工使用曹操专车后需手动下载发票上传，操作繁琐且易遗漏。

客户预期：
新增对曹操专车（曹操出行）发票邮件的自动识别与处理。用户绑定邮箱后，收到曹操出行电子发票邮件，系统自动识别、下载发票PDF、OCR识别、生成消费记录，同时下载行程单作为非发票附件挂载到消费记录。无需用户手动操作，与现有滴滴/高德体验一致。

## 三、需求详情

### 3.1 【曹操专车邮件识别与解析】

**流程：** 绑定邮箱 → 收到曹操出行邮件 → 识别发件人和标题 → 判定为曹操发票邮件 → 解析HTML正文 → 提取发票PDF链接和行程单链接 → 下载文件 → 分别保存至我的-发票和附件管理

**位置：** 邮件拉取阶段

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/Yvenve510YvG1loy/img/da2afee4-1d99-40cd-9b6f-45d965519ff7.png?Expires=1779975181&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=si84XjFi5rZS6uhZ0qq3XgV%2BeCo%3D "")

**功能描述：**

• 识别规则（需同时满足）：
- 发件人包含"caocao"（不区分大小写）或"曹操出行"
- 邮件标题包含"发票"或"电子票"（复用现有标题过滤）
• HTML正文解析提取两类链接：
- 发票PDF文件链接：正文含"发票PDF文件"
- 行程单链接：正文含"所关联行程单"
• 多格式优先级处理：同一发票存在PDF \+ OFD \+ XML多种格式链接时，按PDF \> OFD \> XML优先级，仅下载优先级最高的一种，其余跳过
• 发票PDF下载后走OCR识别流程
• 行程单下载后跳过OCR，作为非发票附件处理

**曹操专车邮件特征：**

| 特征项 | 值 |
|---------|---|
| 发件人 | notify@email.caocaokeji.cn（核心识别特征：caocao） |
| 主题 | 曹操出行电子发票 |
| 发票获取方式 | 正文HTML下载链接（非附件） |
| 发票格式 | PDF \+ XML \+ OFD（3种链接，按PDF\>OFD\>XML优先级保存一种） |
| 行程单 | 单独链接（"所关联行程单"） |
| 链接有效期 | 行程单标注72小时（发票链接待确认） |

### 3.2 【行程单文件名处理】

**流程：** 下载行程单文件 → 检查文件名 → 若不含关键字则追加前缀 → 保存

**位置：** 邮件附件保存环节

**功能描述：**

• 下载行程单文件后，若原始文件名不含"行程"或"行程单"，则在文件名前追加前缀"曹操出行行程单\_"
• 若原始文件名已含"行程"或"行程单"，保持原文件名不变
• 确保行程单能被现，确保在开启“邮件内发票和行程单合并生成一笔消费”功能后能别识别出行程单身份，并和发票合并；

**处理示例：**

| 原始文件名 | 处理后文件名 | 说明 |
|---------------|------------------|------|
| 2740482198513913.pdf | 曹操出行行程单\_2740482198513913.pdf | 追加前缀，确保被识别 |
| 曹操出行行程单.pdf | 曹操出行行程单.pdf | 已含关键字，不变 |

### 3.3 【兼容同一封邮件内发票和行程单合并生成一笔消费功能】

**流程：** 开启发票自动生成消费 → 开启发票行程单合并生成消费 → 获取曹操专车邮件内发票 → 拉取发票和邮件并处理保存 → 识别曹操出行行程单 → 与发票合并生成一条消费

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/Yvenve510YvG1loy/img/da2afee4-1d99-40cd-9b6f-45d965519ff7.png?Expires=1779975181&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=si84XjFi5rZS6uhZ0qq3XgV%2BeCo%3D "")

**功能描述：**

• 行程单判定：文件名含"曹操出行行程单"或"行程单"，且关联发件人名称含"caocao"
• 行程单直接跳过OCR，不调用发票识别接口
• 未开启合并生成-单条模式：发票PDF生成一条消费记录，行程单作为附件存入附件管理
• 开启合并生成-合并模式：同一封曹操邮件的发票PDF \+ 行程单合并为一条消费记录，发票PDF为发票组件数据，行程单为附件组件数据

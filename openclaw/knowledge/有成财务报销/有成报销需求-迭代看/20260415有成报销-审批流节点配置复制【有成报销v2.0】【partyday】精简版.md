## 一、研发速览

| 项 | 内容 |
|---|------|
| **需求摘要** | 审批流设计器支持节点复制粘贴，减少重复配置操作 |
| **涉及页面** | 有成管理后台-自定义审批流设计器 |
| **数据库变更** | 否，复用现有节点配置存储结构 |
| **存量兼容** | 是，不影响已有审批流数据 |
| **关联系统** | 无 |
> **关键边界**：复制缓存仅在当前页面有效，刷新后清空。   

---

## 二、需求背景

### 2.1 现状
- **功能位置**：基础设置 → 自定义审批流
- **当前逻辑**：鼠标悬停节点显示【删除】图标按钮，点击蓝色圆形【\+】按钮弹出九宫格/网格布局的悬浮面板选择节点类型生成空白节点
- **问题**：同类型节点配置高度相似时，每次添加新节点都需从头勾选10\+个配置项，操作繁琐易遗漏

### 2.2 目标

在节点卡片右上角增加【复制】图标按钮，点击后缓存该节点配置；点击蓝色圆形【\+】按钮新增节点时，九宫格/网格布局的悬浮面板中显示"粘贴节点"格子，一键生成配置相同的节点。

## 四、功能描述

### 4.1 节点复制按钮

**位置** 审批流设计器中任意节点卡片的右上角（与【删除】按钮并列）

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/3BMqYyb4rvK3bqwZ/img/ac7fbeda-ae85-4fb7-ac54-6f8b3ee2bf9c.png?Expires=1778234820&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=V2cf%2Bt1hp1lXj%2B2Drq%2B0Al4D9X8%3D "")

**变更内容**
- 鼠标悬停节点时，右上角显示【删除】和【复制】两个操作按钮
- 点击【复制】后，系统将该节点全部配置缓存到当前页面会话中

**处理逻辑**
1. 用户将鼠标悬停在目标节点卡片上
2. 节点右上角浮现【删除】和【复制】按钮
3. 用户点击【复制】，系统读取该节点全部配置并缓存
4. 弹出"已复制'XX'的配置"轻量提示

**边界处理**

| 场景 | 处理方案 |
|------|------------|
| 刷新页面或切换审批流 | 复制缓存自动清空 |
| 复制源节点被删除 | 缓存配置保留，仍可继续粘贴；粘贴后节点独立存在 |
| 删除提示效果 | 保持原状不变，不因为该节点被复制而增加额外提示或变更删除确认弹窗 |

---

### 4.2 粘贴节点

**位置** 审批流设计器 → 点击蓝色圆形【\+】新增节点按钮 → 弹出九宫格/网格布局悬浮面板

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/3BMqYyb4rvK3bqwZ/img/092ad2bb-3aa6-4442-afbc-1ee1c5dbe6f5.png?Expires=1778234820&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=qz4yyLaJqM0jsjjV%2F3iYDRCun4I%3D "")

**变更内容**
- 存在复制缓存时，在九宫格/网格布局的悬浮面板中增加"粘贴节点"格子，并显示复制源节点名称
- 无复制缓存时，"粘贴节点"格子完全不显示
- 点击"粘贴节点"后，在当前位置插入新节点并加载复制的配置

**处理逻辑**
1. 用户点击蓝色圆形【\+】新增节点按钮
2. 系统弹出九宫格/网格布局悬浮面板，展示节点类型：审批（人像图标）、抄送（飞机图标）、签收（对勾图标）、AI审批（N\+图标）、分支条件（树状图）、出纳（¥图标）、同步（循环箭头）等
3. 系统判断是否存在复制缓存： 
    - 若存在：网格中增加"粘贴节点（复制自：XX）"格子
    - 若不存在：面板仅显示常规节点类型，无"粘贴节点"
4. 用户点击"粘贴节点"，系统生成与复制源同类型的新节点
5. 新节点完整加载复制源的配置（除节点名称外），名称自动添加"副本"后缀，同个节点复制多个，粘贴的新节点名称按副本1，副本2，副本N递增；

**边界处理**

| 场景 | 处理方案 |
|------|------------|
| 无复制缓存 | "粘贴节点"格子隐藏，面板保持现有网格样式 |
| 跨分支粘贴 | 在条件分支内点击【\+】时，若缓存来自主分支，仍可正常粘贴 |
| 重名保存 | 允许保存，不强制校验节点名称唯一性 |

**变更内容**
- 工具栏右侧增加【全屏】按钮
- 点击后隐藏顶部导航栏、二级Tab栏、左侧窄边栏，审批流画布扩展至占据整个浏览器可视区域
- 全屏状态下按钮变为【退出全屏】，点击后恢复原有布局
- 全屏/退出全屏过程中，节点配置和当前操作状态保持不变
- 刷新页面后恢复默认非全屏状态

**边界处理**

| 场景 | 处理方案 |
|------|------------|
| 全屏时刷新页面 | 恢复为默认非全屏布局 |
| 全屏时切换审批流 | 保持全屏状态或恢复默认布局均可接受，以技术实现成本最低为准 |

### 4.4 配置复制与独立编辑

**复制范围**

| 节点类型 | 可复制内容 | 不复制内容 |
|------------|---------------|---------------|
| 审批节点 | 节点基础设置、审批人设置、审批方式、操作设置、审批意见设置、超时设置、按钮权限、消息通知设置等 | 节点名称（自动加"副本"后缀） |
| 抄送节点 | 抄送人设置、抄送方式、抄送范围、消息通知设置、字段权限等 | 节点名称（自动加"副本"后缀） |
| 出纳节点 | 节点基础设置、出纳人设置、审批方式、付款设置、操作设置、按钮权限、消息通知设置等 | 节点名称（自动加"副本"后缀） |
| 发起人节点 | 发起人设置、发起条件、表单字段权限、附件设置、消息通知设置等 | 节点名称（自动加"副本"后缀） |

**编辑规则**
- 粘贴后的节点与原节点完全独立，修改互不影响
- 所有配置项均可自由编辑，无锁定限制

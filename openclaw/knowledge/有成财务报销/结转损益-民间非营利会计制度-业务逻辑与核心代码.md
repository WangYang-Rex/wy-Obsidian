# 有成财务 v2 - 结转损益业务逻辑与核心代码（民间非营利会计制度）

> 来源：李亮亮 2026-04-29 提供，yqb 后端结转损益完整梳理

## 一、整体流程

结转损益入口 → SettleController → CorpSettle4WebServiceImpl（编排）→ CorpCarryOverService（民间非营利专用分类+发生额）→ CorpSettle4WebServiceImpl（凭证组装）→ CorpCertificate4WebService（凭证保存）

## 二、民间非营利专用科目分类（carryOver3）

与一般企业准则的核心区别：民间非营利制度不使用「收入/成本/费用」的损益类科目，而是使用**「费用」和「收入」两大类**，且需区分限定性与非限定性。

### 关键分类逻辑

| 科目类别 | restriction 字段 | 归入组 |
|---|---|---|
| 费用类 | 不限 | 结转收入组（jzsr） |
| 收入类 + 非限定 | false | 结转收入组（jzsr） |
| 收入类 + 限定 | true | 结转支出组（jzzc） |

注：民间非营利中，"费用"类科目发生额结转到非限定性净资产的借方（减少），"收入"类按限定性区分结转方向。

## 三、发生额计算（dealSubjectMap3）

支持外币（按 currencyId 分组计算）和数量/单价。

## 四、凭证组装（carryOver 方法中 reportTemplate=3 的特殊处理）

民间非营利生成两种结转凭证：

| 凭证类型 | 结转科目分录 |
|---|---|
| 结转收入/费用（JZSR） | 费用类 + 非限定性收入科目 |
| 结转支出（JZZC） | 限定性收入科目 |

### 民间非营利与一般企业的关键差异

- **一般企业**：结转到「本年利润」（JZSR+JZCB→本年利润，12月→未分配利润）
- **民间非营利**：不结转本年利润，直接结转到「非限定性净资产」和「限定性净资产」

```java
// 非限定性净资产科目：settleSetting.subjectCodes[0]
// 限定性净资产科目：settleSetting.subjectCodes[1]
```

## 五、结转损益公式

### 符号定义

| 符号 | 含义 |
|---|---|
| F_i | 第 i 个费用类科目 |
| R_u | 第 j 个非限定性收入科目 |
| R_r | 第 k 个限定性收入科目 |
| NA_u | 非限定性净资产科目（settleSetting.subjectCodes[0]）|
| NA_r | 限定性净资产科目（settleSetting.subjectCodes[1]）|

### 凭证1：结转费用与非限定性收入 → 非限定性净资产

**分录：**
- 费用类科目：贷方 = 本期发生净额（冲减）
- 非限定性收入科目：借方 = 本期发生净额（冲减）
- 非限定性净资产：借贷差额

**公式：**
```
NA_u = Σ(贷方) - Σ(借方) = Σ(F_i发生额) + Σ(R_u发生额)
```

代码位置：dealSubjectMap3() 中 jzsr 分支，borrorMoney = Σ(贷方) - Σ(借方)；carryOver() 中 JZSR 分支，NA_u.loanMoney = count

### 凭证2：结转限定性收入 → 限定性净资产

**分录：**
- 限定性收入科目：借方 = 本期发生净额（冲减）
- 限定性净资产：贷方 = 限定性收入发生额合计

**公式：**
```
NA_r = Σ(R_r发生额) = Σ(贷方) - Σ(借方)
```

代码位置：dealSubjectMap3() 中 jzzc 分支，loanMoney = Σ(借方) - Σ(贷方)；carryOver() 中 JZZC 分支，NA_r.loanMoney = count.negate()

## 六、现有结转损益处理方式（旧制度）

费用科目全部当作非限定性处理，不需要区分。

```java
// carryOver3() 的逻辑：费用科目不判断 restriction，全部归入 jzsr 组
```

| 科目类型 | restriction | 结转方向 |
|---|---|---|
| 费用（5xxx） | null → 不判断 | 非限定性净资产借方 |
| 收入（4xxx） | false/空 | 非限定性净资产贷方 |
| 收入（4xxx） | true | 限定性净资产贷方 |

旧制度原因：限定性资金使用时已重分类为非限定性，所以费用天然属于非限定性，不需要标记。

## 七、新制度变化

取消了重分类，限定性资金直接消耗、费用直接冲减限定性净资产。所以**费用科目也需要知道「这笔费用是花哪类钱的」**，即需要 restriction 标记。

### 新制度结转损益公式

```
凭证1（JZSR）：
  借：非限定性收入科目（冲减）
  贷：费用类科目 - 非限定性（冲减）
  借/贷：非限定性净资产 = Σ(R_u贷方) - Σ(F_非限定借方)

凭证2（JZZC）：
  借：限定性收入科目（冲减）
  贷：费用类科目 - 限定性（冲减）
  借/贷：限定性净资产 = Σ(R_r贷方) - Σ(F_限定借方)
```

### 新旧制度差异

| 维度 | 旧制度 | 新制度 |
|---|---|---|
| 费用处理 | 全部非限定性 | 按限定性区分 |
| 限定性费用 | 绕道非限定性净资产 | 直接冲减限定性净资产 |
| 结转凭证 | 2张（JZSR+JZZC） | 2张（JZSR+JZZC），但费用分组逻辑不同 |

### 数值示例

假设：非限定性收入 +100万，限定性收入 +50万，非限定性费用 -60万，限定性费用 -20万

**旧制度结果：**
- 非限定性净资产 = 100 - 60 - 20 = +20万（限定性费用绕道）
- 限定性净资产 = 50万

**新制度结果：**
- 非限定性净资产 = 100 - 60 = +40万
- 限定性净资产 = 50 - 20 = +30万

差额 20万 = 限定性费用直接冲减限定性净资产，不再绕道非限定性。

## 八、新制度方案对比

### 方案一：费用科目加 restriction 标记（最简改动）

思路：复用现有收入科目的 restriction 字段，费用科目（5xxx）也允许设置。

| 维度 | 评价 |
|---|---|
| 改动量 | 小 |
| 精细度 | 科目级 |
| 用户操作成本 | 设置科目属性即可 |
| 存量兼容 | 需迁移 |
| 风险 | 低 |

改动点：
- AddSubjectInfoServiceImpl：放开 firstCode == 5 时也允许设置 restriction
- carryOver3()：费用分类逻辑改为跟收入一致，按 restriction 归组
- 数据库无结构变更
- 缺点：一个费用科目只能属于一种属性，实际中同一科目可能同时有限定性和非限定性支出

### 方案二：按凭证分录的辅助核算/项目自动识别

思路：科目本身不标记限定性，通过凭证分录上的辅助核算项（如"资金来源"辅助类别）自动判断。

| 维度 | 评价 |
|---|---|
| 改动量 | 大 |
| 精细度 | 分录级 |
| 用户操作成本 | 必须选资金来源辅助核算 |
| 存量兼容 | 无需迁移 |
| 风险 | 中 |

改动点：
- 结转时查凭证分录，按辅助核算维度分组
- dealSubjectMap3 需按辅助核算拆分发生额
- 结转分录需带上辅助核算
- 缺点：要求用户做凭证时必须选资金来源辅助核算，否则无法判断

### 方案三：科目标记 + 辅助核算混合（推荐）

思路：科目级别设置默认限定性属性（方案一），同时支持通过辅助核算覆盖。优先用辅助核算，没有辅助核算时回退到科目默认属性。

| 维度 | 评价 |
|---|---|
| 改动量 | 最大 |
| 精细度 | 科目级 + 分录级 |
| 用户操作成本 | 简单场景设科目，复杂场景用辅助核算 |
| 存量兼容 | 需迁移 |
| 风险 | 低（向下兼容） |

建议：如果时间紧先上**方案一**（最小可用），预留辅助核算扩展口，后续迭代到方案三。

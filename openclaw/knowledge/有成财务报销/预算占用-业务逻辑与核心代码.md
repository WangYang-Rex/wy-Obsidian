# 有成报销 - 预算占用 业务逻辑与核心代码

> 来源：李亮亮 2026-04-28 提供，yqb 后端预算占用完整梳理

## 一、预算占用生命周期

1. **提交送审** → 占用预算（status=1）
2. **审批完成** → 使用预算（status=2）
3. **驳回/撤回** → 释放占用（status=3）
4. **修改重提** → 重新占用（status=4）
5. **无审批流程** → 直接完成 → status=2

## 二、核心调用链

### 提交互审（占用预算）
提交 → BudgetOrderInfoRouteServiceImpl → BudgetOrderInfoServiceNewImpl → IBudgetNewApi（Dubbo） → BudgetConsumerNew（阻塞队列消费）

### 审批完成（使用预算）
审批最终节点通过 → 更新 status=2

### 驳回/撤回（释放占用）
驳回/撤回 → 释放占用(status=3) + 清除超标标记

## 三、核心类清单

| 类 | 职责 |

| BudgetOrderInfoRouteServiceImpl | 路由层：根据企业配置走新版/老版 |

### 预算控制表 budget_control_info

| 字段 | 说明 |
|---|---|
| budget_money | 预算总额 |
| used_money | 已使用金额（已审批通过的单据） |
| occupied_money | 占用金额（审批中的单据） |
| excessive_operation | 1=提示(弱控制), 2=禁止(强控制), 3=不提示 |

| cycle_type | 1=年, 2=半年, 3=季度, 4=月, 5=无周期 |

## 五、预算控制方式（超额策略）

| excessive_operation | 含义 | 行为 |
|---|---|---|
| 1 | 提示（弱控制） | 超标但允许提交 |
| 2 | 禁止（强控制） | 超标拦截，不可提交 |
| 3 | 不提示 | 超标无感，正常通过 |

**校验流程**：`checkBudget()` → `IBudgetNewApi.excess()` → 返回三类结果：
- `allowed(1)` → 正常
- `excessButAllowed(2)` → 超标但允许（弱控制）
- `prohibited(3)` → 超标且禁止（强控制，前端拦截提交）

## 七、审批各节点与预算操作

| 审批节点 | 预算操作 |
|---|---|
| 提交送审 | 占用 budget(status=1) |
| 审批同意（最终节点） | 使用 budget(status=2) |
| 审批驳回 | 释放占用(status=3) |
| 单据撤回 | 释放占用(status=3) + 清除超标标记 |
| 修改重提 | 重新占用(status=4) |
| 变更审批（已完成单据） | 重新占用 → 完成后再使用 |
| 无审批流程 | 直接完成 → status=2 |

## 八、并发与幂等机制

1. **阻塞队列**：`ArrayBlockingQueue(1000)` + 单消费者（BudgetConsumerNew），保证同一时刻只有一个预算请求被处理
2. **缓存锁**：`saveOrderInfoBudgetBeanCache` / `waitBudgetResponse()` 轮询检查缓存清除（最多5次，每次等2秒），预算微服务处理完清缓存，相当于分布式锁
3. **重试**：`assignTask()` 中 Retryer 最多重试10次（每次500ms）
4. **失败回滚**：预算占用失败会将单据改回待提交状态（`cancelOrderByBudget()`）

## 九、新版 vs 老版路由

企业开通 `newBudget` 模块且单据无老版预算记录 → 走新版微服务（IBudgetNewApi），否则走老版本地实现。

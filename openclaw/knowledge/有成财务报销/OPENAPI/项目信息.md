# 项目信息

## 1. 获取项目信息

- **请求地址：** `POST /reimburse/project/getProjectList.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/project/getProjectList.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/project/getProjectList.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/x-www-form-urlencoded;charset=utf-8`

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| startCreateDateTime | string | 否 | 创建起始时间（yyyy-MM-dd HH:mm:ss） |
| endCreateDateTime | string | 否 | 创建结束时间（yyyy-MM-dd HH:mm:ss） |
| projectIdList | num_array | 否 | 项目id列表 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ id | int64 | 否 | 项目id |
| └ parentId | int64 | 否 | 父项目id |
| └ projectCode | string | 否 | 项目code |
| └ projectName | string | 否 | 项目名称 |
| └ projectStatus | int32 | 否 | 项目状态 1-5（参照 statusEnum） |
| └ statusEnum | string | 否 | 状态枚举：1:启动中 2:进行中 3:验收中 4:已完成 5:已中断 |
| └ parentProjectCode | string | 是 | 父项目编码 |
| └ parentProjectName | string | 是 | 父项目名称 |
| └ created | number | 是 | 项目创建时间（时间戳） |
| └ projectType | string | 是 | 项目类型名称 |
| └ responsibility | string | 是 | 负责人名称 |

### 请求示例

无请求体参数（可通过表单传入筛选条件）。

### 响应示例

```json
{
    "apiName": "youlanda.roob",
    "result": 100,
    "message": "success",
    "data": [
        {
            "id": 664,
            "parentId": 677,
            "projectCode": "93040",
            "projectName": "youlanda.roob",
            "projectStatus": 2,
            "statusEnum": "进行中",
            "parentProjectCode": "",
            "parentProjectName": "",
            "created": 120000000000,
            "projectType": "测试项目类",
            "responsibility": "xxx"
        }
    ]
}
```

## 2. 批量删除项目单据

- **请求地址：** `POST /reimburse/project/deleteProjectOrder.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/project/deleteProjectOrder.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/project/deleteProjectOrder.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json; charset=utf-8`

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| idList | array | 是 | ID列表（number类型） |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |

### 请求示例

```json
{
    "idList": [324, 981]
}
```

### 响应示例

```json
{
    "apiName": "christinia.gottlieb",
    "result": 100,
    "message": "success",
    "data": {}
}
```

## 3. 推送项目单据

- **请求地址：** `POST /reimburse/project/v3/pushProjectOrder.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/project/v3/pushProjectOrder.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/project/v3/pushProjectOrder.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json`

> **说明：**
> 1. 新版本支持子表单推送
> 2. 单据模版及组件信息，请参考接口"报销基础数据" -> "拉取单据/费用模板"

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| submitUserId | string | 是 | 提交人ID |
| orderList | array | 是 | 项目单据推送列表 |
| └ id | string | 是 | 第三方单据唯一id |
| └ templateCode | string | 否 | 单据对应的模板code |
| └ approvalUser | string | 否 | 首个节点审批人ID |
| └ pushOrderStatus | string | 否 | 推送单据状态 1:草稿 2:审批中 3:完成 |
| └ orderComponentList | array | 否 | 单据组件列表（项目名称，编码，状态） |
|   └ code | string | 是 | 组件code |
|   └ bindingVal | string | 否 | 组件值 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |
| └ corpId | string | 否 | 企业ID |
| └ pushSize | int32 | 否 | 推送条目 |
| └ successSize | int32 | 否 | 成功条目 |
| └ created | string | 否 | 推送时间 |
| └ successList | array | 否 | 成功记录 |
|   └ id | int64 | 否 | 记录id |
|   └ corpId | string | 否 | 企业id |
|   └ orderId | string | 否 | 推送过来的单据id |
|   └ orderCode | string | 否 | 单据生成成功的orderCode |
|   └ failReason | string | 否 | 单据生成失败原因 |
|   └ status | int32 | 否 | 单据审批状态 |
|   └ created | string | 否 | 创建时间 |
| └ failList | array | 否 | 失败记录（字段同 successList） |

### 请求示例

```json
{
    "submitUserId": "98",
    "orderList": [
        {
            "id": "98",
            "templateCode": "83983",
            "approvalUser": "9go9xt",
            "pushOrderStatus": "2",
            "orderComponentList": [
                {
                    "code": "83983",
                    "bindingVal": "ahwtud"
                }
            ]
        }
    ]
}
```

### 响应示例

```json
{
    "apiName": "tu.hilpert",
    "result": 100,
    "message": "success",
    "data": {
        "corpId": "98",
        "pushSize": 10,
        "successSize": 10,
        "created": "2022-09-16 17:43:41",
        "successList": [
            {
                "id": 324,
                "corpId": "98",
                "orderId": "98",
                "orderCode": "83983",
                "status": 339,
                "created": "2022-09-16 17:43:41"
            }
        ],
        "failList": []
    }
}
```

## 4. 根据项目获取项目报表

- **请求地址：** `POST /reimburse/project/getProjectForm.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/project/getProjectForm.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/project/getProjectForm.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json`

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| beginTime | string | 否 | 获取统计数据开始时间 |
| endTime | string | 否 | 获取统计数据结束时间 |
| projectCode | string | 否 | 项目编码 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |
| └ skhtAmount | object | 否 | 应收款金额（收款合同） |
| └ yskhtAmount | object | 否 | 已收款金额（收款合同） |
| └ dskhtAmount | object | 否 | 待收款金额（收款合同） |
| └ skhtInvoiceAmount | object | 否 | 收款已开票金额（收款合同） |
| └ skhtUninvoiceAmount | object | 否 | 收款未开票金额（收款合同） |
| └ fkhtAmount | object | 否 | 应付款金额（付款合同） |
| └ yfkhtAmount | object | 否 | 已付款金额（付款合同） |
| └ dfkhtAmount | object | 否 | 待付款金额（付款合同） |
| └ fkhtInvoiceAmount | object | 否 | 付款已开票金额（付款合同） |
| └ fkhtUninvoiceAmount | object | 否 | 付款未开票金额（付款合同） |
| └ reimburseAmount | object | 否 | 报销总金额（报销单） |
| └ reimburseOverAmount | object | 否 | 报销已支付金额（报销单） |
| └ reimburseUnoverAmount | object | 否 | 报销中金额（报销单） |
| └ payAmount | object | 否 | 付款总金额（付款单） |
| └ payOverAmount | object | 否 | 付款已支付金额（付款单） |
| └ payUnoverAmount | object | 否 | 付款支付中金额（付款单） |
| └ unRelaContractpayOverAmount | object | 否 | 未关联合同的付款已支付金额 |
| └ loanAmount | object | 否 | 借款总金额（借款单） |
| └ repayAmount | object | 否 | 已还款金额（借款单） |
| └ remainAmount | object | 否 | 待还款金额（借款单） |
| └ profitAmount | object | 否 | 盈利 |
| └ received | object | 否 | 已收款 |
| └ loanRepayment | object | 否 | 借款还款（已收款） |
| └ contractRepayment | object | 否 | 合同收款（已收款） |
| └ otherRepayment | object | 否 | 其他（已收款） |

### 请求示例

```json
{
    "beginTime": "2025-08-08 10:43:35",
    "endTime": "2025-08-08 10:43:35",
    "projectCode": "25620"
}
```

### 响应示例

```json
{
    "apiName": "jimmie.powlowski",
    "result": 100,
    "message": "success",
    "data": {
        "skhtAmount": "64",
        "yskhtAmount": "477",
        "dskhtAmount": "292",
        "fkhtAmount": "991",
        "reimburseAmount": "325",
        "payAmount": "31",
        "loanAmount": "883",
        "repayAmount": "321",
        "remainAmount": "909",
        "profitAmount": "667"
    }
}
```

# 组织架构（部门）信息

## 1. 获取部门列表

- **请求地址：** `POST /reimburse/dept/getDeptList.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/dept/getDeptList.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/dept/getDeptList.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json; charset=utf-8`

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| deptName | string | 否 | 部门名称 |
| showDisabled | boolean | 否 | 是否查询禁用部门，默认 false |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ deptId | int64 | 否 | 部门ID |
| └ deptCode | string | 是 | 部门编码 |
| └ deptName | string | 否 | 部门名称 |
| └ parentId | int64 | 否 | 父级部门ID |
| └ status | number | 是 | 部门状态: 1 启用 2 禁用（禁用部门请不要使用） |
| └ manageUserInfoList | array | 是 | 部门主管列表 |
|   └ id | number | 是 | 主管记录id |
|   └ userId | string | 是 | 用户ID |
|   └ jobNumber | string | 是 | 工号 |
|   └ userName | string | 是 | 用户名称 |

### 请求示例

```json
{
    "deptName": "helaine.tremblay",
    "showDisabled": false
}
```

### 响应示例

```json
{
    "apiName": "helaine.tremblay",
    "result": 100,
    "message": "success",
    "data": [
        {
            "deptId": 749,
            "deptCode": "",
            "deptName": "helaine.tremblay",
            "parentId": 349,
            "status": 1,
            "manageUserInfoList": [
                {
                    "id": 1,
                    "userId": "xxx",
                    "jobNumber": "001",
                    "userName": "张三"
                }
            ]
        }
    ]
}
```

## 2. 批量新增/更新组织架构

- **请求地址：** `POST /reimburse/dept/mutil/save.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/dept/mutil/save.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/dept/mutil/save.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json; charset=utf-8`

> **说明：** 依据部门编号 code 进行更新，如部门编号相同，会对原有部门信息进行更新，不支持更新部门编号。

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| code | string | 是 | 部门编号 |
| name | string | 是 | 部门名称 |
| parentCode | string | 是 | 上级部门编号 |
| type | number | 是 | 部门类型 1部门 2公司 |
| managerUserNo | string | 否 | 部门主管工号 |
| status | number | 否 | 状态: 1正常 2禁用（停用后子部门同时禁用，员工转移到上级部门） |
| level | string | 否 | 部门等级（层级），根部门是层级1 |
| sort | string | 否 | 排序 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |
| └ corpId | string | 是 | 企业ID |
| └ created | string | 是 | 操作时间 |
| └ size | number | 是 | 推送部门记录总数 |
| └ successSize | number | 是 | 成功记录数 |
| └ failList | array | 是 | 失败条目信息 |
|   └ code | string | 是 | 部门编号 |
|   └ name | string | 是 | 部门名称 |
|   └ failReason | string | 是 | 失败原因 |

### 请求示例

```json
{
    "code": "1001",
    "name": "财务部",
    "parentCode": "1000",
    "type": 1,
    "managerUserNo": "001",
    "status": 1,
    "level": "2",
    "sort": "1"
}
```

### 响应示例

```json
{
    "apiName": "german.rosenbaum",
    "result": 100,
    "message": "success",
    "data": {
        "corpId": "yiqb22d4bec4c35c59aec7a789785c7fe884",
        "created": "2021-11-15T17:50:01.658",
        "size": 10,
        "successSize": 9,
        "failList": [
            {
                "code": "1001",
                "name": "财务部",
                "failReason": "部门编码已存在，请重新填写"
            }
        ]
    }
}
```

# 预算开放接口

共 7 个接口。基础路径：`https://yiqbdata.superboss.cc`（线上）/ `https://yiqbdatatest.superboss.cc`（测试）。

通用查询参数：`appKey=xxx&accessToken=xxxx`

---

## 1. 获取预算列表

**`GET /reimburse/budget/list.rjson`**

描述：获取预算列表

ContentType：`application/x-www-form-urlencoded;charset=UTF-8`

### 请求参数

无额外请求参数。

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码，1xx 表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ budgetId | string | 否 | 预算 ID |
| └ name | string | 否 | 预算名称 |
| └ children | array | 否 | 下级维度 |
| &nbsp;&nbsp;└ name | string | 否 | 维度名称 |
| &nbsp;&nbsp;└ children | array | 否 | 下级维度 |
| └ cycleRanks | array | 否 | 预算期间 |
| &nbsp;&nbsp;└ cycleRankName | string | 否 | 预算期间名称 |
| &nbsp;&nbsp;└ year | int32 | 否 | 预算期间年份 |

### 响应示例

```json
{
  "apiName": "string",
  "result": 100,
  "message": "success",
  "data": [
    {
      "budgetId": "147",
      "name": "预算名称",
      "children": [
        {
          "name": "维度名称",
          "children": []
        }
      ],
      "cycleRanks": [
        {
          "cycleRankName": "期间名称",
          "year": 2024
        }
      ]
    }
  ]
}
```

## 2. 获取预算详情

**`POST /reimburse/budget/detail.rjson`**

描述：获取预算详情

ContentType：`application/json`

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| year | int32 | 是 | 预算年份 |
| budgetName | string | 否 | 预算名称 |
| budgetId | string | 否 | 预算 ID |
| cycleRankStart | string | 否 | 预算开始期间 |
| cycleRankEnd | string | 否 | 预算结束期间 |
| ruleName | string | 否 | 维度名称 |

### 请求示例

```json
{
  "year": 2024,
  "budgetName": "年度预算",
  "budgetId": "112",
  "cycleRankStart": "202401",
  "cycleRankEnd": "202412",
  "ruleName": "部门维度"
```

| └ occupiedMoney | object | 否 | 预算预占用金额 |
| └ remainMoney | object | 否 | 预算剩余金额 |
| └ children | array | 否 | 下级维度 |

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| budgetId | int64 | 否 | 预算 ID |

## 4. 根据单号获取预算占用详情

**`POST /reimburse/ysfk/budget/queryOccupyResultByOrderCode.rjson`**

描述：根据单号获取预算占用详情

ContentType：`application/json`

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| orderCode | string | 否 | 单据编码 |

### 请求示例

```json
{
  "orderCode": "87220"
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码，1xx 表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ budgetId | int64 | 否 | 预算 ID |
| └ useDate | object | 否 | 占用预算时间 |
| └ ruleCodesAll | string | 否 | 预算维度编码路径（逗号分割，最后一个为最顶级维度编码） |
| └ extUniqueCodeMap | map | 否 | ruleCodesAll 与外部编码的映射关系 |
| └ orderAmount | object | 否 | 当前单据占用预算 |
| └ name | string | 否 | 预算名称 |
| └ budgetAmount | object | 否 | 总预算总额 |
| └ usedAmount | object | 否 | 预算单元已用金额 |
| └ residueAmount | object | 否 | 预算单元剩余金额 |
| └ remainTotalAmount | object | 否 | 总预算剩余金额 |
| └ controlType | int32 | 否 | 预算类型：1-按周期 2-按周期与年度总额 3-自定义时间 |
| └ cycleType | int32 | 否 | 周期类型：1-年 2-半年 3-季度 4-月 5-自定义 |

| └ ruleNameAll | string | 否 | 预算规则全路径名 |
| └ usedTotalAmount | object | 否 | 总预算已用金额 |
| └ usedUnitTotalAmount | object | 否 | 预算维度已用金额 |
| └ unitTotalAmount | object | 否 | 预算维度总额 |
| └ remmindUnitTotalAmount | object | 否 | 预算维度剩余金额 |
| └ parentBudgetName | string | 否 | 父预算名称 |
| └ budgetOver | boolean | 否 | 预算超标标识 |
| └ excessInfo | string | 否 | 预算超标提示文本 |
| └ parentOver | boolean | 否 | 父预算超标标识 |
| └ parentUsedUnitTotalAmount | object | 否 | 父预算单元使用总金额 |
| └ parentRemmindUnitTotalAmount | object | 否 | 父预算单元剩余总金额 |
| └ parentUsedAmount | object | 否 | 父预算维度总已用金额 |
| └ parentResidueAmount | object | 否 | 父预算维度剩余金额 |
| └ unitBudgetMoney | object | 否 | 预算单元总额 |
| └ cycleTypeUseDMoney | object | 否 | 当前期间预算已用金额 |
| └ cycleTypeUnUseMoney | object | 否 | 当前期间预算剩余金额 |
| └ cycleTypeTotalMoney | object | 否 | 当前期间预算总额 |

## 5. 获取预算列表基本信息

**`POST /reimburse/ysfk/budget/getBaseInfoListByAdmin.rjson`**

描述：用管理员身份获取预算基本信息

ContentType：`application/json`

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| userId | string | 否 | 用户 ID |
| admin | string | 否 | 为 `true` 时以管理员身份查数据 |

### 请求示例

```json
{
  "userId": "112",
  "admin": "true"
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | number | 否 | 响应码，1xx 表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ id | string | 是 | 预算 ID |
| └ name | string | 是 | 预算名称 |

### 响应示例

```json
{
  "apiName": "string",
  "result": 100,
  "message": "操作成功",
  "data": [
    {
      "id": "11",
      "name": "xxx预算"
    }
  ]
}
```

# 用户信息

## 1. 获取用户列表

- **请求地址：** `POST /reimburse/user/getUserList.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/user/getUserList.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/user/getUserList.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json; charset=utf-8`

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| jobNumber | string | 否 | 工号 |
| jobNumbers | array | 否 | 工号列表 |
| userName | string | 否 | 员工姓名 |
| ifAppAuth | string | 否 | 是否授权用户 true/false，不传查所有 |
| showDeptAllPath | number | 否 | 部门全路径 1-是，0-否 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ id | int64 | 否 | 用户ID |
| └ userAccountId | string | 否 | 用户账号id |
| └ corpId | string | 否 | 企业ID |
| └ userId | string | 否 | 用户ID |
| └ userName | string | 否 | 用户名称 |
| └ userPicUrl | string | 否 | 用户头像链接 |
| └ ifAdmin | boolean | 否 | 是否管理员 |
| └ ifBoss | boolean | 否 | 是否老板 |
| └ position | string | 否 | 职位信息 |
| └ ifLeaderInDepts | string | 否 | 是否部门领导，map格式的json |
| └ defaultDepartment | int64 | 否 | 默认部门 |
| └ department | string | 否 | 所属部门id列表 |
| └ deptList | array | 否 | 部门id列表 |
| └ enableStatus | int32 | 否 | 在职状态 |
| └ created | string | 否 | 创建时间 |
| └ modified | string | 否 | 修改时间 |
| └ ifAppAdmin | boolean | 否 | 是否系统管理员 |
| └ ifAppAuth | boolean | 否 | 是否激活应用 |
| └ ifAuthUser | boolean | 否 | 是否为授权应用用户 |
| └ jobNumber | string | 否 | 工号 |
| └ email | string | 否 | 邮件 |
| └ permanentAddr | string | 否 | 常驻地 |
| └ realUserName | string | 否 | 真实姓名 |
| └ gender | int32 | 否 | 性别 1男 2女 |
| └ idCard | string | 否 | 身份证号 |
| └ passport | string | 否 | 护照号 |
| └ accessToken | string | 否 | 支付宝电子发票员工access_token |
| └ appId | string | 否 | 对接应用id |
| └ custNo | string | 否 | 银企直连支付时所需的客户号 |
| └ operatorNo | string | 否 | 银企直连支付时所需的操作员信息 |
| └ chartConfigId | int32 | 否 | 报表id，权限用 |
| └ phone | string | 否 | 手机号码 |
| └ permanentAddrList | array | 否 | 常驻地列表 |
|   └ id | int32 | 否 | id |
|   └ title | string | 否 | 用户编号 |
| └ departmentRouter | boolean | 否 | 是否显示全路径(0:否;1:是;)默认1 |
| └ indexCreate | boolean | 否 | 是否为首页的新建单据入口 |
| └ ifActive | boolean | 否 | 是否激活 |
| └ userExt | string | 否 | 扩展 |
| └ lastLoginTime | string | 否 | 最后登录时间 |
| └ departmentAllPathList | array | 是 | 部门全路径（二维数组） |
|   └ code | string | 是 | 部门编码 |
|   └ id | number | 是 | 部门id |
|   └ name | string | 是 | 部门名称 |
|   └ parentId | number | 是 | 上级，没有或者0代表顶级 |

## 2. 批量新增/更新企业人员

- **请求地址：** `POST /reimburse/user/mutil/save.rjson`
- **测试环境：** `https://yiqbdatatest.superboss.cc/reimburse/user/mutil/save.rjson?appKey=xxx&accessToken=xxxx`
- **线上环境：** `https://yiqbdata.superboss.cc/reimburse/user/mutil/save.rjson?appKey=xxx&accessToken=xxxx`
- **ContentType：** `application/json; charset=utf-8`

> **注意：**
> 1. 更新推送依赖jobNumber 工号字段，当工号相同时，会对原有数据进行更新
> 2. 所属部门编码和部门名称不能同时为空，避免部门同名，建议使用部门编号
> 3. 常驻地，可使用市名进行匹配，避免同名地级市名相同匹配歧义，建议使用"省名，市名"匹配，多个常驻地使用｜分隔

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| userName | string | 是 | 姓名 |
| mobile | string | 是 | 手机号 |
| jobNumber | string | 是 | 工号 |
| realUserName | string | 否 | 真实姓名 |
| department | string | 是 | 部门编号，多个用逗号分隔 |
| departmentName | string | 否 | 部门名称，多个用逗号分隔 |
| defaultDepartment | string | 是 | 默认部门编号 |
| defaultDepartmentName | string | 否 | 默认部门名称 |
| email | string | 否 | 邮箱 |
| gender | number | 否 | 性别 1男 2女 |
| idCard | string | 否 | 身份证号 |
| passport | string | 否 | 护照号 |
| permanentAddr | string | 否 | 常驻地（城市名），多个用｜分隔 |
| managerUserNo | string | 否 | 直属上级工号 |
| enableStatus | number | 否 | 在职状态 0离职 1在职 默认为1 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | api 名称 |
| result | int32 | 否 | 响应码 1xx表示调用成功 |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |
| └ corpId | string | 是 | 企业ID |
| └ created | string | 是 | 创建时间 |
| └ size | number | 是 | 条目总数 |
| └ successSize | number | 是 | 成功条目数 |
| └ failList | array | 是 | 失败条目列表 |
|   └ failedReason | string | 是 | 失败信息 |
|   └ jobNumber | number | 是 | 工号 |
|   └ userName | string | 是 | 姓名 |

### 请求示例

```json
{
    "userName": "german.rosenbaum",
    "mobile": "13115736445",
    "jobNumber": "exx8pr",
    "realUserName": "张丽珊",
    "department": "1001,1002",
    "departmentName": "财务部，销售部",
    "defaultDepartment": "1001",
    "email": "asdasd@1663.com",
    "gender": 1,
    "enableStatus": 1
}
```

### 响应示例

```json
{
    "apiName": "german.rosenbaum",
    "result": 100,
    "message": "success",
    "data": {
        "corpId": "yiqb22d4bec4c35c59aec7a789785c7fe884",
        "created": "2021-11-15T19:52:39.506",
        "size": 10,
        "successSize": 9,
        "failList": [
            {
                "failedReason": "序号是[1]的数据有误:工号'exx8pr008'已重复",
                "jobNumber": 10001,
                "userName": "张三"
            }
        ]
    }
}
```

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| groupList | array | 是 | 对象集合 |
| └ groupCode | string | 是 | 组编码 |
| └ groupName | string | 是 | 组名称 |

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| result | number | 是 | 100 表示成功 |

### 请求示例

```json
{
    "groupList": [
        {
            "groupCode": "321",
            "groupName": "系统管理"
        }
    ]
}
```

### 响应示例

```json
{
    "result": 100
}
```

# 单据

## 1. 生成单据（推送）

### 请求地址

`POST /reimburse/order/createOrder.rjson`

- 测试环境: `https://yiqbdatatest.superboss.cc/reimburse/order/createOrder.rjson?appKey=xxx&accessToken=xxxx`
- 线上环境: `https://yiqbdata.superboss.cc/reimburse/order/createOrder.rjson?appKey=xxx&accessToken=xxxx`

### ContentType

`application/json; charset=utf-8`

### 说明

1. 新版本支持子表单推送
2. 单据模版及组件信息，请参考接口 "报销基础数据" -> "拉取单据/费用模板"
3. `submitterRequest` / `approvalUserRequest` / `bindingValExpand.componentValueAuxiliaryExpand`：
   - 辅助映射取值字段 和原字段取一必填，如果都填写按照原字段优先
   - 使用辅助字段 需要在 **基础设置/辅助核算映射** 菜单中配置相应的映射关系
   - `auxiliaryCategoryCode` / `auxiliaryCategoryName` 是配置中 辅助类别一栏
   - `auxiliaryItemCode` / `auxiliaryItemName` 是单条配置详情中辅助项目数据
   - 如果一个辅助项目配置了多条映射，后端会随机进行取值匹配

**2023.5.18 更新：**

1. 单据中收款信息组件支持通过银行名称和银行卡号推送收款信息
   - 组件 code 为 `collectionInfo`（收款信息）/ `payAccountInfo`（收款账户 - 收款单使用）
   - 入参例：新版本支持 `bankName` + `bankCardNum` 确定唯一收款账户
   - 单选情况下：`"bindingVal": "{\"bankName\":\"招商银行\",\"bankCardNum\":\"1234567\"}"` 或 `"bindingVal": 1001`
   - 多选情况下：`"bindingVal": "[{\"amount\":100,\"bankName\":\"中国农业银行\",\"bankCardNum\":\"1234567\"}]"` 或 `"bindingVal": "[{id: 100, amount: 1}, {id: 101, amount: 2}]"`
2. 单据如果是推送即完成（即 `pushOrderStatus` = 3）支持接收支付信息（`orderPayInfoList`）用于财务使用会计科目映射生成凭证日记账

**2023-06-01 更新 - 支持合同单推送：**

新增参数 `contractPlanInfoList`，详情见请求参数表。

**2026-04-08 更新 - 税率税额组件推送：**

消费的税率税额组件支持推送，参数结构参考如下：

```json
{
    "code": "taxRateAmount",
    "bindingValExpand": {
        "expandType": "taxRateAmount",
        "componentTaxRateAmountExpand": {
            "valueName": "税率税额信息",
            "valList": [
                {
                    "taxRate": "1%",
                    "amount": 1000,
                    "taxValue": 10,
                    "subInvoiceTypeName": "增值税电子专用发票",
                    "deductibleTaxValue": 9
                },
                {
                    "taxRate": "6%",
                    "amount": "2000.00",
                    "taxValue": "120.00",
                    "subInvoiceTypeName": "增值税电子普通发票"
                }
            ]
        }
    }
}
```

## 2. 获取推送结果

### 请求地址

`POST /reimburse/order/getPushResult.rjson`

- 测试环境: `https://yiqbdatatest.superboss.cc/reimburse/order/getPushResult.rjson?appKey=xxx&accessToken=xxxx`
- 线上环境: `https://yiqbdata.superboss.cc/reimburse/order/getPushResult.rjson?appKey=xxx&accessToken=xxxx`

### ContentType

`application/json; charset=utf-8`

### 说明

获取推送结果

### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| orderIdList | array | 是 | 第三方单据 ID 列表 |

### 请求示例

```json
{
    "orderIdList": [
        "4tk8id",
        "kvo27h"
    ]
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| apiName | string | 否 | API 名称 |
| result | int32 | 否 | 响应码，1xx 表示调用成功 |
| message | string | 否 | 响应信息 |
| data | array | 否 | 响应实体信息 |
| └ id | int64 | 否 | 记录 ID |
| └ corpId | string | 否 | 企业 ID |
| └ orderId | string | 否 | 推送过来的单据 ID |
| └ orderCode | string | 否 | 单据生成成功的 orderCode |
| └ failReason | string | 否 | 如果失败，返回失败原因 |
| └ created | string | 否 | 创建时间 |

### 响应示例

```json
{
    "apiName": "youlanda.roob",
    "result": 100,
    "message": "success",
    "data": [
        {
            "id": 38,
            "corpId": "45",
            "orderId": "45",
            "orderCode": "93040",
```

# 辅助核算

> 辅助核算类别与选项的管理接口，包含类别增改、选项增查改、列表查询及映射查询。

---

## 1. 新增辅助类别

**请求地址：**
- 测试环境：`POST` https://yiqbdatatest.superboss.cc/reimburse/auxiliary/v2/category/add.rjson?appKey=xxx&accessToken=xxxx
- 线上环境：`POST` https://yiqbdata.superboss.cc/reimburse/auxiliary/v2/category/add.rjson?appKey=xxx&accessToken=xxxx

**描述：** 新增辅助核算类别

**ContentType：** `application/json`

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| code | string | 否 | 原辅助类别编码 |
| categoryCode | string | 是 | 辅助类别编码 |
| categoryName | string | 是 | 辅助核算类别名称 |
| visibleScope | object | 否 | 可见范围配置 |
| └ isFullVisible | boolean | 否 | true=全部可见，false=部门可见 |
| └ roles | array | 否 | 角色控制列表 |
|   └ id | string | 否 | 实体ID |
|   └ name | string | 否 | 实体名称 |
| └ departments | array | 否 | 部门控制列表 |
|   └ id | string | 否 | 实体ID |
|   └ name | string | 否 | 实体名称 |
| └ users | array | 否 | 员工控制列表 |
|   └ id | string | 否 | 实体ID |
|   └ name | string | 否 | 实体名称 |
| relationAccountSwitch | boolean | 否 | 是否关联收款账户（默认false） |
| relationTitleSwitch | boolean | 否 | 是否关联发票抬头（默认false） |
| relationAuxiliarySwitch | boolean | 否 | 是否关联辅助核算（默认false） |
| relationAuxiliaryCategoryCode | string | 否 | 关联的具体辅助类别编码 |
| relationAuxiliaryCategoryCodeList | array | 否 | 关联的具体辅助类别编码列表 |

### 请求示例

```json
{
    "code": "84867",
    "categoryCode": "84867",
    "categoryName": "项目",
    "visibleScope": {
        "isFullVisible": true,
        "roles": [{ "id": "153", "name": "财务" }],
        "departments": [{ "id": "153", "name": "财务部" }],
        "users": [{ "id": "153", "name": "张三" }]
    },
    "relationAccountSwitch": true,
    "relationTitleSwitch": true,
    "relationAuxiliarySwitch": true,
    "relationAuxiliaryCategoryCode": "84867",
    "relationAuxiliaryCategoryCodeList": ["01"]
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | 接口名称 |
| result | number | 否 | 响应码（1xx表示成功） |
| message | string | 否 | 响应信息 |
| data | object | 否 | 响应实体信息 |

### 响应示例

```json
{
    "apiName": "addCategory",
    "result": 100,
    "message": "success",
    "data": {}
}
```

## 4. 查询选项

**请求地址：**
- 测试环境：`POST` https://yiqbdatatest.superboss.cc/reimburse/auxiliary/v2/item/get.rjson?appKey=xxx&accessToken=xxxx
- 线上环境：`POST` https://yiqbdata.superboss.cc/reimburse/auxiliary/v2/item/get.rjson?appKey=xxx&accessToken=xxxx

**描述：** 查询辅助核算选项

**ContentType：** `application/json`

### 请求参数（Body）

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| itemName | string | 是* | 选项名称（编码和名称二传一） |
| categoryCode | string | 是 | 辅助类别编码 |
| itemCode | string | 是* | 选项编码（编码和名称二传一） |

> *itemCode 和 itemName 至少传一个

### 请求示例

```json
{
    "itemName": "",
    "categoryCode": "84867",
    "itemCode": "84867"
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| apiName | string | 否 | 接口名称 |
| result | number | 否 | 响应码（1xx表示成功） |
| message | string | 否 | 响应信息 |
| data | array | 否 | 选项列表 |
| └ categoryId | number | 是 | 辅助类别ID |
| └ categoryName | string | 是 | 辅助类别名称 |
| └ externalId | string | 是 | 外部ID |
| └ itemCode | string | 是 | 辅助选项编码 |
| └ itemName | string | 是 | 辅助选项名称 |

### 响应示例

```json
{
    "apiName": "getItem",
    "result": 100,
    "message": "success",
    "data": [
        {
            "categoryId": 11942,
            "categoryName": "项目",
            "externalId": "",
            "itemCode": "001",
            "itemName": "项目A"
        }
    ]
}
```

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| categoryMapId | number | 否 | 辅助核算映射ID |
| content | string | 否 | 关键字 |
| pageNo | number | 否 | 页码（默认1） |
| pageSize | number | 否 | 每页大小（默认10） |

### 请求示例

```json
{
    "categoryMapId": 614,
    "content": "项目",
    "pageNo": 1,
    "pageSize": 10
}
```

### 响应参数

| 参数名 | 类型 | 必填 | 描述 |

    "result": 100,
    "message": "success",
    "data": {

## <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">在单据推送接口（Zd2qAr2w）的</span>`feeComponentList`<span style="color: rgba(0, 0, 0, 0.9);">中新增税率税额组件（taxRate/taxAmount/netAmount），支持外部系统通过OpenAPI推送完整的增值税发票税务信息，实现凭证进项税分录的自动生成。</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**需求方**</span><span style="color: rgba(0, 0, 0, 0.9);">：财务管理员、系统对接开发工程师（氚云对接客户）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**现状问题**</span><span style="color: rgba(0, 0, 0, 0.9);">：
•</span> `feeComponentList`<span style="color: rgba(0, 0, 0, 0.9);">缺少税率税额组件部分字段，无法存储税率、税额、不含税金额
•</span> <span style="color: rgba(0, 0, 0, 0.9);">氚云业务单推送至有成单据，无法将单据中消费下的税率税额等推送过去，导致</span><span style="color: rgba(0, 0, 0, 0.9);">凭证生成时缺少进项税额分录，需要人工补录发票信息
• 无法实现"推送即完成"的全流程自动化对接</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**解决思路**</span><span style="color: rgba(0, 0, 0, 0.9);">：在</span>`feeComponentList`<span style="color: rgba(0, 0, 0, 0.9);">中新增税率税额组件，与现有发票组件结构保持一致，支持字段自动计算与校验</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程**</span>


```plain
氚云/ERP系统创建业务单据（含发票信息）
    ↓
调用有成报销单据推送接口（Zd2qAr2w）
    ↓
接口解析feeComponentList中的税率税额组件
    ↓
存储费用明细（含税率税额组件值）
    ↓
单据审批流转
    ↓
凭证自动生成（推送即完成模式）
    ↓
完整凭证分录（含进项税分录）
```




## <span style="color: rgba(0, 0, 0, 0.9);">**2\. 功能现状**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能位置**</span><span style="color: rgba(0, 0, 0, 0.9);">：有成报销 OpenAPI → 单据推送接口（Zd2qAr2w）→ 请求参数 → feeList → feeComponentList</span>

- <span style="color: rgba(0, 0, 0, 0.9);">invoiceCode：发票代码</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">invoiceNo：发票号码</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">invoiceMoney：发票金额</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现有操作**</span><span style="color: rgba(0, 0, 0, 0.9);">：

• 凭证生成时使用</span>`amount`<span style="color: rgba(0, 0, 0, 0.9);">组件值生成费用科目和资金科目分录</span>

<span style="color: rgba(0, 0, 0, 0.9);">**缺失能力**</span><span style="color: rgba(0, 0, 0, 0.9);">：
•</span> `feeComponentList`<span style="color: rgba(0, 0, 0, 0.9);">中缺少税率税额组件（taxRate/taxAmount/netAmount）
• 无法通过接口推送税率信息，不能自动生成进项税分录
• 无法实现价税分离，金额口径可能不一致</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**3.2 详细功能描述**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.1 feeComponentList税率税额组件扩展**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能概述**</span> <span style="color: rgba(0, 0, 0, 0.9);">在现有</span>`feeList[].feeComponentList`<span style="color: rgba(0, 0, 0, 0.9);">数组中，新增支持税率税额组件，与现有发票组件结构保持一致。</span>

<span style="color: rgba(0, 0, 0, 0.9);">**配置入口**</span> <span style="color: rgba(0, 0, 0, 0.9);">【有成报销 OpenAPI → 单据推送接口（Zd2qAr2w）→ 请求参数 → feeList → feeComponentList】</span>

<span style="color: rgba(0, 0, 0, 0.9);">**接口请求示例**</span>
```json
{
  "orderList": [{
    "id": "CY20240317001",
    "templateCode": "FKD001",
    "pushOrderStatus": "3",
    "submitAmount": 10000,
    "feeList": [{
      "templateCode": "FY001",
      "amount": 10000,
      "feeComponentList": [
        { "code": "expenseType", "bindingVal": "CLF001" },
        { "code": "amount", "bindingVal": "10000" },
        { "code": "invoiceCode", "bindingVal": "4400121131" },
        { "code": "invoiceNo", "bindingVal": "12345678" },
        { "code": "taxRate", "bindingVal": "13" },
        { "code": "taxAmount", "bindingVal": "1150.44" },
        { "code": "netAmount", "bindingVal": "8849.56" }
      ]
    }]
  }]
}
```

#### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.2 字段自动计算与校验**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能概述**</span> <span style="color: rgba(0, 0, 0, 0.9);">根据传入的税率税额组件自动计算缺失字段，并校验金额勾稽关系。</span>

<span style="color: rgba(0, 0, 0, 0.9);">**计算与校验逻辑**</span>
```text
接收 feeComponentList 数组
    ↓
识别税率税额组件（通过code匹配）
    ↓
┌────────────────────────────────────────────────────────────┐
│ 场景1：全部组件传入（推荐）                                  │
│ 传入：amount=10000, taxRate=13, taxAmount=1150.44, netAmount=8849.56 │
│ 校验：amount = netAmount + taxAmount（误差<0.01元）          │
│ 校验：taxRate ≈ taxAmount / netAmount × 100（误差<0.01%）    │
│ 结果：全部字段有效，标记为"已校验"                           │
├────────────────────────────────────────────────────────────┤
│ 场景2：只传入税率和含税金额                                  │
│ 传入：amount=10000, taxRate=13                              │
│ 计算：netAmount = ROUND(amount / (1 + taxRate/100), 2)      │
│ 计算：taxAmount = amount - netAmount                        │
│ 结果：自动计算并存储netAmount和taxAmount                     │
├────────────────────────────────────────────────────────────┤
│ 场景3：只传入税额和不含税金额                                │
│ 传入：netAmount=8849.56, taxAmount=1150.44                  │
│ 计算：amount = netAmount + taxAmount                        │
│ 计算：taxRate = ROUND(taxAmount / netAmount × 100, 2)       │
│ 结果：自动计算并存储amount和taxRate                          │
├────────────────────────────────────────────────────────────┤
│ 场景4：存量兼容（不传税率税额组件）                          │
│ 处理：taxRate/taxAmount/netAmount为空                        │
│ 结果：只存储amount，标记为"未拆分税额"                       │
│ 兼容：原有逻辑保持不变                                       │
└────────────────────────────────────────────────────────────┘
```



<span style="color: rgba(0, 0, 0, 0.9);">**边界处理**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**触发条件**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理方案**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">金额勾稽不符</span> | <span style="color: rgba(0, 0, 0, 0.9);">amount ≠ netAmount \+ taxAmount（误差\>0.01元）</span> | <span style="color: rgba(0, 0, 0, 0.9);">返回错误码210，提示"费用金额不等于不含税金额与税额之和"</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">税率计算不符</span> | <span style="color: rgba(0, 0, 0, 0.9);">taxRate与taxAmount/netAmount计算结果不一致（误差\>0.01%）</span> | <span style="color: rgba(0, 0, 0, 0.9);">返回错误码211，提示"税率与税额/不含税金额计算结果不一致"</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">税率超出范围</span> | <span style="color: rgba(0, 0, 0, 0.9);">taxRate \< 0 或 taxRate \> 100</span> | <span style="color: rgba(0, 0, 0, 0.9);">返回错误码212，提示"税率应在0-100之间"</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">金额精度错误</span> | <span style="color: rgba(0, 0, 0, 0.9);">金额字段小数位数\>2位</span> | <span style="color: rgba(0, 0, 0, 0.9);">返回错误码213，提示"金额字段最多支持2位小数"</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">组件code未定义</span> | <span style="color: rgba(0, 0, 0, 0.9);">传入的code未在系统中配置</span> | <span style="color: rgba(0, 0, 0, 0.9);">返回错误码214，提示"税率税额组件code未在系统中定义"</span> |

<span style="color: rgba(0, 0, 0, 0.9);">**校验错误码**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**错误码**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**错误信息**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理建议**</span> |
|-------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">210</span> | <span style="color: rgba(0, 0, 0, 0.9);">费用金额不等于不含税金额与税额之和</span> | <span style="color: rgba(0, 0, 0, 0.9);">请检查传入的金额字段计算是否正确</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">211</span> | <span style="color: rgba(0, 0, 0, 0.9);">税率与税额/不含税金额计算结果不一致</span> | <span style="color: rgba(0, 0, 0, 0.9);">请检查税率或重新计算税额</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">212</span> | <span style="color: rgba(0, 0, 0, 0.9);">税率应在0-100之间</span> | <span style="color: rgba(0, 0, 0, 0.9);">请检查taxRate组件值</span> |

#### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.3 存量兼容策略**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能概述**</span> <span style="color: rgba(0, 0, 0, 0.9);">新组件为可选字段，完全向后兼容，不影响存量接口调用。</span>

<span style="color: rgba(0, 0, 0, 0.9);">**兼容策略**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理方案**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">存量接口调用（不传税率税额组件）</span> | <span style="color: rgba(0, 0, 0, 0.9);">原有逻辑保持不变，接口完全向后兼容</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">部分组件传入</span> | <span style="color: rgba(0, 0, 0, 0.9);">系统自动计算缺失字段，按"传入优先"原则存储</span> |

<span style="color: rgba(0, 0, 0, 0.9);">**场景：氚云采购付款单推送**</span>

<span style="color: rgba(0, 0, 0, 0.9);">配置步骤（氚云系统）：</span>

2. <span style="color: rgba(0, 0, 0, 0.9);">配置有成报销单据推送接口调用</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">在feeComponentList中构造税率税额组件</span>

<span style="color: rgba(0, 0, 0, 0.9);">使用步骤（日常推送）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">采购人员在氚云录入采购付款单，关联增值税专用发票</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">发票信息：金额10,000元，税率13%，税额1,150.44元，不含税金额8,849.56元</span>

4. <span style="color: rgba(0, 0, 0, 0.9);">有成报销自动校验金额勾稽关系，存储费用明细</span>
5. <span style="color: rgba(0, 0, 0, 0.9);">财务审核通过后，系统自动生成完整凭证（含进项税分录）</span>

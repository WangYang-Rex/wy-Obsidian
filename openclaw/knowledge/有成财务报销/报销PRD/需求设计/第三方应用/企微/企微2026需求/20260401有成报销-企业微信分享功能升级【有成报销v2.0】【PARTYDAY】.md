### <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">企业微信版本有成报销，移动端与PC端单据详情均支持通过企业微信官方分享接口分享至会话，被分享人点击后实时判断权限，根据权限展示查看或审批操作。</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**需求方：**</span> <span style="color: rgba(0, 0, 0, 0.9);">报销申请人/审批人</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**现状问题：**</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">移动端（企微H5）有分享功能，但通过缓存判断权限，非实时</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">PC端（企微工作台PC版）无分享功能</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**解决思路：**</span> <span style="color: rgba(0, 0, 0, 0.9);">统一移动端与PC端分享入口，走企业微信官方分享接口，被分享人点击时实时判断权限</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程：**</span>
```plain
单据详情页点击分享
  ↓
调用企业微信分享接口 → 选择会话/联系人
  ↓
被分享人收到卡片消息
  ↓
点击进入单据详情
  ↓
实时判断权限
  ├─ 无权限 → 提示"无权查看"
  ├─ 仅查看 → 展示单据详情
  └─ 可审批 → 展示详情 + 审批操作
```



### <span style="color: rgba(0, 0, 0, 0.9);">**2\. 功能现状**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能所在页面路径：**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">移动端：企微H5 → 单据详情</span>
- <span style="color: rgba(0, 0, 0, 0.9);">PC端：企微工作台 → 有成报销 → 单据详情（无分享）</span>

<span style="color: rgba(0, 0, 0, 0.9);">**当前能力：**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">移动端有分享按钮，分享链接至企微会话</span>
- <span style="color: rgba(0, 0, 0, 0.9);">缓存权限判断</span>

<span style="color: rgba(0, 0, 0, 0.9);">**当前缺失的能力：**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">PC端无分享功能</span>
- <span style="color: rgba(0, 0, 0, 0.9);">未使用企业微信官方分享接口</span>
- <span style="color: rgba(0, 0, 0, 0.9);">权限判断非实时</span>

##### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.1 分享入口统一**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**移动端（H5）**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**PC端（工作台）**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">位置：单据详情页右上角\[分享\]按钮</span>
- <span style="color: rgba(0, 0, 0, 0.9);">交互：点击唤起企业微信分享弹窗</span>

<span style="color: rgba(0, 0, 0, 0.9);">**分享内容（卡片消息）：**</span>
```plain

│                         │
│ 【差旅报销单】          │

│ 金额：¥3,500.00         │
```

<span style="color: rgba(0, 0, 0, 0.9);">**接口调用：**</span> <span style="color: rgba(0, 0, 0, 0.9);">使用企业微信JS-SDK分享接口</span>

- <span style="color: rgba(0, 0, 0, 0.9);">imgUrl：有成报销logo</span>

##### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.3 实时权限判断**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**判断时机：**</span> <span style="color: rgba(0, 0, 0, 0.9);">被分享人点击链接进入页面时实时查询</span>

<span style="color: rgba(0, 0, 0, 0.9);">**权限判定规则：**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**权限级别**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**判定条件**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**页面展示**</span> |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">无权限</span> | <span style="color: rgba(0, 0, 0, 0.9);">无单据查看权限</span> | <span style="color: rgba(0, 0, 0, 0.9);">"无权查看该单据"提示页</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">仅查看</span> | <span style="color: rgba(0, 0, 0, 0.9);">有查看权限但非当前审批人</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据详情（无操作按钮）</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">可审批</span> | <span style="color: rgba(0, 0, 0, 0.9);">当前审批节点处理人</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据详情 \+ 审批操作区</span> |

<span style="color: rgba(0, 0, 0, 0.9);">**判定逻辑：**</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">校验用户是否有该单据查看权限（按数据权限规则）</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">校验用户是否为当前审批节点处理人</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**边界场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**触发条件**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理方案**</span> |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">审批完成</span> | <span style="color: rgba(0, 0, 0, 0.9);">分享后单据审批完成</span> | <span style="color: rgba(0, 0, 0, 0.9);">被分享人权限满足时可查看，仅展示无审批操作</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">单据撤回</span> | <span style="color: rgba(0, 0, 0, 0.9);">分享后申请人撤回单据</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据回到申请人草稿箱，能否查看按产品原逻辑处理</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">单据驳回</span> | <span style="color: rgba(0, 0, 0, 0.9);">分享后被驳回至申请人</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据回到申请人草稿箱，能否查看按产品原逻辑处理</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">分享人离职</span> | <span style="color: rgba(0, 0, 0, 0.9);">分享人已不在企业</span> | <span style="color: rgba(0, 0, 0, 0.9);">被分享人仍可正常查看，按被分享人权限判断</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">被分享人权限变更</span> | <span style="color: rgba(0, 0, 0, 0.9);">查看时权限已失效</span> | <span style="color: rgba(0, 0, 0, 0.9);">实时判断，无权限时提示"无权查看"</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">链接过期</span> | <span style="color: rgba(0, 0, 0, 0.9);">分享链接超过有效期</span> | <span style="color: rgba(0, 0, 0, 0.9);">走产品原逻辑</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">并发审批</span> | <span style="color: rgba(0, 0, 0, 0.9);">被分享人查看时节点已处理</span> | <span style="color: rgba(0, 0, 0, 0.9);">提示"当前节点已处理"，刷新后按最新状态展示</span> |

### <span style="color: rgba(0, 0, 0, 0.9);">**4\. 前后端工作任务**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**4.1 前端负责内容**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">【移动端】改造分享按钮，调用企业微信JS-SDK分享接口</span>
- <span style="color: rgba(0, 0, 0, 0.9);">【PC端】新增分享按钮，调用企业微信PC端分享接口</span>
- <span style="color: rgba(0, 0, 0, 0.9);">【单据详情页】进入页面时调用实时权限查询接口，根据权限渲染页面</span>
```plain
技术实现方案
1. 分享到企业微信会话
使用企业微信 JS-SDK 的 onMenuShareAppMessage 接口：

wx.onMenuShareAppMessage({
  title: '报销单详情',        // 分享标题
  desc: '单号：EXP20260402001', // 分享描述
  link: 'https://your-app.com/share/expense/123?corpId=wwxxx', // 分享链接
  imgUrl: 'https://your-app.com/logo.png', // 分享图标
  success: function() {
    // 用户确认分享后回调
  },
  cancel: function() {
    // 用户取消分享后回调
  }
});
关键点：

link 参数必须携带 corpId（企业ID），用于企业微信识别企业
建议在链接中携带业务参数（如单据ID）但不携带敏感信息
2. 点击链接打开对应页面
用户点击分享卡片后，需要进行免登授权：

// 1. 引入企业微信 JS-SDK
<script src="https://res.wx.qq.com/open/js/jweixin-1.2.0.js"></script>

// 2. 使用 agentConfig 进行免登
wx.agentConfig({
  corpid: 'wwxxxx', // 企业ID
  agentid: '1000001', // 应用ID
  timestamp: res.timestamp,
  nonceStr: res.nonceStr,
  signature: res.signature,
  jsApiList: ['onMenuShareAppMessage']
});

// 3. 获取当前用户身份（用于权限校验）
wx.getCorpAppContactInfo({
  success: function(res) {
    // res.userId 就是用户的企微账号ID
    // 用这个ID去校验权限，返回对应的页面内容
  }
});
```

### <span style="color: rgba(0, 0, 0, 0.9);">**6\. 附录**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**6.1 典型使用场景**</span>

场景1：PC端分享催审批

申请人在PC端打开单据详情，点击分享发送给审批人，审批人收到卡片消息，点击进入直接审批。

场景2：审批完成分享确认

审批完成后，申请人分享单据给财务，财务查看确认后做后续入账处理。

场景3：跨部门协作文档

业务人员分享报销单给项目负责人，项目负责人查看消费明细，确认项目成本归属。

### <span style="color: rgba(0, 0, 0, 0.9);">**1.3.2 现有打印逻辑现状**</span>

<span style="color: rgba(0, 0, 0, 0.9);">当前系统存在两条打印路径：</span>

<span style="color: rgba(0, 0, 0, 0.9);">**路径A：直接预览打印**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">触发条件：仅勾选【单据】\+ 选中数量≤50张（见20260306需求）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">执行方式：浏览器直接调起打印预览窗口</span>
- <span style="color: rgba(0, 0, 0, 0.9);">适用场景：纯面单打印，无附件</span>

<span style="color: rgba(0, 0, 0, 0.9);">**路径B：下载中心导出**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">触发条件：不满足路径A的所有情况</span>
- <span style="color: rgba(0, 0, 0, 0.9);">执行方式：生成PDF/ZIP文件，用户去下载中心领取</span>
- <span style="color: rgba(0, 0, 0, 0.9);">适用场景：含附件打印、或数量\>50张</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现存问题**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">当用户勾选了附件（如图片、发票等），但附件总大小并不大（如只有几张图片），仍被强制走下载中心导出，操作链路长</span>
- <span style="color: rgba(0, 0, 0, 0.9);">财务反馈：小额附件场景下，直接打印体验更流畅</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**1.4 核心痛点**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**痛点1：附件大小不敏感，一律走导出流程**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户勾选了【单据】\+【图片】，共3张单据，附件图片总计5MB</span>
- <span style="color: rgba(0, 0, 0, 0.9);">当前逻辑：因为勾选了【图片】（非纯面单），强制跳转下载中心导出</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户需等待异步生成 → 去下载中心 → 下载文件 → 本地打开打印</span>

<span style="color: rgba(0, 0, 0, 0.9);">**影响**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">操作链路长（4步 vs 1步）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">小额附件场景用户体验差</span>
- <span style="color: rgba(0, 0, 0, 0.9);">下载中心任务堆积（本可直接打印的小任务占用队列资源）</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**2.2 附件总大小计算逻辑**</span>


```plain
附件总大小计算流程
    ↓
获取【打印设置】中勾选的附件类型集合
    ↓
遍历选中单据列表
    ↓
对每个单据，获取其勾选类型的附件文件
    ↓
累加文件大小（单位：字节 → 转换为MB）
    ↓
IF 总大小 ≤ 150MB THEN
    允许直接打印
ELSE
    提示用户去下载中心导出
END IF
```



<span style="color: rgba(0, 0, 0, 0.9);">**计算公式**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>


```plain
附件总大小(MB) = Σ(单据i的勾选附件类型文件大小) / 1024 / 1024

其中：
- 单据i ∈ 选中单据集合
- 勾选附件类型 ∈ {图片, 纸质发票, 电子发票, 关联单据, 附件电子回单}
- 仅计算实际存在的文件大小（空文件/已删除文件按0计算）
```

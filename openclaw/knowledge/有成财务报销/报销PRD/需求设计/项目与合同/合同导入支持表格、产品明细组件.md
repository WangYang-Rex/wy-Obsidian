# **1\. 需求背景**

用户需要把原来的线下合同导入到有成中进行统一管理，原合同中记载了产品明细。现在我们的导入功能只支持合同里的收付款计划导入，一行代表一份合同，不支持子表格、产品明细的导入。

# **4\. 需求详情**

![image.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/YdgOk2bAvNpdLq4B/img/9e079a96-ee34-4f55-981f-9fc253afadb5.png?Expires=1774157535&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=0lNbhQhC3G%2BoL21mfVsjPbPnHkI%3D "")
1. 合同导入模板最前面增加一列非必填字段“单据序号”，单据序号可以为空，多行之间单据序号不得重复。格式为整数型。
    1. 若有重复导入提示语“单据序号不得重复”
2. 若该合同表单中包含产品明细组件或表格组件，则以组件名称在模板中新增sheet页。
    2. 各组件单独的sheet页中，最前面一列为必填的“单据序号”，后面各列为**子表单字段名称**，支持导入的子表单字段类型有：文本、数字、金额、下拉选项、项目、辅助核算、部门、日期、城市、同行人/员工、成本中心（同主表字段类型）。其中：
        1. 金额：金额币种填写币种编码，填写则默认为本位币；币种编码和金额之间用“\_”连接
        2. 下拉选项：按照文本内容匹配
        3. 项目：填写项目编码
        4. 辅助核算：填写辅助核算的编码
        5. 部门：填写部门的全路径，例如：xx测试集团/xx事业部/财务部
        6. 日期：格式为年/月/日；起止日期之间使用～连接；
        7. 城市：格式为省，市，区
        8. 同行人/员工：个人之间使用“，”号隔开，填写姓名
        9. 成本中心：填写成本中心编码
    3. 若产品明细组件或表格组件必填，且子表单组件中存在必填字段不在上述支持的子表单字段类型中，则限制该模板不支持导入，提示语“\[产品明细组件或表格组件名称\]存在暂不支持导入的必填字段\[必填字段名称\]”
    4. 导入时，产品明细组件或表格组件的数据按照单据序号匹配到合同主表中。若填写的单据序号在合同主表中找不到对应的单据序号，则本次导入全部失败，提示“\[产品明细组件或表格组件名称\]第\[x\]行数据找不到匹配的单据序号”
3. 导入合同弹框提示文案修改：12.产品明细组件/表格组件：在模板新的sheet页中填写对应子表明细，能够支持的字段及格式要求同上。

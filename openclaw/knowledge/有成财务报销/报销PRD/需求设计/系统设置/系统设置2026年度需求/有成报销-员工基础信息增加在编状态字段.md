## <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">在员工基础信息中新增"在编状态"字段（编内/编外），并支持在公式计算中引用该字段，实现表格组件内员工组件与其他组件的数据联动。</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**需求方**</span><span style="color: rgba(0, 0, 0, 0.9);">：系统管理员、财务人员、业务单据配置人员</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**现状问题**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">员工基础信息缺少"在编状态"字段，无法区分编内/编外人员身份</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">表格组件内添加的员工组件，其字段信息在公式计算中无法被其他组件引用</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">无法实现"选择员工→自动带出在编状态→根据在编状态计算其他值"的联动场景</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">第三方系统无法通过接口获取和推送员工的【在编状态】字段</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**解决思路**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">新增员工基础字段"在编状态"</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">扩展公式计算能力，支持员工组件【常用组件和自定义组件】的子字段（包括在编状态）在公式计算中被引用</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">兼容场景：自定义单据【员工组件，文本组件和表格组件】，自定义费用【员工组件，文本组件和表格组件】</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">扩展有成报销</span><span style="color: rgba(0, 0, 0, 0.9);">**【获取用户列表】和【批量新增/更新企业人员】接口的【在编状态】字段**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程**</span>
```plain
HR维护员工在编状态
    ↓
表单设计器配置表格组件（员工组件+文本组件）
    ↓
文本组件配置公式引用员工组件的在编状态
    ↓
用户选择员工后，文本组件自动带出在编状态
```

## <span style="color: rgba(0, 0, 0, 0.9);">**2\. 功能现状**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能位置1**</span><span style="color: rgba(0, 0, 0, 0.9);">：基础设置 → 员工管理 → 员工信息 → 编辑员工</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现有字段**</span><span style="color: rgba(0, 0, 0, 0.9);">：姓名、工号、手机号、邮箱、角色、直属上级、常驻地、所属部门、默认部门、在职状态</span>

<span style="color: rgba(0, 0, 0, 0.9);">**缺失字段**</span><span style="color: rgba(0, 0, 0, 0.9);">：在编状态</span>

<span style="color: rgba(0, 0, 0, 0.9);">**功能位置2**</span><span style="color: rgba(0, 0, 0, 0.9);">：自定义单据 → 表单设计器 → 左侧组件库</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现有组件**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">申请人组件</span>
- <span style="color: rgba(0, 0, 0, 0.9);">员工组件</span>
- <span style="color: rgba(0, 0, 0, 0.9);">文本、数字、日期等基础组件</span>
- <span style="color: rgba(0, 0, 0, 0.9);">表格组件</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现有能力限制**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">公式计算中，表格组件内添加的员工组件缺少在编状态信息</span>
- <span style="color: rgba(0, 0, 0, 0.9);">表格组件内文本组件无法通过公式计算引用员工组件的在编状态</span>

<span style="color: rgba(0, 0, 0, 0.9);">**典型配置示例**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**示例1：表格内带出员工在编状态**</span>

<span style="color: rgba(0, 0, 0, 0.9);">单据结构：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">表格组件，包含两列：员工（员工组件-支持多选）、在编状态（文本组件）</span>

<span style="color: rgba(0, 0, 0, 0.9);">配置：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">"在编状态"列公式：</span>`【CONCAT（-，在编状态（文本数组），剩余参数）】`

<span style="color: rgba(0, 0, 0, 0.9);">效果：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户在"员工"列选择员工后，"在编状态"列自动显示该员工的在编状态</span>

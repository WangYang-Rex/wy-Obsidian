# 高级版单据打印设置中增加【条形码】

> 🔗 **原文链接**：[高级版单据打印设置中增加【条形码】 - 钉钉文档](https://alidocs.dingtalk.com/i/nodes/ZgpG2NdyVXrOqRxzHABO3jZ28MwvDqPk?utm_scene=team_space)

| 修订时间 | 版本 | 修订说明 |
|------------|------|------------|
| 20250226 | v2.0 | 新增条形码系统组件，扫码获取单据号 |
| 20250226 | v2.0 | 条形码仅支持系统默认单据号 |

# **1\. 需求背景**
1. <span style="color: rgba(0, 0, 0, 0.9);">现有打印自定义功能已支持单据字段、系统字段、审批信息等内容，但缺少条形码组件，无法满足客户快速扫条形码识别单据的需求；</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">用户线下流转单据时，需要通过扫码枪快速获取单据信息，提升审批和核对效率；</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">客户江苏省人民医院需要单据有条形码；</span>

# **2\. 业务流程/架构**

用户配置打印模板（高级版）

↓

【系统字段】处拖拽【条形码】组件

↓

设置【字段设置】

↓

保存模板并打印单据

↓

扫码枪扫描单据条形码

↓

系统解析条形码内容，获取单据号，定位对应单据

# **3\. 功能清单**

| 模块 | 功能点 | 备注 |
|------|---------|------|
| 系统组件 | 增加【条形码】 |  |
| 自定义编码 | 新增和编辑自定义编码里选择的适用单据并保存时，增加红色文字提示“启用自定义编码生成的单据，打印的条形码信息固定16个0” |  |
| 打印配置 | 【条形码】组件拖拽至打印区，右侧【字段设置】显示固定提示文字标红“启用自定义编码生成的单据，打印的条形码信息固定16个0” |  |

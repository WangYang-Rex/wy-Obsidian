## <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">可抵扣发票税额分录设置增加「生效账套」配置项，支持按账套维度独立配置可抵扣发票的价税分离规则，实现同一可抵扣发票在不同账套生成凭证时分别应用各自匹配的分录规则。</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**需求方**</span><span style="color: rgba(0, 0, 0, 0.9);">：企业财务管理员/系统管理员</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**现状问题**</span><span style="color: rgba(0, 0, 0, 0.9);">：
• 当前可抵扣发票税额分录设置为全局生效，所有账套共用同一套规则
• 客户实际业务中，不同账套（如总公司账套与分公司账套）对可抵扣发票的税额处理要求可能不同
• 无法实现同一可抵扣发票在多账套凭证生成时按各账套规则分别处理</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**解决思路**</span><span style="color: rgba(0, 0, 0, 0.9);">：在可抵扣发票税额分录配置中增加「生效账套」字段，支持多选，存量规则默认全部账套生效，实现账套级差异化配置</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程**</span>
```plain
配置阶段：
开启税额分录功能开关
↓
配置生效范围规则（单据范围 + 费用类型 + 发票类别 + 生效账套）
──────────────────────────────────────────────────────────────
差旅报销 交通费 增值税专票 账套A、账套B
日常报销 办公费 增值税普票 全部账套

执行阶段：
可抵扣发票进入系统
↓
单据按账套生成凭证时，开关已开启
↓
匹配规则：单据范围 + 费用类型 + 发票类别 + 生效账套
↓
账套命中规则 → 按价税分离生成税额分录（借：费用 借：进项税 贷：应付/银行）
账套未命中规则 → 按原有逻辑生成（借：费用含税金额 贷：应付/银行）
```

- <span style="color: rgba(0, 0, 0, 0.9);">功能开关：总控开关（税额分录是否启用）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">规则配置表格：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">单据范围：下拉选择（差旅报销/日常报销/借款单等）</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现有操作**</span>

- <span style="color: rgba(0, 0, 0, 0.9);">开关开启后，可新增、编辑、删除税额分录规则</span>
- <span style="color: rgba(0, 0, 0, 0.9);">开关关闭时，规则配置区域隐藏</span>
- <span style="color: rgba(0, 0, 0, 0.9);">规则按配置顺序从上到下匹配，命中即停</span>
- <span style="color: rgba(0, 0, 0, 0.9);">匹配维度：单据范围 \+ 费用类型 \+ 发票类别</span>
- <span style="color: rgba(0, 0, 0, 0.9);">匹配到的规则：可抵扣发票按价税分离生成凭证分录</span>
- <span style="color: rgba(0, 0, 0, 0.9);">未匹配到规则：按原有逻辑生成凭证（无价税分离）</span>

<span style="color: rgba(0, 0, 0, 0.9);">**缺失能力**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">无法指定规则对哪些账套生效</span>
- <span style="color: rgba(0, 0, 0, 0.9);">无法实现不同账套用不同规则控制价税分离</span>

<span style="color: rgba(0, 0, 0, 0.9);">**配置界面**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**位置**</span><span style="color: rgba(0, 0, 0, 0.9);">：会计科目映射 → 基础设置 → 税额分录价税分离生成税额分录设置 → 规则配置区域</span>

<span style="color: rgba(0, 0, 0, 0.9);">**新增配置项**</span>

![画布 56.png](https://alidocs2.oss-cn-zhangjiakou.aliyuncs.com/res/AJdl65ADzx0vmOke/img/c581faf5-e110-4bf7-85fe-ee7fab5b6008.png?Expires=1775709736&OSSAccessKeyId=LTAI5tKTjg4Kq1HCdBJ8qpSp&Signature=2H5hMH6AgQcrm4FOGySkRqbLTyw%3D "")



<span style="color: rgba(0, 0, 0, 0.9);">**交互规则**</span> <span style="color: rgba(0, 0, 0, 0.9);">• 点击「新增」按钮，表格新增一行空规则，可填写单据范围、费用类型、发票类别、生效账套
• 点击每行「删除」按钮，删除该行规则

• 配置保存后即时生效，影响后续可抵扣发票生成的凭证</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**3.2.3 凭证生成-可抵扣发票账套独立匹配规则**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**触发条件**</span><span style="color: rgba(0, 0, 0, 0.9);">：税额分录功能开关已开启</span>

<span style="color: rgba(0, 0, 0, 0.9);">**行为分支**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**匹配逻辑**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**分录生成方式**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">功能开关关闭</span> | <span style="color: rgba(0, 0, 0, 0.9);">不进入税额分录逻辑</span> | <span style="color: rgba(0, 0, 0, 0.9);">按原有逻辑生成凭证（借：费用含税金额 贷：应付/银行）</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">账套命中规则</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据范围\+费用类型\+发票类别匹配，且该账套在规则生效账套列表中</span> | <span style="color: rgba(0, 0, 0, 0.9);">按价税分离生成（借：费用不含税金额 借：进项税 贷：应付/银行）</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">账套未命中规则</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据范围\+费用类型\+发票类别不匹配，或该账套不在规则生效账套列表中</span> | <span style="color: rgba(0, 0, 0, 0.9);">按原有逻辑生成凭证（无价税分离）</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">多规则同时命中</span> | <span style="color: rgba(0, 0, 0, 0.9);">按配置顺序从上到下，全部生效</span> | <span style="color: rgba(0, 0, 0, 0.9);">按价税分离生成</span> |

<span style="color: rgba(0, 0, 0, 0.9);">**核心逻辑说明**</span> <span style="color: rgba(0, 0, 0, 0.9);">• 税额分录功能开关关闭时，所有账套均按原有逻辑生成凭证
• 开关开启后，同一可抵扣发票在多个账套生成凭证时，每个账套独立匹配自己的规则
• 匹配维度：单据范围 \+ 费用类型 \+ 发票类别 \+ 生效账套
• 先筛选「生效账套包含当前账套」的规则，再按原有顺序匹配单据范围、费用类型、发票类别
• 命中规则：按价税分离生成凭证分录
• 未命中规则：按原有逻辑生成凭证（无价税分离）</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程**</span>
```plain
可抵扣发票关联到单据费用明细（含费用类型）
↓
单据触发凭证生成
↓
判断税额分录功能开关
↓
关闭 → 按原有逻辑生成凭证（无价税分离）
↓
开启 → 遍历该单据关联的所有目标账套
↓
账套A：
提取单据下所有可抵扣发票及关联的费用类型
↓
筛选生效账套包含「账套A」的规则（或全部账套）
↓
按原顺序匹配单据范围、费用类型、发票类别
↓
命中规则 → 按价税分离生成凭证分录
未命中 → 按原有逻辑生成凭证

账套B：
（同上，独立匹配账套B生效的规则）
```



### <span style="color: rgba(0, 0, 0, 0.9);">**3.3 逻辑互斥与边界处理**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**边界情况处理**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**边界场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**触发条件**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理方案**</span> |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">存量兼容-开关</span> | <span style="color: rgba(0, 0, 0, 0.9);">已开启税额分录功能</span> | <span style="color: rgba(0, 0, 0, 0.9);">升级后开关保持开启状态</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">存量兼容-规则</span> | <span style="color: rgba(0, 0, 0, 0.9);">已配置的税额分录规则无账套信息</span> | <span style="color: rgba(0, 0, 0, 0.9);">升级时自动填充「全部账套」，保持原有行为</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">功能开关关闭</span> | <span style="color: rgba(0, 0, 0, 0.9);">用户关闭税额分录功能开关</span> | <span style="color: rgba(0, 0, 0, 0.9);">规则配置区域隐藏，已配置规则保留但不生效，凭证按原有逻辑生成</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">空规则保存</span> | <span style="color: rgba(0, 0, 0, 0.9);">新增规则未填写完整字段</span> | <span style="color: rgba(0, 0, 0, 0.9);">校验提示，必填字段不能为空</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">空值处理</span> | <span style="color: rgba(0, 0, 0, 0.9);">新增规则未选择生效账套</span> | <span style="color: rgba(0, 0, 0, 0.9);">保存时自动设为「全部账套」</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">互斥逻辑</span> | <span style="color: rgba(0, 0, 0, 0.9);">同时选择「全部账套」和具体账套</span> | <span style="color: rgba(0, 0, 0, 0.9);">选择「全部账套」时清空具体账套；选择具体账套时禁用「全部账套」</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">账套删除</span> | <span style="color: rgba(0, 0, 0, 0.9);">已配置的账套被删除</span> | <span style="color: rgba(0, 0, 0, 0.9);">该规则对该账套不再生效，其他账套不受影响</span> |

## <span style="color: rgba(0, 0, 0, 0.9);">**附录**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**A. 典型使用场景**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**场景一：总公司与分公司差异化价税分离处理**</span>

<span style="color: rgba(0, 0, 0, 0.9);">配置步骤（首次）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">进入会计科目映射 → 基础设置 → 税额分录</span><span style="color: rgba(0, 0, 0, 0.9);">价税分离生成税额分录</span><span style="color: rgba(0, 0, 0, 0.9);">设置</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">开启「启用税额分录功能」开关</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">点击「新增」，添加规则：单据范围=差旅报销，费用类型=交通费，发票类别=增值税专票，生效账套=总公司账套</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">点击「新增」，添加规则：单据范围=差旅报销，费用类型=交通费，发票类别=增值税专票，生效账套=分公司账套A、分公司账套B</span>
5. <span style="color: rgba(0, 0, 0, 0.9);">点击「保存」</span>

<span style="color: rgba(0, 0, 0, 0.9);">使用步骤（日常）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">员工提交差旅报销单，费用明细包含交通费（关联增值税专票）</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">单据同时推送至总公司账套和分公司账套生成凭证</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">总公司账套：命中规则，交通费的可抵扣发票按价税分离生成凭证（借：费用 借：进项税 贷：应付）</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">分公司账套：命中规则，交通费的可抵扣发票按价税分离生成凭证（借：费用 借：进项税 贷：应付）</span>

<span style="color: rgba(0, 0, 0, 0.9);">结果：同一可抵扣发票在不同账套按各自配置决定是否进行价税分离处理</span>

<span style="color: rgba(0, 0, 0, 0.9);">**场景二：部分账套不需要价税分离**</span>

<span style="color: rgba(0, 0, 0, 0.9);">配置步骤（首次）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">进入会计科目映射 → 基础设置 → 税额分录设置</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">开启「启用税额分录功能」开关</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">点击「新增」，添加规则：单据范围=日常报销，费用类型=办公费，发票类别=增值税专票，生效账套=账套A</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">不配置账套B的规则</span>
5. <span style="color: rgba(0, 0, 0, 0.9);">点击「保存」</span>

<span style="color: rgba(0, 0, 0, 0.9);">使用步骤（日常）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">员工提交日常报销单，费用类型=办公费，关联增值税专票</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">单据推送至账套A和账套B生成凭证</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">账套A：命中规则，按价税分离生成凭证</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">账套B：未命中规则，按原有逻辑生成凭证（无价税分离）</span>

<span style="color: rgba(0, 0, 0, 0.9);">结果：实现部分账套价税分离、部分账套保持原样的差异化处理</span>

<span style="color: rgba(0, 0, 0, 0.9);">**场景三：存量规则保持兼容**</span>

<span style="color: rgba(0, 0, 0, 0.9);">配置步骤：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">原有规则：单据范围=日常报销，费用类型=办公费，发票类别=增值税普票</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">系统升级后：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">功能开关自动开启</span>
- <span style="color: rgba(0, 0, 0, 0.9);">规则生效账套字段自动回填「全部账套」</span>

<span style="color: rgba(0, 0, 0, 0.9);">使用步骤：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">员工提交日常报销单，费用类型=办公费，关联可抵扣发票</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">无论在哪个账套生成凭证，可抵扣发票均按价税分离处理</span>

<span style="color: rgba(0, 0, 0, 0.9);">结果：存量配置无需调整，行为保持一致</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">审批流连续直属领导审批功能增强，支持</span><span style="color: rgba(0, 0, 0, 0.9);">**三个独立功能点**</span><span style="color: rgba(0, 0, 0, 0.9);">，在保留原有「指定角色不参与审批」逻辑的基础上，新增「指定角色必须实际审批」逻辑：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">**依次审批**</span> <span style="color: rgba(0, 0, 0, 0.9);">→ 「申请人连续直属领导审批」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">**依次审批**</span> <span style="color: rgba(0, 0, 0, 0.9);">→ 「通过单据员工连续直属领导匹配」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">**非会签**</span> <span style="color: rgba(0, 0, 0, 0.9);">→ 「连续部门主管审批」 → 【审批到指定角色时终止】：支持配置「指定角色是否需要审批」</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**需求方**</span><span style="color: rgba(0, 0, 0, 0.9);">：企业管理员/审批流配置人员</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状问题**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">「申请人连续直属领导审批」匹配到指定角色时</span><span style="color: rgba(0, 0, 0, 0.9);">**流程提前结束，被指定角色不参与审批**</span><span style="color: rgba(0, 0, 0, 0.9);">（旧逻辑）</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">「通过单据员工连续直属领导匹配」匹配到指定角色时</span><span style="color: rgba(0, 0, 0, 0.9);">**流程提前结束，被指定角色不参与审批**</span><span style="color: rgba(0, 0, 0, 0.9);">（旧逻辑）</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">「连续部门主管审批」（非会签）匹配到指定角色时</span><span style="color: rgba(0, 0, 0, 0.9);">**流程提前结束，被指定角色不参与审批**</span><span style="color: rgba(0, 0, 0, 0.9);">（旧逻辑）</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">部分业务场景需要指定角色</span><span style="color: rgba(0, 0, 0, 0.9);">**实际参与审批**</span><span style="color: rgba(0, 0, 0, 0.9);">，而非自动结束</span>

<span style="color: rgba(0, 0, 0, 0.9);">**解决思路**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">**保留旧逻辑**</span><span style="color: rgba(0, 0, 0, 0.9);">：匹配到指定角色时流程结束，该角色不参与审批</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">**新增新逻辑**</span><span style="color: rgba(0, 0, 0, 0.9);">：匹配到指定角色时，该角色必须实际审批后才能结束</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">通过界面配置项「指定角色审批方式」让用户自主选择</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程对比**</span>
```plain
【旧逻辑：指定角色不参与审批】
员工提交单据
    ↓
逐级查找：B（一级）→ C（二级）→ D（三级，指定角色）
    ↓
B审批 → C审批 → 流程结束（D不参与）

【新逻辑：指定角色必须审批】
员工提交单据
    ↓
逐级查找：B（一级）→ C（二级）→ D（三级，指定角色）
    ↓
B审批 → C审批 → D审批（D必须审批）→ 流程结束
```

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程 - 旧逻辑**</span>
```plain
申请人提交单据
    ↓
逐级查找直属领导链
    ↓
判断当前领导是否属于指定角色
    ↓
是 → 该领导**不加入**审批链，流程直接结束
否 → 继续向上查找
```



<span style="color: rgba(0, 0, 0, 0.9);">**业务流程 - 新逻辑**</span>
```plain
申请人提交单据
    ↓
逐级查找直属领导链
    ↓
判断当前领导是否属于指定角色
    ↓
是 → 该领导加入审批链，作为最后一个审批人
    ↓
生成完整审批链，按层级依次审批
    ↓
指定角色审批人收到待办，审批通过后流程结束
```

    ↓

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程 - 新逻辑**</span>

申请人提交单据
    ↓

    ↓
判断当前主管是否属于指定角色
    ↓

    ↓

    ↓

```

#### <span style="color: rgba(0, 0, 0, 0.9);">**场景二：新逻辑 - 指定角色需要审批**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**场景三：按项目成员链，指定角色需要审批**</span>

- <span style="color: rgba(0, 0, 0, 0.9);">勾选「审批到指定角色时终止」，选择「部门负责人」</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**指定角色审批方式：指定角色需要审批，审批后结束**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**流程**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>

员工提交（项目成员=B）
    ↓
查找B的直属领导链：C（组长）→ D（经理）→ E（部门负责人）
    ↓
依次审批：C审批 → D审批 → E（部门负责人）审批 → 结束
                                    ↑
                            部门负责人必须审批
```

### <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**1.1 需求描述**</span>

<span style="color: rgba(0, 0, 0, 0.9);">预算报表中，当统计调整筛选仅勾选"预算申请"时，调整金额列支持点击下探，在新页面展示关联的预算申请单明细。</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求背景**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**需求方：**</span> <span style="color: rgba(0, 0, 0, 0.9);">财务/预算管理员</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**现状问题：**</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">预算报表展示调整金额，但无法追溯金额调整来源</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">用户需手动去预算申请模块查询，操作路径长</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">缺乏从报表数据到原始单据的穿透能力</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**解决思路：**</span> <span style="color: rgba(0, 0, 0, 0.9);">特定筛选条件下，调整金额变为可点击链接，一键下探查看关联预算申请单</span>

<span style="color: rgba(0, 0, 0, 0.9);">**业务流程：**</span>
```plain
进入预算报表 → 统计调整勾选"预算申请"（仅勾选此项）
  ↓
调整金额变为蓝色可点击
  ↓
点击调整金额 → 新页面打开预算申请单列表
  ↓
查看关联单据明细
```

### <span style="color: rgba(0, 0, 0, 0.9);">**6\. 附录**</span>

6.1 典型使用场景

场景：追溯预算调整来源

财务查看2026年某部门预算报表，发现调整金额异常。仅勾选"预算申请"后点击调整金额，新页面展示该部门所有预算申请单，快速定位是哪几笔申请导致了调整。

6.2 相关数据模型

调整金额与预算申请单关系：

一对多：一个调整金额可能对应多笔预算申请单

关联维度：年度 \+ 预算维度（部门/成本中心/项目等）\+ 调整类型=预算申请

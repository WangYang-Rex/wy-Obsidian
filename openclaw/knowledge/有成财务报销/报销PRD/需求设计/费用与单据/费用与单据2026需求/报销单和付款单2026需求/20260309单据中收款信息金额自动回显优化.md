## <span style="color: rgba(0, 0, 0, 0.9);">**2、需求背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**2.1 需求来源**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**客户**</span><span style="color: rgba(0, 0, 0, 0.9);">：周至联合医院</span>

<span style="color: rgba(0, 0, 0, 0.9);">**目标用户**</span><span style="color: rgba(0, 0, 0, 0.9);">：财务</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**2.2 现状问题**</span>

<span style="color: rgba(0, 0, 0, 0.9);">员工在报销单或付款单的"收款信息"组件处添加收款信息后，添加完费用明细信息，在提交单据前，"收款信息"内"金额"处仍然显示0。财务人员需要在提交前确认收款金额与单据支付金额一致，当前无法在提交前查看正确收款金额，影响财务核对效率。</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**2.3 使用限制**</span>

<span style="color: rgba(0, 0, 0, 0.9);">用户仅要求在"收款信息"组件</span><span style="color: rgba(0, 0, 0, 0.9);">**未勾选"支持多选"**</span><span style="color: rgba(0, 0, 0, 0.9);">的情况下，收款信息的金额在提交单据前能反映真实的单据支付金额。</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**3、现有逻辑**</span>



| <span style="color: rgba(0, 0, 0, 0.9);">**序号**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**逻辑说明**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">a</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据内"收款信息"组件未勾选"支持多选"时，单据内仅能添加一条收款信息</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">b</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据内"收款信息"组件未勾选"支持多选"时，单据提交后回显至"收款信息"的"金额"优先取单据内"支付金额"组件的值，若没有"支付金额"组件，就取"费用明细"合计金额的值</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">c</span> | <span style="color: rgba(0, 0, 0, 0.9);">单据未提交或保存草稿前，"收款信息"的"金额"不更新且不能编辑</span> |

## <span style="color: rgba(0, 0, 0, 0.9);">**4、核心痛点**</span>

<span style="color: rgba(0, 0, 0, 0.9);">单据内"收款信息"的"金额"在单据未保存或未提交前没有显示支付金额，用户需要"收款金额"与该单据此时需要支付的金额一致。</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**7、需求详情**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**7.1、需求说明**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**一、触发条件**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**生效条件**</span><span style="color: rgba(0, 0, 0, 0.9);">（需同时满足）：</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">单据模板中"收款信息"组件</span><span style="color: rgba(0, 0, 0, 0.9);">**未勾选"支持多选"**</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">单据处于</span><span style="color: rgba(0, 0, 0, 0.9);">**编辑状态**</span><span style="color: rgba(0, 0, 0, 0.9);">（未提交、未保存为草稿）</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**二、金额计算规则**</span>

<span style="color: rgba(0, 0, 0, 0.9);">单据处于编辑状态时，"收款信息"的"金额"字段按以下优先级自动回显：</span>

<span style="color: #FE0300;">"收款信息"的"金额"=单据提交金额-核销金额-企业月结</span>



#### <span style="color: rgba(0, 0, 0, 0.9);">**三、展示规则**</span>



| <span style="color: rgba(0, 0, 0, 0.9);">**状态**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**金额显示**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**是否可编辑**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|-------------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">新建或编辑中（未保存）</span> | <span style="color: rgba(0, 0, 0, 0.9);">按上述规则自动计算回显</span> | <span style="color: rgba(0, 0, 0, 0.9);">不可编辑（保持原逻辑）</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">保存草稿后</span> | <span style="color: rgba(0, 0, 0, 0.9);">按上述规则自动计算回显</span> | <span style="color: rgba(0, 0, 0, 0.9);">不可编辑</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">提交后</span> | <span style="color: rgba(0, 0, 0, 0.9);">按原有逻辑回显</span> | <span style="color: rgba(0, 0, 0, 0.9);">不可编辑</span> |

#### <span style="color: rgba(0, 0, 0, 0.9);">**四、边界情况处理**</span>



| <span style="color: rgba(0, 0, 0, 0.9);">**场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**处理逻辑**</span> |
|----------------------------------------------------------|----------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">费用明细为空</span> | <span style="color: rgba(0, 0, 0, 0.9);">收款信息金额显示0</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">收款信息已存在金额（历史数据）</span> | <span style="color: rgba(0, 0, 0, 0.9);">按新逻辑覆盖显示计算值</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">多币种单据</span> | <span style="color: rgba(0, 0, 0, 0.9);">按单据主币种计算合计</span> |

### <span style="color: rgba(0, 0, 0, 0.9);">**7.2、交互说明**</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**PC端**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**1\. 费用明细编辑时**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户添加/修改/删除费用明细行</span>
- <span style="color: rgba(0, 0, 0, 0.9);">系统自动计算费用合计</span>
- <span style="color: rgba(0, 0, 0, 0.9);">收款信息"金额"字段</span><span style="color: rgba(0, 0, 0, 0.9);">**实时更新**</span><span style="color: rgba(0, 0, 0, 0.9);">显示计算值</span>

<span style="color: rgba(0, 0, 0, 0.9);">~~**2\. 支付金额组件变动时**~~</span>
- <span style="color: rgba(0, 0, 0, 0.9);">~~用户修改支付金额~~</span>
- <span style="color: rgba(0, 0, 0, 0.9);">~~收款信息"金额"字段~~</span><span style="color: rgba(0, 0, 0, 0.9);">~~**实时同步**~~</span><span style="color: rgba(0, 0, 0, 0.9);">~~显示相同值~~</span>

<span style="color: rgba(0, 0, 0, 0.9);">**3\. 视觉提示**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">自动回显的金额以</span><span style="color: rgba(0, 0, 0, 0.9);">**灰色字体**</span><span style="color: rgba(0, 0, 0, 0.9);">展示（表示系统自动计算）</span>

#### <span style="color: rgba(0, 0, 0, 0.9);">**MB端（移动端）**</span>
1. <span style="color: rgba(0, 0, 0, 0.9);">逻辑与PC端保持一致</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">收款信息卡片中金额字段实时更新</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">不可编辑状态保持现有样式</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**7.3、互斥逻辑说明**</span>

| <span style="color: rgba(0, 0, 0, 0.9);">**场景**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**是否生效本需求**</span> | <span style="color: rgba(0, 0, 0, 0.9);">**说明**</span> |
|----------------------------------------------------------|-------------------------------------------------------------------------|----------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">收款信息-支持多选（勾选）</span> | <span style="color: rgba(0, 0, 0, 0.9);">不生效</span> | <span style="color: rgba(0, 0, 0, 0.9);">保持原逻辑，不自动回显金额</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">收款信息-单选（未勾选）</span> |  <span style="color: rgba(0, 0, 0, 0.9);">生效</span> | <span style="color: rgba(0, 0, 0, 0.9);">按本需求逻辑自动回显金额</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">已提交单据</span> |  <span style="color: rgba(0, 0, 0, 0.9);">不生效</span> | <span style="color: rgba(0, 0, 0, 0.9);">保持提交时的金额不变</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">已保存草稿后打开</span> | <span style="color: rgba(0, 0, 0, 0.9);">生效</span> | <span style="color: rgba(0, 0, 0, 0.9);">重新按当前数据计算回显</span> |

### <span style="color: rgba(0, 0, 0, 0.9);">**7.4、数据校验**</span>

<span style="color: rgba(0, 0, 0, 0.9);">单据提交时保持现有校验逻辑：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">收款信息金额应与支付金额（或费用合计）一致</span>
- <span style="color: rgba(0, 0, 0, 0.9);">不一致时按原有校验规则提示错误</span>

|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|

## <span style="color: rgba(0, 0, 0, 0.9);">**9、历史数据处理**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**已保存草稿的单据**</span><span style="color: rgba(0, 0, 0, 0.9);">：下次打开编辑时按新逻辑重新计算显示</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**已提交的单据**</span><span style="color: rgba(0, 0, 0, 0.9);">：保持原有显示，不做变更</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**正在编辑的单据**</span><span style="color: rgba(0, 0, 0, 0.9);">：实时按新逻辑计算显示</span>

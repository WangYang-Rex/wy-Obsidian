<span style="color: rgba(0, 0, 0, 0.9);">1.3.2 火车票电子发票特殊性
               铁路电子客票（火车票）作为数电票的一种，具有特殊业务属性：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**开票日期**</span><span style="color: rgba(0, 0, 0, 0.9);">：铁路部门开具发票的日期（通常与乘车日期不同，可能提前或延后开具）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**出发时间**</span><span style="color: rgba(0, 0, 0, 0.9);">**：**</span><span style="color: rgba(0, 0, 0, 0.9);">旅客实际乘坐列车的日期（票面标注的"2024年03月15日"等）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**业务场景差异**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">财务审核时关注开票日期（入账期间、进项税抵扣期间）</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">业务审核时关注</span><span style="color: rgba(0, 0, 0, 0.9);">出发时间</span><span style="color: rgba(0, 0, 0, 0.9);">（差旅日期真实性、行程合理性）</span>

<span style="color: rgba(0, 0, 0, 0.9);">1.3.3 客户业务场景
               客户公司差旅管理场景：</span><span style="color: rgba(0, 0, 0, 0.9);">报销周期差异：员工可能集中报销（月末/季末），火车票可能提前30天购买（开票日期早），但实际乘车日期与出差申请单日期需要核对</span>


1. <span style="color: rgba(0, 0, 0, 0.9);">**审核痛点：**</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">财务审核时看到开票日期为"2024-02-01"，但实际</span><span style="color: rgba(0, 0, 0, 0.9);">出发时间</span><span style="color: rgba(0, 0, 0, 0.9);">为"2024-03-15"</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">无法直观判断发票对应的实际行程日期，需人工打开PDF核对</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">退票/改签场景下，开票日期与乘车日期差异更大，易产生混淆</span>

<span style="color: rgba(0, 0, 0, 0.9);">2.1 核心痛点
痛点：火车票电子发票识别后仅显示"开票日期"，缺失"出发时间"关键字段
现状：OCR/解析引擎提取了开票日期，但未提取票面标注的出发时间
影响：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">财务审核效率低：需人工打开PDF核对实际行程日期</span>
- <span style="color: rgba(0, 0, 0, 0.9);">合规风险：无法快速核验发票与出差申请单日期一致性</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户体验差：报销人无法确认系统识别的发票对应正确的行程</span>

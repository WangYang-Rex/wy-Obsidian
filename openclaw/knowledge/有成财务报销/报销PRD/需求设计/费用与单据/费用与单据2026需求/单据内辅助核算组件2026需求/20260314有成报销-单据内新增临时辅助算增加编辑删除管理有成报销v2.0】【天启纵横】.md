# <span style="color: rgba(0, 0, 0, 0.9);">**1\. 需求背景**</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**1.1 目标用户**</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**财务管理员**</span><span style="color: rgba(0, 0, 0, 0.9);">：负责辅助核算基础数据配置、权限分配、数据治理</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**报销员工**</span><span style="color: rgba(0, 0, 0, 0.9);">：在单据填报时使用辅助核算组件，临时新增辅助核算项</span>
- <span style="color: rgba(0, 0, 0, 0.9);">**辅助核算专员**</span><span style="color: rgba(0, 0, 0, 0.9);">：仅拥有临时数据管理权限，负责用户层辅助核算数据的日常维护</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**1.2 需求来源**</span>

<span style="color: rgba(0, 0, 0, 0.9);">天启纵横客户反馈 \+ 内部权限管理优化需求</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**1.3 背景**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.3.1 辅助核算双层级架构说明**</span>

<span style="color: rgba(0, 0, 0, 0.9);">有成报销内辅助核算数据分为两个层级存储，满足不同场景的管理需求：</span>


```plain
┌─────────────────────────────────────────────────────────────────────┐
│                     辅助核算双层级架构                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【基础档案-辅助核算】                                                │
│  ├── 配置入口：基础档案 → 辅助核算设置                                │
│  ├── 管理权限：财务管理员（所有数据）/辅助核算专员（仅临时数据）         │
│  ├── 数据层级：企业层（正式数据）                                     │
│  ├── 数据特性：长期保留，影响全系统业务逻辑                            │
│  └── 应用场景：标准化的辅助核算项，供全企业使用                        │
│                                                                      │
│  【单据内辅助核算组件】                                               │
│  ├── 配置入口：基础档案 → 辅助核算设置 → 允许新增开启                  │
│  ├── 配置入口：单据 → 辅助核算组件 → 新建                              │
│  ├── 管理权限：创建人可在基础档案-辅助核算内编辑删除                         │
│  ├── 数据层级：用户层（临时数据）                                     │
│  ├── 数据特性：24小时自动清理，审批通过后同步至企业层                   │
│  └── 应用场景：临时新增的辅助核算项，解决紧急填报需求                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```



### <span style="color: rgba(0, 0, 0, 0.9);">**1.3.2 现有功能说明**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**基础设置-辅助核算（企业层正式数据）**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">仅财务管理员可进入配置</span>
- <span style="color: rgba(0, 0, 0, 0.9);">配置完成后被授权用户可见可用</span>
- <span style="color: rgba(0, 0, 0, 0.9);">辅助核算可配置辅助核算表单模版，模版内可添加收款信息，关联项目，发票抬头，项目和公司账户等支持填单时关联带入信息的组件</span>
- <span style="color: rgba(0, 0, 0, 0.9);">数据变更影响所有关联业务（报表、审批流、财务核算等）</span>
- <span style="color: rgba(0, 0, 0, 0.9);">支持启用/停用，停用后单据内不可选择</span>

<span style="color: rgba(0, 0, 0, 0.9);">**单据内辅助核算组件（用户层临时数据）**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
```plain
单据辅助核算组件弹窗
    ├── 选择已有辅助核算项（被授权的企业层数据）
    └── 新建辅助核算项（用户层临时数据）
            ├── 数据层级：用户层（临时）
            ├── 共享范围：单据内有同一辅助类别的企业内用户共享
            ├── 24小时自动清理（未审批通过/草稿状态）
            └── 审批通过后同步至企业层（成为正式数据）
                    └── 同步后等同于基础设置中配置的辅助核算
```



<span style="color: rgba(0, 0, 0, 0.9);">**系统设置-角色与权限（数据权限）**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">现有数据权限缺少对辅助核算模块的独立数据权限控制</span>

1. <span style="color: rgba(0, 0, 0, 0.9);">**基础设置配置场景**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">财务管理员在【基础档案-辅助核算】配置可用的辅助核算项</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">标准化的辅助核算项通过权限分配给企业员工使用</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">停用的辅助核算项在单据内不可见</span>
2. <span style="color: rgba(0, 0, 0, 0.9);">**单据内临时新增场景**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">员工在填报报销单/付款单时选择辅助核算组件</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">若基础设置中未配置所需核算项，临时新建</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">单据內选择"主体公司"辅助核算项可带入"收款信息"</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">若该员工无此"收款信息"权限，则单据内不带入"收款信息"</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">新建的数据24小时内有效，审批通过后成为正式数据</span>
3. <span style="color: rgba(0, 0, 0, 0.9);">**数据纠错场景**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">员工临时新建时填写错误（如公司名称、税号、收款账户等）</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">员工需要在基础档案模块统一管理和清理临时辅助核算数据</span>
4. <span style="color: rgba(0, 0, 0, 0.9);">**示例主体公司关联收款账户场景**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">辅助核算类别为新增的"主体公司"数据可配置对应"收款信息"</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">员工需从账户列表中选择正确的收款信息</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">需按账户类型（全部、个人账户、对公帐户、现金账户、承兑汇票、其他账户和虚拟账户）筛选</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">目前账户类型的条件过滤异常</span>
5. <span style="color: rgba(0, 0, 0, 0.9);">**新增-权限管控场景**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">财务管理员需要严格控制谁可以管理全部辅助核算数据和临时辅助核算数据</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">部分用户（如辅助核算专员）仅需管理临时数据，不应看到正式数据</span>
    - <span style="color: rgba(0, 0, 0, 0.9);">临时数据需要明显标识，便于区分和管理</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**1.3.4 数据流转关系**</span>


```plain
单据内新建辅助核算（用户层临时数据）
    │
    ├── 24小时内未审批通过 ───→ 自动清理
    │
    └── 审批通过 ─────────────→ 同步至企业层正式数据并删除用户层该辅助核算
                                     │
                                     ↓
                              基础设置-辅助核算
                                     │
                                     ↓
                              全企业用户根据辅助核算权限可见可用
```



<span style="color: rgba(0, 0, 0, 0.9);">**新增数据管理入口**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>


```plain
【基础档案-辅助核算管理】
    │
    ├── 所有数据权限（财务管理员）
    │       ├── 企业层正式数据：完整管理权限
    │       └── 用户层临时数据：可查看、编辑、删除，带"临时"或“占用中”标识
    │
    └── 临时数据权限（辅助核算专员）
            └── 用户层临时数据：仅可查看、编辑、删除临时数据，带"临时"或“占用中”标识
```

## <span style="color: rgba(0, 0, 0, 0.9);">**1.4 核心痛点**</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**痛点1：单据内新增的辅助核算缺失编辑删除功能**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">员工在单据辅助核算组件弹窗内新建辅助核算项后，填写错误无法修改或删除</span>
- <span style="color: rgba(0, 0, 0, 0.9);">财务管理员也无法在单据内干预临时数据</span>
- <span style="color: rgba(0, 0, 0, 0.9);">错误数据24小时内持续存在，影响其他用户选择</span>
- <span style="color: rgba(0, 0, 0, 0.9);">审批通过后错误数据同步至企业层，成为基础设置中的正式脏数据</span>

<span style="color: rgba(0, 0, 0, 0.9);">**影响**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">数据质量下降，财务核算准确性受影响</span>
- <span style="color: rgba(0, 0, 0, 0.9);">员工需废弃当前单据重新填报，效率降低</span>
- <span style="color: rgba(0, 0, 0, 0.9);">脏数据进入基础设置，增加财务管理员数据治理成本</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**痛点2：用户层辅助核算为主体公司时选择收款账户信息页面账户类型过滤条件失效**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">在单据辅助核算组件弹窗内选择收款账户信息页面，账户类型筛选条件不生效</span>
- <span style="color: rgba(0, 0, 0, 0.9);">未按【账户信息设置】中的用户授权权限过滤</span>
- <span style="color: rgba(0, 0, 0, 0.9);">显示全部账户信息，包括无权限查看的敏感数据</span>

<span style="color: rgba(0, 0, 0, 0.9);">**影响**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">数据安全风险：员工可见未授权的账户信息</span>
- <span style="color: rgba(0, 0, 0, 0.9);">操作效率降低：需从大量无关账户中人工筛选</span>
- <span style="color: rgba(0, 0, 0, 0.9);">合规隐患：违反最小权限原则</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**痛点3：基础档案缺乏对用户层临时数据的统一管理入口**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">用户层临时数据仅能在单据内查看和管理</span>
- <span style="color: rgba(0, 0, 0, 0.9);">财务管理员无法在企业层面统一监控临时数据情况</span>
- <span style="color: rgba(0, 0, 0, 0.9);">临时数据与正式数据无区分标识，易造成混淆</span>

<span style="color: rgba(0, 0, 0, 0.9);">**影响**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">数据治理困难：无法及时发现和处理问题临时数据</span>
- <span style="color: rgba(0, 0, 0, 0.9);">管理效率低：需逐单检查，无法批量管理</span>
- <span style="color: rgba(0, 0, 0, 0.9);">数据安全隐患：临时数据长期滞留未被清理</span>

### <span style="color: rgba(0, 0, 0, 0.9);">**痛点4：缺乏细粒度的辅助核算数据权限控制**</span>

<span style="color: rgba(0, 0, 0, 0.9);">**现状**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">系统设置中无辅助核算模块的独立数据权限</span>
- <span style="color: rgba(0, 0, 0, 0.9);">只能控制"能否进入模块"，无法控制"能看到哪些数据"</span>
- <span style="color: rgba(0, 0, 0, 0.9);">辅助核算专员需要管理临时数据，但被迫看到全部正式数据</span>

<span style="color: rgba(0, 0, 0, 0.9);">**影响**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">权限过大：辅助核算专员可看到全部企业层敏感数据</span>
- <span style="color: rgba(0, 0, 0, 0.9);">职责不清：临时数据管理与正式数据管理权限混杂</span>
- <span style="color: rgba(0, 0, 0, 0.9);">合规风险：违反数据最小可见原则</span>

# <span style="color: rgba(0, 0, 0, 0.9);">**2\. 业务流程/架构**</span>

## <span style="color: rgba(0, 0, 0, 0.9);">**2.1 辅助核算双层级管理流程（优化后）**</span>


```plain
┌─────────────────────────────────────────────────────────────────────┐
│                    辅助核算双层级管理流程（优化后）                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  【基础档案入口】按权限区分                                            │
│       ↓                                                              │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  财务管理员（所有数据权限）                                    │    │
│  │       ↓                                                      │    │
│  │  基础档案 → 辅助核算设置                                       │    │
│  │       ↓                                                      │    │
│  │  ├─ 企业层正式数据（完整管理）                                 │    │
│  │  └─ 用户层临时数据（可编辑删除，带"临时"标识；带“占用中”标识不可编辑删除）                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│       ↓                                                              │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  辅助核算专员（临时数据权限）                                  │    │
│  │       ↓                                                      │    │
│  │  基础档案 → 辅助核算设置                                       │    │
│  │       ↓                                                      │    │
│  │  └─ 用户层临时数据（仅可编辑删除临时数据；带“占用中”标识不可编辑删除）                      │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  【单据内入口】报销员工                                              │
│       ↓                                                              │

│       ↓                                                              │
```

|----------------------------------------------------------------|----------------------------------------------------------------|-------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------|
| <span style="color: rgba(0, 0, 0, 0.9);">企业层正式数据</span> | <span style="color: rgba(0, 0, 0, 0.9);">启用/停用</span> | <span style="color: rgba(0, 0, 0, 0.9);">✅ 可删除（需二次确认）</span> | <span style="color: rgba(0, 0, 0, 0.9);">❌ 不可见</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">用户层临时数据</span> | <span style="color: rgba(0, 0, 0, 0.9);">草稿</span> | <span style="color: rgba(0, 0, 0, 0.9);">✅ 可删除</span> | <span style="color: rgba(0, 0, 0, 0.9);">✅ 可删除</span> |
| <span style="color: rgba(0, 0, 0, 0.9);">用户层临时数据</span> | <span style="color: rgba(0, 0, 0, 0.9);">占用中</span> | <span style="color: rgba(0, 0, 0, 0.9);">❌ 不可删除</span> | <span style="color: rgba(0, 0, 0, 0.9);">❌ 不可删除</span> |

<span style="color: rgba(0, 0, 0, 0.9);">**删除交互流程**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>


```plain
在基础档案-辅助核算列表页
    ↓
点击临时数据列表项的【删除】按钮
    ↓
【系统校验】数据状态
    ↓
IF 状态 = 占用中 THEN
    弹出提示"该临时数据正在被单据引用且审批中，不可删除"
ELSE
    弹出确认弹窗
        ┌─────────────────────────────────────────────────────────────┐
        │  确认删除                                                    │
        │  ─────────────────────────────────────────────────────────  │
        │  辅助核算名称: 临时客户A                                      │
        │  删除后，引用该辅助核算的草稿单据内辅助核算项清空           │
        │                                                              │
        │  【取消】                        【确认删除】                │
        └─────────────────────────────────────────────────────────────┘
        ↓
    点击【确认删除】
        ↓
    执行物理删除
        ↓
    刷新列表 → 给提示"删除成功"
END IF
```



<span style="color: rgba(0, 0, 0, 0.9);">**删除影响说明**</span><span style="color: rgba(0, 0, 0, 0.9);">：</span>
- <span style="color: rgba(0, 0, 0, 0.9);">临时数据删除后，引用该数据的草稿单据内该辅助核算字段显示清空</span>
- <span style="color: rgba(0, 0, 0, 0.9);">临时数据删除后，正在审批中的单据不受影响（因数据已被冻结）</span>

2、竞品分析（易快报）

[<span style="color: #0000FF;"><u>https://gykj.yuque.com/docs/share/8b224f8d-7b66-4457-bd35-839054859f16?#</u></span>](https://gykj.yuque.com/docs/share/8b224f8d-7b66-4457-bd35-839054859f16?#) 《竞品分析【易快报】》

| **模块** | **功能概述** | **目的** |
|----------|----------------|----------|
| 发票管理 | 发票录入-发票校验-开票收票-认证抵扣-发票复核-发票导出 | 结合当前系统功能与开发中的云票，为后续产品迭代作参考 |
| 对公-采购链路 | 项目管理-采购合同-对公付款-预付到票/到票核销-做账 | 对比业务场景与业务流程，提炼亮点，记录不足；   同时走查提单审批流程，与以前功能做对比 |
| 对公-销售链路 | 项目管理-销售合同-回款计划/催收-收款-做账 |  |
| 预算费控 | 预算申请、节假日差标等亮点功能 | 有成报销预算费控较为完善，提炼亮点功能   同时走查报销核销审批流程，与以前功能做对比 |
| 单据与报表分析 | 主要：应收应付、自定义报表 | 调研两端自定义报表场景；提炼应收应付亮点 |
| 电子会计档案 | 档案采集、检索、装册等 | 调研产品功能，为有成报销电档做准备 |
| 平台互联 | OA系统、ERP、CRM等对接   包含财务系统的对接 | 形成有成报销自有到一体化方案归纳会计科目映射两端能支持场景的区别 |
| 消费集成 | 差旅平台、行程管理 |  |
| 统一支付 | 支付、回单 |  |
| 运营/应用市场 | 应用中心、解决方案、标杆案例 | 提炼以上调研的功能与亮点，统一规划，形成有成报销自己的应用市场 |
| 业务自定义 | 表单、打印、辅助核算、审批流 | 主要是审批流的区别 |
| 其他 | 初始化流程、信用管理、钉钉、企微、飞书等 |  |
| 视觉设计 | 贯穿到功能调研中 |  |
| 交互设计 | 贯穿到功能调研中 |  |

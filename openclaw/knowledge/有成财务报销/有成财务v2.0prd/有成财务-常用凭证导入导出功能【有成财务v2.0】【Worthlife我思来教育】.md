## **1\. 需求背景**

### **1.1 需求描述**

财务人员需要在不同公司账套之间迁移常用凭证数据，以及批量创建常用凭证。当前常用凭证功能仅支持单条手动创建，不支持批量导入和导出，导致多账套场景下重复录入工作量大。

**重要约束：不同账套的会计科目设置可能不同，跨账套导入时需人工核对科目映射关系，系统不做自动科目映射。**

### **1.2 需求背景**
- **需求方**：财务人员
- **现状问题**：
• 常用凭证仅支持单条手动创建，无法批量导入
• 无法导出常用凭证数据，跨账套迁移需手动重建
• 集团型企业多账套场景下，相同常用凭证需重复录入
- **解决思路**：在常用凭证功能中增加导入导出能力，支持Excel批量操作和跨账套数据迁移

**业务流程**
```plain
财务人员整理常用凭证清单
↓
使用导入模板批量创建常用凭证
↓
或导出已有常用凭证 → 在其他账套导入使用
```

## **2\. 功能现状**

**功能位置**：基础设置 → 常用凭证

**现有功能**：
- 常用凭证列表展示
- 新增/编辑常用凭证（单条）
- 类型管理（提现、工资、销售、采购、日常支出等）
- 凭证内容包含：摘要、会计科目、借方金额、贷方金额
- 录入凭证时可调用常用凭证
- 录入凭证时可保存为常用凭证

**当前缺失能力**：
- 常用凭证导出功能
- 常用凭证批量导入功能
- 跨账套数据迁移能力

### **3.2 详细功能描述**

#### **3.2.1 常用凭证导出功能**

**位置**：基础设置 → 常用凭证 → 导出

**界面调整**

在常用凭证页面增加"导出"按钮（与"新增"按钮同级，位于页面右上角区域）。
```plain
常用凭证                                    [新增] [导出]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**导出字段**

| **序号** | **字段名称** | **说明** | **数据示例** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|
| 1 | 类型 | 常用凭证类型 | 日常支出 |
| 2 | 名称 | 常用凭证名称 | 报销差旅费 |
| 3 | 摘要 | 凭证摘要 | 报销差旅费 |
| 4 | 会计科目编码 | 会计科目编码 | 520106 |
| 5 | 会计科目名称 | 会计科目名称 | 管理费用\_差旅费 |
| 6 | 借方金额 | 借方金额（空表示不固定） | 10000 或 空 |
| 7 | 贷方金额 | 贷方金额（空表示不固定） | 10000 或 空 |

**导出逻辑**
- 导出范围为当前筛选条件下的所有常用凭证
- 无筛选时导出全部常用凭证
- 每条分录占一行，同一常用凭证多条分录显示为多行，类型和名称列重复显示
- 金额为空时表示该常用凭证不固定金额，由使用时填写

**导出文件格式**
- 文件名：`常用凭证_导出_YYYYMMDD_HHMMSS.xlsx`
- 格式：Excel (.xlsx)
- 编码：UTF-8

#### **3.2.2 常用凭证导入功能**

**位置**：基础设置 → 常用凭证 → 导入

**界面调整**

```plain
常用凭证                                    [新增] [导入] [导出]

```


**导入弹窗**
```plain
┌─────────────────────────────────────────────────────────┐
│ 导入常用凭证                                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  步骤1：下载导入模板                                      │
│  [下载模板] 下载标准Excel模板，按模板格式填写数据          │
│                                                         │
│  步骤2：上传文件                                          │
│  [点击上传或拖拽文件至此]                                 │
│  支持 .xlsx 格式，文件大小不超过 5MB                      │
│                                                         │

│  • 不同账套的会计科目设置可能不同，导入前请确认科目匹配    │

│  • 建议先导出本账套科目对照表，核对后再导入               │
│                                                         │
│                              [取消]  [开始导入]          │
└─────────────────────────────────────────────────────────┘
```


**导入模板字段**

| **序号** | **字段名称** | **必填** | **数据类型** | **说明** |
|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------|
| 1 | 类型 | 是 | 文本 | 常用凭证类型，不存在时自动创建 |
| 2 | 名称 | 是 | 文本 | 常用凭证名称，同类型下不可重复 |
| 3 | 摘要 | 是 | 文本 | 凭证摘要 |
| 4 | 会计科目编码 | 是 | 文本 | 会计科目编码，须为系统中存在的科目 |
| 5 | 会计科目名称 | 是 | 文本 | 会计科目名称，须与编码匹配 |
| 6 | 借方金额 | 否 | 数值 | 固定借方金额，空表示不固定 |
| 7 | 贷方金额 | 否 | 数值 | 固定贷方金额，空表示不固定 |

**导入逻辑**
1. **数据校验**
    - 类型、名称、摘要、会计科目编码、会计科目名称 为必填项，为空时该行导入失败
    - 会计科目编码须为系统中存在的科目，不存在时报错
    - 会计科目名称须与编码匹配，不匹配时报错
    - 借方金额和贷方金额不能同时为空（至少有一个有值）
    - 借方金额和贷方金额不能同时有值（只能借方或贷方）
2. **类型处理**
    - 类型不存在时，自动创建新类型
    - 类型已存在时，使用已有类型
3. **常用凭证处理**
    - 同一类型\+名称视为同一条常用凭证
    - 多条分录（多行）表示该常用凭证包含多个会计科目
    - 名称在同一类型下不可重复，重复时报错
4. **科目校验**
    - 校验科目编码是否存在
    - 校验科目名称是否与编码匹配
    - 校验科目是否允许用于凭证（非末级科目等校验）
5. **错误处理**
    - 校验不通过时，整批阻断，返回错误文件
    - 错误文件中标识每行错误原因
    - 用户修正后可重新导入

### **3.3 逻辑互斥与边界处理**

**边界情况处理**

| **边界场景** | **触发条件** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 类型不存在 | 导入的类型在系统中不存在 | 自动创建新类型，类型名称为导入值 |
| 名称重复 | 同一类型下名称重复 | 报错提示："同一类型下名称不可重复" |
| 科目不存在 | 会计科目编码不存在 | 报错提示："会计科目编码不存在"，阻断导入 |
| 科目不匹配 | 编码和名称不匹配 | 报错提示："会计科目名称与编码不匹配" |
| 金额校验 | 借贷方同时有值或同时为空 | 报错提示："借方金额和贷方金额不能同时为空或同时有值" |
| 跨账套科目不一致 | 源账套和目标账套科目设置不同 | **严格校验阻断**：目标账套科目编码不存在时，该行导入失败，需用户手动调整科目映射后重新导入 |
| 科目非末级 | 导入的科目不是末级科目 | 报错提示："会计科目须为末级科目" |
| 模板列缺失 | 使用错误模板导入 | 报错提示："导入模板格式不正确，请下载标准模板" |
| 空行处理 | 导入文件包含空行 | 跳过空行，不报错 |
| 金额格式 | 金额包含非数字字符 | 报错提示："金额格式不正确" |

**与其他功能的兼容性**

| **相关功能** | **交互关系** | **处理方案** |
|----------------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|
| 录入凭证-常用凭证 | 数据读取 | 导入的常用凭证在录入凭证页面可正常使用 |

| 类型管理 | 数据写入 | 导入时自动创建不存在的类型 |
| **跨账套科目映射** | **关键约束** | **不同账套科目设置可能不同，导入前需人工核对科目映射关系，系统不做自动科目映射** |

## **附录**

**A. 典型使用场景**

场景：集团型企业，总公司已维护好常用凭证，需要同步到分公司账套

操作步骤：
1. 总公司财务进入基础设置 → 常用凭证
2. 点击"导出"，下载常用凭证数据
3. **导出分公司科目表，与总公司科目表核对编码一致性**
4. **如科目编码不一致，手动调整导入文件中的科目编码**
5. 分公司财务登录分公司账套
6. 进入基础设置 → 常用凭证 → 导入
7. 上传调整后的文件，完成导入
8. 导入失败时，下载错误文件查看科目不匹配项，修正后重新导入

结果：实现常用凭证的跨账套迁移（需人工处理科目映射）

**注意事项**：
- 不同账套科目编码可能不同，系统不做自动映射
- 建议集团统一会计科目编码体系
- 导入前务必核对科目编码匹配情况

**B. 导入模板示例**
```plain
| 类型 | 名称 | 摘要 | 会计科目编码 | 会计科目名称 | 借方金额 | 贷方金额 |
| 日常支出 | 报销差旅费 | 报销差旅费 | 520106 | 管理费用_差旅费 | 10000 | |
| 日常支出 | 报销差旅费 | 报销差旅费 | 1002 | 银行存款 | | 10000 |
| 工资 | 计提工资 | 计提工资 | 520103 | 管理费用_工资 | 50000 | |
| 工资 | 计提工资 | 计提工资 | 2211 | 应付职工薪酬 | | 50000 |
```

说明：
- 第1、2行为同一条常用凭证（日常支出-报销差旅费），包含两个分录
- 第3、4行为同一条常用凭证（工资-计提工资），包含两个分录
- 借方金额和贷方金额只能有一个有值

**C. 跨账套导入注意事项**

由于不同账套的会计科目设置可能存在差异，跨账套导入常用凭证时需注意：
1. **科目编码核对**
    - 导出后，先核对源账套和目标账套的科目编码是否一致
    - 如不一致，需手动调整导入文件中的科目编码和名称
2. **科目映射建议**
    - 建议在导入前，先导出目标账套的科目表作为对照
    - 建立源账套到目标账套的科目映射关系
    - 手动修改导入文件中的科目编码后，再执行导入
3. **系统行为**
    - 系统**不做自动科目映射**，严格按编码匹配
    - 编码不存在时，该行数据导入失败
    - 失败数据可在错误文件中查看，修正后可重新导入
4. **最佳实践**
    - 集团型企业建议统一会计科目编码体系
    - 如无法统一，建立科目对照表作为中间映射层
    - 大批量迁移前，建议先导入少量数据验证科目匹配情况

**D. 字段校验规则**

| **字段** | **必填** | **校验规则** |
|----------------------------------------------------------|----------------------------------------------------------|----------------------------------------------------------------|
| 类型 | 是 | 保持产品原逻辑 |
| 名称 | 是 | 保持产品原逻辑，同类型下唯一 |
| 摘要 | 是 | 保持产品原逻辑 |
| 会计科目编码 | 是 | 须为系统中存在的科目编码 |
| 会计科目名称 | 是 | 须与编码匹配 |
| 借方金额 | 否 | 数值，精度2位小数，与贷方金额互斥 |
| 贷方金额 | 否 | 数值，精度2位小数，与借方金额互斥 |
